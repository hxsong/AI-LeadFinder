重要提示

租户资源充值页面

对应业务流程节点：P01（平台管理员审核订单、开通账号、下发资源包）

页面概述

本页面是 SYS 运营系统后台的核心功能页面，供平台管理员 (SAD) 为企业客户开通 LeadFinder 服务并下发资源包。系统支持两种资源 SKU 的组合下发：Global Credits（全局获客算力积分） 和 BOT Seats（获客引擎实例配额）。

可见功能逻辑

1. 租户搜索与选择

搜索功能

  • • 支持按租户名称或 Tenant ID 模糊搜索
  • • 输入时实时显示匹配结果下拉列表
  • • 选中后显示租户信息卡片，支持清除重选

2. 资源包选择

LeadFinder 旗舰版基础包

  • • 3 个 BOT 席位
  • • 50,000 Global Credits

LeadFinder 算力加油包

  • • 10,000 Global Credits

资源包可多选，系统自动计算汇总数量

3. 资源汇总展示

资源类型 计算规则
BOT 席位 仅基础包包含，累加计算
Global Credits 基础包 + 加油包累加

4. 提交按钮状态

条件 按钮状态
未选择租户 或 未选择资源包 禁用（灰色）
已选择租户 且 已选择资源包 启用（蓝色）

不可见功能逻辑

1. 资源下发 API

POST /internal/v1/tenant/resources/provision

Request Body:
{
  "tenant_id": "T-8821",
  "packages": ["basic", "credits"],
  "total_seats": 3,
  "total_credits": 60000,
  "operator": "admin"
}

Response:
{
  "code": 200,
  "data": {
    "order_id": "ORD-20260227-001",
    "status": "success",
    "synced_to_cls": true
  }
}

2. 系统间流转

  1. SYS 系统生成充值流水记录
  2. SYS 调用 CLS 数据获客主系统的内部 Open API
  3. CLS 接收指令，初始化该企业租户的数据库表
  4. 将 total_credits 更新为 60,000，max_bot_seats 更新为 3

页面跳转关系

客户管理列表
租户资源充值.html
资源下发确认.html
