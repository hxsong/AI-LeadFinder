P37-P38 节点

访客雷达

对应业务流程节点：P37-P38（展示所有访问记录列表 → 查看 IP 行为分析与历史记录）

页面概述

访客雷达是动态流量捕捉模块的核心展示页面，为企业管理员提供一个极具科技感的"暗网雷达图"。页面实时展示独立站访客的匿名行为数据，通过 AI 翻译访客轨迹背后的采购动机，用极具说服力的"旁白"诱导客户触发企业身份揭秘。所有数据来源于【访客表】，仅包含匿名行为资产，不包含企业实体字段。

筛选与过滤

热度筛选

默认行为：进入页面时默认展示「极高」热度的访客数据

  • • 全部：显示所有访客
  • • 极高：行为分 ≥ 80，红色标签（默认）
  • • 较高：行为分 60-79，橙色标签
  • • 一般：行为分 < 60，绿色标签

规则说明：

  • • 极高热度（红色）：规则打分 ≥ 80 分
  • • 较高热度（橙色）：规则打分 60-79 分
  • • 一般热度（绿色）：规则打分 < 60 分

地区筛选

  • • 下拉选择：全部地区 / 北美 / 欧洲 / 东南亚
  • • 基于 GeoIP 定位数据过滤
  • • 右侧显示「实时更新中」状态指示

时间筛选

默认行为：进入页面时默认选择当日日期，自动加载今日访客数据

快捷选项：

今日（默认） 近7天 近30天 全部

自定义日期范围：

  • • 开始日期选择器：选择日期范围起点
  • • 结束日期选择器：选择日期范围终点
  • • 点击「应用」按钮确认自定义日期范围
  • • 日期格式：YYYY-MM-DD
  • • 结束日期不能晚于当日

交互效果：

  • • 切换日期后自动触发数据查询
  • • 显示加载指示器（旋转动画）
  • • 查询完成后更新访客列表和统计卡片
  • • 右上角显示当前选择的日期范围文本

API 接口 - 获取访客列表

GET /api/v1/visitors

Request Headers:
  Authorization: Bearer {enterprise_token}

Query Parameters:
  page: 1
  page_size: 20
  heat_level: all | high | medium | low
  region: all | north_america | europe | southeast_asia
  date_mode: today | last7days | last30days | all | custom
  date_from: 2024-02-01
  date_to: 2024-02-26

Date Mode 说明：
  - today: 当日日期（date_from = date_to = 当天）

  - all: 不限制日期，返回所有数据
  - custom: 使用用户自定义的 date_from 和 date_to

异步处理流程

任务提交后进入后台队列：① IP 解析服务（IRS）反查企业信息 → ② 企业联系人数据服务（ECS）挖掘联系人 → ③ 动态数据分析服务（DDA）生成动态线索报告 → ④ 写入动态线索池 → ⑤ 通知用户完成。

规则打分机制

设计原则：只根据可明确识别的行为操作类型加分，不做内容判断

行为类型 加分值 说明
页面浏览 +5分 任何页面浏览行为
复制文本 +15分 任何复制操作（体现主动分析意图）
文件下载 +10分 任何文件下载（体现深度兴趣）
表单提交 +25分 完成表单提交（有明确转化意向）
表单弃填 +15分 有表单交互但未提交（有需求但存在顾虑）
点击行为 +8分 点击关键元素（按钮、链接、CTA等，体现主动交互意向）
多次访问 +10分 历史访问次数 ≥ 2

极限值处理：规则打分上限 100 分封顶，取 min(计算分, 100)

AI 模型处理逻辑 - 关键动作洞察

数据入库逻辑 - P36

DDA 返回动态行为分析数据后，系统执行以下处理：

1

写入访客表

访客行为数据写入 Visitor_Log_Table，仅含匿名行为资产

2

进入线索池

点击"深度挖掘"后，访客数据直接添加至动态线索池

数据流动说明

访客数据仅包含匿名行为资产，不包含企业实体字段。点击"深度挖掘"后，执行 IP 解析与联系人挖掘，完成后自动添加至动态线索池。

限制规则

Credits 余额检查

触发深度挖掘前检查账户余额 > 0，余额 ≤ 0 时弹窗提示充值，阻止任务提交。

余额规则：余额可为负，≤ 0 时禁止消耗操作；挖掘执行中不扣费，完成后按实际消耗结算，允许余额为负。

热度判断逻辑

热度等级由规则打分结果决定，用于快速识别高价值访客：

  • • 极高热度（红色）：规则打分 ≥ 80 分
  • • 较高热度（橙色）：规则打分 60-79 分
  • • 一般热度（绿色）：规则打分 < 60 分

AI 分析触发条件

AI 只在达到触发阈值后生成洞察分析，不再参与打分：

  • • 触发阈值：规则打分 ≥ 60 分
  • • 低于阈值：不触发 AI 分析，不显示 AI 洞察区域
  • • 高于阈值：触发 AI 生成关键动作洞察和综合研判建议

日期范围限制

时间筛选的日期范围限制规则：

  • • 结束日期：不能晚于当日（today）
  • • 开始日期：不能晚于结束日期
  • • 日期格式：严格遵循 YYYY-MM-DD 格式
  • • 快捷选项：切换时自动填充对应日期范围
  • • 自定义范围：需点击「应用」按钮确认

重复挖掘限制

已挖掘的访客记录会标记为「已挖掘」状态，不存在重复挖掘的概念。访客一旦被挖掘过，系统直接显示其已挖掘状态，不会再次触发挖掘流程，也不会产生重复扣费。

深度挖掘准入规则

访客数据无论分数高低均可进行深度挖掘，逻辑说明如下：

  • • 规则打分 < 60 分：仍可进行深度挖掘，触发 IP 企业解析与联系人挖掘流程，Credits 按标准计费
  • • 规则打分 ≥ 60 分：触发深度挖掘的同时，额外生成 AI 行为洞察分析
  • • 核心逻辑：深度挖掘是独立于规则打分的操作，分数仅影响是否触发 AI 分析，不作为挖掘准入门槛

数据保留周期

访客表数据默认保留 90 天，超期自动归档。已挖掘访客数据永久保留。

权限控制

当前版本系统暂无权限控制逻辑，所有登录用户均可查看访客雷达数据。

IP 解析处理逻辑

深度挖掘时调用 IP 解析服务，处理逻辑如下：

  • • 解析成功：返回企业名称、地址、行业等信息，生成动态线索
  • • 解析失败（无法识别）：提示用户「该 IP 无法解析到企业」，不扣除 Credits
  • • 解析失败（服务异常）：提示用户「IP 解析服务暂时不可用」，不扣除 Credits

AI 行为分析异常检测

AI 模型存在幻觉风险，可能导致行为意向分与实际行为不符。本模块对 AI 行为分析输出进行专项检测。

  • • 输出为空检测：AI 返回空分析时触发，提示「AI 分析结果为空」
  • • 格式校验：分析 JSON 解析失败时触发，提示「AI 返回数据格式异常」
  • • 一致性校验：分析与原始行为数据矛盾时触发，提示「AI 分析仅供参考」
  • • 数值范围校验：行为意向分超出 0-100 范围时触发，提示「AI 评分异常，已使用默认值」

AI 行为分析异常处理策略

异常类型 处理方式 提示文案
输出为空 显示默认热度等级 「AI 分析结果为空」
格式错误 使用规则计算热度 「AI 返回数据格式异常」
内容幻觉 标记仅供参考 「AI 分析仅供参考，以实际数据为准」
数值异常 使用业务默认值 「AI 评分异常，已使用默认值」
