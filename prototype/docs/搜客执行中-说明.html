功能说明文档
P16-P20 节点

搜客执行中页面

对应业务流程节点：P16-P20（确认开始搜客 → 返回企业数据分析结果列表）

页面概述

本页面是搜客任务的执行监控页面，实时展示任务进度和各步骤执行状态。系统按照瀑布流计划依次调用第三方 API 获取原始数据，并根据四维评分框架计算潜在客户得分。用户可离开页面，任务完成后从搜客记录中查看结果。

可见功能逻辑

1. UI 组件

1

Credits 冻结状态指示

导航栏显示冻结的 Credits 数量，带旋转动画图标

2

中心动画区域

三层同心圆动画，最内层带旋转加载图标，外层带脉冲扩散效果

3

整体进度条

显示百分比进度（0%-100%），带渐变色和波浪动画效果

4

执行步骤列表

5 个步骤卡片：Google Maps 企业扫描、海关数据交叉验证、展会数据交叉验证、官网内容富化、AI 四维评分计算

5

预计耗时提示

显示预计耗时（按 API 调用次数预估，每次约 1 分钟），提示用户可离开页面，任务完成后从搜客记录查看

2. 交互效果

  • • 步骤状态切换：等待中（灰色）→ 执行中（蓝色+旋转图标）→ 已完成（绿色+勾选图标）
  • • 进度条动画：平滑递增，带波浪流动效果（progress-wave）
  • • 中心动画：三层同心圆，最内层旋转，外层脉冲扩散
  • • 页面进入：组件依次淡入上滑（fadeInUp）
  • • 自动跳转：所有步骤完成后自动跳转到结果页面

4. AI 四维评分计算（加权汇总）

计算公式

潜在客户得分 = 需求匹配度 × W1 + 购买能力 × W2 + 采购意向 × W3 + 活跃意向 × W4

动态权重配置（根据业务类型）

业务类型 需求匹配度 购买能力 采购意向 活跃意向
外贸型 0.32 0.20 0.35 0.13
生产型 0.40 0.25 0.20 0.15
服务型 0.35 0.25 0.15 0.25
渠道型 0.35 0.20 0.30 0.15

计算示例（外贸型客户）：

7. 异常处理流程

异常类型与处理策略

API 调用失败
  • • 重试策略：最多重试 3 次，间隔 5s、10s、30s
  • • 降级处理：跳过该步骤，继续执行后续步骤
  • • 日志记录：记录失败原因、时间、请求参数
数据解析失败
  • • 原始数据保存：将原始响应保存至数据库，便于后续排查
  • • 默认值处理：缺失字段使用默认值，不影响整体流程
AI 调用失败
  • • 重试策略：最多重试 2 次，切换备用模型
  • • 降级处理：使用规则引擎替代 AI 计算
  • • Credits 退还：AI 调用失败时退还已扣除的 Credits
任务超时
  • • 接口超时：每次 API 调用超时时间 1 分钟，超时后跳过继续执行
  • • 任务不阻塞：确保任务最终完成，不会因单个接口失败而阻塞
  • • 结果查看：任务完成后可在搜客记录中查看结果

异常日志格式：

8. AI 评分异常检测

AI 模型存在幻觉风险，可能导致评分结果与实际情况不符。本模块对 AI 评分输出进行专项检测。

  • • 输出为空检测：AI 返回空评分时触发，提示「AI 评分结果为空，请重试」
  • • 格式校验：评分 JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」
  • • 一致性校验：评分与原始数据矛盾时触发，提示「AI 评分存在异常，建议重新分析」
  • • 数值范围校验：评分超出 0-100 范围时触发，提示「AI 评分异常，已使用默认值」

AI 评分异常处理策略

异常类型 处理方式 提示文案
输出为空 自动重试 1 次，失败后使用规则评分 「AI 评分结果为空，已使用默认评分」
格式错误 自动重试 1 次，失败后使用默认值 「AI 返回数据格式异常」
内容幻觉 标记异常，切换备用模型重新评分 「AI 评分存在异常，正在重新分析」
数值异常 使用业务默认值，通知风控 「AI 评分异常，已使用默认值」

9. Credits 结算规则

执行中不扣费：搜客执行过程中不扣除 Credits，所有 API 调用费用在任务完成后统一结算。

冻结机制

任务开始时冻结预估 Credits（含 30% 溢价），导航栏显示冻结数量

结算机制

任务完成后按实际 API 成功调用次数结算，退还冻结差额，允许余额为负
