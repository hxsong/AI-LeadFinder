功能说明文档
P27 节点

线索详情

对应业务流程节点：P27（展示静态线索分析报告）

页面概述

线索详情页是静态线索挖掘流程的终局呈现页面，展示完整的线索分析报告。页面分为上下两层结构：上层展示企业核心信息、最终线索评分、AI 判词和联系人矩阵；下层展示该线索被收录的原始静态证据（海关数据、展会数据、地图数据、网页数据）。页面底部提供"使用智能触达 (ITA) 生成全渠道开发信"按钮，引导用户进入下一阶段流程。

可见功能逻辑

1. 企业核心信息区

企业基础信息

  • • 企业名称（可点击跳转官网）
  • • 企业域名（超链接）
  • • 所在地
  • • 企业规模（员工数）

2. 联系人矩阵

联系人标识

  • • 联系人卡片统一样式
  • • 姓名 + 职位标签
  • • 按重要性排序展示

联系人信息

  • • 头像（姓名首字母）
  • • 姓名 + 职位标签
  • • 邮箱（可点击发送邮件）
  • • 邮箱验证状态徽章
  • • 社媒链接（LinkedIn/Twitter）

操作按钮

  • • 所有联系人：统一按钮样式
  • • 点击后跳转至智能触达 (ITA) 服务

3. 原始静态证据

展示该线索被收录的背景依据，保留当初打分时的原始证明数据

海关数据卡片

  • • 匹配状态徽章（已匹配/未匹配）
  • • 时间范围
  • • 进口次数
  • • HS Code + 产品描述

展会数据卡片

  • • 匹配状态徽章
  • • 参展次数
  • • 展会名称 + 地点
  • • 参与类型（参展商/观众）

地图数据卡片

  • • 验证状态徽章
  • • 详细地址
  • • 营业状态

网页数据卡片

  • • 抓取状态徽章
  • • 抓取时间
  • • 内容摘要（前 800 字）

4. 账单摘要

费用明细（基于 Apollo.io Credits 消耗）

  • • 基础联系人数据：不消耗 Credits
  • • 邮箱揭示（reveal_personal_emails）：每个有效邮箱 1 Credit
  • • 电话号码揭示（reveal_phone_number）：每个有效电话 1 Credit
  • • LinkedIn 主页附加：包含在基础数据中
  • • 本次消耗总计（加粗高亮）

5. 分支流程

A

返回线索列表

点击底部"返回线索列表"按钮，跳转至 静态线索池.html

B

使用智能触达

点击底部主按钮，跳转至智能触达 (ITA) 服务页面，进入触达流程

C

发送邮件

点击联系人卡片的"发送邮件"按钮，跳转至智能触达页面并预选该联系人

D

社媒链接

点击 LinkedIn/Twitter 链接，新标签页打开对应社媒主页

6. 异常流程

无联系人结果

联系人矩阵区域显示"未找到关键联系人"，账单摘要显示消耗 0 Credits

数据缺失

若某类原始证据数据缺失，对应卡片显示"暂无数据"状态

7. 交互效果

卡片悬停

联系人卡片悬停时边框颜色变化

链接悬停

邮箱/社媒链接悬停时显示下划线

入场动画

各区块依次淡入上滑（delay 递增）

限制规则

权限控制

当前版本系统暂无权限控制逻辑，所有登录用户均可查看线索详情。

数据只读

线索详情页为只读模式，不支持编辑联系人信息

线索状态

点击深度挖掘的线索，无论是否挖掘成功，都会进入线索池。线索池中没有线索评分字段。

AI 综合研判异常检测

AI 模型存在幻觉风险，可能导致综合研判结论与实际情况不符。本模块对 AI 综合研判输出进行专项检测。

  • • 输出为空检测：AI 返回空研判时触发，提示「AI 研判结果为空」
  • • 格式校验：研判 JSON 解析失败时触发，提示「AI 返回数据格式异常」
  • • 一致性校验：研判与原始数据矛盾时触发，提示「AI 研判仅供参考」
  • • 完整性校验：必要研判字段缺失时触发，提示「AI 研判不完整」

AI 综合研判异常处理策略

异常类型 处理方式 提示文案
输出为空 显示默认研判模板 「AI 研判结果为空」
格式错误 显示原始数据摘要 「AI 返回数据格式异常」
内容幻觉 标记仅供参考，提示用户核实 「AI 研判仅供参考，以实际数据为准」
字段缺失 使用默认值填充 「AI 研判不完整」
