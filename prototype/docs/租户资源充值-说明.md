> **状态**：N（初稿）
> **对应 PRD 节点**：P01

---

## 概述

租户资源充值页面用于管理员为入驻租户分配 Global Credits 和 BOT 引擎席位。实际充值操作使用现有服务商系统，本页面作为操作引导入口，配置和开通逻辑与独立应用一致。

---

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 多BOT工作台.html | 点击"购买更多算力"按钮 | `bot_id` |

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 多BOT工作台.html | 充值成功后返回 | `bot_id` |

## 功能逻辑

### 场景一：租户搜索与选择

**触发条件**：管理员进入充值页面，需指定目标租户。

**执行逻辑**：
1. 管理员输入租户名称或 Tenant ID 进行模糊搜索
2. 输入时实时显示匹配结果下拉列表
3. 选中后显示租户信息卡片（公司名称、当前余额、到期时间）
4. 支持清除重选

**输出**：已选定目标租户的展示卡片。

---

### 场景二：资源包选择

**触发条件**：管理员已选定目标租户。

**执行逻辑**：
1. 展示可选资源包列表：
   - LeadFinder 旗舰版基础包：3 个 BOT 席位 + 50,000 Global Credits
   - LeadFinder 算力加油包：10,000 Credits（无 BOT 席位）
2. 管理员点击选择资源包，被选中包显示高亮状态
3. 选中后展示费用汇总信息
4. 支持取消选择

**输出**：已选资源包的汇总信息（可进入下单确认流程）。

---

### 场景三：资源下发执行

**触发条件**：管理员确认选择后，点击确认按钮。

**执行逻辑**：
1. 前端调用内部资源下发接口
2. 后端校验租户信息有效性
3. 扣除对应 Credits，增加租户可用余额
4. 返回成功结果，前端刷新余额显示

**输出**：充值成功提示，余额实时更新。

## 页面状态

| 状态 | 触发条件 | UI 表现 |
|-----|---------|---------|
| 正常 | 页面加载完成 | 展示租户搜索区、资源包列表 |
| 搜索中 | 输入搜索词 | 下拉列表显示匹配结果 |
| 选择中 | 点击资源包 | 选中包高亮，汇总区更新 |

| 充值失败 | 接口返回错误 | Toast 错误提示 |

## 数据流

### 请求：资源下发

```
POST /internal/v1/tenant/resources/provision
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求体**：

```json
{
  "tenant_id": "tenant_001",
  "package_type": "flagship_basic",
  "credits_amount": 50000,
  "bot_seats": 3
}
```

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tenant_id": "tenant_001",
    "new_balance": 75000,
    "bot_seats_available": 3,
    "transaction_id": "TXN-202401-001"
  }
}
```

| code | 含义 |
|------|------|
| 200 | 下发成功 |
| 400 | 参数错误 |
| 403 | 无权限操作该租户 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 搜索范围 | 模糊搜索 | 支持按租户名称或 Tenant ID 搜索 |
| 选择限制 | 每次仅选一个资源包 | 选中新的自动取消旧的 |
| 余额更新 | 充值成功后 | 实时刷新页面余额显示 |
| 重复提交 | 充值中按钮点击 | 按钮禁用，防止重复扣费 |

---

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 搜索无结果 | API 返回空数组 | 显示空态 | 「未找到匹配的租户」 |
| 充值失败 | code 非 200 | 失败提示 | 「充值失败，请稍后重试」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
