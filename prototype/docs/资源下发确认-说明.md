## 概述

资源下发确认页面用于在资源正式发放前展示下发详情，供管理员确认。该页面作为操作示例参考，实际资源配置和开通逻辑与独立应用一致，参考访客雷达的开通和配置过程。

## 功能逻辑

### 场景一：查看资源下发详情

**触发条件**：从充值页面跳转进入。

**执行逻辑**：
1. 页面展示本次下发的资源详情：租户名称、Credits 数量、BOT 席位数量、生效时间
2. 展示下发状态进度（下发中 → 已确认 → 已生效）
3. 管理员确认信息无误后，点击"确认生效"按钮

**输出**：资源下发确认状态。

---

### 场景二：确认资源生效

**触发条件**：管理员确认资源信息无误。

**执行逻辑**：
1. 前端发送确认请求
2. 后端更新资源状态为"已生效"
3. 租户可用余额和席位实时更新

**输出**：资源生效提示，页面跳转回工作台。

---

### 场景三：取消下发

**触发条件**：管理员发现信息有误。

**执行逻辑**：
1. 点击"取消"按钮
2. 后端撤销本次下发记录，不扣除 Credits
3. 页面跳转回充值页面

**输出**：撤销成功提示。

## 页面状态

| 状态 | 触发条件 | UI 表现 |
|-----|---------|--------|
| 正常 | 数据完整返回 | 展示下发详情 |
| 加载中 | 请求发出后 | 骨架屏或 loading 动画 |
| 确认中 | 点击确认按钮 | 按钮 loading 状态 |
| 已生效 | 确认成功 | 成功提示 + 跳转 |
| 已取消 | 取消操作成功 | 撤销提示 + 跳转 |

## 数据流

### 请求：确认资源下发

```
POST /internal/v1/tenant/resources/confirm
```

**请求体**：

```json
{
  "transaction_id": "TXN-202401-001",
  "tenant_id": "tenant_001"
}
```

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "transaction_id": "TXN-202401-001",
    "status": "active",
    "activated_at": "2024-01-20T10:30:00Z"
  }
}
```

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 撤销操作 | 确认前取消 | 撤销下发，不扣除 Credits |
| 重复确认 | 重复点击确认按钮 | 按钮禁用，防止重复确认 |
| 超时返回 | 30 分钟无操作 | 提示返回工作台 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /internal/v1/tenant/resources/confirm | POST | 确认资源生效 | 点击确认按钮 |
| /internal/v1/tenant/resources/cancel | POST | 取消下发 | 点击取消按钮 |

---

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 资源已生效 | 状态校验 | 提示并跳转 | 「该资源已生效，无需重复确认」 |
| 资源不存在 | code 404 | 错误提示 | 「该下发记录不存在或已过期」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
