> **版本**：v1.0.0
> **状态**：N（初稿）
> **对应 PRD 节点**：全局
> **对应原型页面**：提示词语言管理.html
> **最后更新**：2026-04-10

---

## 概述

提示词语言管理模块为 AI 模型提供中英文双语提示词支持，解决三方接口调用时的传参语言问题。系统同时存储中英文两个版本的提示词，根据调用的模型类型自动选择对应语言版本；同时在调用国外 API 时自动将中文关键字翻译为英文，保证接口调用成功率。

## 功能逻辑

### 场景一：提示词双语化架构

**触发条件**：系统调用 AI 模型时。

**执行逻辑**：
1. 语言选择引擎根据模型类型判断适用语言
2. 英文优先模型（GPT-4、Claude 3.5、Gemini 1.5 等）：使用英文提示词
3. 中文优先模型（文心一言、通义千问、讯飞星火等）：使用中文提示词
4. 默认使用英文

**输出**：自动选择对应语言版本的提示词。

---

### 场景二：参数语言转换

**触发条件**：用户使用中文搜索需求，系统调用国外 API 时。

**执行逻辑**：
1. 用户输入中文搜索需求（如"欧洲登山装备品牌代理商"）
2. AI 策略规划师生成中文搜索关键字
3. 系统根据 API 类型路由：
   - Google Maps API：中文关键字翻译为英文后调用
   - 10times API：中文关键字翻译为英文后调用
   - 出海派 API：保持中文（国内 API）
   - Jina Reader API：无需翻译（URL 格式）

**输出**：每个 API 使用正确语言的关键字调用，结果 JSON 同时包含英文（API 调用用）和中文（前端展示用）两套关键字。

---

### 场景三：成本优化模型适配

**触发条件**：系统需选择 AI 模型处理任务时。

**执行逻辑**：
1. 简单任务（ETL 数据清洗、联系人排序）：使用 GPT-4o-mini 或 Claude 3 Haiku
2. 复杂推理任务（ICP 建模、深度分析）：使用 GPT-4o 或 Claude 3.5 Sonnet
3. 模型选择结合语言偏好（见场景一）

**输出**：根据任务复杂度自动选择最合适的模型。

```json
{
  "prompt_id": "P07_ETL_EXTRACTOR",
  "prompt_name": "ETL数据清洗抽取引擎",
  "versions": {
    "zh": {
      "role": "你是一个严谨的企业数据清洗与结构化抽取引擎...",
      "task": "请阅读以下文本内容，严格提取其中的客观业务事实...",
      "strict_rules": ["零捏造原则", "空值处理", "去营销化"],
      "output_schema": {}
    },
    "en": {
      "role": "You are a rigorous enterprise data cleaning...",
      "task": "Read the following text content, strictly extract...",
      "strict_rules": ["Zero Fabrication Principle", "Empty Value Handling", "De-marketing"],
      "output_schema": {}
    }
  },
  "applicable_models": ["gpt-4o-mini", "gemini-1.5-flash", "claude-3-haiku"]
}
```

### 语言选择逻辑

```javascript
function selectPromptLanguage(modelType: string): "zh" | "en" {
  const englishPreferredModels = [
    "gpt-4", "gpt-4o", "gpt-4o-mini",
    "claude-3-opus", "claude-3-sonnet", "claude-3-5-sonnet",
    "claude-3-haiku", "claude-3-5-haiku",
    "gemini-1.5-pro", "gemini-1.5-flash"
  ];
  const chinesePreferredModels = [
    "ernie-4", "ernie-3.5", "ernie-bot",
    "qwen-max", "qwen-plus", "qwen-turbo",
    "spark-4.0", "spark-3.5"
  ];
  if (englishPreferredModels.some(m => modelType.includes(m))) return "en";
  if (chinesePreferredModels.some(m => modelType.includes(m))) return "zh";
  return "en"; // 默认英文
}
```

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 提示词版本管理 | 每次更新提示词 | 必须同时更新中英文两个版本，记录版本号 |
| 翻译一致性 | 中英文提示词需同步更新 | 语义保持一致，避免翻译差异导致模型输出不一致 |
| 参数验证 | 调用国外 API 前 | 必须验证搜索关键字已正确翻译为英文 |
| 日志记录 | 每次 API 调用 | 记录语言转换情况，便于问题排查和优化 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/prompts | GET | 获取提示词配置列表 | 页面加载 |
| /api/v1/prompts/{id} | GET | 获取单个提示词详情 | 点击配置项 |
| /api/v1/prompts/{id} | PUT | 更新提示词配置 | 保存修改 |
| /api/v1/prompts/{id}/switch-version | POST | 切换提示词语言版本 | 用户点击版本切换 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 提示词配置加载失败 | HTTP 错误 | 错误提示 + 重试按钮 | 「提示词配置加载失败，请刷新页面重试」 |
| 版本加载失败 | 请求返回错误 | 失败提示 | 「无法加载该版本配置，请重试」 |
| 网络连接异常 | 请求失败 | 全局错误处理 | 「网络连接异常，请检查网络后重试」 |
| 配置加载超时 | 请求超时 | 超时提示 | 「配置加载超时，请稍后重试」 |
| 服务暂时不可用 | HTTP 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| 保存失败 | PUT 请求失败 | 失败提示 | 「配置保存失败，请稍后重试」 |
| 提示词格式不合法 | 格式校验失败 | 格式错误提示 | 「提示词格式不合法，请检查后重试」 |
| 内容超出长度限制 | 长度校验失败 | 长度超限提示 | 「提示词内容超出长度限制」 |
| 当前模型不支持该提示词 | 版本不支持 | 警告提示 | 「当前模型 {model_name} 不支持该提示词类型」 |
| 检测到语言不匹配 | 语言检测异常 | 警告提示 | 「检测到语言不匹配，建议切换至 {language} 版本」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 提示词 ID | prompt_id | 系统生成，如 P07_ETL_EXTRACTOR |
| 提示词名称 | prompt_name | 配置时定义，如 ETL数据清洗抽取引擎 |
| 中文角色设定 | versions.zh.role | 配置时填写 |
| 英文角色设定 | versions.en.role | 配置时填写 |

| 英文任务描述 | versions.en.task | 配置时填写 |
| 中文规则集 | versions.zh.strict_rules | 配置时填写，数组 |
| 英文规则集 | versions.en.strict_rules | 配置时填写，数组 |
| 输出结构 | output_schema | JSON Schema 定义 |
| 适用模型列表 | applicable_models | 模型 ID 数组 |
| 当前选择语言 | selected_version | 自动判断或用户手动切换 |
| 模型匹配语言 | matched_language | en（英文优先模型）或 zh（中文优先模型） |
