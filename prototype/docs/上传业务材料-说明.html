P04 节点

上传业务材料页面

对应业务流程节点：P04（为当前 BOT 自助上传特定企业/产品业务材料）

页面概述

本页面用于接收用户本地非结构化业务凭证，作为 AI 画像分析的物理语料。用户可上传产品目录、公司介绍、服务介绍等材料。此流程与 P05/P06 互斥，防止实体信息冲突。

不可见功能逻辑

1. 文件上传接口

POST /api/v1/materials/upload

Request Headers:
  Authorization: Bearer {tenant_token}
  Content-Type: multipart/form-data

Request Body:
  files: File[]           // 文件数组，最多3个

Response:
{
  "code": 200,
  "data": {
    "upload_id": "UPLOAD-2024-001",
    "files": [
      {
        "file_id": "FILE-001",
        "file_name": "产品画册.pdf",
        "file_size": 5242880,
        "file_type": "application/pdf",
        "storage_url": "s3://materials/tenant-001/FILE-001.pdf",
        "upload_time": "2024-01-15T14:30:00Z"
      }
    ],
    "total_size": 15728640,
    "status": "uploaded"
  }
}

服务端处理逻辑：

  • • 验证文件类型和大小限制
  • • 生成唯一 file_id 并存储到 AWS S3 / 阿里云 OSS
  • • 记录文件元数据到数据库
  • • 返回存储 URL 供后续处理使用

2. 异常流程处理

异常场景 错误码 前端处理
文件格式不支持 400 弹窗提示"不支持的文件格式"
文件超过 5MB 400 弹窗提示"文件过大，最大支持 5MB"
超过 3 个文件 400 弹窗提示"最多只能上传 3 个文件"
URL 格式错误 400 弹窗提示"请输入有效的网址"
上传失败 500 显示错误提示，支持重试

3. AI 解析异常检测

AI 模型存在幻觉风险，可能导致材料解析结果与实际内容不符。本模块对 AI 解析输出进行专项检测。

  • • 输出为空检测：AI 解析返回空结果时触发，提示「AI 解析结果为空，请重试或检查材料内容」
  • • 格式校验：JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」
  • • 一致性校验：解析结果与原文矛盾时触发，提示「AI 解析结果存在异常，建议重新上传」
  • • 完整性校验：必要业务字段缺失时触发，提示「AI 解析不完整，部分数据缺失」

AI 解析异常处理策略

异常类型 处理方式 提示文案
输出为空 自动重试 1 次，失败后提示用户 「AI 解析结果为空，请重试」
格式错误 自动重试 1 次，失败后使用默认值 「AI 返回数据格式异常」
内容幻觉 标记异常，提示用户重新上传 「AI 解析结果存在异常，建议重新上传」
字段缺失 使用默认值填充，标记数据不完整 「AI 解析不完整，部分数据缺失」
