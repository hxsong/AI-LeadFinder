功能说明文档
P12.1 节点

搜客记录页面

对应业务流程节点：P12.1（查看历史搜客任务及结果）

页面概述

搜客记录页面用于展示用户的历史搜客任务及结果。用户可以在此页面查看所有搜客任务的执行状态、消耗的 Credits、发现的企业数量等信息。支持按状态和时间筛选，点击已完成任务可跳转至搜客结果列表页面查看详细结果。

可见功能逻辑

1. UI 组件

1

统计卡片

顶部三个统计卡片：总搜客次数、累计消耗 Credits、发现企业数量

2

筛选栏

状态筛选（全部/已完成/进行中/失败）+ 时间筛选（全部时间/近7天/近30天/近90天）

3

记录列表

按时间倒序展示搜客记录，每条记录包含：搜索时间、任务名称、搜索需求摘要、目标数量、消耗 Credits、数据源、状态标签

4

分页组件

底部分页导航，支持翻页查看更多记录

5

新建搜客按钮

页面右上角「新建搜客」按钮，点击跳转至搜客需求输入页面

2. 任务状态展示

状态 图标 标签样式 交互行为
已完成 绿色勾选图标 已完成 可点击跳转至搜客结果列表
进行中 蓝色旋转加载图标 进行中 显示进度条，不可点击
失败 红色叉号图标 失败 显示错误原因 + 重试按钮

3. 交互效果

  • • 筛选切换：点击状态/时间筛选按钮，列表实时更新
  • • 记录点击：已完成任务点击跳转至搜客结果列表页面
  • • 重试按钮：失败任务点击重试，重新发起搜客请求
  • • 页面进入动画：各组件依次淡入上滑（fadeInUp）
  • • 悬停效果：记录卡片悬停时显示阴影效果

2. 重试接口

POST /api/v1/search/retry

Request Headers:
  Authorization: Bearer {tenant_token}
  Content-Type: application/json

Request Body:
{
  "tenant_id": "tenant_001",
  "task_id": "task_20240115_001"
}

Response (Success):
{
  "code": 200,
  "message": "重试任务已创建",
  "data": {
    "new_task_id": "task_20240120_002",
    "status": "pending"
  }
}

3. 数据存储

搜索任务表

记录所有搜客任务的元数据、状态、结果统计

Credits 流水表

记录每次搜客的 Credits 消耗明细

结果缓存

Redis 缓存搜客结果，有效期 7 天

限制规则

数据保留

  • • 搜客记录保留 90 天
  • • 搜客结果缓存 7 天
  • • 超期数据自动归档

权限限制

  • • 仅可查看本租户的搜客记录
  • • 重试功能需 Credits 余额 > 0

余额规则：余额可为负，≤ 0 时禁止消耗操作；重试任务执行中不扣费，完成后按实际消耗结算，允许余额为负。

分页限制

  • • 每页最多显示 10 条记录
  • • 最大翻页深度 100 页

重试限制

  • • 仅失败任务可重试
  • • 单个任务最多重试 3 次
