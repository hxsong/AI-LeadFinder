P13-P15 节点

搜客策略预览页面

对应业务流程节点：P13-P15（AI 智能生成搜客策略 → 展示搜客策略和预估消耗费用）

页面概述

本页面展示 AI 智能生成的搜客策略，包括策略解读、瀑布流执行计划、费用预估明细。策略生成基于用户输入的搜索需求和目标客户数量，结合三方 API 配置智能规划最优执行方案。用户确认后系统冻结预估 Credits 并开始执行搜客任务。

详细处理流程见「不可见功能逻辑」模块。

可见功能逻辑

1. UI 组件

1

AI 策略解读卡片

展示 AI 生成的策略逻辑说明，向用户解释这套策略的执行思路

2

瀑布流执行计划

动态展示执行步骤（1-4步），根据用户需求智能选择需要调用的接口，每步显示 API 来源和参数配置

3

目标对齐模块

展示目标企业数量、预计候选企业、最终输出企业的数量对比。预计候选企业 = 各接口预估获取数量之和

4

费用预估明细

展示各 API 服务的预估费用和总消耗，隐藏单价和计算细节

2. 交互效果

  • • 页面进入动画：各组件依次淡入上滑（fadeInUp），带延迟效果
  • • 返回按钮：点击返回 搜客需求输入.html 修改策略
  • • 确认按钮：点击后显示 "冻结 Credits..." 加载状态，按钮禁用
  • • 成功状态徽章：顶部显示 "AI 策略规划完成" 绿色徽章

3. 分支流程

Credits 余额不足：若用户 Credits 余额 < 预估消耗，显示充值提示，禁用确认按钮

API 服务不可用：若某第三方 API 服务离线，策略中对应步骤标记为"服务异常"，提示用户稍后重试

不可见功能逻辑

1. 参数校验与任务创建

触发条件：用户提交 user_query（需求内容）+ target_count（目标客户最大数量）

1

参数校验

验证 tenant_id 有效性、user_query 长度（≤500）、target_count 范围（20-200）

2

创建待确认搜索任务

语言选择说明：根据使用模型自动选择对应语言版本。国外模型(GPT-4, Claude)使用英文提示词，国内模型使用中文提示词。特别注意：调用国外 API 时，search_keywords 必须输出英文版本。

// AI 策略规划师 System Prompt

# Role
你是一个智能 API 调度网关与搜客策略规划师。

# Task
根据用户的搜索需求和目标客户数量，制定一套瀑布流式的数据抓取策略。
你必须决定：先调用哪个 API 铺网？再调用哪个 API 过滤？生成搜索关键字列表和调用顺序。

# Input
{
  "user_query": "我想找欧洲的登山装备的品牌代理商",
  "target_count": 20
}

# Available APIs (可用数据源)

## 1. GOOGLE_MAPS_API
**用途**：按地理位置和实体类型寻找企业
**核心参数**：
- textQuery: 搜索文本（需翻译为英文）
- includedType: 地点类型筛选
- locationRestriction: 区域限制
- pageSize: 每页数量（最大20）
- languageCode: 返回语言
- **target_count**: 目标企业数量（必传）
**调用限制**：单次最多返回 20 条，需根据 target_count 计算调用次数

## 2. CHUHAIPAI_API
**用途**：查询海关进出口记录，获取有采购意向的企业
**核心参数**：
- searchField: 搜索字段（product/shipname/consname）
- modifier: 匹配方式（contains/exactly/any）
- searchTerm: 搜索关键词
- hs_codes: HS 编码数组
- country: 国家代码
- date_range: 日期范围
- **target_count**: 目标企业数量（必传）
**调用限制**：批量查询一次可处理 100 个域名

## 3. 10TIMES_EXPO_API
**用途**：获取展会参展商名单
**核心参数**：
- location: 地理位置
- query: 活动名称搜索关键词（需翻译为英文）
- category: 行业类别 ID
- eventType: 活动类型
- date_range: 日期范围
- **target_count**: 目标企业数量（必传）
**调用限制**：批量查询一次可处理最多 1000 个活动

## 4. JINA_READER_API
**用途**：解析企业官网内容
**核心参数**：
- url: 要解析的 URL
- extract_mode: 提取模式
- **target_count**: 目标企业数量（必传）
**调用限制**：单次请求对应 1 个 URL，需根据 target_count 计算调用次数

# Execution Rules
1. 目标寻客数量上限设为 {target_count} 家
2. **动态接口选择**：
   - 根据用户需求智能选择需要调用的接口，不强制调用所有接口
   - 示例：用户只想要海关数据 → 仅调用 CHUHAIPAI_API
   - 示例：用户需要欧洲地区企业 → 调用 GOOGLE_MAPS_API + 其他验证接口
3. **起手式判断**：
   - 意图包含明确区域 → 优先用 GOOGLE_MAPS_API 起手
   - 意图强调采购某类产品 → 优先用 CHUHAIPAI_API 起手
   - 意图强调展会参展商 → 优先用 10TIMES_EXPO_API 起手
4. **target_count 传递**：
   - 每个被调用的接口都必须传入 target_count 参数
   - 确保每个接口都能获取到目标数量的数据
   - 对于有返回数量限制的接口，需要计算调用次数

# 参数语言转换规则 (重要!)
- GOOGLE_MAPS_API: textQuery 必须为英文
- CHUHAIPAI_API: searchTerm 可为中文或英文
- 10TIMES_EXPO_API: query 必须为英文
- JINA_READER_API: 不涉及搜索关键字

3. 脚本生成与费用预估

核心逻辑：根据 AI 策略规划师输出的 waterfall_plan，自动生成可执行的 API 调用脚本，并根据调用次数预估费用。

脚本生成逻辑：

1

解析 waterfall_plan

遍历每个 step，提取 api_source、parameters

2

生成 API 调用脚本

根据 API 类型，生成对应的 curl 命令或 SDK 调用代码

3

计算调用次数

根据 target_count 和 API 返回限制，计算每个 API 需要调用的次数

4

预估费用

调用次数 × API 单价 × (1 + 30% 动态溢价)

AI 策略规划异常检测

AI 模型存在幻觉风险，可能导致生成的搜客策略与用户需求不符或逻辑矛盾。本模块对 AI 策略输出进行专项检测。

  • • 输出为空检测：AI 返回空策略时触发，提示「AI 策略生成结果为空，请重试」
  • • 格式校验：策略 JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」
  • • 一致性校验：策略与用户需求矛盾时触发，提示「AI 策略与需求不符，建议重新生成」
  • • 完整性校验：必要策略字段缺失时触发，提示「AI 策略生成不完整」

AI 策略异常处理策略

异常类型 处理方式 提示文案
输出为空 自动重试 1 次，失败后提示用户 「AI 策略为空，请重试」
格式错误 自动重试 1 次，失败后使用默认策略 「AI 返回数据格式异常」
内容幻觉 标记异常，提示用户重新生成 「AI 策略存在异常，建议重新生成」
字段缺失 使用默认值填充，标记数据不完整 「AI 策略不完整，已使用默认值」
