# 动态线索池

> **版本**：v1.0.0
> **状态**：N（初稿）
> **对应 PRD 节点**：P39-P46
> **对应原型页面**：动态线索池.html
> **最后更新**：2026-04-10

---

## 概述

动态线索池是管理第一方动态流量的核心工作台，承载历史线索的回溯与新线索的任务追踪。页面展示两类内容：进行中的挖掘任务（带动画状态）和历史挖掘记录。用户在访客行为详情页点击确认挖掘后，系统立即在此页面创建一条 Resolving 状态的占位记录，后台异步执行 IP 解析和联系人挖掘。

---

## 页面入口与出口

### 入口

| 来源 | 触发条件 | 携带参数 |

## 功能逻辑

### 场景一：查看进行中的挖掘任务

**触发条件**：存在 Resolving 状态的线索挖掘任务。

**执行逻辑**：
1. 页面展示进行中任务卡片，带脉冲动画和高亮边框
2. 卡片显示：IP 地址（脱敏）、访客基础信息（地区、设备、来源）、"挖掘解析中"状态标签
3. 进度动画展示：IP 企业解析状态 / 企业信息验证状态 / 联系人挖掘状态
4. 卡片边框高亮 + 背景渐变 + 脉冲动画

**输出**：进行中任务卡片，实时反映任务进度。

---

### 场景二：查看历史挖掘记录 - 成功线索

**触发条件**：挖掘任务已完成（状态为 Completed）。

**执行逻辑**：
1. 展示已完成线索卡片：企业名称、企业域名（超链接）、所在地、行业类型
2. 展示卡片核心信息：联系人数量、Credits 消耗、入库时间
3. 点击卡片跳转至动态线索详情.html

**输出**：历史线索卡片列表，按入库时间倒序排列。

---

### 场景三：查看历史挖掘记录 - ISP 宽带线索

**触发条件**：IP 解析结果为公共宽带（ISP），无法精准定位企业。

**执行逻辑**：
1. 企业名称显示"ISP 匿名访客"
2. 显示"ISP 宽带"警告标签和 ISP 运营商名称
3. 保留行为数据和地理位置信息
4. 联系人矩阵显示"未找到联系人"

**输出**：ISP 宽带线索卡片，特殊标识展示。

---

### 场景四：导出线索列表

**触发条件**：用户点击"导出线索列表"按钮。

**执行逻辑**：
1. 前端发送 `GET /api/v1/leads/dynamic/export` 请求
2. 后端返回 Excel 下载链接（有效期 24 小时）
3. 浏览器自动下载或前端触发下载

**输出**：线索列表 Excel 文件。

## 数据流

### 请求一：创建动态线索占位记录

```
POST /api/v1/dynamic-leads
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求体**：

```json
{
  "visitor_log_id": "VIS-99482X",
  "base_behavior_score": 95,
  "ip_address": "67.43.156.99",
  "geo_location": "Florida, USA",
  "device_info": "Desktop / Windows",
  "traffic_source": "Google 搜索"
}
```

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "dynamic_lead_id": "DYN-202602-001",
    "status": "Resolving",
    "created_at": "2024-02-27T10:05:00Z"
  }
}
```

### 请求二：获取动态线索列表

```
GET /api/v1/dynamic-leads?status=Resolving,Completed,ISP&grade=S,A,B,C&sort=score_desc
```

**请求头**：`Authorization: Bearer {tenant_token}`

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | 状态筛选（多选），逗号分隔：Resolving/Completed/ISP |
| grade | string | 否 | 等级筛选（多选），逗号分隔：S/A/B/C |
| sort | string | 否 | 排序方式：score_desc / score_asc / created_desc / created_asc |
| page | number | 否 | 页码，默认 1 |
| limit | number | 否 | 每页数量，默认 20 |

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 8,
    "items": [
      {
        "dynamic_lead_id": "DYN-202602-001",

        "created_at": "2024-02-27T10:05:00Z"
      },

        "dynamic_lead_id": "DYN-202602-002",
        "status": "Completed",
        "enterprise_info": { "name": "Orlando Mega Stage AV", "domain": "orlando-av.com", "industry": "Events & AV Integration" },
```

### 请求三：导出线索列表

```
GET /api/v1/leads/dynamic/export
```

**Query 参数**：`status=all | resolving | completed | isp`

**响应体**：

```json
{
  "code": 200,
  "data": {
    "download_url": "https://cdn.example.com/exports/dynamic_leads_20240120.xlsx",
    "expires_at": "2024-01-21T10:30:00Z"
  }
}
```

| code | 含义 |
|------|------|
| 200 | 操作成功 |
| 400 | 参数错误 |
| 404 | 线索不存在 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 任务异步执行 | 用户关闭页面 | 挖掘任务在后台继续执行，不中断 |
| 接口超时机制 | 单次接口调用超时 1 分钟 | 跳过本次调用继续执行，不阻塞整体任务 |
| 实时刷新 | 进行中任务 | 需实时轮询或 WebSocket 推送状态更新 |
| 数据隔离 | 动态线索池 vs 静态线索池 | 两类数据独立管理，互不干扰 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/dynamic-leads | POST | 创建动态线索占位记录 | 深度挖掘任务创建时 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| IP 地址解析失败 | IP 解析服务返回失败 | ISP 线索卡片展示 | 「IP 地址解析失败，无法识别企业信息」 |
| 企业信息验证未通过 | 验证服务返回失败 | 失败提示 | 「企业信息验证未通过，线索质量不达标」 |
| Credits 余额不足 | 联系人挖掘时余额不足 | 任务中断，提示充值 | 「Credits 余额不足，无法完成联系人挖掘」 |
| 挖掘任务意外中断 | 任务状态异常 | 自动重试尝试恢复 | 「挖掘任务意外中断，正在尝试恢复」 |
| IP 无法解析到企业 | IP 解析服务无结果 | ISP 线索展示 | 「该 IP 无法解析到企业」 |
| 解析服务不可用 | IP 解析服务异常 | 任务暂停，提示 | 「IP 解析服务暂时不可用，任务将自动重试」 |
| 网络请求超时 | 单次 API 调用超时 | 自动重试 | 「网络请求超时，任务将自动重试，请稍候」 |
| 无数据可导出 | 筛选条件下无数据 | 导出失败提示 | 「当前筛选条件下无数据可导出，请调整筛选条件」 |
| 导出任务创建失败 | 后端异常 | 导出失败提示 | 「导出任务创建失败，请稍后重试」 |
| 收藏列表达上限 | 收藏数超过系统限制 | 收藏失败提示 | 「收藏列表已达上限（{limit} 条），请清理后重试」 |
| 实时连接中断 | WebSocket 断开 | 提示重连 | 「实时连接已断开，正在尝试重新连接」 |
| 数据已过期 | 缓存过期 | 提示刷新 | 「数据已过期，请刷新页面获取最新数据」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 动态线索 ID | dynamic_lead_id | 系统生成，UUID |
| 线索状态 | status | Resolving（进行中）/ Completed（已完成）/ ISP（宽带匿名） |
| 基础行为评分 | base_behavior_score | 打分引擎计算，0-100 |
| IP 地址（脱敏） | ip_address | GeoIP 脱敏处理 |
| 地理位置 | geo_location | GeoIP 定位数据 |
| 设备信息 | device_info | 探针 JS 采集 UA 解析 |
| 流量来源 | traffic_source | 流量归因数据 |
| IP 解析进度 | progress.ip_resolution | pending / in_progress / completed / failed |
| 企业验证进度 | progress.enterprise_verification | pending / in_progress / completed / failed |
| 联系人挖掘进度 | progress.contact_mining | pending / in_progress / completed / failed |
| 企业名称 | enterprise_info.name | IRS IP 解析服务返回 |
| 企业域名 | enterprise_info.domain | IRS IP 解析服务返回 |
| 行业类型 | enterprise_info.industry | IRS IP 解析服务返回 |
| 最终动态评分 | final_dynamic_score | 行为分 + ICP 匹配分综合计算 |
| 线索等级 | lead_grade | S/A/B/C，由评分映射 |
| 联系人数量 | contacts_count | Apollo.io 联系人挖掘结果 |
| 联系人预览 | contacts_preview | 前 1-3 个联系人简要信息 |
| Credits 消耗 | credits_consumed | Apollo.io Credits 实际消耗 |
| ISP 运营商名称 | isp_name | IRS IP 解析服务返回宽带 ISP 名称 |
| 入库时间 | created_at | ISO 8601 时间戳 |
