功能说明文档
P39-P46 节点

动态线索池

对应业务流程节点：P39-P46（展示动态线索分析报告，承载 P39 触发的占位创建到 P45 状态更新的全过程展示）

页面概述

动态线索池是客户管理第一方动态流量的核心工作台，承载了历史线索的回溯与新线索的任务追踪。页面展示两类内容：进行中的挖掘任务（带动画状态）和历史挖掘记录。用户在 P38 点击确认挖掘后，系统立即在此页面创建一条 Resolving 状态的占位记录，后台异步执行 IP 解析和联系人挖掘。

可见功能逻辑

1. 进行中的任务卡片

状态展示

  • • IP 地址脱敏显示
  • • "挖掘解析中"状态标签
  • • 访客基础信息（地区、设备、来源）

进度动画

  • • IP 企业解析状态（完成/进行中）
  • • 企业信息验证状态
  • • 联系人挖掘状态
  • • 进度条动画（渐变色流动效果）

动画效果

卡片边框高亮 + 脉冲动画 + 进度条流动效果，提示用户任务正在执行中

2. 历史挖掘记录 - 成功高价值线索

企业信息

  • • 企业名称
  • • 企业域名（超链接）
  • • 所在地
  • • 行业类型

卡片核心信息

  • • 企业名称 / IP 地址
  • • 企业简介 / 访问信息
  • • 联系人数量 / Credits 消耗
  • • 入库时间

3. 历史挖掘记录 - ISP 宽带线索

当 IP 解析结果为公共宽带（ISP）时，企业名称显示"ISP 匿名访客"，但保留行为数据价值

ISP 标识

  • • 企业名称显示"ISP 匿名访客"
  • • "ISP 宽带"警告标签
  • • ISP 运营商名称（如 Comcast）

保留数据

  • • 地理位置信息

3. 状态更新机制

Resolving

挖掘解析中，显示进度动画

Completed

挖掘完成，展示完整报告

ISP

宽带解析，未找到企业

限制规则

权限控制

当前版本系统暂无权限控制逻辑，所有登录用户均可查看动态线索池。

异步任务执行规则

  • • 页面关闭继续执行：挖掘任务在后台异步执行，用户关闭页面不影响任务进度
  • • 接口超时机制：每次接口调用超时时间 1 分钟，超时后跳过本次调用继续执行
  • • 任务不阻塞：确保任务最终完成，不会因单个接口失败而阻塞
  • • 预估时长：按接口调用次数预估，每次调用约 1 分钟

实时刷新

进行中的任务需实时轮询或 WebSocket 推送状态更新

数据隔离

动态线索池与静态线索池数据隔离，独立管理

导出线索列表 API

导出功能说明

GET /api/v1/leads/dynamic/export

Query Parameters:
  status: "all" | "resolving" | "completed" | "isp"

Response:
{
  "code": 200,
  "data": {
    "download_url": "https://cdn.example.com/exports/dynamic_leads_20240120.xlsx",
    "expires_at": "2024-01-21T10:30:00Z"
  }
}

导出功能说明

  • • 导出格式：Excel（.xlsx）
  • • 版本限制：无，所有版本均可使用
  • • 导出字段：
    • - 企业名称
    • - 官网
    • - 行业
    • - 线索等级（S/A/B/C）
    • - 任务状态（挖掘解析中/挖掘完成/ISP宽带）
    • - 基础行为评分
    • - 最终动态评分
    • - ISP 名称（仅 ISP 任务）
    • - 联系人数量
    • - 主要联系人姓名
    • - 主要联系人职位
    • - Credits 消耗
    • - 创建时间
