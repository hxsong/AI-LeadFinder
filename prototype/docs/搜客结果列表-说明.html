功能说明文档
P20-P21 节点

搜客结果列表页面

对应业务流程节点：P20-P21（返回企业数据分析结果列表 → 展示企业数据分析结果列表）

页面概述

本页面是静态数据获客流程的结果展示页面，展示经过 AI 分析和打分的企业线索列表。用户可以查看匹配度评分、AI 推荐理由、企业徽章等信息，并选择目标企业进行联系人挖掘。

可见功能逻辑

1. UI 组件

1

成功横幅

显示任务完成状态、实际消耗 Credits、退回差额、发现企业数量

2

筛选排序栏

排序选项（匹配度高到低/低到高/企业名称A-Z）、筛选标签（全部/有海关记录/有参展记录）、导出列表按钮

导出功能说明

  • • 导出格式：Excel（.xlsx）
  • • 版本限制：无，所有版本均可使用
  • • 导出字段：企业名称、官网、匹配度评分、AI推荐理由、徽章标签、企业简介、员工规模、联系方式、海关记录、展会记录、地理位置
3

企业结果卡片

匹配度环形评分图、企业名称和官网链接、徽章标签、企业简介、AI 推荐理由、员工规模、挖掘联系人按钮

4

分页组件

显示总企业数量，支持页码跳转

5

原始数据查看弹窗

展示海关数据、展会数据、地图数据、网页数据四类原始抓取信息

6

挖掘企业联系人按钮

点击后弹出联系人挖掘确认弹窗，显示计费规则和异步执行提示，确认后跳转至静态线索池页面

7

联系人挖掘确认弹窗

显示计费规则（验证邮箱 2 Credits/个，LinkedIn +1 Credit，Twitter/FB +1 Credit/个）、异步执行提示、当前账户余额

8

重新搜索按钮

点击返回搜客需求输入页面，当前搜客结果仍可在「搜客记录」中查看

2. 交互效果

  • • 卡片悬停：阴影加深（hover:shadow-md），鼠标变为手型
  • • 评分环形图：根据分数显示不同颜色（≥80 绿色，60-79 蓝色，<60 灰色）
  • • 筛选标签切换：选中态蓝色背景，未选中灰色背景
  • • 挖掘按钮：点击打开弹窗，弹窗内显示加载状态和成功状态
  • • 页面进入动画：卡片依次淡入上滑（fadeInUp）

3. 徽章类型说明

海关记录

企业有海关进出口记录，命中出海派数据库

参展记录

企业有展会参展记录，命中 10times 数据库

地理位置

企业地理位置符合搜索条件（如加州）

4. 原始数据弹窗内容说明

海关数据
  • • HS Code：海关编码
  • • 产品描述：进口产品名称
  • • 进口次数：近12个月进口次数
  • • 最近日期：最后一次进口时间
  • • 供应商国家：进口来源国
  • • 数据来源：出海派
展会数据
  • • 展会名称：参展活动名称
  • • 展会日期：参展时间
  • • 展会地点：举办城市
  • • 参与类型：参展商/观众
  • • 数据来源：10times
地图数据
  • • 企业名称：地图注册名称
  • • 详细地址：完整地址信息
  • • 用户评分：Google评分及评价数
  • • 营业状态：营业中/已关闭
  • • 坐标位置：经纬度信息
  • • 数据来源：Google Maps
网页数据
  • • 抓取时间：内容抓取日期
  • • 内容长度：抓取字符数（无限制)
  • • 正文内容:官网首页及关键页面摘要
  • • 数据来源:Jina Reader

不可见功能逻辑

1. P20：AI 推理判词生成 - AI 模型处理

// AI 推理判词生成 Prompt

# Role
你是一位拥有 10 年经验的 B2B 商业分析师，擅长从企业数据中提炼关键洞察。

# Task
根据企业的匹配度得分、海关记录、展会记录、地理位置等信息，生成一段简洁有力的 AI 推荐理由（不超过 50 字）。

# Input
- 企业名称：{enterprise_name}
- 综合得分：{total_score}
- 维度得分：{score_breakdown}
- 海关记录：{has_customs_record}
- 展会记录：{has_exhibitor_record}
- 地理位置：{region}
- 业务描述：{description}
- 数据完整度：{data_completeness}

# Output Rules
1. 必须包含至少一个具体的数据点（如海关记录、展会名称）
2. 必须突出企业与客户 ICP 的匹配点
3. 语言简洁有力，不超过 50 字

# Example Output
"欧洲登山装备渠道商，展会曝光积极"

推理模型

gpt-4o-mini（快速推理）

输出结果

简洁的 AI 推荐理由文本（≤50 字）

6. 服务端处理逻辑

1

Credits 结算（P20）

按实际 API 成功调用次数重新核算，从冻结 Credits 中扣除实际消耗，退还差额

2

结果排序与筛选

支持按匹配度排序、按徽章筛选，分页返回结果

3

原始数据查询

支持按企业 ID 查询海关、展会、地图、网页四类原始抓取数据

4

联系人挖掘任务创建（P22）

创建异步挖掘任务，返回任务 ID，用户跳转至静态线索池查看进度

限制规则

联系人挖掘限制

  • • 单次挖掘消耗 15 Credits
  • • 单企业最多挖掘 5 次
  • • 联系人数据有效期 90 天

余额规则：余额可为负，≤ 0 时禁止消耗操作；挖掘执行中不扣费，完成后按实际消耗结算，允许余额为负。

数据访问限制

  • • 搜索结果保留 90 天
  • • 导出功能无版本限制

分页限制

  • • 每页最大 20 条记录
  • • 总结果最大 200 条

第三方服务依赖

  • • 联系人挖掘依赖 ECS 服务
  • • 服务不可用时显示提示

AI 推荐理由异常检测

AI 模型存在幻觉风险，可能导致推荐理由与企业信息不符。本模块对 AI 推荐理由输出进行专项检测。

  • • 输出为空检测：AI 返回空推荐理由时触发，显示「暂无推荐理由」
  • • 格式校验：推荐理由格式异常时触发，显示原始数据摘要
  • • 一致性校验：推荐理由与企业信息矛盾时触发，提示「推荐理由仅供参考」
  • • 长度校验：推荐理由超过 500 字时截断，显示「查看更多」展开

AI 推荐理由异常处理策略

异常类型 处理方式 提示文案
输出为空 显示默认推荐理由模板 「暂无推荐理由」
格式错误 显示原始数据摘要作为替代 「推荐理由生成异常」
内容幻觉 标记数据仅供参考，提示用户核实 「推荐理由仅供参考」
长度超限 截断显示，提供展开功能 「查看更多」
