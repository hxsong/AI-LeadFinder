# 独立站授权

> **版本**：v1.0.0
> **状态**：N（初稿）
> **对应 PRD 节点**：P06-P07
> **对应原型页面**：独立站授权.html
> **最后更新**：2026-04-10

---

## 概述

独立站授权页面利用企业已有的建站系统，通过 API 毫秒级获取高纯度、高结构化的客户商业意图数据。用户授权后，系统从独立站 API 自动拉取企业介绍、主营品类、SEO 关键词等结构化数据，无需手动上传文件。

## 功能逻辑

### 场景一：站点列表展示与选择

**触发条件**：用户进入独立站授权页面。

**执行逻辑**：
1. 前端调用接口获取当前租户下已绑定的站点列表
2. 每个站点卡片显示：站点名称、站点域名、状态标签（已上线/未上线）
3. 用户点击站点卡片，选中后显示橙色边框 + 勾选图标
4. 展开数据同步预览区域，显示该站点的企业信息摘要
5. 选择限制：每次仅可选 1 个站点作为核心主站

**输出**：已选站点的数据预览。

---

### 场景二：数据同步预览

**触发条件**：用户选中某个站点后。

**执行逻辑**：
1. 前端自动调用数据同步接口，实时获取站点数据
2. 摘要卡片展示：公司简介、成立时间/规模、目标市场、资质证书、主营品类（一/二级分类标签）、SEO 关键词
3. 同步状态指示：正在同步... → 同步完成

**输出**：站点数据预览摘要。

---

### 场景三：确认授权

**触发条件**：用户确认站点选择，点击"确认授权"按钮。

**执行逻辑**：
1. 前端调用建站系统内部只读 API 网关
2. 毫秒级获取高纯度、高结构化的客户商业意图数据
3. 数据包括：企业介绍（含目标市场、资质证书）、主营品类、SEO 关键词
4. 无需用户手动输入，自动完成数据采集
5. 后端存储数据，触发 AI 分析任务
6. 跳转到 AI 业务分析中页面

**输出**：授权完成，跳转至分析处理页。

## 数据流

### 请求一：获取站点列表

```
GET /api/v1/tenant/sites
```

**请求头**：`Authorization: Bearer {tenant_token}`

**响应体**：

```json
{
  "code": 200,
  "data": {
    "sites": [
      {
        "site_id": "SITE-001",
        "site_name": "XX科技官方网站",
        "site_domain": "xx-tech.example.com",
        "status": "online"
      }
    ]
  }
}
```

### 请求二：同步站点数据

```
POST /api/v1/sites/{site_id}/sync
```

**响应体**：

```json
{
  "code": 200,
  "data": {
    "site_id": "SITE-001",
    "sync_status": "completed",
    "company_profile": "专注 LED 显示屏研发与制造...",
    "founded_info": "2015年成立，200人规模",
    "target_markets": ["北美", "欧洲"],
    "certifications": ["ISO9001", "CE"],
    "categories": {
      "primary": ["LED 显示屏", "商业广告屏"],
      "secondary": ["控制系统", "安装配件"]
    },
    "seo_keywords": {
      "core": ["LED screen", "digital signage"],
      "long_tail": ["outdoor LED display", "rental LED panel"]
    }
  }
}
```

### 请求三：确认授权

```
POST /api/v1/sites/{site_id}/authorize
```

**响应体**：

```json
{
  "code": 200,
  "data": {
    "site_id": "SITE-001",
    "authorization_status": "authorized",
    "task_id": "TASK-202401-001",
    "redirect_to": "/ai-analysis.html?bot_id=BOT-001&task_id=TASK-202401-001"
  }
}
```

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 选择限制 | 每次仅选 1 个站点 | 选中新的自动取消旧的 |
| 站点状态 | 未上线站点 | 可选但显示警告提示 |
| 同步失败 | 站点 API 调用失败 | 提示同步失败，可重试 |
| 无可授权站点 | 租户下无站点 | 显示空态引导文案 |
| 授权幂等 | 已授权站点重新授权 | 返回已有 task_id，不重复创建 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/tenant/sites | GET | 获取站点列表 | 页面加载 |
| /api/v1/sites/{site_id}/sync | POST | 同步站点数据 | 选中站点后 |
| /api/v1/sites/{site_id}/authorize | POST | 确认授权 | 点击确认授权按钮 |

---

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 站点列表为空 | sites 数组为空 | 空态引导 | 「暂无已绑定的独立站，请先在建站系统中添加站点」 |
| 站点同步失败 | sync 接口返回错误 | 失败提示 | 「数据同步失败，请稍后重试」 |
| 授权失败 | authorize 接口返回错误 | 错误提示 | 「授权失败，请稍后重试」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| 未登录 | code 401 | 跳转登录页 | 「请先登录」 |
