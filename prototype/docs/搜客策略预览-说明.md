## 2. 可见功能逻辑

### 2.1 功能描述

展示 AI 智能生成的搜客策略，包括策略解读、瀑布流执行计划、费用预估明细

### 2.2 交互细节

- 页面进入动画： 各组件依次淡入上滑（fadeInUp），带延迟效果 • 返回按钮： 点击返回 搜客需求输入.html 修改策略 • 确认按钮： 点击后显示 "冻结 Credits..." 加载状态，按钮禁用 • 成功状态徽章： 顶部显示 "AI 策略规划完成" 绿色徽章 3. 分支流程 Credit
- Credits 余额需 > 0 且 ≥ 预估消耗 • 预估消耗含 30% 动态溢价 余额规则： 余额可为负，≤ 0 时禁止消耗操作；执行中不扣费，完成后按实际消耗结算，允许余额为负。 策略限制 • 目标寻客数量上限：200 家 • 单次执行最多 4 个 API 步骤 API 可用性 • 依赖第三方
- API 离线时策略标记异常 费用结算 • 预估费用仅供参考 • 实际扣费按成功调用结算 AI 策略规划异常检测 AI 模型存在幻觉风险 ，可能导致生成的搜客策略与用户需求不符或逻辑矛盾。本模块对 AI 策略输出进行专项检测。 • 输出为空检测： AI 返回空策略时触发，提示「AI 策略生成结果为空，
- 格式校验： 策略 JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」 • 一致性校验： 策略与用户需求矛盾时触发，提示「AI 策略与需求不符，建议重新生成」 • 完整性校验： 必要策略字段缺失时触发，提示「AI 策略生成不完整」 AI 策略异常处理策略 异常类型 处理方式
