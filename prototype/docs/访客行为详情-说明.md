# 访客行为详情

> **版本**：v1.0.0
> **状态**：N（初稿）
> **对应 PRD 节点**：P39-P42
> **对应原型页面**：访客行为详情.html
> **最后更新**：2026-04-10

---

## 概述

访客行为详情页是访客雷达的深度分析页面，展示单个访客的完整行为画像。页面以时间轴形式呈现高维微观行为轨迹，每个行为节点附带 AI 行为解读。同时展示同一 IP 的历史访问记录，支持溯源分析。右侧面板显示规则打分结果、AI 关键动作洞察和综合研判建议，并提供深度挖掘入口触发企业身份揭秘流程。

| 来源 | 触发条件 | 携带参数 |

| 目标 | 触发条件 | 携带数据 |

## 功能逻辑

### 场景一：正常查看访客行为详情

**触发条件**：用户从访客雷达点击某条访客卡片进入。

**执行逻辑**：
1. 前端请求 `GET /api/v1/visitors/{visitor_log_id}` 获取完整行为报告
2. 页面展示访客基础信息卡片（IP、地理位置、设备、操作系统、流量来源、落地页、访问次数、首次访问）
3. 展示高维微观行为轨迹时间轴（按时间倒序排列每个行为节点）
4. 展示同一 IP 的历史访问记录（按时间倒序排列）
5. 右侧面板展示：规则打分结果 / AI 关键动作洞察（规则打分 ≥ 60 时）/ AI 综合研判建议（规则打分 ≥ 60 时）

**输出**：完整访客行为报告页面。

---

### 场景二：行为轨迹时间轴

**触发条件**：访客有多个行为记录。

**执行逻辑**：
1. 按时间倒序展示行为节点，不同行为类型对应不同 UI 颜色
2. 节点 1 - 页面浏览（蓝色）：展示时间戳、页面标题、URL 路径、停留时长、滚动深度
3. 节点 2 - 复制文本（琥珀色）：展示被复制的具体内容，高优意图信号
4. 节点 3 - 意图流失（红色）：展示表单 ID、表单 Title、填表进度、弃填状态
5. 节点 4 - 文件下载（绿色）：展示文件名、文件类型、文件大小
6. 节点 5 - 点击行为（紫色）：展示元素类型、元素文本、点击位置
7. 节点 6 - 会话结束（灰色）：展示退出时间、总会话时长

**输出**：带 AI 解读的行为轨迹时间轴。

---

### 场景三：深度挖掘（分支 A - 存在有效 UTM）

**触发条件**：访客存在有效 UTM 参数（source/medium/campaign）。

**执行逻辑**：
1. 系统自动解析 UTM 参数中的企业标识
2. 关联静态数据中已有的企业信息，跳过 IP 解析服务调用，节省 Credits
3. 直接进入联系人挖掘阶段

**输出**：直接进入联系人挖掘，Credits 仅消耗联系人挖掘部分。

---

### 场景四：深度挖掘（分支 B - 无 UTM 标识）

**触发条件**：访客无 UTM 参数。

**执行逻辑**：
1. 启动 IP 解析服务（IRS）进行企业识别
2. 通过 IP 反查获取企业域名/名称
3. 验证企业信息有效性
4. 进入联系人挖掘阶段
5. 弹窗展示处理进度：IP 企业解析 → 企业信息验证 → 联系人挖掘

**输出**：异步执行，完成后自动跳转到动态线索详情页。

---

### 场景五：AI 行为解读异常

**触发条件**：AI 模型返回空内容、格式错误、内容幻觉或字段缺失。

**执行逻辑**：
1. 前端检测 AI 解读字段异常状态
2. 按异常类型显示对应降级文案（详见异常处理章节）
3. 页面其余内容正常展示

**输出**：访客行为报告页面，AI 解读区域显示降级文案。

## 页面状态

| 状态 | 触发条件 | UI 表现 |
|-----|---------|---------|
| 正常 | 数据完整返回 | 展示完整行为报告 |
| 加载中 | 请求发出后 | 骨架屏或 loading 动画 |
| AI 已触发 | 规则打分 ≥ 60 | 右侧面板显示 AI 洞察和综合研判 |

| 数据过期 | 缓存数据过期 | 提示刷新 |

### 请求二：触发深度挖掘

```
POST /api/v1/visitors/{visitor_log_id}/reveal
```

**请求头**：`Authorization: Bearer {enterprise_token}`

**请求体**：

```json
{
  "visitor_log_id": "VIS-99482X",
  "options": { "include_linkedin": true, "include_social": ["twitter", "facebook"] }
}
```

**响应体**：

```json
{
  "code": 200,
  "data": {
    "task_id": "task_abc123",
    "status": "processing",
    "estimated_credits": 15,
    "processing_steps": [
      { "step": "ip_resolution", "status": "pending" },
      { "step": "company_verification", "status": "pending" },
      { "step": "contact_mining", "status": "pending" }
    ],
```

| code | 含义 |
|------|------|
| 200 | 任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 404 | 访客不存在 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 热度判断 | 规则打分 ≥ 80 | 极高热度（红色脉冲动画） |
| 热度判断 | 规则打分 60-79 | 较高热度（橙色静态标签） |
| 热度判断 | 规则打分 < 60 | 一般热度（绿色静态标签） |
| AI 分析触发 | 规则打分 ≥ 60 分 | 触发 AI 行为洞察和综合研判建议 |
| AI 分析不触发 | 规则打分 < 60 分 | 不触发 AI 分析，不显示 AI 洞察区域 |
| 深度挖掘准入 | 不限分数 | 任何分数均可触发，分数仅影响是否触发 AI 分析 |
| 重复挖掘限制 | 已挖掘的访客 | 显示"已挖掘"状态，不重复触发，不重复扣费 |
| 历史记录关联 | 同一 IP | 无时间限制，多用户场景统一关联 |
| Credits 余额 | 余额 ≤ 0 | 阻止深度挖掘任务提交，弹窗提示充值 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 访客不存在 | code 404 | 提示并返回列表 | 「该访客记录不存在或已被清除」 |
| 网络超时 | 请求超时 30s | 显示错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| Credits 不足 | 余额 ≤ 0 | 弹窗提示充值，阻止提交 | 「Credits 余额不足（当前：{balance}），请充值后再试」 |
| 挖掘已达上限 | 已挖掘记录重复点击 | 按钮禁用 | 「该访客已达到最大挖掘次数」 |
| 挖掘请求超时 | HTTP 超时 | 弹窗提示重试 | 「挖掘请求超时，请稍后重试」 |
| 数据过期 | 缓存过期 | 提示刷新获取最新数据 | 「部分数据已过期，请刷新获取最新数据」 |
| AI 解读为空 | ai_insights 为空数组 | 显示默认解读模板 | 「暂无解读」 |
| AI 格式错误 | JSON 解析失败 | 显示原始行为描述 | 「AI 解读生成异常」 |
| AI 内容幻觉 | 与行为事实矛盾 | 标记仅供参考 | 「AI 解读仅供参考」 |
| AI 字段缺失 | 关键洞察缺失 | 显示分析中状态 | 「AI 分析不完整」 |
| 页面跳转失败 | 导航操作失败 | 提示重试 | 「页面跳转失败，请重试」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 访客 ID | visitor_log_id | 系统生成，UUID |
| IP 地址（脱敏） | visitor_identity.ip_masked | GeoIP 脱敏处理 |
| 地理位置 | visitor_identity.geo_location | GeoIP 定位数据 |
| 设备类型 | visitor_identity.device_type | UA 解析：Desktop/Mobile/Tablet |
| 操作系统 | visitor_identity.os | UA 解析：Windows/macOS/iOS/Android |
| 流量来源 | traffic_context.source | 流量归因：Google 搜索/直接访问/推广链接 |
| 落地页 | traffic_context.landing_page | 首次访问 URL 路径 |
| UTM 参数 | traffic_context.utm_params | URL 解析获取 |
| 访问次数 | visit_stats.visit_count | 同一 IP 历史来访总次数 |
| 首次访问 | visit_stats.first_visit | ISO 8601 日期 |
| 最近访问 | visit_stats.last_visit | ISO 8601 日期 |
| 规则打分 | behavior_analysis.rule_based_score | 行为打分引擎，0-100 |
| 热度等级 | behavior_analysis.heat_level | 极高/较高/一般 |
| AI 分析触发 | behavior_analysis.ai_analysis_triggered | boolean |
| AI 关键洞察 | behavior_analysis.ai_insights | LLM 生成，需做空值校验 |
| AI 综合研判 | behavior_analysis.ai_reasoning | LLM 生成，需做空值和格式校验 |
| 行为时间戳 | current_session.events[].timestamp | Unix 时间戳 |
| 行为类型 | current_session.events[].event_type | page_view / copy_text / file_download / click / form_submit / form_abandon / session_end |
| 行为对象 | current_session.events[].target_object | 各类型携带不同字段 |
| 停留时长 | current_session.events[].metrics.stay_duration_ms | 毫秒 |
| 滚动深度 | current_session.events[].metrics.scroll_depth | 百分比 |
| AI 单行为解读 | current_session.events[].ai_interpretation | LLM 生成 |
| 会话时长 | current_session.duration_seconds | 秒 |
| 历史会话日期 | historical_sessions[].date | ISO 8601 日期 |
| 历史会话打分 | historical_sessions[].score | 该会话规则打分 |
| 历史会话摘要 | historical_sessions[].summary | 会话关键行为摘要 |
