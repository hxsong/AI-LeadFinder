P28-P32 节点

探针部署

对应业务流程节点：P28-P32 探针代码分发、授权与验证部署

页面概述

本页面是动态流量捕捉模块的入口，负责向企业管理员提供域名绑定与 JS 探针代码的部署功能。系统采用「域名绑定优先」机制，探针代码针对绑定域名生成，部署时校验域名一致性，确保数据采集的准确性与安全性。

核心约束规则：域名绑定机制

设计原则

探针代码的生成与绑定以「网站域名」为最小单位，而非企业账号。

约束项 规则说明 业务理由
一账号一域名 每个企业账号仅能绑定一个网站域名 防止多站点访客数据重叠
探针域名唯一 探针代码针对域名生成，同一域名对应唯一探针 ID 确保数据采集的准确性与可追溯性
部署域名校验 探针部署后，系统验证部署域名是否与绑定域名一致 防止探针代码被滥用至未授权域名
域名变更限制 域名绑定后如需更换，需联系平台管理员重置 避免历史数据断层，保障数据连续性

可见功能逻辑

步骤一：域名绑定入口 (二选一)

方式 A：独立站授权绑定

通过 OAuth 授权独立站，系统自动提取域名并绑定。支持后续一键部署探针。

方式 B：手动填写域名

EBA 手动输入网站域名，系统验证域名格式与可访问性后完成绑定。需手动嵌入代码。

分支 A：已授权独立站 - 一键安装路径

1

域名自动绑定

系统检测到已授权独立站，自动提取域名并完成绑定，生成专属探针 ID。

2

一键安装按钮

显示「一键安装追踪探针」主按钮，点击后弹出部署进度弹窗，实时展示：连接独立站 API → 注入追踪代码 → 校验域名一致性 → 验证部署状态。

3

部署成功反馈

部署完成后，弹窗自动关闭，页面底部显示成功卡片「数据已回传，探针已生效」，并提供跳转至「访客雷达」的入口。

分支 B：手动填写域名 - 手动埋点路径

1

域名输入与验证

EBA 手动输入网站域名，系统验证域名格式（正则校验）与可访问性（HTTP 请求）后完成绑定，生成专属探针 ID。

2

代码展示区

页面展示 JS 探针代码片段，包含针对绑定域名生成的专属探针 ID。代码区块右上角提供「复制代码」按钮。

3

三步操作指引

以卡片形式展示操作步骤：① 复制上方代码 → ② 粘贴到网站 <head> 标签 → ③ 等待系统验证生效。

4

验证按钮

提供「验证代码是否生效」按钮，点击后弹出验证进度弹窗，系统检测网站是否已正确嵌入追踪代码，并校验请求来源域名是否与绑定域名一致。

不可见功能逻辑

API 接口 - 域名绑定

POST /api/v1/domain/bind

Request Headers:
  Authorization: Bearer {enterprise_token}

Request Body:
{
  "domain": "example.com",
  "bind_type": "manual"  // manual | oauth
}

Response:
{
  "code": 200,
  "data": {
    "domain": "example.com",
    "probe_id": "LF-X7Y8Z9",
    "bind_type": "manual",
    "bind_at": "2024-02-26T10:00:00Z",
    "status": "bound"
  }
}

校验逻辑

验证域名格式、可访问性、是否已被其他账号绑定、当前账号是否已有绑定域名。

返回内容

包含绑定域名、生成的探针 ID、绑定类型、绑定时间。

验证逻辑

服务端发起 HTTP 请求访问绑定域名网站，解析 HTML 内容检测 <head> 标签中是否包含探针脚本。同时校验请求来源域名是否与绑定域名一致，若不一致则拒绝数据上报。

探针代码核心采集逻辑

探针脚本加载后，自动初始化以下采集维度：

行为类型 触发条件 采集内容
Click 元素点击 href、innerText、data-product-sku
Scroll 滚动深度 50%/100% 视区内停留最久元素的标题
Hover 悬停 > 3秒 停留区域核心关键词/按钮文案
Copy 文本复制 被复制的 Selection_Text
Form 表单交互 Form_ID、Title（不采集输入内容）

GDPR 合规红线

绝对不采集用户在 input 框中输入的具体字符内容，仅记录「发生过输入行为」。所有采集行为需用户 Cookie 同意后触发。

限制规则

一账号一域名

每个企业账号仅能绑定一个网站域名，不可同时绑定多个站点。如需更换域名，需联系平台管理员。

域名唯一性

同一域名仅能被一个账号绑定，若域名已被其他账号绑定，将拒绝当前绑定请求。

部署域名校验

探针代码仅对绑定域名生效，若部署到未绑定域名，数据将无法正常上报，系统会返回域名不一致错误。

验证频率

手动验证按钮有 30 秒冷却时间，避免频繁请求对目标网站造成压力。

异常处理机制

异常场景 系统行为 用户提示
域名已绑定其他账号 拒绝绑定请求 「该域名已被其他账号绑定，如需协助请联系客服」
探针部署至未绑定域名 拒绝数据上报 「检测到探针部署域名与绑定域名不一致，请检查部署位置」
账号已有绑定域名 阻止新域名绑定 「当前账号已绑定域名 xxx.com，如需更换请联系管理员」
域名格式无效 拒绝绑定 「请输入有效的网站域名，如 example.com」
