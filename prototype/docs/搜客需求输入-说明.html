功能说明文档
P12 节点

搜客需求输入页面

对应业务流程节点：P12（在当前 BOT 下，输入自然语言描述补充搜客需求）

页面概述

本页面是静态数据获客流程的入口，用户可以输入自然语言描述补充搜客需求，或直接使用已生成的 ICP 画像基准进行搜索。系统将根据用户输入智能规划最优搜客策略。

与评分系统的关系：搜客需求输入是对 ICP 画像的短期修正。在 P08 阶段生成的 ICP 画像包含客户的基础目标客户信息，但客户在短期搜客时可能有更具体的需求（如特定区域、特定品类、有海关记录等）。本页面接收用户补充的短期搜客需求，与 ICP 画像结合后形成完整的评分参数，用于 P19 阶段潜在客户打分计算。

可见功能逻辑

1. UI 组件

1

ICP 画像基准卡片

展示已加载的 ICP 画像模型，包含四个维度的动态权重配置，根据业务类型自动调整权重分配：

业务类型 需求匹配度 购买能力 采购意向 活跃意向
外贸型 32% 20% 35% 13%
生产型 40% 25% 20% 15%
服务型 35% 25% 15% 25%
渠道型 35% 20% 30% 15%

点击「查看报告」可跳转至业务分析报告页面，回顾之前生成的 ICP 分析结果

3

搜客需求文本框

支持自然语言输入，最大 500 字符，实时显示字符计数

4

快速筛选标签

预设标签： 美国、欧洲、有海关记录、近期参展、大型企业，点击自动追加到文本框

与评分维度对应关系：

  • • 美国/欧洲 → 影响「购买能力」评分的地理匹配
  • • 有海关记录 → 影响「采购意向」评分筛选
  • • 近期参展 → 影响「活跃意向」评分筛选
  • • 大型企业 → 影响「购买能力」评分中的规模得分
5

目标企业数量滑块

范围 20-200 家，默认值 100 家

2. 交互效果

  • • 文本输入：实时字符计数，超过 500 字符自动截断
  • • 快速标签点击：自动追加标签文本到输入框，标签样式变为选中态（蓝色背景）
  • • 滑块拖动：实时更新显示的目标企业数量
  • • 提交按钮：点击后显示加载动画 "AI 规划中..."，按钮禁用
  • • 页面进入动画：各组件依次淡入上滑（fadeInUp）

3. 分支流程

降级兜底逻辑：若 user_query 为空，前端隐式传递 use_default_icp=true 标识，系统将完全调取 P08 阶段的 ICP 画像作为默认搜索指令。

不可见功能逻辑

1. API 接口调用

POST /api/v1/search/input

Request Headers:
  Authorization: Bearer {tenant_token}
  Content-Type: application/json

Request Body:
{
  "tenant_id": "tenant_001",
  "user_query": "我想找欧洲的登山装备的品牌代理商",
  "target_count": 20,
  "use_default_icp": false
}

Response (Success):
{
  "code": 200,
  "message": "success",
  "data": {
    "task_id": "task_20240115_001",
    "status": "pending"
  }
}

Response (Error):
{
  "code": 400,
  "message": "错误信息",
  "data": {}
}

2. 数据存储

搜索任务表

记录 task_id、tenant_id、user_query、target_count、status

ICP 缓存

Redis 缓存 ICP 画像数据，有效期 24 小时

操作日志

记录用户搜索行为，用于后续漏斗分析

限制规则

输入限制

  • • 文本输入最大 500 字符
  • • 目标企业数量范围：20-200 家

权限限制

  • • 需完成 P08 业务分析阶段
  • • 需有有效的 ICP 画像数据
  • • Credits 余额需 > 0

余额规则：余额可为负，≤ 0 时禁止消耗操作；执行中不扣费，完成后按实际消耗结算，允许余额为负。

频率限制

  • • 一个 BOT 同时只能执行一个搜客任务

数据依赖

  • • 依赖 P08 阶段生成的 ICP 建模数据
