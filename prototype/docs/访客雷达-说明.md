> **对应 PRD 节点**：P29-P38
> **对应原型页面**：访客雷达.html
> **最后更新**：2026-04-10

---

## 概述

访客雷达是动态流量获客模块的核心工作台，实时展示所有访问过企业独立站的访客数据。用户通过热度筛选、地区筛选和时间筛选快速定位高价值访客，点击卡片可进入访客行为详情页，对高价值访客触发深度挖掘，解锁企业身份和联系人信息。

## 功能逻辑

### 场景一：正常浏览访客列表

**触发条件**：页面正常加载，无进行中任务。

**执行逻辑**：
1. 页面加载时调用 `GET /api/v1/visitors` 获取访客列表
2. 展示统计卡片：今日访客数 / 近7日访客数 / 高热度访客数 / 已挖掘访客数
3. 默认显示全部地区访客，默认选择当日日期
4. 右侧显示"实时更新中"状态指示器

**输出**：访客卡片列表，支持筛选和翻页。

---

### 场景二：热度筛选

**触发条件**：用户点击热度标签（全部/极高/较高/一般）。

**执行逻辑**：
1. 热度定义：极高热度（红色，规则打分 ≥ 80）、较高热度（橙色，规则打分 60-79）、一般热度（绿色，规则打分 < 60）
2. 用户点击热度标签，列表自动过滤为对应热度等级
3. 刷新统计卡片中的高热度访客计数

**输出**：按热度等级过滤后的访客列表。

---

### 场景三：地区筛选

**触发条件**：用户从下拉菜单选择地区。

**执行逻辑**：
1. 地区选项：全部地区 / 北美 / 欧洲 / 东南亚
2. 基于 GeoIP 定位数据过滤
3. 筛选后刷新列表和统计卡片

**输出**：按地区过滤后的访客列表。

---

### 场景四：时间筛选

**触发条件**：用户选择日期范围。

**执行逻辑**：
1. 快捷选项：今日（默认）/ 近7天 / 近30天 / 全部
2. 自定义日期范围：开始日期选择器 + 结束日期选择器，点击"应用"确认
3. 结束日期不能晚于当日（today）
4. 日期格式：YYYY-MM-DD
5. 切换日期后自动触发数据查询

**输出**：按日期范围过滤后的访客列表。

---

### 场景五：深度挖掘访客

**触发条件**：用户点击卡片右侧"深度挖掘"按钮。

**执行逻辑**：
1. 弹出深度挖掘弹窗，展示计费规则：IP 企业解析 = 5 Credits，每获取 1 个验证邮箱 = 2 Credits，附加 LinkedIn = +1 Credit，附加 Twitter/FB = +1 Credit/个
2. 若检测到 UTM 参数，显示"系统将优先尝试关联静态数据中已有的企业信息"提示
3. 用户确认后，前端发送 `POST /api/v1/visitors/{visitor_log_id}/reveal`
4. 进入页面时查询任务状态，若存在进行中任务自动跳转到动态线索池

**输出**：异步执行，IP 解析 + 联系人挖掘在后台进行，完成后自动进入动态线索池。

## 数据流

### 请求一：获取访客列表

```
GET /api/v1/visitors
```

**请求头**：`Authorization: Bearer {tenant_token}`

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| heat_level | string | 否 | 热度等级：very_high / high / normal / all（默认 all） |
| region | string | 否 | 地区：NA / EU / SEA / all（默认 all） |
| date_range | string | 否 | 日期范围：today / 7d / 30d / all（默认 today） |
| start_date | string | 否 | 自定义开始日期，YYYY-MM-DD |
| end_date | string | 否 | 自定义结束日期，YYYY-MM-DD |
| page | number | 否 | 页码，默认 1 |
| limit | number | 否 | 每页数量，默认 20 |

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 47,
    "today_count": 12,
    "items": [
      {
        "visitor_log_id": "VIS-99482X",
        "ip_masked": "67.43.156.**",
        "geo_location": "Florida, USA",
        "heat_level": "very_high",
        "rule_score": 95,
        "device_type": "Desktop",
        "traffic_source": "Google 搜索",
        "visit_count": 3,
        "first_visit": "2024-02-24",
        "last_visit": "2024-02-27",
        "is_mined": false,
        "ai_analysis_triggered": true
      }
    ]
  }
}
```

### 请求二：触发深度挖掘

```
POST /api/v1/visitors/{visitor_log_id}/reveal
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求体**：

```json
{
  "visitor_log_id": "VIS-99482X",
  "options": {
    "include_linkedin": true,
    "include_social": ["twitter", "facebook"]
  }
}
```

**响应体**：

```json
{
  "code": 200,
  "data": {
    "task_id": "task_abc123",
    "status": "processing",
    "estimated_credits": 15,
    "message": "深度挖掘任务已启动"
  }
}
```

| code | 含义 |
|------|------|
| 200 | 任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 热度等级 | 规则打分 ≥ 80 | 极高热度（红色） |
| 热度等级 | 规则打分 60-79 | 较高热度（橙色） |
| 热度等级 | 规则打分 < 60 | 一般热度（绿色） |
| AI 分析触发 | 规则打分 ≥ 60 分 | 触发 AI 行为洞察和综合研判，不触发 AI 打分 |
| AI 分析不触发 | 规则打分 < 60 分 | 不触发 AI 分析，不显示 AI 洞察区域 |
| 深度挖掘准入 | 不限分数 | 任何分数均可触发深度挖掘，分数仅影响是否触发 AI 分析 |
| 日期范围 | 结束日期不能晚于当日 | 超出范围自动截断 |
| 重复挖掘 | 已挖掘的访客 | 显示"已挖掘"状态，不重复触发 |

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 网络超时 | 请求超时 30s | 显示错误提示 | 「网络连接异常，请检查网络后重试」 |
| Credits 不足 | 余额 ≤ 0 | 按钮禁用，弹窗提示充值 | 「Credits 余额不足（当前：{balance}），请充值后再试」 |
| 服务不可用 | HTTP 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| 访客不存在 | HTTP 404 | 提示并返回列表 | 「该访客记录不存在或已被清除」 |
| 深度挖掘已达上限 | 已挖掘访客重复点击 | 按钮禁用 | 「该访客已达到最大挖掘次数」 |
| 挖掘请求超时 | 请求超时 | 弹窗提示重试 | 「挖掘请求超时，请稍后重试」 |
| 数据过期 | 缓存数据过期 | 提示刷新 | 「部分数据已过期，请刷新获取最新数据」 |
| 实时连接中断 | WebSocket 断开 | 提示重连 | 「实时连接已断开，正在尝试重新连接」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 访客 ID | visitor_log_id | 系统生成，UUID |
| IP 地址（脱敏） | ip_masked | GeoIP 脱敏处理 |
| 地理位置 | geo_location | GeoIP 定位数据 |
| 热度等级 | heat_level | very_high / high / normal |
| 规则打分 | rule_score | 行为打分引擎计算 |
| 设备类型 | device_type | 探针 JS 采集 UA 解析 |
| 操作系统 | os | 探针 JS 采集 UA 解析 |
| 流量来源 | traffic_source | 流量归因解析（Google 搜索/直接访问等） |
| 落地页 | landing_page | 首次访问 URL 路径 |
| 访问次数 | visit_count | 同一 IP 历史访问次数 |
| 首次访问 | first_visit | ISO 8601 日期 |
| 最近访问 | last_visit | ISO 8601 日期 |
| 是否已挖掘 | is_mined | boolean |
| AI 分析是否触发 | ai_analysis_triggered | boolean（规则打分 ≥ 60 时为 true） |
| 深度挖掘预估 Credits | estimated_credits | 系统计算（IP 解析 + 邮箱揭示等） |
