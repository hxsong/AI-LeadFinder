P21-P27 节点

静态线索池

对应业务流程节点：P21-P27（选择目标企业挖掘联系人 → 展示静态线索分析报告）

页面概述

静态线索池是用户管理已挖掘线索的核心页面。页面展示所有已完成挖掘和正在挖掘中的静态线索，支持按评分、等级、状态进行筛选和排序。每条线索卡片实时展示挖掘进度，挖掘完成后显示联系人摘要信息。点击卡片可进入线索详情页查看完整分析报告。

可见功能逻辑

1. 线索卡片展示

挖掘中状态

  • • 卡片边框高亮显示（蓝色边框）
  • • 顶部显示"挖掘中"标签（带旋转动画）
  • • 显示进度条和剩余时间预估
  • • 联系人数量显示"挖掘中..."

已完成状态

  • • 展示已获取的联系人头像和姓名
  • • 显示消耗的 Credits 数量

卡片核心信息

  • • 企业名称
  • • 企业简介
  • • 联系人数量 / Credits 消耗
  • • 入库时间

2. 分支流程

A

点击线索卡片

无论挖掘中还是已完成，点击卡片均跳转至线索详情页（线索详情.html）

B

返回搜客结果

点击底部"返回搜客结果"按钮，跳转至 搜客结果列表.html

C

筛选条件切换

点击筛选按钮后，列表实时刷新，保持当前排序方式

3. 异常流程

挖掘失败

若 Apollo API 调用失败，卡片显示"挖掘失败"状态，支持重新挖掘操作

无联系人结果

若未找到任何联系人，卡片显示"未找到联系人"，消耗 0 Credits

4. 交互效果

卡片悬停

阴影加深，cursor 变为 pointer

进度条动画

渐变色流动效果，模拟加载状态

入场动画

卡片依次淡入上滑（delay 递增）

5. P26：价值精算与持久化入库

计费规则（基于 Apollo.io Credits 消耗）

  • • 基础联系人数据：不消耗 Credits
  • • 邮箱揭示（reveal_personal_emails）：每个有效邮箱 1 Credit
  • • 电话号码揭示（reveal_phone_number）：每个有效电话 1 Credit

限制规则

Credits 余额检查

挖掘前需确认 Credits 余额 > 0，余额 ≤ 0 时禁止启动新挖掘任务

余额规则：余额可为负，≤ 0 时禁止消耗操作；挖掘执行中不扣费，完成后按实际消耗结算，允许余额为负。

异步任务执行规则

  • • 页面关闭继续执行：挖掘任务在后台异步执行，用户关闭页面不影响任务进度
  • • 接口超时机制：每次接口调用超时时间 1 分钟，超时后跳过本次调用继续执行
  • • 任务不阻塞：确保任务最终完成，不会因单个接口失败而阻塞
  • • 预估时长：按接口调用次数预估，每次调用约 1 分钟

并发挖掘限制

单租户同时最多 5 个挖掘中任务，超出需等待任务完成

线索去重

同一企业域名不可重复入库，已存在的线索显示"已入库"标识

权限控制

当前版本系统暂无权限控制逻辑，所有登录用户均可查看静态线索池。

AI 线索评分异常检测

AI 模型存在幻觉风险，可能导致线索评分与实际情况不符。本模块对 AI 评分输出进行专项检测。

  • • 输出为空检测：AI 返回空评分时触发，提示「AI 评分结果为空，请重试」
  • • 格式校验：评分 JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」
  • • 一致性校验：评分与原始数据矛盾时触发，提示「AI 评分存在异常，建议重新分析」
  • • 数值范围校验：评分超出 0-100 范围时触发，提示「AI 评分异常，已使用默认值」

AI 评分异常处理策略

异常类型 处理方式 提示文案
输出为空 使用基础评分，不显示 AI 判词 「AI 评分结果为空」
格式错误 自动重试 1 次，失败后使用默认值 「AI 返回数据格式异常」
内容幻觉 标记异常，显示基础评分 「AI 评分仅供参考」
数值异常 使用业务默认值，通知风控 「AI 评分异常，已使用默认值」
