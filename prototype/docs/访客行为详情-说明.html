功能说明文档
P39 节点

访客行为详情

对应业务流程节点：P39（人工查看特定 IP 行为分析详情并选择挖掘 → 展示动态线索分析报告）

页面概述

访客行为详情页是访客雷达的深度分析页面，展示单个访客的完整行为画像。页面以时间轴形式呈现高维微观行为轨迹，每个行为节点附带 AI 行为解读。同时展示同一 IP 的历史访问记录，支持溯源分析。右侧面板显示规则打分结果、AI 关键动作洞察、综合研判建议，并提供深度挖掘入口触发企业身份揭秘流程。

时间轴说明：按时间倒序展示访客行为轨迹，每个节点代表一个独立的用户动作。行为类型不同，UI 展示区块和携带信息也有所差异。

1

页面浏览节点

展示信息：时间戳、页面标题、URL 路径、停留时长、滚动深度

UI 区块：蓝色边框卡片，序号圆圈为蓝色背景

AI 解读：节点下方附加 AI 行为解读卡片（当规则打分 ≥ 60 分时显示）

2

复制文本节点（高优意图）

展示信息：被复制的具体文本内容（如技术参数、联系方式、价格信息）

UI 区块：琥珀色背景高亮卡片，序号圆圈为琥珀色背景，标识高价值意图信号

AI 解读：节点下方附加 AI 行为解读卡片（当规则打分 ≥ 60 分时显示）

3

意图流失节点（转化漏斗）

展示信息：表单 ID、表单 Title、填表进度、弃填状态（已放弃/未完成）

UI 区块：红色边框卡片，序号圆圈为红色背景，标识关键转化流失点

AI 解读：节点下方附加 AI 行为解读卡片（当规则打分 ≥ 60 分时显示）

4

文件下载节点

展示信息：文件名、文件类型、文件大小

UI 区块：绿色边框卡片，序号圆圈为绿色背景，标识深度兴趣信号

AI 解读：节点下方附加 AI 行为解读卡片（当规则打分 ≥ 60 分时显示）

5

展示信息：元素类型、元素文本/描述、点击位置

UI 区块：紫色边框卡片，序号圆圈为紫色背景，标识主动交互意向

AI 解读：节点下方附加 AI 行为解读卡片（当规则打分 ≥ 60 分时显示）

6

会话结束节点

展示信息：访客离开时间、总会话时长、退出页面

UI 区块：灰色背景卡片，序号圆圈为灰色背景

AI 解读：不显示（会话结束节点不携带意图信号）

数据处理逻辑：

  • • 页面浏览：实时采集，每次页面切换生成新节点
  • • 复制文本：通过 JS 监听 copy 事件采集，仅记录复制内容，不记录来源页面 URL
  • • 文件下载：监听文件下载链接点击或资源请求，采集文件名、类型和大小
  • • 点击行为：监听关键元素点击事件（按钮、CTA、导航链接等），采集元素标识和位置
  • • 表单流失：监听表单聚焦和失焦事件，结合页面可见性变化判断弃填
  • • 会话结束：基于页面可见性 API（visibilitychange）或心跳超时判定

企业识别分支逻辑 - P39/P40

分支 A：存在有效 UTM 标识

• 系统自动解析 UTM 参数中的企业标识

• 关联静态数据中已有的企业信息

• 跳过 IP 解析服务调用，节省 Credits

• 直接进入联系人挖掘阶段

分支 B：无 UTM 标识

• 启动 IP 解析服务（IRS）进行企业识别

• 通过 IP 反查获取企业域名/名称

• 验证企业信息有效性

• 进入联系人挖掘阶段

AI 模型处理逻辑 - 关键动作洞察（统一版本）

设计说明：使用统一提示词一次性生成单行为分析、关键动作洞察和 AI 综合研判建议。AI 不再参与打分，只在达到触发阈值后生成洞察分析。

# AI 分析触发条件
当规则打分达到 60 分及以上时，触发 AI 分析

# AI 输入数据
- 访客行为序列: {behavior_events}
  - 每个行为节点包含: 时间戳、行为类型(event_type)、交互对象(target_object)、行为指标(metrics)
  - 行为类型: page_view, copy_text, file_download, click, form_submit, form_abandon
- 规则打分结果: {rule_based_score}
- 热度等级: {heat_level}

# AI 任务
一次性生成以下内容：
1. 单行为分析 (ai_interpretation): 对每个高价值行为节点生成意图解读
2. 关键动作洞察 (ai_insights): 识别高价值行为并解读意图
3. 综合研判建议 (ai_reasoning): 给出跟进建议

# AI Output JSON Format
{

触发阈值

规则打分 ≥ 60 分

输出内容

单行为解读 + 关键洞察 + 综合研判

ai_interpretations 字段说明

字段名 类型 说明
event_index int 行为节点在序列中的索引
event_type string 行为类型
summary string 行为意图的一句话总结，15-30字

限制规则

Credits 余额检查

触发深度挖掘前检查账户余额 > 0，余额 ≤ 0 时弹窗提示充值，阻止任务提交。

余额规则：余额可为负，≤ 0 时禁止消耗操作；挖掘执行中不扣费，完成后按实际消耗结算，允许余额为负。

热度判断逻辑

热度等级由规则打分结果决定，用于快速识别高价值访客：

  • • 极高热度（红色）：规则打分 ≥ 80 分
  • • 较高热度（橙色）：规则打分 60-79 分
  • • 一般热度（绿色）：规则打分 < 60 分

AI 分析触发条件

AI 只在达到触发阈值后生成洞察分析，不再参与打分：

  • • 触发阈值：规则打分 ≥ 60 分
  • • 低于阈值：不触发 AI 分析，不显示 AI 洞察区域
  • • 高于阈值：触发 AI 生成关键动作洞察和综合研判建议

重复挖掘限制

已挖掘的访客记录会标记为「已挖掘」状态，不存在重复挖掘的概念。访客一旦被挖掘过，系统直接显示其已挖掘状态，不会再次触发挖掘流程，也不会产生重复扣费。

深度挖掘准入规则

访客数据无论分数高低均可进行深度挖掘，逻辑说明如下：

  • • 规则打分 < 60 分：仍可进行深度挖掘，触发 IP 企业解析与联系人挖掘流程，Credits 按标准计费
  • • 规则打分 ≥ 60 分：触发深度挖掘的同时，额外生成 AI 行为洞察分析
  • • 核心逻辑：深度挖掘是独立于规则打分的操作，分数仅影响是否触发 AI 分析，不作为挖掘准入门槛

IP 解析处理逻辑

深度挖掘时调用 IP 解析服务，处理逻辑如下：

  • • 解析成功：返回企业名称、地址、行业等信息，生成动态线索
  • • 解析失败（无法识别）：提示用户「该 IP 无法解析到企业」，不扣除 Credits
  • • 解析失败（服务异常）：提示用户「IP 解析服务暂时不可用」，不扣除 Credits

权限控制

当前版本系统暂无权限控制逻辑，所有登录用户均可查看访客行为详情数据。

AI 行为解读异常检测

AI 模型存在幻觉风险，可能导致行为解读与实际行为不符。本模块对 AI 行为解读输出进行专项检测。

  • • 输出为空检测：AI 返回空解读时触发，显示「暂无解读」
  • • 格式校验：解读 JSON 解析失败时触发，显示原始行为描述
  • • 一致性校验：解读与行为事实矛盾时触发，提示「AI 解读仅供参考」
  • • 完整性校验：关键动作洞察缺失时触发，显示「分析中」

AI 行为解读异常处理策略

异常类型 处理方式 提示文案
输出为空 显示默认解读模板 「暂无解读」
格式错误 显示原始行为描述 「AI 解读生成异常」
内容幻觉 标记仅供参考 「AI 解读仅供参考」
字段缺失 显示分析中状态 「AI 分析不完整」
