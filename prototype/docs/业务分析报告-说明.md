## 概述

业务分析报告页面是业务初始化流程的最终展示页面，基于 P08 阶段生成的四维评分模型（需求匹配度、购买能力、采购意向、活跃意向）展示深度业务分析结果。报告包含企业定位、业务类型、四维雷达图、目标客户画像、核心优势标签等模块，供用户确认后进入搜客流程。

## 功能逻辑

### 场景一：查看完整业务分析报告

**触发条件**：从 AI 业务分析中页面跳转进入。

**执行逻辑**：
1. 前端请求报告详情数据
2. 页面展示完整报告内容：
   - 企业定位区：企业名称、业务类型标签、企业定位描述
   - 四维雷达图：需求匹配度、购买能力、采购意向、活跃意向
   - 主营品类标签区：一级/二级分类
   - 核心优势标签区
   - 目标客户画像区：客户类型、市场区域、主营品类、企业规模
   - 评分权重说明

**输出**：完整业务分析报告页面。

---

### 场景二：四维评分权重说明

**触发条件**：用户查看评分权重配置。

**执行逻辑**：
1. 展示当前报告使用的四维权重值
2. 展示各维度定义说明：
   - 需求匹配度：候选企业是否需要客户的产品
   - 购买能力：企业规模评分（60%）+ 地理评分（40%）
   - 采购意向：海关 HS Code 匹配度 × 时间衰减系数
   - 活跃意向：展会记录 × 时间衰减系数
3. 权重总和必须等于 100

**输出**：权重说明模块。

---

### 场景三：重新分析

**触发条件**：用户点击"重新分析"按钮。

**执行逻辑**：
1. 弹出确认提示："重新分析会清除当前报告和向量数据"
2. 用户确认后，调用重新分析接口
3. 后端清除当前报告和向量数据
4. 跳转至数据来源选择页面

**输出**：跳转至业务初始化页面。

---

### 场景四：开始搜客

**触发条件**：用户确认报告无误，点击"开始搜客"。

**执行逻辑**：
1. 前端跳转到搜客需求输入页面
2. 使用当前报告的 ICP 画像作为搜客输入条件

**输出**：跳转至搜客需求输入页面。

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 重新分析 | 点击重新分析 | 清除当前报告和向量数据，需重新选择数据来源 |
| 核心优势标签 | 无数量限制 | 按实际数量展示 |
| 四维权重总和 | 四个维度 | 总和必须等于 1.00（100%） |
| 目标客户规模 | 可为空 | 为空时前端显示"待定" |
| 报告不存在 | 报告数据丢失 | 跳转至初始化页面重新开始 |
| 向量数据丢失 | 数据损坏 | 提示数据异常，建议重新分析 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 报告不存在 | code 404 | 跳转至初始化页面 | 「报告不存在或已过期，请重新开始」 |
| 向量数据丢失 | code 500 | 异常提示 | 「数据异常，建议重新分析」 |
| AI 输出为空 | 四维评分为空 | 显示默认评分 0 | 「AI 分析结果为空，请重新分析」 |
| AI 格式错误 | JSON 解析失败 | 显示原始数据 | 「AI 返回数据格式异常」 |
| AI 内容幻觉 | 与原始数据矛盾 | 标记仅供参考 | 「AI 分析结果仅供参考，以实际数据为准」 |
| AI 字段缺失 | 必要字段缺失 | 使用默认值 | 「AI 分析不完整」 |
| 评分超出范围 | 评分 <0 或 >100 | 默认值替代 | 「AI 评分异常，已使用默认值」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
