提示词语言管理架构

为 AI 模型提供中英文双语提示词支持，解决三方接口调用时的传参语言问题

文档概述

本文档说明了 LeadFinder 系统中提示词语言管理的完整技术架构，包括：

  • • 提示词双语化架构：中英文提示词库的存储和选择机制
  • • 参数语言转换机制：解决调用三方 API 时传参语言的问题
  • • 模型适配策略：根据模型类型自动选择合适的提示词语言

3. 语言选择逻辑

function selectPromptLanguage(modelType: string): "zh" | "en" {
  const englishPreferredModels = [
    "gpt-4", "gpt-4o", "gpt-4o-mini",
    "claude-3-opus", "claude-3-sonnet", "claude-3-5-sonnet",
    "claude-3-haiku", "claude-3-5-haiku",
    "gemini-1.5-pro", "gemini-1.5-flash"
  ];

  const chinesePreferredModels = [
    "ernie-4", "ernie-3.5", "ernie-bot",
    "qwen-max", "qwen-plus", "qwen-turbo",
    "spark-4.0", "spark-3.5"
  ];

  if (englishPreferredModels.some(m => modelType.includes(m))) {
    return "en";
  }

  if (chinesePreferredModels.some(m => modelType.includes(m))) {
    return "zh";
  }

  return "en"; // 默认使用英文
}

中文优先模型

参数语言转换机制

问题背景

当用户输入中文搜索需求（如"我想找欧洲的登山装备品牌代理商"）时，AI 策略规划师会生成相应的搜索关键字。但如果直接将中文关键字传给国外三方 API（如 Google Maps），会导致调用失败或返回错误结果。

解决方案：在调用国外 API 时，自动将中文关键字翻译为英文；国内 API 保持中文。

1. API 语言要求分析

API 服务 提供商 语言要求 关键字字段
Google Maps API Google 必须英文 textQuery
10times API 10times 必须英文 query
Jina Reader API Jina AI 不限 URL (无需翻译)
ChuHaipai API 出海派 中英文皆可 searchTerm

2. 转换流程

     ▼

     │
     ▼

技术注意事项

提示词版本管理

每次更新提示词时，需要同时更新中英文两个版本，并记录版本号便于追溯。

翻译一致性

中英文提示词需要在语义上保持一致，避免因翻译差异导致模型输出不一致。

参数验证

调用国外 API 前，必须验证搜索关键字已正确翻译为英文。

日志记录

记录每次 API 调用的语言转换情况，便于问题排查和优化。
