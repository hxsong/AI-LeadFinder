功能说明文档
P03 节点

多BOT工作台页面

对应业务流程节点：P03（企业管理员登录后的"指挥中心"）

页面概述

本页面是企业管理员 (EBA) 登录后的首个"指挥中心 (Command Center)"。页面清晰展示全局资源状态（算力积分余额、BOT 席位额度），并提供 BOT 工作空间矩阵入口。系统采用"账号级统管算力资金池 + BOT 级绝对隔离业务数据"架构，支撑多品牌、多产品线的企业级客户场景。

可见功能逻辑

1. 全局资源看板

算力积分余额

  • • 显示当前账号的 Global Credits 余额
  • • 全账号名下所有 BOT 共享此资金池
  • • 用于支付搜客、IP 解析、联系人挖掘等 API 成本

引擎实例额度

  • • 显示"已创建 X / 共 Y 个 BOT"
  • • 限制可创建的获客 BOT 实例数量
  • • 不同 BOT 对应不同的独立站或业务线

2. 购买扩展按钮

点击 [购买更多算力]

  • • 按钮位于"全局资源看板"标题右侧
  • • 弹出【购买更多算力】模态框
  • • 提示"请联系销售或客服人员"，并显示官网链接
  • • 注意：点击不直接跳支付，而是引导线下联系

UI 文案："如需扩充获客引擎实例席位，或补充全局获客算力积分，请联系销售或客服人员为您配置专属升级方案。"

3. BOT 工作空间矩阵

已激活的 BOT 卡片

  • • 展示 BOT 名称（如"北美区-户外屏主站"）
  • • 显示当前运行状态（待初始化/正常运行/运行异常）
  • • 显示该 BOT 累计获取的线索数（静态/动态）
  • • 显示该 BOT 消耗的 Credits
  • • 提供 [进入获客引擎] 主按钮
状态显示逻辑

状态以动态数据获客（JS 探针）的运行状态为准

状态 显示样式 触发条件
待初始化 灰色徽章 JS 探针脚本未配置（未完成探针部署流程）
正常运行 绿色徽章 JS 探针已配置且数据上报正常（最近 24 小时有数据上报）
运行异常 红色徽章 JS 探针已配置但数据上报异常（最近 24 小时无数据上报 或 最近 500 次上报中错误率 > 5%）
自动恢复机制

当 BOT 进入"运行异常"状态后，系统会持续监控探针上报数据：

  • • 若最近 500 次上报中错误率恢复至 ≤5%，自动重置为"正常运行"
  • • 若 24 小时后恢复数据上报，自动重置为"正常运行"
  • • 无需人工干预，状态自动同步

创建新 BOT 占位符

  • • 带有 + 号的虚线框卡片
  • • 若剩余额度 > 0，允许点击创建
  • • 若额度不足，置灰并提示联系销售

4. 核心交互逻辑

动作 校验/处理逻辑 跳转目标
点击 [进入获客引擎] 设置 current_active_bot_id 搜客需求输入.html
点击 [+ 创建新 BOT] 校验 bot_seats_available > 0，显示创建弹窗（只需输入 BOT 名称） 无跳转（弹窗交互）
点击 [创建并初始化] 显示"初始化中..."loading 动画，禁用按钮，轮询后端初始化状态 初始化完成后跳转业务初始化.html
点击 [购买更多算力] 弹出购买更多算力模态框（显示官网链接） 无跳转

3. 初始化状态查询接口

轮询机制说明

  • • 前端在创建 BOT 后，轮询此接口查询初始化状态
  • • 轮询间隔：2 秒
  • • 轮询终止条件：status = "initialized"（成功）或 "failed"（失败）
  • • 成功后跳转业务初始化页面，失败后提示用户重试
GET /api/v1/bot/{bot_id}/init-status

Response:
{
  "code": 200,
  "data": {
    "bot_id": "BOT-1002",
    "status": "initializing",  // initializing | initialized | failed
    "init_progress": 45,       // 0-100
    "error_message": null      // 仅 status=failed 时有值
  }
}

4. 上下文切换机制

关键设计：

  • • 点击 [进入获客引擎] 时，前端将 current_active_bot_id 设置为选中的 ID
  • • 此后该用户进行的所有操作均被严格死锁在该 BOT 的数据作用域内
  • • 不同 BOT 的配置不会发生交叉污染

限制规则

BOT 席位限制

bot_seats_available <= 0 时，创建按钮置灰禁用

数据隔离

不同 BOT 的业务数据完全隔离，不可交叉访问

Credits 共享

全局 Credits 池被所有 BOT 共享消耗

购买流程

扩展资源需联系销售，不支持在线支付
