P07-P08 节点

AI 业务分析处理页面

对应业务流程节点：P07-P08（文本提取与结构化 → 深度分析与 ICP 建模）

页面概述

本页面是 AI 业务分析的核心处理页面，负责将原始输入数据转换为结构化的客户信息。处理结果将用于业务分析报告展示和后续潜在客户评分计算。

系统通过异步任务队列处理大模型长耗时任务，前端实时展示处理进度。处理流程分为两个阶段：文本提取与结构化（P07）、深度分析与 ICP 建模（P08）。

输入数据链路

路径 1：文档上传

来源页面：上传业务材料.html

数据形态：PDF / WORD / MD 文档

处理步骤

步骤 处理内容 输出格式
1. 文件解析 PDF/WORD/MD → 纯文本 Markdown 长文本
2. ETL 清洗 去除噪音，提取结构化数据 JSON 实体字典

路径 2：独立站授权

来源页面：独立站授权.html

数据形态：网站结构化数据

处理步骤

步骤 处理内容 输出格式
1. 数据同步 从独立站 API 获取数据 JSON 数据包
2. 数据解析 提取核心业务信息 JSON 实体字典

URL 参数识别

后续评分调用（P19）

P19 阶段进行潜在客户打分时，将调用 P08 生成的客户信息：

  • • 读取客户信息：获取企业定位、业务类型、主营品类、核心优势等字段
  • • 读取权重配置：根据业务类型获取动态权重
  • • AI 模型判断需求匹配度：将客户信息与候选企业信息输入 AI 模型，判断供需关系
  • • 计算维度得分：
    • - 需求匹配度：AI 模型判断候选企业是否是客户的潜在客户
    • - 购买能力：规模评分（60%）+ 地理评分（40%）
    • - 采购意向：海关 HS Code 匹配度 × 时间衰减系数
    • - 活跃意向：展会记录 × 时间衰减系数
  • • 加权计算：最终得分 = Σ(维度得分 × 权重)

异常处理规则

1. 必填字段缺失处理

当必填字段（企业名称、企业定位、业务类型、主营品类、目标客户类型、目标客户市场区域）缺失时，系统将：

  • • 中断处理流程：停止后续处理，不进入 P08 阶段
  • • 提示客户补充：显示缺失字段列表，引导客户返回上一步补充信息
  • • 保存已提取数据：将已成功提取的字段暂存，避免重复处理

2. 系统异常处理

异常场景 错误码 前端处理
任务不存在 404 跳转至初始化页面重新开始
AI 模型调用失败 500 显示错误提示，支持重试
任务超时 408 显示超时提示，支持重试
数据解析失败 422 提示数据格式问题，返回上一步

3. AI 输出异常检测

AI 模型存在幻觉风险，可能导致输出结果与输入材料不符或逻辑矛盾。本模块对 AI 输出进行专项检测。

  • • 输出为空检测：AI 返回空结果或 null 时触发，提示「AI 分析结果为空，请重试或检查输入材料」
  • • 格式校验：JSON 解析失败时触发，提示「AI 返回数据格式异常，正在自动重试...」
  • • 一致性校验：输出与输入材料矛盾时触发，提示「AI 分析结果存在异常，建议重新分析」
  • • 完整性校验：必要字段（如企业名称、ICP描述）缺失时触发，提示「AI 分析不完整，部分数据缺失」
  • • 数值范围校验：评分超出 0-100 范围时触发，提示「AI 评分异常，已使用默认值」

AI 异常处理策略

异常类型 处理方式 提示文案
输出为空 自动重试 1 次，失败后提示用户 「AI 分析结果为空，请重试」

限制规则

不可中断

分析任务一旦开始不可中断，需等待完成或失败

超时限制

单任务最长执行时间 10 分钟，超时自动标记失败

并发限制

同一租户同时只能有 1 个分析任务执行中

重试限制

失败后最多重试 3 次，超过需联系客服

异步任务执行规则

  • • 页面关闭继续执行：异步任务在后台执行，用户关闭页面不影响任务进度
  • • 接口超时机制：每次接口调用超时时间 1 分钟，超时后跳过本次调用继续执行
  • • 任务不阻塞：确保任务最终完成，不会因单个接口失败而阻塞
  • • 预估时长：按接口调用次数预估，每次调用约 1 分钟
