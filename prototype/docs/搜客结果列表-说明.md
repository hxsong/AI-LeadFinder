> **版本**：v1.0.0
> **状态**：N（初稿）
> **对应 PRD 节点**：P20-P22
> **对应原型页面**：搜客结果列表.html
> **最后更新**：2026-04-10

---

## 概述

搜客结果列表是静态数据获客流程的结果展示页面，展示经过 AI 分析和 ICP 匹配打分排序的企业线索列表。用户可浏览、筛选、导出结果，点击企业卡片查看原始数据证据，并对高价值企业执行联系人挖掘操作。

---

## 功能逻辑

### 场景一：浏览搜客结果列表

**触发条件**：用户进入搜客结果列表页面。

**执行逻辑**：
1. 前端请求搜客结果数据
2. 以卡片形式展示候选企业列表，默认按评分倒序
3. 每张卡片展示：匹配度环形评分图、企业名称、官网链接（超链接）、徽章标签、企业简介、AI 推荐理由、员工规模
4. 评分颜色：≥80 绿色（S级）/ 60-79 蓝色（A/B级）/ <60 灰色（C级）

**输出**：按评分排序的企业卡片列表。

---

### 场景二：筛选与排序

**触发条件**：用户切换筛选条件。

**执行逻辑**：
1. 评分筛选：全部 / ≥80 / 60-79 / <60
2. 徽章标签筛选：按标签类型过滤（S级潜力 / 有海关数据 / 有展会数据 / 有联系方式）
3. 列表实时刷新，保持当前排序

**输出**：按筛选条件过滤后的企业列表。

---

### 场景三：导出搜客结果

**触发条件**：用户点击"导出结果"按钮。

**执行逻辑**：
1. 前端请求导出接口
2. 后端生成 Excel 文件，返回下载链接（有效期 24 小时）
3. 导出字段：企业名称、官网、匹配度评分、AI推荐理由、徽章标签、企业简介、员工规模、联系方式、海关记录、展会记录、地理位置
4. 无版本限制，所有版本均可使用

**输出**：Excel 文件下载。

---

### 场景四：查看企业原始数据

**触发条件**：用户点击企业卡片的"查看详情"。

**执行逻辑**：
1. 弹出原始数据查看弹窗
2. 展示四个数据来源的原始证据：
   - 海关数据：HS Code、产品描述、进口次数、最近日期、供应商国家（来源：出海派）
   - 展会数据：展会名称、日期、地点、参与类型（来源：10times）
   - 地图数据：企业名称、地址、Google评分、营业状态、坐标（来源：Google Maps）
   - 网站数据：域名、内容摘要（来源：Jina Reader）

**输出**：原始数据证据弹窗。

---

### 场景五：联系人挖掘

**触发条件**：用户点击企业卡片的"深度挖掘"按钮。

**执行逻辑**：
1. 弹出联系人挖掘弹窗，展示计费规则
2. Credits 规则：单次挖掘消耗 15 Credits，每获取 1 个验证邮箱 = 2 Credits，附加 LinkedIn = +1 Credit，附加 Twitter/FB = +1 Credit/个
3. 用户确认后，任务异步执行
4. 前端跳转至静态线索池页面查看进度

**输出**：联系人挖掘任务创建，跳转至静态线索池。

## 数据流

### 请求一：获取搜客结果

```
GET /api/v1/search/results/{task_id}
```

**请求头**：`Authorization: Bearer {tenant_token}`

**响应体**：

```json
{
  "code": 200,
  "data": {
    "task_id": "TASK-202401-001",
    "total": 98,
    "high_potential_count": 8,
    "items": [
      {
        "enterprise_id": "ENT-001",
        "company_name": "Alpine Gear International",
        "website": "https://alpine-gear.com",
        "match_score": 88,
        "score_grade": "S",
        "badges": ["S级潜力", "有海关数据"],
        "company_profile": "Wholesale distributor specializing in outdoor equipment...",
        "employee_scale": "50-200人",
        "ai_reasoning": "该企业长期进口户外攀岩设备，与客户主营品类高度匹配...",
        "data_sources": { "customs": true, "exhibition": false, "maps": true, "website": true }
      }
    ]
  }
}
```

### 请求二：联系人挖掘

```

```

### 请求三：导出结果

```
GET /api/v1/search/results/{task_id}/export
```

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 数据保留期限 | 搜索结果保留 | 90 天 |
| 导出功能 | 无版本限制 | 所有版本均可使用 |
| 分页限制 | 每页最大 20 条记录 | 总结果最大 200 条 |
| 联系人挖掘消耗 | 单次挖掘 | 15 Credits |
| 联系人邮箱揭示 | 每获取 1 个验证邮箱 | 2 Credits |
| LinkedIn 附加 | 附加 LinkedIn 主页 | +1 Credit |
| 社交媒体附加 | 附加 Twitter/FB | +1 Credit/个 |
| 单企业挖掘次数 | 最多次数 | 5 次 |
| 联系人数据有效期 | 有效期 | 90 天 |
| Credits 余额规则 | 余额可为负 | ≤0 禁止消耗操作，挖掘中不扣费，完成后结算 |
| AI 推荐理由超长 | 超过 500 字 | 截断 + "查看更多"展开 |
| AI 推荐理由幻觉 | 与企业信息矛盾 | 标记仅供参考 | 「推荐理由仅供参考」 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/search/results/{task_id} | GET | 获取搜客结果列表 | 页面加载 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 任务不存在 | code 404 | 错误提示并返回 | 「该任务不存在或已过期」 |
| 结果数据为空 | items 数组为空 | 空态引导 | 「未找到符合条件的候选企业」 |
| Credits 不足 | 余额 ≤ 0 | 按钮禁用 | 「Credits 余额不足（当前：{balance}），无法执行挖掘」 |
| 挖掘已达上限 | 已挖掘 5 次 | 按钮禁用 | 「该企业已达到最大挖掘次数（5 次）」 |
| 导出失败 | code 非 200 | 失败提示 | 「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | 失败提示 | 「当前筛选条件下无数据可导出」 |
| AI 推荐理由为空 | ai_reasoning 为空 | 显示默认文案 | 「暂无推荐理由」 |
| AI 推荐理由格式错误 | JSON 解析失败 | 显示原始摘要 | 「推荐理由生成异常」 |
| AI 推荐理由超长 | 超过 500 字 | 截断展示 | - |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
