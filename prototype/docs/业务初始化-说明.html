P03-P11 节点

业务初始化页面

对应业务流程节点：P03-P11（登录工作台 → 展示业务分析报告）

页面概述

本页面是 LeadFinder 业务初始化流程的入口页面，用户在此选择数据来源方式。系统提供两种互斥的数据获取途径：上传业务材料（P04）或授权独立站（P05/P06）。此选择将决定后续业务分析的数据输入方式，且选择后不可切换。

可见功能逻辑

1. 数据来源选择卡片

上传业务材料（P04）

  • • 点击卡片或"开始上传材料"按钮进入 上传业务材料.html 页面
  • • 适用于无独立站或材料更全面的企业
  • • 支持格式：PDF、WORD、MD、TXT，单文件最大 5MB

授权独立站（P05/P06）

  • • 点击卡片或"授权独立站"按钮进入 独立站授权.html 页面
  • • 适用于已使用建站系统的企业
  • • 自动同步产品信息、企业介绍等结构化数据

2. 互斥选择机制

!

选择任一卡片后，显示黄色警告提示框："两种数据来源方式互斥，选择其中一种后将无法切换。"

此设计防止实体信息冲突，确保数据一致性。

3. 按钮状态变化

状态 按钮样式 按钮文案
未选择 灰色禁用 "请选择数据来源"
选择上传 蓝色可点击 "开始上传材料"
选择授权 橙色可点击 "授权独立站"

4. 交互效果

卡片悬停：上浮 4px + 蓝色阴影
选中状态：蓝色边框 + 渐变背景
进度指示器：步骤 1 高亮，步骤 2、3 灰色

不可见功能逻辑

1. 页面初始化接口

GET /api/v1/tenant/init-status

Request Headers:
  Authorization: Bearer {tenant_token}

Response:
{
  "code": 200,
  "data": {
    "tenant_id": "TENANT-001",
    "company_name": "深圳市XX科技有限公司",
    "has_site": true,
    "site_list": [
      {
        "site_id": "SITE-001",
        "site_name": "XX科技官方网站",
        "domain": "xx-tech.example.com",
        "status": "online"
      }
    ],
    "has_uploaded_materials": false,
    "analysis_status": null
  }
}

处理逻辑：

  • • 验证租户登录状态，未登录跳转登录页
  • • 检查租户是否已有独立站（决定授权卡片是否可用）
  • • 检查是否已选择数据来源（已选择则直接跳转对应页面）

2. 数据来源选择提交接口

POST /api/v1/tenant/data-source/select

Request Headers:
  Authorization: Bearer {tenant_token}

Request Body:
{
  "source_type": "upload" | "site",
  "selected_site_id": "SITE-001"  // 仅当 source_type 为 site 时必填
}

Response:
{
  "code": 200,
  "data": {
    "selection_id": "SEL-2024-001",
    "source_type": "upload",
    "created_at": "2024-01-15T14:30:00Z",
    "next_step": "/upload"
  }
}

处理逻辑：

  • • 记录用户选择的数据来源类型
  • • 设置租户数据来源标记，后续不可更改
  • • 返回下一步跳转路径

3. 异常流程处理

异常场景 错误码 处理方式
租户未登录 401 跳转登录页
租户无独立站 403 授权卡片置灰禁用
已选择数据来源 302 自动跳转对应页面
服务异常 500 显示错误提示，支持重试

限制规则

互斥限制

两种数据来源方式互斥，选择后不可切换，防止实体信息冲突

权限限制

无独立站的租户无法选择授权方式，卡片置灰显示

状态限制

已完成业务分析的租户跳过此页面，直接进入报告页

会话限制

需要有效的租户登录态，超时需重新登录

页面跳转关系

登录页
业务初始化.html
上传业务材料.html
独立站授权.html
