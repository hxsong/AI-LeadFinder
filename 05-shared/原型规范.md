# AI-LeadFinder 产品原型规范

**版本：** v1.0.0
**依据：** Phase 10 原型规范建设
**最后更新：** 2026-04-17

---

## 1. 目的

本文档定义 AI-LeadFinder 产品原型（HTML 交互原型）的制作和维护规范，供产品经理、前端开发、设计评审参考。

本文档聚焦当前项目实际用到的规范（实用轻量），行业通用最佳实践作为参考链接，不做过度展开。

---

## 2. 目录结构规范

原型资产统一存放在 `03-prototypes/` 目录下，按以下结构组织：

```
03-prototypes/
├── README.md              # 顶层索引，含三向映射表入口
├── catalog-panel.js      # 目录面板工具（可选引入）
├── entry-nav.js          # 侧边导航工具（可选引入）
├── nav-config.json       # 导航配置（entry-nav.js 依赖）
├── pages/                # 原型 HTML 页面（按模块分目录）
│   ├── 01-账号/
│   ├── 02-静态数据/
│   └── 03-企业管理/
└── styles/               # 样式文件（仅供原型使用）
    └── global.css        # 全局样式 + 设计 Token
```

**规则：**
- `pages/` 按产品模块组织，不按页面类型组织
- `styles/` 和 `components/` 保持在原型目录下，不外迁到 `resources/`（职责分离：原型样式 ≠ 文档图片）
- 新增模块目录时，同步更新 `03-prototypes/README.md` 的页面列表

---

## 3. HTML 原型文件命名规范

**规则：**
- 文件名使用中文命名，如 `搜客需求输入.html`、`访客雷达.html`
- 文件名应与功能名称一致，避免缩写或编号
- 不在文件名中包含页面状态（如 `-进行中`、`-成功`），状态通过原型内容体现

**示例：**
```
# 正确
搜客需求输入.html
访客雷达.html
线索详情.html

# 错误
P12-搜客需求.html        （不应使用编号）
search_input.html        （不应使用英文）
```

## 4. CSS 样式规范

**核心文件：** `03-prototypes/styles/global.css`

**样式引用方式：** 原型 HTML 文件使用相对路径引用全局样式文件：

```html
<link rel="stylesheet" href="../styles/global.css">
```

**设计 Token（来自 global.css）：**

| Token | 值 | 用途 |
|-------|----|------|
| 主色调 | `#3B82F6` | 按钮、链接、强调 |
| 次色调 | `#8B5CF6` | 渐变、图标 |
| 成功色 | `#10B981` | 成功状态 |
| 警告色 | `#F59E0B` | 警告状态 |
| 危险色 | `#EF4444` | 错误/删除状态 |
| 背景色 | `#FFFFFF` | 页面背景 |
| 卡片背景 | `#F8FAFC` | 卡片/容器背景 |
| 文字主色 | `#334155` | 正文 |
| 文字次色 | `#64748B` | 次要文字 |
| 边框色 | `#E2E8F0` | 分隔线、边框 |

**规则：**
- 优先使用 global.css 中定义的设计 Token，避免硬编码颜色值
- 页面级样式可通过内联 `<style>` 标签添加，但应尽量复用 global.css 中的类
- 不在 `styles/` 外创建额外的样式文件

## 5. 页面间导航规范

### 5.1 左侧固定导航（entry-nav.js）

通过 `entry-nav.js` 提供左侧固定导航栏：

```html
<script src="../entry-nav.js"></script>
```

配置通过 `nav-config.json` 管理，修改配置后无需改 JS 文件。

### 5.2 目录面板（catalog-panel.js）

通过 `catalog-panel.js` 提供浮动目录面板（右上角，点击展开）：

```html
<script src="../catalog-panel.js"></script>
```

页面和功能说明文档的链接在 JS 中硬编码（如需修改，编辑 catalog-panel.js 第 4-48 行）。

### 5.3 原型页面内跳转

**规则：**
- 页面内跳转使用相对路径，如 `<a href="搜客结果列表.html">`
- 跳转跨模块页面使用完整相对路径，如 `<a href="../02-静态数据/搜客需求输入.html">`
- 外部链接（如功能说明文档）使用 `../04-specs/{模块}/{文件名}.md` 路径

## 6. 原型与功能说明文档对应关系（三向映射）

原型页面、功能说明文档、PRD 节点三者的对应关系通过**三向映射表**维护。

### 6.1 映射表位置

- 各模块三向映射表：位于 `02-modules/{模块}/README.md`
- 原型顶层索引：`03-prototypes/README.md`（含三向映射表入口）
- 功能说明顶层索引：`04-specs/README.md`

### 6.2 映射表格式

```markdown
## {模块名} — 三向映射表

| PRD 节点 | 原型页面 | 功能说明文档 |
|----------|----------|-------------|
| [节点名称] | [页面名](../03-prototypes/pages/{模块}/xxx.html) | [文档名](../04-specs/{模块}/xxx.md) |
```

### 6.3 维护规则

- 新增原型页面时：更新 `03-prototypes/README.md` + 对应模块的三向映射表
- 新增功能说明时：更新 `04-specs/` + 对应模块的三向映射表
- 新增 PRD 节点时：更新 `02-modules/{模块}/README.md` 的三向映射表

## 7. 原型评审和发布流程

### 7.1 制作阶段

1. 产品经理根据 PRD 创建或更新 HTML 原型页面
2. 原型页面存放在 `03-prototypes/pages/{模块}/`
3. 同步更新对应模块的三向映射表（`02-modules/{模块}/README.md`）

### 7.2 评审阶段

1. 通过本地静态服务器查看原型（确保 JS 导航工具正常工作）
   ```bash
   python -m http.server 8080
   # 或
   npx serve .
   ```
2. 评审通过后在 CHANGELOG 中记录变更（`05-shared/changelog/`）

### 7.3 发布说明

- 原型页面无需"正式发布"——三向映射表即为权威索引
- 更新 CHANGELOG 时，在对应模块变更记录中添加原型变更说明
