## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 系统生成 UUID |
| 初始化状态 | `init_status` | `string` | 接口返回 | `pending` / `completed` |
| 业务分析报告状态 | `report_status` | `string` | 接口返回 | `not_started` / `processing` / `completed` / `failed` |
| 报告 ID | `report_id` | `string` | 接口返回 | 系统生成 UUID |
| 报告生成时间 | `report_created_at` | `string` | 接口返回 | ISO 8601 时间戳 |
| 报告过期时间 | `report_expires_at` | `string` | 接口返回 | ISO 8601 时间戳 |
| 报告是否过期 | `is_report_expired` | `boolean` | 接口返回 | 基于 `report_expires_at` 计算 |
| 报告数据 | `report_data` | `object` | 接口返回 | 完整的业务分析报告数据 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 多BOT工作台.html | 点击 [+ 创建新 BOT] 或点击未初始化 BOT | `bot_id` |

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 上传业务材料.html | 点击"上传业务材料"卡片 | `bot_id` |
| 独立站授权.html | 点击"授权独立站"卡片 | `bot_id` |
| 业务分析报告.html | 报告已完成且未过期时点击"查看报告" | `bot_id`, `report_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后 | 页面加载 | 数据返回 |
| 正常 | 数据完整返回 | 数据返回成功 | 用户操作 |
| 报告已完成 | 检测到已有完成的报告 | `report_status = completed` | 用户操作 |
| 报告已过期 | 报告超过有效期 | `is_report_expired = true` | 用户操作 |

## 数据流

### 获取 BOT 初始化状态

```
GET /api/v1/bot/{bot_id}/init-status
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | 路径参数，BOT 唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "bot_id": "BOT-001",
    "init_status": "completed",
    "report_status": "completed",
    "report_id": "RPT-202401-001",
    "report_created_at": "2024-01-20T10:35:00Z",
    "report_expires_at": "2024-02-20T10:35:00Z",
    "is_report_expired": false
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 查询成功 |
| 401 | 未登录 |
| 404 | BOT 不存在 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：首次初始化

**触发条件**：新创建的 BOT 首次进入业务初始化页面。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id`
2. 系统执行处理：
   - 查询 BOT 初始化状态
   - 若 `init_status = pending`，展示两个初始化选项卡片：上传业务材料、授权独立站
   - 两个选项为互斥选择，用户只能二选一
3. 系统返回响应：返回初始化状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| BOT 不存在 | 提示并返回工作台 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功 |

**输出**：初始化选项页面。

---

### 场景二：已有报告未过期

**触发条件**：BOT 已有完成的业务分析报告且未过期。

**执行逻辑**：

1. 系统接收请求：页面加载，检测到 `report_status = completed` 且 `is_report_expired = false`
2. 系统执行处理：
   - 展示"查看报告"按钮，点击跳转至业务分析报告页面
   - 同时展示"重新分析"选项
3. 系统返回响应：页面渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 报告已过期 | 提示报告已过期，需重新分析 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 报告已完成 | `report_status = completed` |

**输出**：展示查看报告入口。

---

### 场景三：选择初始化方式

**触发条件**：用户点击"上传业务材料"或"授权独立站"卡片。

**执行逻辑**：

1. 系统接收请求：用户点击初始化选项
2. 系统执行处理：
   - 用户选择上传业务材料：跳转至上传业务材料页面
   - 用户选择授权独立站：跳转至独立站授权页面
   - 两个选项互斥，不可同时选择
3. 系统返回响应：页面跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 已有一种方式在进行中 | 禁用另一种方式 |

**状态变更**：无

**输出**：跳转至对应初始化页面。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| BOT 不存在 | code 404 | 返回错误 | Toast「该 BOT 不存在或已被删除」 |
| 网络超时 | 请求超时 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
