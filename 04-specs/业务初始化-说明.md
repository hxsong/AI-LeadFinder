## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 系统生成 UUID，从 URL 参数获取 |
| 初始化状态 | `init_status` | `string` | 接口返回 | 枚举值 pending/completed，服务端从 bot 表查询 |
| 数据来源类型 | `data_source_type` | `string` | 接口返回 | 枚举值 material/site/null，表示数据来源方式 |
| 业务分析报告状态 | `report_status` | `string` | 接口返回 | 枚举值 not_started/processing/completed/failed |
| 报告 ID | `report_id` | `string` | 接口返回 | 系统生成 UUID，从 bot_analysis_report 表获取 |
| 报告生成时间 | `report_created_at` | `string` | 接口返回 | ISO 8601 时间戳，从 bot_analysis_report 表获取 |
| 报告是否可查看 | `is_report_viewable` | `boolean` | 接口返回 | 基于 report_status=completed 计算，报告永久有效 |
| 独立站绑定状态 | `has_bound_sites` | `boolean` | 接口返回 | 服务端查询 tenant_sites 表判断是否有绑定站点 |
| 独立站列表 | `sites` | `array` | 接口返回 | 已绑定的独立站简要信息列表 |

**状态枚举说明**：

- 初始化状态：pending（待初始化）、completed（已完成）
- 数据来源类型：material（上传业务材料）、site（授权独立站）、null（未选择）
- 业务分析报告状态：not_started（未开始）、processing（分析中）、completed（已完成）、failed（失败）

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 多BOT工作台.html | 点击 [+ 创建新 BOT] 按钮 | 无（创建新 BOT 后跳转） |

**入口前置条件**：
- 用户已完成登录认证，持有有效的 tenant_token
- URL 中包含有效的 bot_id 参数（除创建新 BOT 场景外）

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 上传业务材料.html | 点击上传业务材料卡片并确认 | `bot_id` |
| 独立站授权.html | 点击授权独立站卡片并确认 | `bot_id` |
| 业务分析报告.html | 报告已完成时点击查看报告按钮 | `bot_id`, `report_id` |
| 多BOT工作台.html | 点击底部回到工作台按钮 | 无 |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 数据请求中 | 页面初始化，请求发出后 | 数据加载成功或失败 |
| 首次初始化 | 新 BOT 或未选择数据来源 | init_status=pending 或 data_source_type=null | 用户选择数据来源并确认 |
| 已有报告 | 检测到已有完成的报告 | report_status=completed | 用户点击重新分析并确认 |

| 错误状态 | 数据加载失败 | 接口返回 404/500 等错误 | 用户点击返回工作台 |

**状态流转图**：

```

[首次初始化] --(点击上传卡片)--> [确认弹窗] --> [上传业务材料]

```

## 业务处理逻辑

### 场景一：首次初始化

**触发条件**：新创建的 BOT 首次进入业务初始化页面，或 init_status=pending / data_source_type=null。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 bot_id
2. 系统执行处理：
   - 调用 GET /api/v1/bot/{bot_id}/init-status 查询 BOT 初始化状态
   - 若 init_status=pending 或 data_source_type=null，进入首次初始化状态
   - 展示两个初始化选项卡片：上传业务材料、授权独立站
   - 若 has_bound_sites=false，授权独立站卡片置灰禁用
   - 两个选项为互斥选择，用户选择后弹出确认弹窗提示选择后不可更改
3. 系统返回响应：页面渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| BOT 不存在 | Toast「该 BOT 不存在或已被删除」，3秒后自动跳转回多BOT工作台 |
| 租户无独立站 | 授权独立站卡片置灰，显示暂无可用独立站提示 |
| 网络超时 | Toast「网络连接异常，请检查网络后重试」，提供刷新按钮 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 首次初始化 | 数据返回成功且 init_status=pending |
| 加载中 | 错误状态 | 接口返回 404/500 |

**输出**：初始化选项页面，用户可选择数据来源方式。

---

### 场景二：已有报告

**触发条件**：BOT 已有完成的业务分析报告，report_status=completed。

**执行逻辑**：

1. 系统接收请求：页面加载，检测到 report_status=completed
2. 系统执行处理：
   - 展示查看报告按钮，点击跳转至业务分析报告页面
   - 同时展示重新分析选项，点击后弹出确认弹窗
   - 确认重新分析后，清除当前报告数据，重置为首次初始化状态
3. 系统返回响应：页面渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 报告生成中 | 若 report_status=processing，自动跳转至 AI业务分析中页面 |
| 报告生成失败 | 若 report_status=failed，提示上次分析失败，提供重新分析入口 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 已有报告 | report_status=completed |
| 已有报告 | 首次初始化 | 用户确认重新分析 |

**输出**：展示查看报告入口和重新分析选项。

---

### 场景三：选择数据来源

**触发条件**：用户点击上传业务材料或授权独立站卡片。

**执行逻辑**：

1. 系统接收请求：用户点击初始化选项卡片
2. 系统执行处理：
   - 用户选择上传业务材料：
     - 显示确认弹窗：选择后将无法切换，确认使用上传业务材料方式？
     - 用户确认后，调用接口更新 BOT 的 data_source_type=material
     - 跳转至上传业务材料页面
   - 用户选择授权独立站：
     - 显示确认弹窗：选择后将无法切换，确认使用授权独立站方式？
     - 用户确认后，调用接口更新 BOT 的 data_source_type=site
     - 跳转至独立站授权页面
   - 两个选项互斥，不可同时选择
3. 系统返回响应：页面跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 已有一种方式在进行中 | 禁用另一种方式，提示已选择数据来源，不可切换 |
| 用户取消确认 | 关闭弹窗，保持当前页面状态 |

**状态变更**：无（页面跳转）

**输出**：跳转至对应初始化页面。

---

### 场景四：数据来源选择确认

**触发条件**：用户点击卡片后确认选择。

**执行逻辑**：

1. 系统接收请求：用户点击确认按钮
2. 系统执行处理：
   - 调用 POST /api/v1/bot/{bot_id}/set-data-source 接口
   - 请求体：{ data_source_type: material | site }
   - 服务端更新 bot 表的 data_source_type 字段
   - 更新成功后跳转至对应页面
3. 系统返回响应：页面跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 接口调用失败 | Toast「设置失败，请稍后重试」，留在当前页面 |
| 已被其他方式初始化 | Toast「该 BOT 已选择数据来源，不可更改」 |

**状态变更**：首次初始化 → 跳转至子页面

**输出**：数据来源设置完成，进入对应数据采集流程。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 初始化状态 | 查询 | 从 bot 表查询 bot_id 对应记录的 init_status 字段 | MySQL bot 表 |
| 数据来源类型 | 查询 | 从 bot 表查询 bot_id 对应记录的 data_source_type 字段 | MySQL bot 表 |
| 业务分析报告状态 | 查询 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at DESC 取第一条的 status 字段 | MySQL bot_analysis_report 表 |
| 报告 ID | 查询 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at DESC 取第一条的 report_id 字段 | MySQL bot_analysis_report 表 |
| 报告生成时间 | 查询 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at DESC 取第一条的 created_at 字段 | MySQL bot_analysis_report 表 |
| 报告是否可查看 | 计算 | 服务端计算 report_status===completed | 内存计算 |
| 独立站绑定状态 | 查询+计算 | 从 tenant_sites 表查询 tenant_id 对应且 status=online 的记录数，判断 COUNT>0 | MySQL tenant_sites 表 |
| 独立站列表 | 查询 | 从 tenant_sites 表查询 tenant_id 对应且 status=online 的站点列表 | MySQL tenant_sites 表 |

### 数据来源深度说明

| 字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|------|-----------|------|----------|------------------------|
| 初始化状态 | `init_status` | `string` | 接口返回 | 从 bot 表查询 init_status 字段，枚举值 pending/completed |
| 数据来源类型 | `data_source_type` | `string` | 接口返回 | 从 bot 表查询 data_source_type 字段，枚举值 material/site/null |
| 业务分析报告状态 | `report_status` | `string` | 接口返回 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at 倒序取第一条的 status 字段 |
| 报告 ID | `report_id` | `string` | 接口返回 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at 倒序取第一条的 report_id 字段 |
| 报告生成时间 | `report_created_at` | `string` | 接口返回 | 从 bot_analysis_report 表按 bot_id 查询，按 created_at 倒序取第一条的 created_at 字段 |
| 报告是否可查看 | `is_report_viewable` | `boolean` | 系统生成 | 服务端计算字段 report_status===completed，仅 completed 状态可查看 |
| 独立站绑定状态 | `has_bound_sites` | `boolean` | 系统生成 | 服务端查询 tenant_sites 表，统计 tenant_id 对应且 status=online 的记录数，判断 COUNT>0 |
| 独立站列表 | `sites` | `array` | 接口返回 | 从 tenant_sites 表查询 tenant_id 对应且 status=online 的站点列表 |

### 数据来源设置逻辑

**设置数据来源类型**：

1. 校验 BOT 是否存在且属于当前租户
2. 校验 data_source_type 是否为 null（已设置则拒绝修改）
3. 更新 bot 表：UPDATE bot SET data_source_type=?, updated_at=NOW() WHERE bot_id=?
4. 记录操作日志到 bot_operation_log 表

## 数据流

### API 一：获取 BOT 初始化状态

```
GET /api/v1/bot/{bot_id}/init-status
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | 路径参数，BOT 唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "bot_id": "BOT-001",
    "init_status": "pending",
    "data_source_type": null,
    "report_status": "not_started",
    "report_id": null,
    "report_created_at": null,
    "is_report_viewable": false,
    "has_bound_sites": true,
    "sites": [
      {
        "site_id": "SITE-001",
        "site_name": "XX科技官方网站",
        "site_domain": "xx-tech.example.com"
      }
    ]
  }
}
```

**响应码说明**：

| code | 含义 | 处理方式 |
|------|------|---------|
| 200 | 查询成功 | 正常渲染页面 |
| 401 | 未登录 | 跳转登录页面 |
| 404 | BOT 不存在 | Toast「该 BOT 不存在或已被删除」，引导返回工作台 |
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」 |

### API 二：设置数据来源类型

```
POST /api/v1/bot/{bot_id}/set-data-source
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求体**：

```json
{
  "data_source_type": "material"
}
```

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| data_source_type | `string` | 是 | 数据来源类型：material / site |

**响应结构**：

```json
{
  "code": 200,
  "message": "设置成功",
  "data": {
    "bot_id": "BOT-001",
    "data_source_type": "material"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 设置成功 |
| 400 | 参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 404 | BOT 不存在 |
| 409 | 已设置数据来源，冲突 |
| 500 | 服务异常 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| BOT 不存在 | code 404 | 返回 404 错误 | Toast「该 BOT 不存在或已被删除」，3秒后跳转回多BOT工作台 |
| 无权限访问 | code 403 | 返回 403 错误 | Toast「无权限访问该 BOT」 |
| 未登录 | code 401 | 返回 401 错误 | 跳转登录页面，Toast「请先登录」 |
| 网络超时 | 请求超时 30s | 返回 504 | Toast「网络连接异常，请检查网络后重试」，显示刷新按钮 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
| 已选数据来源 | code 409 | 返回 409 冲突 | Toast「该 BOT 已选择数据来源，不可更改」 |
| 设置失败 | code 500 | 返回 500 错误 | Toast「设置失败，请稍后重试」 |
