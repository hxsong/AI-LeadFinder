## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 访客 ID | `visitor_log_id` | `string` | 系统生成 | 系统生成 UUID |
| IP 地址（脱敏） | `ip_masked` | `string` | GeoIP 脱敏处理 | — |
| 地理位置 | `geo_location` | `string` | GeoIP 定位数据 | — |
| 热度等级 | `heat_level` | `string` | 系统映射 | very_high / high / normal |
| 规则打分 | `rule_score` | `number` | 行为打分引擎计算 | 0-100 |
| 设备类型 | `device_type` | `string` | 探针 JS 采集 UA 解析 | Desktop/Mobile/Tablet |
| 操作系统 | `os` | `string` | 探针 JS 采集 UA 解析 | Windows/macOS/iOS/Android |
| 流量来源 | `traffic_source` | `string` | 流量归因解析 | Google 搜索/直接访问等 |
| 落地页 | `landing_page` | `string` | 首次访问 URL 路径 | — |
| 访问次数 | `visit_count` | `number` | 同一 IP 历史访问次数 | — |
| 首次访问 | `first_visit` | `string` | 接口返回 | ISO 8601 日期 |
| 最近访问 | `last_visit` | `string` | 接口返回 | ISO 8601 日期 |
| 是否已挖掘 | `is_mined` | `boolean` | 接口返回 | — |
| AI 分析是否触发 | `ai_analysis_triggered` | `boolean` | 系统判断 | 规则打分 ≥ 60 时为 true |
| 深度挖掘预估 Credits | `estimated_credits` | `number` | 系统计算 | IP 解析 + 邮箱揭示等 |

## 数据流

### 获取访客列表

```
GET /api/v1/visitors
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| heat_level | `string` | 否 | 热度等级：very_high / high / normal / all（默认 all） |
| region | `string` | 否 | 地区：NA / EU / SEA / all（默认 all） |
| date_range | `string` | 否 | 日期范围：today / 7d / 30d / all（默认 today） |
| start_date | `string` | 否 | 自定义开始日期，YYYY-MM-DD |
| end_date | `string` | 否 | 自定义结束日期，YYYY-MM-DD |
| page | `number` | 否 | 页码，默认 1 |
| limit | `number` | 否 | 每页数量，默认 20 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 47,
    "today_count": 12,
    "items": [
      {
        "visitor_log_id": "VIS-99482X",
        "ip_masked": "67.43.156.**",
        "geo_location": "Florida, USA",
        "heat_level": "very_high",
        "rule_score": 95,
        "device_type": "Desktop",
        "traffic_source": "Google 搜索",
        "visit_count": 3,
        "first_visit": "2024-02-24",
        "last_visit": "2024-02-27",
        "is_mined": false,
        "ai_analysis_triggered": true
      }
    ]
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 查询成功 |
| 401 | 未登录 |
| 500 | 服务异常 |

### 触发深度挖掘

```
POST /api/v1/visitors/{visitor_log_id}/reveal
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| visitor_log_id | `string` | 是 | 路径参数，访客记录 ID |
| options | `object` | 否 | 包含 include_linkedin, include_social 等选项 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "task_abc123",
    "status": "processing",
    "estimated_credits": 15,
    "message": "深度挖掘任务已启动"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 404 | 访客不存在 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：正常浏览访客列表

**触发条件**：页面正常加载，无进行中任务。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id`
2. 系统执行处理：
   - 页面加载时调用 `GET /api/v1/visitors` 获取访客列表
   - 展示统计卡片：今日访客数 / 近7日访客数 / 高热度访客数 / 已挖掘访客数
   - 默认显示全部地区访客，默认选择当日日期
   - 右侧显示"实时更新中"状态指示器
3. 系统返回响应：返回访客列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选条件下无数据 | 显示空态引导文案 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功 |
| 加载中 | 空态 | 数据为空 |

**输出**：访客卡片列表，支持筛选和翻页。

---

### 场景二：热度筛选

**触发条件**：用户点击热度标签（全部/极高/较高/一般）。

**执行逻辑**：

1. 系统接收请求：用户选择热度标签
2. 系统执行处理：
   - 热度定义：极高热度（红色，规则打分 ≥ 80）、较高热度（橙色，规则打分 60-79）、一般热度（绿色，规则打分 < 60）
   - 用户点击热度标签，列表自动过滤为对应热度等级
   - 刷新统计卡片中的高热度访客计数
3. 系统返回响应：返回过滤后的访客列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按热度等级过滤后的访客列表。

---

### 场景三：地区筛选

**触发条件**：用户从下拉菜单选择地区。

**执行逻辑**：

1. 系统接收请求：用户选择地区
2. 系统执行处理：
   - 地区选项：全部地区 / 北美 / 欧洲 / 东南亚
   - 基于 GeoIP 定位数据过滤
   - 筛选后刷新列表和统计卡片
3. 系统返回响应：返回过滤后的访客列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按地区过滤后的访客列表。

---

### 场景四：时间筛选

**触发条件**：用户选择日期范围。

**执行逻辑**：

1. 系统接收请求：用户选择日期范围
2. 系统执行处理：
   - 快捷选项：今日（默认）/ 近7天 / 近30天 / 全部
   - 自定义日期范围：开始日期选择器 + 结束日期选择器，点击"应用"确认
   - 结束日期不能晚于当日（today）
   - 日期格式：YYYY-MM-DD
   - 切换日期后自动触发数据查询
3. 系统返回响应：返回过滤后的访客列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 结束日期晚于当日 | 自动截断至当日 |
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按日期范围过滤后的访客列表。

---

### 场景五：深度挖掘访客

**触发条件**：用户点击卡片右侧"深度挖掘"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 弹出深度挖掘弹窗，展示计费规则：IP 企业解析 = 1 Credit，每获取 1 个验证邮箱 = 1 Credit，附加 LinkedIn = 0 Credit（免费），附加 Twitter/FB = 0 Credit/个（免费）
   - 若检测到 UTM 参数，显示"系统将优先尝试关联静态数据中已有的企业信息"提示
   - 用户确认后，前端发送 `POST /api/v1/visitors/{visitor_log_id}/reveal`
   - 进入页面时查询任务状态，若存在进行中任务自动跳转到动态线索池
3. 系统返回响应：返回挖掘任务状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| Credits 余额 ≤ 0 | 按钮禁用，弹窗提示充值 |
| 已挖掘的访客 | 显示"已挖掘"状态，不重复触发 |
| 挖掘请求超时 | 弹窗提示重试 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 深度挖掘中 | 用户点击深度挖掘按钮 |
| 深度挖掘中 | 正常 | 挖掘接口返回失败 |

**输出**：异步执行，IP 解析 + 联系人挖掘在后台进行，完成后自动进入动态线索池。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| Credits 不足 | 余额 ≤ 0 | 拒绝挖掘请求 | 按钮禁用，弹窗提示充值，Toast「Credits 余额不足（当前：{balance}），请充值后再试」 |
| 服务不可用 | HTTP 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
| 访客不存在 | HTTP 404 | 返回错误 | 提示并返回列表，Toast「该访客记录不存在或已被清除」 |
| 深度挖掘已达上限 | 已挖掘访客重复点击 | — | 按钮禁用，Toast「该访客已达到最大挖掘次数」 |
| 挖掘请求超时 | 请求超时 | — | 弹窗提示重试，Toast「挖掘请求超时，请稍后重试」 |
| 数据过期 | 缓存数据过期 | — | 提示刷新，Toast「部分数据已过期，请刷新获取最新数据」 |
| 实时连接中断 | WebSocket 断开 | 尝试重连 | 提示重连，Toast「实时连接已断开，正在尝试重新连接」 |
