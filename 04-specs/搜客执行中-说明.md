## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 任务 ID | `task_id` | `string` | 路由参数传入 | 搜客任务唯一标识 |
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 整体进度 | `overall_progress` | `number` | 接口返回 | 0-100 整数 |
| 整体状态 | `overall_status` | `string` | 接口返回 | processing / completed / failed |
| 阶段编码 | `stages[].stage` | `string` | 接口返回 | P16-P18 / P19-P20 / P20-P21 |
| 阶段名称 | `stages[].name` | `string` | 接口返回 | 企业信息提取 / ICP 匹配打分 / 企业信息增强 |
| 阶段状态 | `stages[].status` | `string` | 接口返回 | pending / in_progress / completed / failed |

| 候选企业数量 | `stages[].stats.candidates_found` | `number` | 接口返回 | API 返回总数 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 搜客策略预览.html | 用户点击"确认执行"按钮 | `bot_id`, `task_id` |
| 搜客需求输入.html | 进入页面时检测到已有任务在执行 | `bot_id`, `task_id` |

**入口前置条件**：搜客任务已创建且状态为 running 或 processing。

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 搜客结果列表.html | 所有步骤完成后自动跳转 | `bot_id`, `task_id` |
| 搜客记录.html | 用户主动点击侧边栏"搜客记录" | — |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 初始化 | 页面加载，开始轮询 | 页面加载 | 首次轮询请求发出 |
| 企业信息提取中 | P16-P18 阶段执行中 | stages[0].status = in_progress | 阶段完成或失败 |
| ICP 匹配打分中 | P19-P20 阶段执行中 | stages[1].status = in_progress | 阶段完成或失败 |
| 企业信息增强中 | P20-P21 阶段执行中 | stages[2].status = in_progress | 阶段完成或失败 |
| 执行完成 | 所有阶段执行完毕 | overall_status = completed | 自动跳转 |
| 执行失败 | 任一阶段执行失败 | 任一 stages[].status = failed | 用户点击重试或返回 |

**状态流转图**：
```
初始化 --(P16开始)--> 企业信息提取中 --(完成)--> ICP 匹配打分中 --(完成)--> 企业信息增强中 --(完成)--> 执行完成 --(自动跳转)--> 搜客结果列表.html
   │                          │                       │                        │
   └──────────────────────────┴───────────────────────┴────────────────────────┘
                              (任一阶段失败)
                                      │
                                      ▼
                                执行失败 --(重试)--> 初始化
```

## 数据流

### 获取搜客任务状态

```
GET /api/v1/search/tasks/{task_id}/status
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | `string` | 是 | 路径参数，搜客任务唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "TASK-202401-001",
    "overall_progress": 65,
    "overall_status": "processing",
    "stages": [
      {
        "stage": "P16-P18",
        "name": "企业信息提取",
        "status": "completed",
        "progress": 100,
        "stats": { "candidates_found": 150, "deduplicated": 98 }
      },
      {
        "stage": "P19-P20",
        "name": "ICP 匹配打分",
        "status": "in_progress",
        "progress": 45,
        "stats": { "scored": 44, "total": 98, "above_threshold": 38 }
      },
      {
        "stage": "P20-P21",
        "name": "企业信息增强",
        "status": "pending",
        "progress": 0,
        "stats": { "enhanced": 0, "total": 38 }
      }
    ]
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 查询成功 |
| 401 | 未登录 |
| 404 | 任务不存在 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：企业信息提取（P16-P18）

**触发条件**：搜客任务启动，进入企业信息提取阶段。

**执行逻辑**：

1. 系统接收请求：任务状态接口返回 stages[0].status = in_progress
2. 系统执行处理：
   - 调用数据源 API（出海派 / 10times / Google Maps）提取候选企业列表
   - ETL 数据清洗：去重、补全字段、标准化格式
   - 每提取一批企业，前端更新进度条
   - 最终输出候选企业基础信息列表（含企业名称、官网等）
3. 系统返回响应：更新阶段状态和进度

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 单个 API 调用超时 | 1 分钟超时，跳过本次调用继续执行 |
| API 调用失败 | 最多重试 3 次，间隔 5s/10s/30s |
| 无候选企业 | 标记阶段完成，但 stats.candidates_found = 0 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 初始化 | 企业信息提取中 | stages[0].status = in_progress |
| 企业信息提取中 | ICP 匹配打分中 | stages[0].status = completed |
| 企业信息提取中 | 执行失败 | stages[0].status = failed |

**输出**：清洗后的候选企业列表，进入 ICP 匹配阶段。

---

### 场景二：ICP 匹配打分（P19-P20）

**触发条件**：企业信息提取完成，进入 ICP 匹配打分阶段。

**执行逻辑**：

1. 系统接收请求：任务状态接口返回 stages[1].status = in_progress
2. 系统执行处理：
   - 将候选企业信息与 ICP 画像输入 AI 评分模型
   - AI 模型判断供需关系，计算需求匹配度
   - 结合四维权重（需求匹配度/购买能力/采购意向/活跃意向）加权计算最终得分
   - 每家企业评分完成后前端更新列表进度
   - 过滤低于阈值的企业（可配置，默认 60 分）
3. 系统返回响应：更新阶段状态和进度

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| AI 调用失败 | 使用规则引擎替代 AI 计算 |
| AI 评分超出 0-100 范围 | 强制截断至 0-100 |
| 评分结果为空 | 默认评分 0，不阻断流程 |
| 无企业达到阈值 | stats.above_threshold = 0 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 企业信息提取中 | ICP 匹配打分中 | stages[1].status = in_progress |
| ICP 匹配打分中 | 企业信息增强中 | stages[1].status = completed |
| ICP 匹配打分中 | 执行失败 | stages[1].status = failed |

**输出**：按评分排序的候选企业列表，含 AI 推荐理由。

---

### 场景三：企业信息增强（P20-P21）

**触发条件**：ICP 匹配打分完成，进入企业信息增强阶段。

**执行逻辑**：

1. 系统接收请求：任务状态接口返回 stages[2].status = in_progress
2. 系统执行处理：
   - 对高评分企业调用增强数据接口
   - 补充海关数据（HS Code、进口记录）
   - 补充展会数据（参展记录）
   - 补充地图数据（评分、地址、营业状态）
3. 系统返回响应：更新阶段状态和进度

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 增强接口超时 | 跳过该企业，继续处理其他企业 |
| 增强数据不存在 | 标记该企业无对应数据，不阻断流程 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| ICP 匹配打分中 | 企业信息增强中 | stages[2].status = in_progress |
| 企业信息增强中 | 执行完成 | stages[2].status = completed |
| 企业信息增强中 | 执行失败 | stages[2].status = failed |

**输出**：增强后的企业线索列表，含原始数据证据。

---

### 场景四：处理进度实时展示

**触发条件**：页面加载后，任务执行中。

**执行逻辑**：

1. 系统接收请求：页面加载，开始轮询任务状态
2. 系统执行处理：
   - 展示三大阶段进度：企业信息提取 → ICP 匹配打分 → 企业信息增强
   - 每阶段展示子步骤状态
   - 步骤状态切换：等待中（灰色）→ 执行中（蓝色+旋转图标）→ 已完成（绿色+勾选图标）
   - 进度条动画：平滑递增，带波浪流动效果
   - 中心动画：三层同心圆，最内层旋转，外层脉冲扩散
   - 所有步骤完成后自动跳转至搜客结果列表
3. 系统返回响应：实时更新 UI

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 轮询请求超时 | 继续下一次轮询，前端保持当前状态 |
| 网络断开 | 停止轮询，前端展示网络异常提示 |
| 用户关闭页面 | 异步任务继续在后台执行，不影响处理 |
| 跳转失败 | 前端展示手动跳转按钮 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 初始化 | 企业信息提取中 | stages[0].status = in_progress |
| 企业信息提取中 | ICP 匹配打分中 | stages[0].status = completed |
| ICP 匹配打分中 | 企业信息增强中 | stages[1].status = completed |
| 企业信息增强中 | 执行完成 | stages[2].status = completed |
| 执行完成 | 已跳转 | 自动跳转至搜客结果列表 |

**输出**：实时进度 UI，处理完成自动跳转。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 任务不存在 | 接口返回 404 | 返回错误 | Toast「任务不存在或已过期」 |
| 步骤执行失败 | stages[].status = failed | 记录错误日志，触发重试 | 失败提示 + 重试按钮，Toast「{step_name} 执行失败，正在重试...」 |
| API 超时 | 请求超时 1 分钟 | 跳过继续执行 | Toast「网络请求超时，任务将自动重试，请稍候」 |
| Credits 余额不足 | 余额 ≤ 0 | 任务中断，记录欠费状态 | 任务中断提示，Toast「Credits 余额不足，任务已暂停」 |
| AI 评分为空 | ai_score 为空 | 重试或默认值 | Toast「AI 评分结果为空，请重试」 |
| AI 格式错误 | JSON 解析失败 | 自动重试 | Toast「AI 返回数据格式异常，正在自动重试...」 |
| AI 评分异常 | 评分与数据矛盾 | 标记仅供参考 | Toast「AI 评分存在异常，建议重新分析」 |
| 评分超出范围 | 评分 <0 或 >100 | 默认值替代 | Toast「AI 评分异常，已使用默认值」 |
| 数据解析失败 | 原始数据保存 | 默认值处理 | Toast「数据解析失败，已使用默认值」 |
| 网络超时 | 请求超时 | 错误提示 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | 接口返回 500 | 全局错误处理 | Toast「服务暂时不可用，请稍后再试」 |
| 页面关闭 | 用户关闭页面 | 任务继续在后台异步执行 | — |
