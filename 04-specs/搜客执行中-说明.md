## 概述

搜客执行中页面是搜客任务的执行监控页面，实时展示从企业信息提取、ICP 匹配打分到企业信息增强全流程的各步骤执行状态。前端实时轮询后端任务状态，通过动画展示任务进度，处理完成后自动跳转至搜客结果列表。

## 功能逻辑

### 场景一：企业信息提取（P16-P18）

**触发条件**：搜客任务启动。

**执行逻辑**：
1. 调用数据源 API（出海派 / 10times / Google Maps）提取候选企业列表
2. ETL 数据清洗：去重、补全字段、标准化格式
3. 每提取一批企业，前端更新进度条
4. 最终输出候选企业基础信息列表（含企业名称、官网等）

**输出**：清洗后的候选企业列表，进入 ICP 匹配阶段。

---

### 场景二：ICP 匹配打分（P19-P20）

**触发条件**：企业信息提取完成。

**执行逻辑**：
1. 将候选企业信息与 ICP 画像输入 AI 评分模型
2. AI 模型判断供需关系，计算需求匹配度
3. 结合四维权重（需求匹配度/购买能力/采购意向/活跃意向）加权计算最终得分
4. 每家企业评分完成后前端更新列表进度
5. 过滤低于阈值的企业（可配置，默认 60 分）

**输出**：按评分排序的候选企业列表，含 AI 推荐理由。

---

### 场景三：企业信息增强（P20-P21）

**触发条件**：ICP 匹配打分完成。

**执行逻辑**：
1. 对高评分企业调用增强数据接口
2. 补充海关数据（HS Code、进口记录）
3. 补充展会数据（参展记录）
4. 补充地图数据（评分、地址、营业状态）

**输出**：增强后的企业线索列表，含原始数据证据。

---

### 场景四：处理进度实时展示

**触发条件**：页面加载后任务执行中。

**执行逻辑**：
1. 展示三大阶段进度：企业信息提取 → ICP 匹配打分 → 企业信息增强
2. 每阶段展示子步骤状态
3. 步骤状态切换：等待中（灰色）→ 执行中（蓝色+旋转图标）→ 已完成（绿色+勾选图标）
4. 进度条动画：平滑递增，带波浪流动效果
5. 中心动画：三层同心圆，最内层旋转，外层脉冲扩散
6. 所有步骤完成后自动跳转至搜客结果列表

**输出**：实时进度 UI。

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 页面关闭 | 用户关闭页面 | 任务继续在后台异步执行 |
| 接口超时 | 单次 API 调用超时 | 1 分钟超时，跳过本次调用继续执行 |
| 任务不阻塞 | 单个接口失败 | 确保任务最终完成 |
| 重试策略 | API 调用失败 | 最多重试 3 次，间隔 5s/10s/30s |
| 降级处理 | AI 调用失败 | 使用规则引擎替代 AI 计算 |
| Credits 退还 | AI 调用失败 | 退还已扣除的 Credits |
| 评分阈值 | 默认 60 分 | 低于阈值的候选企业不展示 |
| 评分范围 | AI 评分 | 必须在 0-100 范围内，超出则触发异常 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 任务不存在 | code 404 | 错误提示 | 「任务不存在或已过期」 |
| 步骤执行失败 | status = failed | 失败提示 + 重试按钮 | 「{step_name} 执行失败，正在重试...」 |
| API 超时 | 请求超时 1 分钟 | 跳过继续执行 | 「网络请求超时，任务将自动重试，请稍候」 |
| Credits 余额不足 | 余额 ≤ 0 | 任务中断提示 | 「Credits 余额不足，任务已暂停」 |
| AI 评分为空 | ai_score 为空 | 重试或默认值 | 「AI 评分结果为空，请重试」 |
| AI 格式错误 | JSON 解析失败 | 自动重试 | 「AI 返回数据格式异常，正在自动重试...」 |
| AI 评分异常 | 评分与数据矛盾 | 标记仅供参考 | 「AI 评分存在异常，建议重新分析」 |
| 评分超出范围 | 评分 <0 或 >100 | 默认值替代 | 「AI 评分异常，已使用默认值」 |
| 数据解析失败 | 原始数据保存 | 默认值处理 | 「数据解析失败，已使用默认值」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
