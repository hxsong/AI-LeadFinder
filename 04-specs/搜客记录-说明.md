## 概述

搜客记录页面展示当前 BOT 下所有历史搜客任务的记录列表。用户可通过状态筛选和时间筛选快速定位目标任务，点击已完成任务跳转到搜客结果列表查看详情，对失败任务执行重试操作。

---

## 页面入口与出口

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 多BOT工作台.html | 点击侧边栏"搜客记录" | `bot_id` |
| 搜客需求输入.html | 搜客任务启动后跳转 | `bot_id` |
| 搜客执行中.html | 任务完成后自动跳转 | `bot_id`, `task_id` |

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 搜客需求输入.html | 点击"新建搜客"按钮 | `bot_id` |
| 搜客结果列表.html | 点击已完成的任务记录 | `bot_id`, `task_id` |

## 功能逻辑

### 场景一：查看搜客记录列表

**触发条件**：用户进入搜客记录页面。

**执行逻辑**：
1. 前端请求搜客记录列表（默认按创建时间倒序）
2. 每条记录展示：搜索需求摘要、状态标签、创建时间、结果数量、消耗 Credits
3. 状态标签：进行中（蓝色旋转）/ 已完成（绿色勾选）/ 失败（红色叉号）
4. 分页展示，每页 10 条，最多翻至第 100 页

**输出**：搜客任务记录列表。

---

### 场景二：状态筛选

**触发条件**：用户点击状态筛选按钮。

**执行逻辑**：
1. 选项：全部 / 进行中 / 已完成 / 失败
2. 点击后列表实时刷新，按选定状态过滤
3. 进行中任务显示蓝色旋转加载图标和进度条，不可点击

**输出**：按状态过滤后的记录列表。

---

### 场景三：时间筛选

**触发条件**：用户选择时间范围。

**执行逻辑**：
1. 快捷选项：今日 / 近7天 / 近30天 / 全部
2. 自定义日期：开始日期 + 结束日期，点击"应用"确认
3. 筛选后列表实时刷新

**输出**：按时间范围过滤后的记录列表。

---

### 场景四：重试失败任务

**触发条件**：用户点击失败任务的重试按钮。

**执行逻辑**：
1. 校验 Credits 余额（余额可为负，≤ 0 时禁止重试）
2. 调用重试接口，创建新任务
3. 重试成功后跳转到搜客执行中页面
4. 单个任务最多重试 3 次

**输出**：重试任务创建成功，跳转执行页。

---

### 场景五：查看已完成任务详情

**触发条件**：用户点击已完成的任务记录。

**执行逻辑**：
1. 跳转到搜客结果列表.html，传递 task_id
2. 展示该任务的完整结果

**输出**：跳转至搜客结果列表。

| 状态 | 触发条件 | UI 表现 |
|-----|---------|---------|
| 正常 | 数据加载成功 | 展示记录列表 |

## 数据流

### 请求一：获取搜客记录列表

```
GET /api/v1/search/records
```

**请求头**：`Authorization: Bearer {tenant_token}`

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | string | 是 | BOT ID |
| status | string | 否 | 状态筛选：all / processing / completed / failed |
| start_date | string | 否 | 开始日期 YYYY-MM-DD |
| end_date | string | 否 | 结束日期 YYYY-MM-DD |
| page | number | 否 | 页码，默认 1 |
| limit | number | 否 | 每页数量，默认 10 |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "total": 12,
    "page": 1,
    "limit": 10,
    "items": [

        "high_potential_count": 8,
        "credits_consumed": 45,
        "created_at": "2024-01-20T10:00:00Z"
      },
      {
        "task_id": "TASK-202401-002",
        "status": "processing",
        "progress": 65,

}
```

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 记录保留期限 | 搜客记录保留 | 90 天，超期自动归档 |
| 结果缓存期限 | 搜客结果保留 | 7 天 |
| 数据权限 | 查看权限 | 仅可查看本租户（当前 BOT）下的记录 |
| 重试 Credits | 余额可为负，≤0 禁止重试 | 重试任务执行中不扣费，完成后按实际消耗结算 |
| 重试次数限制 | 单个任务最多重试 | 3 次 |
| 分页深度 | 每页最大记录数 | 10 条，最大翻至第 100 页 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/search/records | GET | 获取搜客记录列表 | 页面加载、筛选条件变化 |
| /api/v1/search/retry | POST | 重试失败任务 | 点击重试按钮 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 网络超时 | 请求超时 30s | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| Credits 不足 | 余额 ≤ 0 时点击重试 | 重试按钮禁用 | 「Credits 余额不足（当前：{balance}），无法执行重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| 重试次数已达上限 | retry_count ≥ 3 | 重试按钮禁用 | 「该任务已达最大重试次数（3 次）」 |
| 无数据可展示 | 筛选条件下无记录 | 空态引导 | 「暂无符合条件的搜客记录」 |
| 数据已过期 | 缓存过期 | 提示刷新 | 「部分数据已过期，请刷新获取最新数据」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 任务 ID | task_id | 系统生成 UUID |
| 搜索需求摘要 | demand_summary | 搜客需求输入内容摘要 |
| 任务状态 | status | processing / completed / failed |
| 结果总数 | result_count | 候选企业数量 |
| 高潜力数量 | high_potential_count | 评分 ≥ 阈值的企业数量 |
| Credits 消耗 | credits_consumed | 任务实际消耗 |
| 任务进度 | progress | 进行中任务当前进度 0-100 |
| 错误原因 | error_message | 失败时记录错误信息 |
| 重试次数 | retry_count | 已重试次数，≤ 3 |
| 创建时间 | created_at | ISO 8601 时间戳 |
