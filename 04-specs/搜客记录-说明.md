## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 任务 ID | `task_id` | `string` | 系统生成 | 搜客任务唯一标识 |
| 搜索需求摘要 | `demand_summary` | `string` | 用户输入摘要 | 搜客需求输入内容摘要 |
| 任务状态 | `status` | `string` | 接口返回 | processing / completed / failed |
| 结果总数 | `result_count` | `number` | 接口返回 | 候选企业数量 |
| 高潜力数量 | `high_potential_count` | `number` | 接口返回 | 评分 ≥ 阈值的企业数量 |
| Credits 消耗 | `credits_consumed` | `number` | 接口返回 | 任务实际消耗 Credits |
| 任务进度 | `progress` | `number` | 接口返回 | 进行中任务当前进度 0-100 |
| 错误原因 | `error_message` | `string` | 接口返回 | 失败时记录错误信息 |
| 重试次数 | `retry_count` | `number` | 接口返回 | 已重试次数，≤ 3 |
| 创建时间 | `created_at` | `string` | 接口返回 | ISO 8601 时间戳 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 多BOT工作台.html | 点击侧边栏"搜客记录" | `bot_id` |

| 搜客执行中.html | 任务完成后自动跳转 | `bot_id`, `task_id` |

### 出口

| 目标 | 触发条件 | 携带数据 |

| 搜客需求输入.html | 点击"新建搜客"按钮 | `bot_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后 | 页面加载 | 数据返回 |
| 正常 | 数据加载成功 | 列表数据返回 | 用户操作 |
| 空态 | 无记录数据 | 列表为空 | 用户操作 |

## 数据流

### 获取搜客记录列表

```
GET /api/v1/search/records
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | BOT ID |
| status | `string` | 否 | 状态筛选：all / processing / completed / failed |
| start_date | `string` | 否 | 开始日期 YYYY-MM-DD |
| end_date | `string` | 否 | 结束日期 YYYY-MM-DD |
| page | `number` | 否 | 页码，默认 1 |
| limit | `number` | 否 | 每页数量，默认 10 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 12,
    "page": 1,
    "limit": 10,
    "items": [
      {
        "task_id": "TASK-202401-001",
        "demand_summary": "欧洲户外屏代理商",
        "status": "completed",
        "result_count": 98,
        "high_potential_count": 8,
        "credits_consumed": 45,
        "created_at": "2024-01-20T10:00:00Z"
      },
      {
        "task_id": "TASK-202401-002",
        "status": "processing",
        "progress": 65,
        "created_at": "2024-01-20T11:00:00Z"
      },
      {
        "task_id": "TASK-202401-003",
        "demand_summary": "北美工具代理商",
        "status": "failed",
        "error_message": "数据源 API 超时",
        "retry_count": 2,
        "created_at": "2024-01-19T14:30:00Z"
      }
    ]
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 查询成功 |
| 401 | 未登录 |
| 500 | 服务异常 |

### 重试失败任务

```
POST /api/v1/search/retry
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | `string` | 是 | 失败任务 ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "new_task_id": "TASK-202401-003-R1",
    "status": "pending"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 重试任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 404 | 任务不存在 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：查看搜客记录列表

**触发条件**：用户进入搜客记录页面。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id`
2. 系统执行处理：
   - 前端请求搜客记录列表（默认按创建时间倒序）
   - 每条记录展示：搜索需求摘要、状态标签、创建时间、结果数量、消耗 Credits
   - 状态标签：进行中（蓝色旋转）/ 已完成（绿色勾选）/ 失败（红色叉号）
   - 分页展示，每页 10 条，最多翻至第 100 页
3. 系统返回响应：返回记录列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无记录数据 | 显示空态引导文案 |
| 进行中任务 | 显示蓝色旋转加载图标和进度条，不可点击 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功 |
| 加载中 | 空态 | 数据为空 |

**输出**：搜客任务记录列表。

---

### 场景二：状态筛选

**触发条件**：用户点击状态筛选按钮。

**执行逻辑**：

1. 系统接收请求：用户选择筛选条件
2. 系统执行处理：
   - 选项：全部 / 进行中 / 已完成 / 失败
   - 点击后列表实时刷新，按选定状态过滤
   - 进行中任务显示蓝色旋转加载图标和进度条，不可点击
3. 系统返回响应：返回过滤后的记录列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按状态过滤后的记录列表。

---

### 场景三：时间筛选

**触发条件**：用户选择时间范围。

**执行逻辑**：

1. 系统接收请求：用户选择时间范围
2. 系统执行处理：
   - 快捷选项：今日 / 近7天 / 近30天 / 全部
   - 自定义日期：开始日期 + 结束日期，点击"应用"确认
   - 筛选后列表实时刷新
3. 系统返回响应：返回过滤后的记录列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 开始日期 > 结束日期 | 提示"开始日期不能晚于结束日期" |
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按时间范围过滤后的记录列表。

---

### 场景四：重试失败任务

**触发条件**：用户点击失败任务的重试按钮。

**执行逻辑**：

1. 系统接收请求：用户点击重试按钮
2. 系统执行处理：
   - 校验 Credits 余额（余额可为负，≤ 0 时禁止重试）
   - 调用重试接口 `POST /api/v1/search/retry`
   - 创建新任务，跳转到搜客执行中页面
   - 单个任务最多重试 3 次
3. 系统返回响应：返回新任务 ID

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| Credits 余额 ≤ 0 | 重试按钮禁用，提示"Credits 余额不足，无法执行重试" |
| 重试次数 ≥ 3 | 重试按钮禁用，提示"该任务已达最大重试次数（3 次）" |
| 任务不存在 | 提示"该任务不存在或已过期" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 重试中 | 用户点击重试按钮 |
| 重试中 | 正常 | 重试接口返回失败 |

**输出**：重试任务创建成功，跳转执行页。

---

### 场景五：查看已完成任务详情

**触发条件**：用户点击已完成的任务记录。

**执行逻辑**：

1. 系统接收请求：用户点击已完成记录
2. 系统执行处理：
   - 跳转到搜客结果列表，传递 `bot_id` 和 `task_id`
   - 展示该任务的完整结果
3. 系统返回响应：页面跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 任务结果已过期（>7天） | 提示"搜索结果已过期，请重新搜客" |

**状态变更**：无

**输出**：跳转至搜客结果列表。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| Credits 不足 | 余额 ≤ 0 时点击重试 | 拒绝重试请求 | 重试按钮禁用，Toast「Credits 余额不足（当前：{balance}），无法执行重试」 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
| 重试次数已达上限 | retry_count ≥ 3 | — | 重试按钮禁用，Toast「该任务已达最大重试次数（3 次）」 |
| 无数据可展示 | 筛选条件下无记录 | — | 空态引导，Toast「暂无符合条件的搜客记录」 |
| 数据已过期 | 缓存过期 | — | 提示刷新，Toast「部分数据已过期，请刷新获取最新数据」 |
| 任务不存在 | code 404 | 返回错误 | Toast「该任务不存在或已过期」 |
