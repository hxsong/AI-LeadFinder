## 数据契约

### 前端展示字段映射

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识，格式：`BOT-{timestamp}-{random}` |
| 任务 ID | `task_id` | `string` | 系统生成 | 搜客任务唯一标识，格式：`TASK-{YYYYMM}-{sequence}` |
| 搜索需求摘要 | `demand_summary` | `string` | 用户输入摘要 | 截取用户输入前 50 字符，超出显示"..." |
| 任务标题 | `task_title` | `string` | 系统生成 | 从搜索需求摘要中截取前 12 个字符作为标题，如"加州音视频集成商..." |
| 任务状态 | `status` | `enum` | 接口返回 | `pending`/`completed`/`failed`，进行中的任务不显示在记录列表 |
| 结果总数 | `result_count` | `number` | 接口返回 | 候选企业数量 |
| 高潜力数量 | `high_potential_count` | `number` | 接口返回 | 评分 ≥ 60 分（A级及以上）的企业数量 |
| MoliAI点数消耗 | `moli_ai_consumed` | `number` | 接口返回 | 任务实际消耗 MoliAI点数 |
| 目标企业数 | `target_count` | `number` | 用户输入 | 用户设置的目标企业数量（20-200） |
| 数据源类型 | `data_sources` | `array` | 系统生成 | 使用的数据源：`map`/`customs`/`exhibition`/`web` |
| 错误原因 | `error_message` | `string` | 接口返回 | 失败时记录错误信息，最大 200 字符 |
| 错误代码 | `error_code` | `string` | 接口返回 | 错误类型标识，如 `DATA_SOURCE_TIMEOUT` |
| 创建时间 | `created_at` | `string` | 系统生成 | ISO 8601 时间戳，如 `2024-01-20T10:00:00Z` |
| 完成时间 | `completed_at` | `string` | 系统生成 | ISO 8601 时间戳，仅 completed/failed 状态有值 |
| 数据过期时间 | `expires_at` | `string` | 系统生成 | 结果数据过期时间，创建时间 + 90 天 |
| 总搜客次数 | `stats.total_tasks` | `number` | 接口返回 | 当前 BOT 历史搜客任务总数 |
| 累计消耗 | `stats.total_moli_ai_consumed` | `number` | 接口返回 | 历史累计消耗 MoliAI点数 总数 |
| 发现企业 | `stats.total_companies_found` | `number` | 接口返回 | 历史累计发现企业数量 |

### 数据来源深度说明

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 任务标题 | `task_title` | `string` | 系统生成 | 从 `demand_summary` 字段截取前 12 个字符作为任务标题，若原始需求不足 12 字符则使用完整内容。任务创建时生成一次，存储于 `search_tasks.task_title` 字段 |
| 搜索需求摘要 | `demand_summary` | `string` | 用户输入摘要 | 服务端对用户原始输入进行截断处理，保留前 50 个字符，超出部分添加"..."后缀。存储于 `search_tasks` 表的 `demand_summary` 字段 |
| 结果总数 | `result_count` | `number` | 接口返回 | 从 `search_task_stats` 表查询，任务完成时写入。统计搜客任务最终入库的候选企业数量 |
| 高潜力数量 | `high_potential_count` | `number` | 接口返回 | 从 `search_task_stats` 表查询，统计评分 ≥ 60 分的企业数量。任务完成时根据 `search_results` 表的 `score` 字段聚合计算 |
| MoliAI点数消耗 | `moli_ai_consumed` | `number` | 接口返回 | 从 `search_task_stats` 表查询，任务完成后按实际 API 调用次数结算。包含地图 API、海关 API、展会 API 等调用消耗 |
| 统计概览数据 | `stats` | `object` | 接口返回 | 实时计算当前 BOT 的历史累计数据，查询 `search_tasks` 和 `search_task_stats` 表聚合得出。Redis 缓存 5 分钟，Key: `task_stats:{bot_id}` |
| 数据源类型 | `data_sources` | `array` | 数组存储 | 数据源类型以 JSON 数组格式存储，如 `["map", "customs", "exhibition"]` |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 搜客需求输入.html | 搜客任务启动后跳转 | `bot_id` |
| 搜客执行中.html | 任务完成后自动跳转 | `bot_id`, `task_id` |
| 搜客结果列表.html | 点击返回按钮 | `bot_id` |

**入口前置条件**：用户已登录且拥有访问该 BOT 的权限

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 搜客需求输入.html | 点击"新建搜客"按钮 | `bot_id` |
| 搜客需求输入.html | 点击失败任务的"重试"按钮 | `bot_id`, `retry_from`, `prefill_data` |
| 搜客结果列表.html | 点击已完成的任务记录 | `bot_id`, `task_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后等待数据返回 | 页面加载或筛选条件变更 | 接口返回成功/失败 |
| 正常 | 数据加载成功且有记录 | 列表数据返回且 `items.length > 0` | 用户操作或筛选条件变更 |
| 空态 | 无记录数据 | 列表数据返回且 `items.length === 0` | 用户点击"新建搜客" |
| 筛选空态 | 筛选条件下无数据 | 筛选后 `items.length === 0` | 用户清除筛选条件 |
| 错误态 | 数据加载失败 | 接口返回错误（非 200） | 用户点击"重新加载" |

**状态流转图**：
```
[加载中] --(数据返回成功且有记录)--> [正常]
   |                                      |
   |--(数据返回成功但无记录)--> [空态]    |
   |                                      |
   |--(接口返回错误)--> [错误态]          |
   |                                      |
   v                                      v
[筛选条件变更] <--(清除筛选条件)--- [筛选空态]
```

## 业务处理逻辑

### 场景一：查看搜客记录列表

**触发条件**：用户进入搜客记录页面。

**执行逻辑**：

1. 系统接收请求：页面加载，从 URL 解析 `bot_id`
2. 系统执行处理：
   - 调用 `GET /api/v1/search/records` 获取记录列表
   - 默认参数：`status=all`, `page=1`, `limit=10`
   - 按 `created_at` 倒序排列
3. 前端渲染处理：
   - 统计概览卡片展示 `stats` 数据
   - 每条记录根据 `status` 渲染不同状态标签
   - `completed` 状态：绿色勾选图标 + "已完成"标签
   - `failed` 状态：红色叉号图标 + "失败"标签 + 错误原因
   - `pending` 状态：灰色等待图标 + "等待中"标签
4. 系统返回响应：返回记录列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无记录数据 | 显示空态引导，隐藏统计卡片和筛选栏 |
| 失败任务 | 显示错误原因和重试按钮 |
| 结果已过期 | 已完成任务显示"数据已过期"提示，仍可查看 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功且 `items.length > 0` |
| 加载中 | 空态 | 数据返回成功且 `items.length === 0` |
| 加载中 | 错误态 | 接口返回非 200 状态码 |

**输出**：搜客任务记录列表。

---

### 场景二：状态筛选

**触发条件**：用户点击状态筛选按钮。

**执行逻辑**：

1. 系统接收请求：用户点击筛选按钮（全部/已完成/失败）
2. 系统执行处理：
   - 更新当前选中状态，高亮对应按钮
   - 调用 `GET /api/v1/search/records` 传入 `status` 参数
   - 重置页码为 1
   - 列表区域显示加载动画
3. 前端渲染处理：
   - 根据新数据重新渲染列表
   - 筛选后无数据时显示筛选空态
4. 系统返回响应：返回过滤后的记录列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示筛选空态，提示"暂无符合条件的搜客记录" |
| 网络异常 | 保持原列表，Toast 提示网络错误 |

**状态变更**：无

**输出**：按状态过滤后的记录列表。

---

### 场景三：时间筛选

**触发条件**：用户选择时间范围。

**执行逻辑**：

1. 系统接收请求：用户选择时间范围
2. 快捷选项处理：
   - "今日"：`start_date=今天`, `end_date=今天`
   - "近7天"：`start_date=今天-7天`, `end_date=今天`
   - "近30天"：`start_date=今天-30天`, `end_date=今天`
   - "全部时间"：不传 `start_date` 和 `end_date`
3. 自定义日期处理：
   - 弹出日期选择器，选择开始日期和结束日期
   - 点击"应用"按钮确认
   - 校验：开始日期不能晚于结束日期
4. 系统执行处理：
   - 调用 `GET /api/v1/search/records` 传入日期参数
   - 重置页码为 1
5. 系统返回响应：返回过滤后的记录列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 开始日期 > 结束日期 | 提示"开始日期不能晚于结束日期"，不发送请求 |
| 筛选后无数据 | 显示筛选空态 |
| 日期范围超过 90 天 | 提示"最多查询 90 天内的记录" |

**状态变更**：无

**输出**：按时间范围过滤后的记录列表。

---

### 场景四：分页加载

**触发条件**：用户点击分页按钮。

**执行逻辑**：

1. 系统接收请求：用户点击页码或上一页/下一页按钮
2. 系统执行处理：
   - 更新当前页码
   - 调用 `GET /api/v1/search/records` 传入 `page` 参数
   - 列表区域显示加载动画（保留筛选栏）
3. 前端渲染处理：
   - 根据新数据重新渲染列表
   - 更新分页器状态（高亮当前页，禁用边界页）
   - 滚动到列表顶部
4. 系统返回响应：返回对应页码的记录列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 页码 > 最大页数 | 自动调整到最后一页 |
| 分页加载失败 | 保持当前页，Toast 提示错误 |

**状态变更**：无

**输出**：对应页码的记录列表。

### 场景五：重试失败任务

**触发条件**：用户点击失败任务的重试按钮。

**执行逻辑**：

1. 系统接收请求：用户点击重试按钮
2. 前置校验：
   - 检查任务状态：`status === 'failed'`
3. 系统执行处理：
   - 调用 `GET /api/v1/search/tasks/{task_id}` 获取任务详情
   - 按钮显示加载状态，禁用重复点击
4. 成功处理：
   - 获取原任务的搜索需求配置
   - 跳转到搜客需求输入页面，携带回填参数
   - URL 格式：`搜客需求输入.html?bot_id={bot_id}&retry_from={task_id}&prefill=1`
   - 通过 URL 参数或 localStorage 传递回填数据：
     - `search_demand`：原始搜索需求文本
     - `target_count`：目标企业数量
     - `data_sources`：数据源配置
     - `icp_criteria`：ICP 匹配条件
5. 失败处理：
   - 根据错误码显示对应提示
   - 恢复按钮状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 任务不存在 | Toast「该任务不存在或已过期」 |
| 任务状态非 failed | Toast「该任务状态不允许重试」 |
| 获取详情失败 | Toast「获取任务信息失败，请稍后重试」 |

**状态变更**：无

**输出**：跳转至搜客需求输入页面，自动回填原任务配置。

---

### 场景六：查看已完成任务详情

**触发条件**：用户点击已完成的任务记录。

**执行逻辑**：

1. 系统接收请求：用户点击已完成记录卡片
2. 前置校验：
   - 检查任务状态：`status === 'completed'`
   - 检查数据是否过期：`new Date() < new Date(expires_at)`
3. 系统执行处理：
   - 跳转到搜客结果列表，传递 `bot_id` 和 `task_id`
   - URL 格式：`搜客结果列表.html?bot_id={bot_id}&task_id={task_id}`
4. 过期数据处理：
   - 数据已过期时仍允许跳转
   - 在结果页面显示"数据已过期"提示

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 任务结果已过期 | 仍可跳转，结果页面显示"搜索结果已过期，部分数据可能不完整" |
| 任务不存在 | Toast「该任务不存在或已过期」 |

**状态变更**：无

**输出**：跳转至搜客结果列表。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| `task_title` | 文本截取 | 从 `demand_summary` 字段截取前 12 个字符作为任务标题。若原始需求不足 12 字符则使用完整内容。任务创建时生成一次 | `search_tasks.task_title` |
| `demand_summary` | 文本截断 | 对用户原始输入进行截断处理，保留前 50 个字符，超出部分添加"..."后缀 | `search_tasks.demand_summary` |
| `result_count` | 聚合计算 | 任务完成时，统计 `search_results` 表中该 `task_id` 的记录数，写入 `search_task_stats` 表 | `search_task_stats.result_count` |
| `high_potential_count` | 聚合计算 | 任务完成时，统计 `search_results` 表中 `score >= 60` 的记录数，写入 `search_task_stats` 表 | `search_task_stats.high_potential_count` |
| `moli_ai_consumed` | 累加计算 | 任务执行过程中记录每次 API 调用消耗，完成后累加写入 `search_task_stats` 表 | `search_task_stats.moli_ai_consumed` |
| `stats` | 实时聚合 | 查询当前 BOT 的历史累计数据：`COUNT(tasks)`, `SUM(moli_ai_consumed)`, `SUM(result_count)`。Redis 缓存 5 分钟 | Redis Key: `task_stats:{bot_id}` |
| `data_sources` | 数组存储 | 数据源类型以 JSON 数组格式存储，如 `["map", "customs", "exhibition"]` | `search_tasks.data_sources` |

Authorization: Bearer {tenant_token}

  "data": {
    "task_id": "TASK-202401-003",
    "bot_id": "BOT-xxx",

    "demand_summary": "中东地区的音响设备进口商...",

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志，发送告警 | Toast「服务暂时不可用，请稍后再试」 |
| 无数据可展示 | 筛选条件下无记录 | — | 空态引导，提示「暂无符合条件的搜客记录」 |
| 数据已过期 | 当前时间 > expires_at | — | 提示「部分数据已过期，请刷新获取最新数据」 |
| 任务不存在 | code 404 | 返回错误 | Toast「该任务不存在或已过期」 |
| 参数错误 | code 400 | 返回具体错误字段 | Toast「请求参数错误，请刷新后重试」 |
| 无权限访问 | code 403 | 返回错误 | 跳转无权限页面 |
| 并发请求限制 | 同一用户 1s 内多次请求 | 限流处理 | Toast「操作过于频繁，请稍后再试」 |

## 性能安全要求

### 性能指标

| 指标类型 | 指标要求 | 说明 |
|---------|---------|------|
| 列表加载时间 | ≤ 500ms（P95） | 从请求发出到数据渲染完成 |
| 筛选响应时间 | ≤ 300ms（P95） | 筛选条件变更后数据返回时间 |
| 分页切换时间 | ≤ 300ms（P95） | 点击分页按钮后数据返回时间 |
| 任务详情接口响应时间 | ≤ 200ms（P95） | 获取单个任务详情的时间 |
| 列表单次返回数量 | 默认 10，最大 50 | 控制单次返回数据量 |
| 最大分页页数 | 100 页 | 防止深分页性能问题 |
| 数据保留期限 | 90 天 | 超过期限的数据自动清理 |

### 安全策略

| 安全维度 | 策略要求 | 说明 |
|---------|---------|------|
| 权限校验 | JWT Token 认证 | 必须校验用户是否有权限访问该 BOT |
| 数据隔离 | 租户隔离 | 只能查询当前租户的数据，SQL 查询必须带 `tenant_id` 条件 |
| 防刷限制 | 接口限流 | 单个用户 1s 内最多 10 次请求 |
| 敏感信息 | 日志脱敏 | 错误日志中不得包含用户敏感信息 |
