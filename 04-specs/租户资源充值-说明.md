## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 租户名称 | `tenant.company_name` | `string` | 租户服务 | 租户注册信息中的企业名称 |
| Tenant ID | `tenant_id` | `string` | 租户服务 | 系统生成的唯一租户标识 |
| 当前系统到期时间 | `current_expiry_date` | `string` | 租户服务 | 格式：YYYY-MM-DD |
| 系统服务剩余年数 | `remaining_years` | `number` | 后端计算 | 根据当前到期时间计算，保留1位小数 |
| 当前 BOT 席位数 | `current_bot_seats` | `number` | 租户配置 | 租户当前拥有的BOT席位总数 |
| 新增服务年数 | `service_years` | `number` | 用户输入 | 范围：0-5年 |
| BOT 席位增购数 | `bot_seats_add` | `number` | 用户输入 | 范围：≥0 |
| 系统服务费 | `service_fee` | `number` | 后端计算 | 单位：元，计算公式见处理逻辑 |
| Credits 资源包类型 | `credits_packages[].type` | `string` | 配置数据 | 枚举：basic/standard/professional/enterprise |
| Credits 额度 | `credits_packages[].amount` | `number` | 配置数据 | 单个资源包包含的Credits数量 |
| Credits 包价格 | `credits_packages[].price` | `number` | 配置数据 | 单个资源包的价格，单位：元 |
| Credits 费用 | `credits_fee` | `number` | 后端计算 | 所有选中资源包价格之和 |
| 总计费用 | `total_fee` | `number` | 后端计算 | 系统服务费 + Credits费用 |
| 交易流水号 | `transaction_id` | `string` | 系统生成 | 格式：TXN-YYYYMMDD-XXX |
| 续费后到期时间 | `new_expiry_date` | `string` | 后端计算 | 根据新增年数计算的新到期时间 |
| 功能开关状态 | `feature_enabled` | `boolean` | 租户配置 | 控制AI-LeadFinder功能启停 |
| 已创建 BOT 数量 | `created_bot_count` | `number` | BOT服务 | 租户已创建的BOT实例数量 |
| 目标席位数 | `target_seats` | `number` | 用户输入 | 减少席位时输入的目标值 |
| 待删除 BOT ID 列表 | `delete_bot_ids` | `array` | 用户选择 | 减少席位时勾选的要删除的BOT |

## 数据流

### 资源下发

```
POST /internal/v1/tenant/resources/provision
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | `string` | 是 | 租户唯一标识 |
| service_years | `number` | 否 | 新增服务年数，0-5 |
| bot_seats_add | `number` | 否 | 增购BOT席位数量 |
| credits_packages | `array` | 否 | 选中的Credits资源包列表 |
| credits_packages[].type | `string` | 是 | 资源包类型 |
| credits_packages[].amount | `number` | 是 | Credits数量 |
| credits_packages[].price | `number` | 是 | 资源包价格 |

**请求体示例**：
```json
{
  "tenant_id": "tenant_001",
  "service_years": 2,
  "bot_seats_add": 2,
  "credits_packages": [
    { "type": "standard", "amount": 5000, "price": 4500 },
    { "type": "basic", "amount": 1000, "price": 1000 }
  ]
}
```

**响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tenant_id": "tenant_001",
    "transaction_id": "TXN-202401-001",
    "service_fee": 42000,
    "service_fee_formula": "(3 × 2 + 2 × 4) × 3,000 = 42,000",
    "credits_fee": 5500,
    "total_fee": 47500,
    "new_expiry_date": "2029-03-15",
    "new_bot_seats_total": 5,
    "new_credits_balance": 76000
  }
}
```

**响应码说明**：

| code | 含义 | 后端处理 |
|------|------|---------|
| 200 | 下发成功 | 返回交易详情 |
| 400 | 参数错误 | 返回具体错误字段 |
| 403 | 无权限操作该租户 | 拒绝请求，记录审计日志 |
| 500 | 服务异常 | 返回通用错误，触发告警 |

### 减少席位

```
POST /internal/v1/tenant/bot-seats/reduce
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | `string` | 是 | 租户唯一标识 |
| target_seats | `number` | 是 | 目标席位数量，必须小于当前席位 |
| delete_bot_ids | `array` | 否 | 需要删除的BOT ID列表 |

**请求体示例**：
```json
{
  "tenant_id": "tenant_001",
  "target_seats": 2,
  "delete_bot_ids": ["bot_001"]
}
```

**响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tenant_id": "tenant_001",
    "previous_seats": 3,
    "current_seats": 2,
    "deleted_bots": ["bot_001"],
    "refund_amount": 0
  }
}
```

**响应码说明**：

| code | 含义 | 后端处理 |
|------|------|---------|
| 200 | 减少成功 | 更新席位，删除指定BOT |
| 400 | 参数错误 | 校验目标席位必须小于当前席位 |
| 403 | 无权限 | 拒绝请求 |
| 409 | 冲突 | 目标席位小于已创建BOT数（未删除足够BOT） |
| 500 | 服务异常 | 返回通用错误 |

## 处理逻辑

### 场景一：系统服务费计算

**触发条件**：前端接收到`service_years`或`bot_seats_add`参数变更

**执行逻辑**：

1. 系统接收参数变更请求
2. 计算总服务年数：`total_years = ceil(remaining_years + service_years)`
3. 计算系统服务费：`service_fee = (current_bot_seats × service_years + bot_seats_add × total_years) × 3000`
4. 计算新的到期时间：`new_expiry_date = current_expiry_date + service_years`
5. 返回计算结果至前端展示

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| service_years = 0 且 bot_seats_add = 0 | 系统服务费显示为0 |
| service_years > 5 | 拒绝，返回参数错误 |
| bot_seats_add < 0 | 拒绝，返回参数错误 |
| remaining_years 为小数 | 按原始值参与计算，仅最终显示时向上取整 |

**状态变更**：无

**输出**：系统服务费金额、计算公式、新的到期时间

**计算示例**：
- 当前 BOT 席位：3个
- 当前剩余年数：1.9年
- 新增年数：2年
- 增购席位：2个
- 总服务年数 = ceil(1.9 + 2) = 4年
- 系统服务费 = (3 × 2 + 2 × 4) × 3,000 = 42,000 元

---

### 场景二：Credits 资源包选择

**触发条件**：前端接收到资源包选择/取消选择事件

**执行逻辑**：

1. 系统接收资源包选择状态变更
2. 累加所有选中资源包的`price`字段
3. 更新Credits费用显示
4. 更新总计费用

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 取消所有选择 | Credits费用显示为0 |
| 同时选择多个资源包 | 费用累加计算 |

**状态变更**：无

**输出**：Credits费用金额

---

### 场景三：资源下发执行

**触发条件**：前端提交资源下发请求

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/resources/provision`请求
2. 校验请求参数合法性
3. 校验操作者权限
4. 执行资源下发事务：
   - 更新租户系统服务到期时间
   - 增加BOT席位数量
   - 增加租户Credits余额
5. 生成交易流水号
6. 返回下发结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| service_years = 0 且 bot_seats_add = 0 且 credits_packages为空 | 拒绝请求，返回"未选择任何资源" |
| 租户不存在 | 返回404错误 |
| 权限不足 | 返回403错误 |
| 服务超时 | 返回500错误，事务回滚 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 充值中 | 点击确认购买 |
| 充值中 | 充值成功 | 接口返回成功 |
| 充值中 | 充值失败 | 接口返回错误 |

**输出**：交易流水号、费用明细、新的资源状态

---

### 场景四：功能开关控制

**触发条件**：前端提交功能开关切换请求

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/feature/toggle`请求
2. 更新租户功能开关配置
3. 同步更新关联服务状态：
   - 系统入口权限
   - 访客雷达数据上传能力
4. 返回更新结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 系统服务已到期 | 自动关闭功能，禁止手动开启 |
| 开关切换失败 | 配置回滚，保持原状态 |

**状态变更**：无

**输出**：功能开关新状态、受影响服务列表

---

### 场景五：减少席位

**触发条件**：前端提交减少席位请求

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/bot-seats/reduce`请求
2. 校验目标席位数有效性：
   - 目标席位 < 当前席位
   - 目标席位 ≥ (已创建BOT数 - 待删除BOT数)
3. 执行事务：
   - 删除指定的BOT实例（数据不可恢复）
   - 更新租户BOT席位数至目标值
4. 返回操作结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| target_seats ≥ current_seats | 拒绝请求，返回参数错误 |
| target_seats < (created_bots - delete_count) | 拒绝请求，返回"需先删除多余BOT" |
| delete_bot_ids 包含不存在的BOT | 忽略不存在的BOT，继续处理 |
| 删除后无BOT剩余 | 允许，席位减少至目标值 |

**状态变更**：无

**输出**：操作结果、已删除BOT列表、退费金额（固定为0）

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 充值失败 | 接口返回code非200 | 记录错误日志，返回具体错误信息 | Toast「充值失败，请稍后重试」 |
| 网络超时 | 请求超时（>30s） | 无 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | 接口返回code 500 | 触发服务告警，记录错误详情 | Toast「服务暂时不可用，请稍后再试」 |
| 减少席位超限 | 校验目标席位 < 删除后剩余BOT数 | 返回409冲突错误 | 弹窗显示警告「需先删除多余 BOT 后才能减少席位」 |
| 功能开关失败 | 接口返回code非200 | 配置回滚，记录审计日志 | 开关状态回滚，Toast「功能状态切换失败，请重试」 |
| 参数错误 | 接口返回code 400 | 返回具体错误字段 | 根据错误字段高亮对应输入框 |
| 无操作权限 | 接口返回code 403 | 记录审计日志 | Toast「无权限执行此操作」 |
