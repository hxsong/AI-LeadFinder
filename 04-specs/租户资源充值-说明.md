## 数据契约

| 系统服务剩余年数 | `remaining_years` | `number` | 系统生成 | 后端计算：`max(0, (expiry_date - today) / 365)`，保留1位小数 |

| Credits 额度 | `credits_packages[].amount` | `number` | 配置数据 | 单个资源包包含的Credits数量 |
| Credits 包价格 | `credits_packages[].price` | `number` | 配置数据 | 单个资源包的价格，单位：元 |
| Credits 费用 | `credits_fee` | `number` | 系统生成 | 所有选中资源包价格之和 |
| 总计费用 | `total_fee` | `number` | 系统生成 | `service_fee + credits_fee` |
| 交易流水号 | `transaction_id` | `string` | 系统生成 | 格式：`TXN-YYYYMMDD-{6位随机数}` |
| 续费后到期时间 | `new_expiry_date` | `string` | 系统生成 | 根据新增年数计算的新到期时间，格式：YYYY-MM-DD |

### 数据来源深度说明

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 系统服务剩余年数 | `remaining_years` | `number` | 系统生成 | 实时计算：`max(0, DATEDIFF(expiry_date, CURDATE()) / 365)`，保留1位小数。计算结果不缓存，每次页面加载时重新计算 |
| 系统服务费 | `service_fee` | `number` | 系统生成 | 实时计算：`(current_bot_seats × service_years + bot_seats_add × total_years) × 6999`，其中 `total_years = CEIL(remaining_years + service_years)`。计算逻辑由前端实时执行，后端在提交时二次校验 |
| 已创建 BOT 数量 | `created_bot_count` | `number` | 接口返回 | 从 `bots` 表执行聚合查询：`SELECT COUNT(*) FROM bots WHERE tenant_id = ? AND status != 'deleted'`，实时查询无缓存 |
| Credits 资源包配置 | `credits_packages` | `array` | 接口返回 | 从 `system_configs` 表查询 `credits_package_config` JSON 配置字段，配置变更后需重启服务生效 |
| 交易流水号 | `transaction_id` | `string` | 系统生成 | 生成规则：`CONCAT('TXN-', DATE_FORMAT(NOW(), '%Y%m%d'), '-', LPAD(FLOOR(RAND() * 1000000), 6, '0'))`，确保当日唯一性 |
| 续费后到期时间 | `new_expiry_date` | `string` | 系统生成 | 计算规则：`DATE_ADD(current_expiry_date, INTERVAL service_years YEAR)`，若当前已到期则从当前日期开始计算 |

## 业务处理逻辑

### 场景一：系统服务费计算

**触发条件**：前端接收到`service_years`或`bot_seats_add`参数变更

**执行逻辑**：

1. 系统接收参数变更请求
2. 计算总服务年数：`total_years = ceil(remaining_years + service_years)`
3. 计算系统服务费：`service_fee = (current_bot_seats × service_years + bot_seats_add × total_years) × 6999`
4. 计算新的到期时间：`new_expiry_date = current_expiry_date + service_years`
5. 返回计算结果至前端展示

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| service_years = 0 且 bot_seats_add = 0 | 系统服务费显示为0 |
| service_years > 5 | 拒绝，返回参数错误 |
| bot_seats_add < 0 | 拒绝，返回参数错误 |
| remaining_years 为小数 | 按原始值参与计算，仅最终显示时向上取整 |

**状态变更**：无

**输出**：系统服务费金额、计算公式、新的到期时间

---

### 场景二：Credits 资源包选择

**触发条件**：前端接收到资源包选择/取消选择事件

**执行逻辑**：

1. 系统接收资源包选择状态变更
2. 累加所有选中资源包的`price`字段
3. 更新Credits费用显示
4. 更新总计费用

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 取消所有选择 | Credits费用显示为0 |
| 同时选择多个资源包 | 费用累加计算 |

**状态变更**：无

**输出**：Credits费用金额

---

### 场景三：创建充值订单

**触发条件**：前端提交充值请求，点击"确认购买"按钮

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/resources/provision`请求
2. 校验请求参数合法性
3. 校验操作者权限
4. 创建交易订单（状态为`pending`）：
   - 生成交易流水号
   - 将请求参数（新增年数、增购席位、Credits包选择）序列化存储到`request_data`字段
   - 计算并存储费用明细（系统服务费、Credits费用、总费用）
5. **注意：此时不执行实际资源下发**，仅创建待确认订单
6. 返回订单创建结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| service_years = 0 且 bot_seats_add = 0 且 credits_packages为空 | 拒绝请求，返回"未选择任何资源" |
| 租户不存在 | 返回404错误 |
| 权限不足 | 返回403错误 |
| 服务超时 | 返回500错误，事务回滚 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 充值中 | 点击确认购买 |
| 充值中 | 充值成功 | 接口返回成功，订单创建完成 |
| 充值中 | 充值失败 | 接口返回错误 |

**输出**：交易流水号、订单详情（费用明细、待确认状态）、跳转至资源下发确认页面

---

### 场景四：功能开关控制

**触发条件**：前端提交功能开关切换请求

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/feature/toggle`请求
2. 更新租户功能开关配置
3. 同步更新关联服务状态：
   - 系统入口权限
   - 访客雷达数据上传能力
4. 返回更新结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 系统服务已到期 | 自动关闭功能，禁止手动开启 |
| 开关切换失败 | 配置回滚，保持原状态 |

**状态变更**：无

**输出**：功能开关新状态、受影响服务列表

---

### 场景五：减少席位

**触发条件**：前端提交减少席位请求

**执行逻辑**：

1. 系统接收`POST /internal/v1/tenant/bot-seats/reduce`请求
2. 校验目标席位数有效性：
   - 目标席位 < 当前席位
   - 目标席位 ≥ (已创建BOT数 - 待删除BOT数)
3. 执行事务：
   - 删除指定的BOT实例（数据不可恢复）
   - 更新租户BOT席位数至目标值
4. 返回操作结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| target_seats ≥ current_seats | 拒绝请求，返回参数错误 |
| target_seats < (created_bots - delete_count) | 拒绝请求，返回"需先删除多余BOT" |
| delete_bot_ids 包含不存在的BOT | 忽略不存在的BOT，继续处理 |
| 删除后无BOT剩余 | 允许，席位减少至目标值 |

**状态变更**：无

**输出**：操作结果、已删除BOT列表、退费金额（固定为0）

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| `remaining_years` | 计算 | 查询 `tenant_subscriptions.expiry_date`，计算 `(expiry_date - CURDATE()) / 365`，保留1位小数 | 实时计算，不存储 |
| `service_fee` | 计算 | 按业务规则计算，详见【计算逻辑】 | 实时计算，不存储 |
| `new_expiry_date` | 计算 | `DATE_ADD(current_expiry_date, INTERVAL service_years YEAR)` | 实时计算，不存储 |
| `created_bot_count` | 聚合查询 | `SELECT COUNT(*) FROM bots WHERE tenant_id = ? AND status != 'deleted'` | `bots` 表 |
| `feature_enabled` | 单表查询 | `SELECT ai_leadfinder_enabled FROM tenant_configs WHERE tenant_id = ?` | `tenant_configs` 表 |
| `credits_packages` | 配置查询 | 从 `system_configs` 表查询 `credits_package_config` JSON 配置 | `system_configs` 表 |

### 计算逻辑

**系统服务费计算公式**：
```
total_years = CEIL(remaining_years + service_years)
service_fee = (current_bot_seats × service_years + bot_seats_add × total_years) × 6999
```

**计算示例**：
- 当前 BOT 席位：3个
- 当前剩余年数：1.9年
- 新增年数：2年
- 增购席位：2个
- 总服务年数 = ceil(1.9 + 2) = 4年
- 系统服务费 = (3 × 2 + 2 × 4) × 6,999 = 97,986 元

**Credits 费用计算**：
```
credits_fee = SUM(selected_packages[].price)
```

**总计费用计算**：
```
total_fee = service_fee + credits_fee
```

**触发时机**：实时计算，用户修改参数时前端即时计算展示，后端提交时二次校验

**存储方式**：计算结果不持久化，仅作为展示和校验使用

### 交易流水号生成规则

```
transaction_id = CONCAT('TXN-', DATE_FORMAT(NOW(), '%Y%m%d'), '-', LPAD(FLOOR(RAND() * 1000000), 6, '0'))
```

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 充值失败 | 接口返回code非200 | 记录错误日志，返回具体错误信息 | Toast「充值失败，请稍后重试」 |
| 网络超时 | 请求超时（>30s） | 无 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | 接口返回code 500 | 触发服务告警，记录错误详情 | Toast「服务暂时不可用，请稍后再试」 |
| 减少席位超限 | 校验目标席位 < 删除后剩余BOT数 | 返回409冲突错误 | 弹窗显示警告「需先删除多余 BOT 后才能减少席位」 |
| 功能开关失败 | 接口返回code非200 | 配置回滚，记录审计日志 | 开关状态回滚，Toast「功能状态切换失败，请重试」 |
| 参数错误 | 接口返回code 400 | 返回具体错误字段 | 根据错误字段高亮对应输入框 |
| 无操作权限 | 接口返回code 403 | 记录审计日志 | Toast「无权限执行此操作」 |
| 租户不存在 | 接口返回code 404 | 记录错误日志 | Toast「租户信息不存在，请刷新页面重试」 |
| 资源已生效 | 重复提交检测 | 返回409冲突错误 | Toast「该资源已生效，请勿重复操作」 |
