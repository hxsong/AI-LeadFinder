## 概述

租户资源充值页面用于管理员为入驻租户分配系统服务年限、BOT 引擎席位和 Credits 资源。实际充值操作使用现有服务商系统，本页面作为操作引导入口，配置和开通逻辑与独立应用一致。

## 功能逻辑

### 场景一：系统服务费购买

**触发条件**：管理员需要延长租户系统服务有效期或增购 BOT 席位。

**执行逻辑**：
1. 页面展示当前系统到期时间和当前 BOT 席位数
2. 管理员设置两项参数：
   - 新增服务年数（1-5年）
   - 增购 BOT 席位数量
3. 系统实时计算系统服务费：
   - 总服务年数 = ceil(当前剩余年数 + 新增年数)
   - 系统服务费 = (当前 BOT 席位数 × 新增年数 + 增购席位 × 总服务年数) × 3,000 元
4. 购买后系统到期时间延长对应年数，BOT 席位增加

**输出**：系统服务费计算结果和计算逻辑展示。

**计算示例**：
- 当前 BOT 席位：3个
- 当前剩余年数：1.9年
- 新增年数：2年
- 增购席位：2个
- 总服务年数 = ceil(1.9 + 2) = 4年
- 系统服务费 = (3 × 2 + 2 × 4) × 3,000 = 42,000 元

---

### 场景二：Credits 资源包选择

**触发条件**：管理员需要为租户补充 Credits 资源。

**执行逻辑**：
1. 展示四种 Credits 资源包：
   - 基础包：1,000 Credits / 1,000 元
   - 标准包：5,000 Credits / 4,500 元（9折）
   - 专业包：20,000 Credits / 16,000 元（8折）
   - 企业包：100,000 Credits / 70,000 元（7折）
2. 管理员可选择多个资源包（多选）
3. 选中后实时计算 Credits 费用

**输出**：已选 Credits 资源包汇总信息。

---

### 场景三：资源下发执行

**触发条件**：管理员确认选择后，点击"确认购买"按钮。

**执行逻辑**：
1. 前端调用内部资源下发接口
2. 后端校验租户信息有效性
3. 执行资源下发：
   - 系统服务：延长到期时间
   - BOT 席位：增加可用席位
   - Credits：增加租户余额
4. 返回成功结果，前端跳转资源下发确认页面

**输出**：充值成功提示，生成交易流水号。

---

### 场景四：功能开关控制

**触发条件**：管理员需要开启或关闭租户 AI-LeadFinder 功能。

**执行逻辑**：
1. 点击功能开关切换状态
2. 开关控制：系统入口权限 + 访客雷达数据上传能力
3. 系统服务到期后自动关闭，关闭后访客雷达不再接受上传数据
4. 手动关闭后同样停止访客雷达数据接收

**输出**：功能状态变更结果。

---

### 场景五：减少席位（独立工具）

**触发条件**：管理员需要减少租户 BOT 席位数量。

**执行逻辑**：
1. 点击"减少席位"按钮，弹出确认弹窗
2. 弹窗展示：
   - 当前席位数量
   - 已创建 BOT 数量
   - 目标席位数输入框
3. 展示已创建 BOT 列表，包含：
   - BOT 名称、创建时间、线索数、运行状态
   - 可勾选需要删除的 BOT
4. 实时校验逻辑：
   - 计算删除后剩余 BOT 数 = 已创建数 - 选中删除数
   - 若目标席位 < 删除后剩余 BOT 数，显示警告提示需继续删除
   - 若目标席位 ≥ 当前席位，禁用确认按钮
   - 仅当目标席位有效且 ≥ 删除后剩余 BOT 数时，启用确认按钮
5. 确认后执行：
   - 删除选中的 BOT（数据不可恢复）
   - 减少席位至目标数量
   - 该操作不退费

**输出**：席位减少结果提示，已删除 BOT 列表。

**API 请求**：
```
POST /internal/v1/tenant/bot-seats/reduce
```

**请求体**：
```json
{
  "tenant_id": "tenant_001",
  "target_seats": 2,
  "delete_bot_ids": ["bot_001"]
}
```

**响应体**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tenant_id": "tenant_001",
    "previous_seats": 3,
    "current_seats": 2,
    "deleted_bots": ["bot_001"],
    "refund_amount": 0
  }
}
```

## 数据流

### 请求：资源下发

```
POST /internal/v1/tenant/resources/provision
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求体**：

```json
{
  "tenant_id": "tenant_001",
  "service_years": 2,
  "bot_seats_add": 2,
  "credits_packages": [
    { "type": "standard", "amount": 5000, "price": 4500 },
    { "type": "basic", "amount": 1000, "price": 1000 }
  ]
}
```

**响应体**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "tenant_id": "tenant_001",
    "transaction_id": "TXN-202401-001",
    "service_fee": 42000,
    "service_fee_formula": "(3 × 2 + 2 × 4) × 3,000 = 42,000",
    "credits_fee": 5500,
    "total_fee": 47500,
    "new_expiry_date": "2029-03-15",
    "new_bot_seats_total": 5,
    "new_credits_balance": 76000
  }
}
```

| code | 含义 |
|------|------|
| 200 | 下发成功 |
| 400 | 参数错误 |
| 403 | 无权限操作该租户 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 系统服务费计算 | 同时设置年限和席位 | (当前BOT×新增年数 + 增购席位×总年数) × 3,000 |
| 年数取整规则 | 任何小数年份 | 向上取整（1.2年、1.9年都按2年计算） |
| Credits 选择 | 支持多选 | 多个资源包费用累加 |
| 重复提交 | 充值中按钮点击 | 按钮禁用，防止重复扣费 |
| 减少席位限制 | 目标席位 < 已创建 BOT | 提示需先删除多余 BOT |
| 减少席位退费 | 任何情况 | 不退费 |
| 功能开关 | 到期或手动关闭 | 关闭系统入口，访客雷达停止接收数据 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 充值失败 | code 非 200 | 失败提示 | 「充值失败，请稍后重试」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
| 减少席位超限 | 已创建 BOT > 目标席位 | 弹窗警告 | 「需先删除多余 BOT 后才能减少席位」 |
| 功能开关失败 | code 非 200 | 开关回滚 | 「功能状态切换失败，请重试」 |
