## 数据契约

### 页面展示数据

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识，从 URL 参数 `?bot_id=xxx` 获取 |
| 自然语言需求 | `user_query` | `string` | 用户输入 | 用户输入的搜客需求文本，最多 500 字符，超出自动截断 |
| 快速填写标签 | - | - | 前端写死 | 前端快捷输入功能，可选值：美国/欧洲/有海关记录/近期参展/大型企业，点击后自动追加到自然语言需求文本框，不单独存储和传输 |
| 目标企业数量 | `target_count` | `number` | 用户输入 | 范围 20-200，默认 100，通过滑块组件选择 |
| 使用默认 ICP | `use_default_icp` | `boolean` | 系统生成 | 当 `user_query` 为空时自动设为 `true`，表示使用 ICP 画像默认搜索 |
| ICP 画像状态 | `icp_status` | `string` | 接口返回 | 当前 BOT 是否存在有效 ICP 画像，值：ready / missing / expired |
| ICP 权重配置 | `icp_weights` | `object` | 数据库查询 | 从 `bot_customer_profiles.scoring_model.weights` 读取四维权重配置 |
| MoliAI点数余额 | `moli_ai_balance` | `number` | 接口返回 | 当前租户 MoliAI点数余额，从 `moli_ai_balances` 表查询 |
| 搜客任务 ID | `task_id` | `string` | 系统生成 | 后端创建搜客任务后返回的唯一标识，格式：`TASK-{YYYYMMDD}-{序号}` |
| 任务状态 | `status` | `string` | 接口返回 | 当前 BOT 的搜客任务状态，值：pending / running / completed / failed / none |
| 进行中任务 ID | `running_task_id` | `string` | 接口返回 | 若存在进行中的任务，返回该任务 ID |

### 数据来源深度说明

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| ICP 画像状态 | `icp_status` | `string` | 接口返回 | 服务端根据 `bot_id` 查询 `bot_customer_profiles` 表，校验 `scoring_model` 字段完整性和 `updated_at` 时间（90 天内有效）。若数据完整且在有效期内返回 `ready`，不存在返回 `missing`，过期返回 `expired` |
| ICP 权重配置 | `icp_weights` | `object` | 接口返回 | 从 `bot_customer_profiles.scoring_model.weights` 读取四维权重配置（需求匹配度、购买能力、采购意向、活跃意向），实时查询无缓存 |
| MoliAI点数余额 | `moli_ai_balance` | `number` | 接口返回 | 从 `moli_ai_balances` 表查询 `balance` 字段，实时查询无缓存 |
| 任务状态 | `status` | `string` | 接口返回 | 查询 `search_tasks` 表中该 BOT 最新任务的状态，按 `created_at` 倒序取第一条 |
| 进行中任务 ID | `running_task_id` | `string` | 接口返回 | 查询 `search_tasks` 表中 `status` = 'running' 的任务记录，若存在则返回 `task_id` |
| 使用默认 ICP | `use_default_icp` | `boolean` | 系统生成 | 前端逻辑：若 `user_query` 为空或仅包含空白字符，自动设为 `true` |

## 页面流转

### 入口

| 来源页面 | 触发条件 | 携带参数 |
|---------|---------|---------|
| 业务分析报告.html | 用户点击"开始搜客"按钮 | `bot_id` |
| 搜客记录.html | 用户点击"新建搜客"按钮 | `bot_id` |

**入口前置条件**：
- 用户已登录（携带有效 `tenant_token`）
- URL 中包含有效的 `bot_id` 参数
- 当前 BOT 已完成 P08 业务分析，存在有效 ICP 画像（`icp_status` = ready）

### 出口

| 目标页面 | 触发条件 | 携带数据 |
|---------|---------|---------|
| 搜客策略预览.html | 用户确认需求并提交成功 | `bot_id`, `task_id` |
| 搜客执行中.html | 进入页面时检测到已有任务在执行中 | `bot_id`, `task_id`（自动跳转） |
| 搜客记录.html | 用户主动点击"搜客记录" | — |
| 业务分析报告.html | ICP 画像不存在时，点击"查看报告" | `bot_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 页面初始化，查询任务状态和 ICP 画像 | 页面加载 | 任务状态查询完成 |
| 有进行中任务 | 检测到当前 BOT 有执行中任务 | 查询到 `status` = running | 自动跳转至搜客执行中 |
| 正常编辑 | 无进行中任务，允许输入需求 | 无 running 任务且 ICP 就绪 | 用户提交后跳转 |
| 提交中 | 已点击确认搜客，等待响应 | 用户点击"确认搜客" | 后端返回后跳转 |
| ICP 缺失 | ICP 画像不存在或已过期 | `icp_status` = missing/expired | 用户跳转至业务分析报告 |
| MoliAI点数不足 | MoliAI点数余额 ≤ 0 | `moli_ai_balance` ≤ 0 | 用户充值后刷新 |

**状态流转图**：

```
加载中 --(无running任务, ICP就绪)--> 正常编辑 --(点击确认)--> 提交中 --(返回task_id)--> 搜客策略预览.html
加载中 --(有running任务)--> 有进行中任务 --(自动跳转)--> 搜客执行中.html
加载中 --(ICP缺失/过期)--> ICP 缺失 --(点击跳转)--> 业务分析报告.html
加载中 --(MoliAI点数≤0)--> MoliAI点数不足 --(充值后刷新)--> 正常编辑
```

## 业务处理逻辑

### 场景一：正常搜客流程

**触发条件**：用户有明确的短期搜客需求，且 MoliAI点数余额充足，ICP 画像有效。

**执行逻辑**：

1. 页面加载时解析 URL 参数，获取 `bot_id`：
   - 参数缺失：跳转至 BOT 列表页面
2. 前端调用接口查询任务状态：
   - 接口：`GET /api/v1/search/status?bot_id={bot_id}`
   - 检查是否存在进行中的任务
   - 检查 ICP 画像状态
   - 获取 MoliAI点数余额
3. 若存在进行中的任务：
   - 自动跳转至搜客执行中页面
   - 携带 `bot_id` 和 `task_id`
4. 若 ICP 画像不存在或已过期：
   - 禁用"确认搜客"按钮
   - 显示提示"ICP 画像不存在，请重新生成业务分析报告"
   - 提供"查看报告"按钮跳转至业务分析报告页面
5. 页面展示 ICP 画像卡片：
   - 展示四维权重配置（需求匹配度、购买能力、采购意向、活跃意向）
   - 提供"查看报告"链接
6. 用户输入需求：
   - 在文本域输入自然语言需求（实时显示字符计数 0/500）
   - 超出 500 字符自动截断
7. 用户可选点击快速筛选标签：
   - 标签内容自动追加到文本框
   - 已选标签高亮显示
8. 用户拖动滑块设置目标企业数量：
   - 范围 20-200，默认 100
   - 实时显示当前数值
9. 用户点击"确认搜客"按钮：
   - 按钮进入 loading 状态，防止重复提交
   - 前端调用 `POST /api/v1/search/input`
   - 请求参数包含 `user_query`、`target_count`、`use_default_icp`（user_query 已包含快速标签追加的内容）
10. 后端返回成功后：
    - 获取 `task_id`
    - 跳转至搜客策略预览页面
    - 携带 `bot_id` 和 `task_id`

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| user_query 超过 500 字符 | 超出部分自动截断，不报错，字符计数显示 500/500 |
| MoliAI点数余额 ≤ 0 | "确认搜客"按钮禁用，提示"额度不足，请充值" |
| 余额 < 预估消耗 | 按钮禁用，Toast「预估需要 {estimated} MoliAI点数，当前余额不足」 |
| ICP 画像不存在 | 按钮禁用，提示"请先完成业务初始化" |
| ICP 画像已过期（>90天） | 按钮禁用，提示"ICP 画像已过期，请重新分析" |
| 当前 BOT 已有搜客任务执行中 | 进入页面时自动跳转至搜客执行中页面 |
| 网络超时（>30s） | Toast「网络连接异常，请检查网络后重试」，支持重试 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常编辑 | 提交中 | 用户点击"确认搜客" |
| 提交中 | 正常编辑 | 接口返回错误（非跳转类错误） |
| 提交中 | 跳转中 | 接口返回成功 |
| 加载中 | 有进行中任务 | 查询到 running 状态任务 |
| 有进行中任务 | 已跳转 | 自动跳转至搜客执行中页面 |

**输出**：跳转到搜客策略预览页面，携带 `bot_id` 和 `task_id`。

### 场景二：使用 ICP 画像默认搜索

**触发条件**：用户未输入任何文字，直接提交空表单。

**执行逻辑**：

1. 系统接收请求：前端检测 `user_query` 为空或仅包含空白字符
2. 前端自动设置 `use_default_icp=true`
3. 请求发送至 `POST /api/v1/search/input`
4. 后端处理：
   - 读取该 BOT 的 ICP 画像作为搜索指令
   - 使用 `scoring_model` 中的配置生成策略
5. 系统返回响应：返回 `task_id`，前端跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| ICP 画像不存在 | 返回错误码 1003，提示重新生成业务分析报告 |
| ICP 画像已过期 | 返回错误码 1004，提示重新分析 |
| ICP 画像数据不完整 | 返回错误码 1003，提示 ICP 画像异常 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常编辑 | 提交中 | 用户提交空表单 |
| 提交中 | 跳转中 | 接口返回成功 |

**输出**：与场景一相同，跳转到搜客策略预览页面。

### 场景三：已有任务在执行

**触发条件**：进入页面时，后端检测到该 BOT 存在状态为"执行中"的搜客任务。

**执行逻辑**：

1. 系统接收请求：前端在进入页面时调用 `GET /api/v1/search/status`
2. 系统执行处理：
   - 查询当前 BOT 的任务状态
   - 若 `has_running_task` = true，获取 `running_task_id`
   - 前端自动跳转到搜客执行中页面
3. 系统返回响应：无表单交互，直接跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 查询任务状态失败 | 提示错误，允许用户重试 |
| 任务状态查询超时 | Toast「查询超时，请刷新重试」 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 有进行中任务 | 查询到 running 状态任务 |
| 有进行中任务 | 已跳转 | 自动跳转至搜客执行中页面 |

**输出**：自动跳转至搜客执行中页面，无任何表单交互。

### 场景四：快速筛选标签使用

**触发条件**：用户点击快速筛选标签（美国/欧洲/有海关记录/近期参展/大型企业）。

**执行逻辑**：

1. 用户点击标签按钮
2. 标签按钮切换为选中状态（高亮样式）
3. 标签内容自动追加到文本框：
   - 若文本框已有内容且不以标点/空格结尾，自动添加"，"
   - 追加标签文本
4. 更新字符计数
5. 再次点击已选中的标签，取消选中状态，但不从文本框删除（避免误操作）

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 追加后超过 500 字符 | 自动截断至 500 字符，Toast「已自动截断至最大长度」 |
| 文本框为空 | 直接追加标签内容，不加标点 |

**状态变更**：无

**输出**：文本框内容更新，标签选中状态变更。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| ICP 画像状态 | 数据查询 | 服务端根据 `bot_id` 查询 `bot_customer_profiles` 表，校验 `scoring_model` 字段完整性和 `updated_at` 时间（90 天内有效） | MySQL `bot_customer_profiles` 表 |
| ICP 权重配置 | 数据查询 | 从 `bot_customer_profiles.scoring_model.weights` 读取四维权重配置 | MySQL `bot_customer_profiles` 表 |
| MoliAI点数余额 | 数据查询 | 从 `moli_ai_balances` 表查询 `balance` 字段 | MySQL `moli_ai_balances` 表 |
| 任务状态 | 数据查询 | 查询 `search_tasks` 表中该 BOT 最新任务的状态 | MySQL `search_tasks` 表 |
| 进行中任务 | 数据查询 | 查询 `search_tasks` 表中 `status` = 'running' 的任务记录 | MySQL `search_tasks` 表 |

| 字段名 | 类型 | 说明 |

| 字段名 | 类型 | 说明 |

| balance | decimal(10,2) | 当前 MoliAI点数余额 |
| updated_at | datetime | 更新时间 |

## 数据流

### API 一：查询任务状态

```
GET /api/v1/search/status?bot_id={bot_id}
```

**请求头**：
- `Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | 查询参数，BOT 唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "has_running_task": true,
    "task_id": "TASK-202401-001",
    "status": "running",
    "icp_status": "ready",
    "moli_ai_balance": 60000
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|----------|-------------|------|------|
| `has_running_task` | 是否有进行中任务 | boolean | true 表示存在进行中的搜客任务 |
| `task_id` | 任务 ID | string | 进行中的任务唯一标识 |
| `status` | 任务状态 | string | running / pending / completed / failed / none |
| `icp_status` | ICP 画像状态 | string | ready / missing / expired |
| `moli_ai_balance` | MoliAI点数余额 | number | 当前租户 MoliAI点数余额 |

**响应码说明**：

| code | 含义 | 处理方式 |
|------|------|---------|
| 200 | 查询成功 | 正常渲染页面 |
| 401 | 未登录 | 跳转登录页面，Toast「请先登录」 |
| 404 | BOT 不存在 | Toast「BOT 不存在或已被删除」 |
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」 |

### API 二：提交搜客需求

```
POST /api/v1/search/input
```

**请求头**：
- `Authorization: Bearer {tenant_token}`
- `Content-Type`: `application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | `string` | 是 | 租户 ID，从登录态获取 |
| bot_id | `string` | 是 | BOT 实例 ID |
| user_query | `string` | 否 | 用户输入的自然语言需求（已包含快速标签追加的内容）；为空时后端使用 ICP 画像 |
| target_count | `number` | 否 | 目标企业数量，默认 100，范围 20-200 |
| use_default_icp | `boolean` | 否 | 是否使用默认 ICP；user_query 为空时自动设为 true |

**请求示例**：

```json
{
  "tenant_id": "TENANT-001",
  "bot_id": "BOT-001",
  "user_query": "找一下加州的音视频集成商，优先考虑有展会记录的企业，美国，近期参展",
  "target_count": 100,
  "use_default_icp": false
}
```

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "TASK-20240525-001",
    "status": "pending",
    "redirect_to": "/搜客策略预览.html?bot_id=BOT-001&task_id=TASK-20240525-001"
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|----------|-------------|------|------|
| `task_id` | 任务 ID | string | 新创建的搜客任务唯一标识 |
| `status` | 任务状态 | string | 固定返回 pending |
| `redirect_to` | 跳转地址 | string | 下一步跳转的页面地址 |

**响应码说明**：

| code | 含义 | 处理方式 |
|------|------|---------|
| 200 | 任务创建成功 | 跳转至搜客策略预览页面 |
| 400 | 参数错误 | 根据错误字段高亮对应输入框 |
| 1002 | MoliAI点数余额不足 | Toast「额度不足，请充值」，按钮禁用 |
| 1003 | ICP 画像不存在 | Toast「ICP 画像不存在，请重新生成业务分析报告」 |
| 1004 | ICP 画像已过期 | Toast「ICP 画像已过期，请重新分析」 |
| 1005 | 已有任务执行中 | 自动跳转至搜客执行中页面 |
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求无响应超过 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」，显示重试按钮 |
| LLM 超时（>60s） | 策略生成调用超时 | 降级返回预置策略模板 | Toast「AI 分析超时，已使用默认策略」 |
| MoliAI点数不足 | 余额 < 预估消耗 | 返回错误码 1002 | "确认搜客"按钮禁用，Toast「额度不足，请充值」 |
| MoliAI点数为负 | 余额 ≤ 0 | 返回错误码 1002 | 按钮禁用，显示"额度已用完，请充值" |
| 任务并发 | search/status 返回有进行中任务 | 返回错误码 1005 | 自动跳转至搜客执行中页面 |
| ICP 数据缺失 | /search/status 返回 icp_status = missing | 返回错误码 1003 | Toast「ICP 画像不存在，请重新生成」，按钮禁用，提供跳转链接 |
| ICP 数据过期 | icp_status = expired | 返回错误码 1004 | Toast「ICP 画像已过期，请重新分析」，按钮禁用 |
| 参数错误 | 接口返回 400 | 返回具体错误字段 | 根据错误字段高亮对应输入框，如 target_count 越界 |
| 服务不可用 | HTTP 500 | 记录错误日志，触发告警 | Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | HTTP 401 | 返回 401 | 跳转登录页面，Toast「请先登录」 |
| BOT 不存在 | HTTP 404 | 返回 404 | Toast「BOT 不存在或已被删除」 |
| 重复提交 | 防抖处理 | 同一任务 5 秒内重复提交返回错误 | 按钮 loading 状态防止重复点击 |
