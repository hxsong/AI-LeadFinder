## 概述

搜客需求输入是静态数据获客的入口页面。用户通过自然语言补充短期搜客需求，系统结合 P09 阶段生成的 ICP 画像，生成完整评分参数，传递给下游 P19 阶段进行潜在客户打分计算。

### 入口

| 来源 | 触发条件 | 携带参数 |

## 功能逻辑

### 场景一：正常搜客流程

**触发条件**：用户有明确的短期搜客需求，且 Credits 余额充足。

**执行逻辑**：

1. 用户输入自然语言需求（最大 500 字符）
2. 用户可选：点击快速筛选标签（美国/欧洲/有海关记录/近期参展/大型企业），标签内容自动追加到文本框
3. 用户拖动滑块设置目标企业数量（默认 100 家，范围 20-200）
4. 用户点击"确认搜客"按钮
5. 前端将请求发至 `/api/v1/search/input`
6. 后端返回 `task_id`，前端跳转至搜客策略预览页面

**输出**：跳转到搜客策略预览页面。

---

### 场景二：使用 ICP 画像默认搜索

**触发条件**：用户未输入任何文字，直接提交空表单。

**执行逻辑**：

1. 前端检测 `user_query` 为空，自动设置 `use_default_icp=true`
2. 请求发送至 `/api/v1/search/input`
3. 后端读取该 BOT 的 P09 ICP 画像作为搜索指令
4. 返回 `task_id`，前端跳转

**输出**：与场景一相同，跳转到搜客策略预览页面。

---

### 场景三：已有任务在执行

**触发条件**：进入页面时，后端检测到该 BOT 存在状态为"执行中"的搜客任务。

**执行逻辑**：

1. 前端在进入页面时调用 `/api/v1/search/status` 查询任务状态
2. 若存在进行中任务，前端自动跳转到搜客执行中页面
3. 用户无法在本页面发起新任务

**输出**：自动跳转至搜客执行中页面，无任何表单交互。

## 数据流

### 请求

```
POST /api/v1/search/input
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | string | 是 | 租户 ID |
| bot_id | string | 是 | BOT 实例 ID |
| user_query | string | 否 | 用户输入的自然语言需求；为空时后端使用 ICP 画像 |
| target_count | number | 否 | 目标企业数量，默认 100，范围 20-200 |
| use_default_icp | boolean | 否 | 是否使用默认 ICP；`user_query` 为空时自动设为 `true` |

**响应体**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "STR-xxx",
    "status": "pending"
  }
}
```

| code | 含义 |
|------|------|
| 200 | 任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 500 | 服务异常 |

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 文本长度 | `user_query` 超过 500 字符 | 超出部分自动截断，不报错 |
| Credits 余额 | 余额 ≤ 0 或余额 < 预估消耗 | "确认搜客"按钮禁用，提示充值 |
| 前置条件 | 未完成 P08 业务分析 | 按钮禁用，提示"请先完成业务初始化" |
| 并发限制 | 当前 BOT 已有搜客任务执行中 | 进入页面时自动跳转至搜客执行中页面 |
| ICP 画像 | ICP 数据不存在或已过期 | 禁止提交，提示"请重新生成业务分析报告" |

## API 清单

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/search/input | POST | 提交搜客需求，创建搜客任务 |
| /api/v1/search/status | GET | 查询当前 BOT 是否有进行中任务 |
| /api/v1/bot/{id}/icp | GET | 获取 ICP 画像（页面初始化时调用） |

## 异常处理

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求无响应超过 30s | 重试 1 次 | Toast "网络异常，请检查网络后重试" |
| LLM 超时（>60s） | gpt-4o 调用超时 | 降级返回预置策略模板 | Toast "AI 分析超时，已使用默认策略" |
| Credits 不足 | 余额 < 预估消耗 | 不发起请求 | 按钮禁用，Toast "额度不足，请充值" |
| 任务并发 | `search/status` 返回有进行中任务 | — | 自动跳转至搜客执行中页面 |
| ICP 数据缺失 | `/bot/{id}/icp` 返回 404 | — | Toast "ICP 画像不存在，请重新生成"，按钮禁用 |
| 服务不可用 | HTTP 500 | 记录错误日志 | Toast "服务暂时不可用，请稍后再试" |

## 字段映射

| 前端数据 | 字段名 | 后端/存储 | 说明 |
|---------|--------|---------|------|
| BOT ID | `bot_id` | `bot_instances.bot_id` | URL 参数或全局状态 |
| ICP 画像权重 | `icp_weights` | AI 分析结果（缓存于 Redis） | 有效期 24h |
| 用户需求文本 | `user_query` | `search_tasks.user_query` | 用户输入，最多 500 字符 |
| 目标数量 | `target_count` | `search_tasks.target_count` | 整数，范围 20-200 |
| 使用默认 ICP | `use_default_icp` | 计算字段 | user_query 为空时为 true |
| 搜客任务 ID | `task_id` | `search_tasks.task_id` | 后端生成，UUID |
| 任务状态 | `status` | `search_tasks.status` | pending / running / completed / failed |
