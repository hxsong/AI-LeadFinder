## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| AI业务分析中.html | 所有阶段处理完成后自动跳转（延迟 1.5 秒） | `bot_id`, `task_id` |

**入口前置条件**：
- 用户已登录（携带有效 tenant_token）
- URL 中包含有效的 `bot_id` 和 `task_id` 参数
- 对应的分析任务状态为 `completed`
- `bot_customer_profiles` 表中存在该 BOT 的有效画像数据

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 业务初始化.html | 点击"重新分析"按钮，确认后跳转 | `bot_id` |
| 搜客需求输入.html | 点击"开始搜客"按钮 | `bot_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 页面加载，请求报告数据 | 页面加载完成 | 数据请求返回 |
| 正常 | 报告数据加载成功且数据完整 | 接口返回 200 且数据完整 | 用户点击跳转按钮 |
| 报告不存在 | 查询不到该 BOT 的画像数据 | 接口返回 404 | 跳转至业务初始化页面 |
| 数据异常 | 必要字段缺失或格式错误 | 接口返回 500 或数据校验失败 | 用户点击重新分析 |
| AI 字段异常 | AI 输出字段不完整 | 部分 AI 字段缺失 | 显示降级文案 |

**状态流转图**：

```
[加载中] --(报告数据加载成功)--> [正常]
[正常] --(点击重新分析)--> [跳转至业务初始化]
[正常] --(点击开始搜客)--> [跳转至搜客需求输入]
[加载中] --(报告不存在)--> [跳转至业务初始化]
[加载中] --(数据异常)--> [异常]
```

## 业务处理逻辑

### 场景一：查看完整业务分析报告

**触发条件**：从 AI 业务分析中页面跳转进入，或用户直接访问报告页面。

**执行逻辑**：

1. 页面加载时解析 URL 参数，获取 `bot_id` 和 `task_id`：
   - 任一参数缺失：跳转至业务初始化页面

2. 前端调用接口获取报告数据：
   - 接口：`GET /api/v1/bots/{bot_id}/analysis-report`
   - 携带 `tenant_token` 认证

3. 页面展示完整报告内容：
   - **企业定位区**：企业名称、业务类型标签、企业定位描述
   - **企业规模区**：员工规模、营收规模
   - **主营品类标签区**：一级/二级分类标签展示
   - **核心优势标签区**：最多 5 条核心优势标签
   - **供应链分析区**：上游依赖 → 自身生态位 → 下游潜客群流程图
   - **目标客户画像区**：客户类型、市场区域、主营品类、企业规模
   - **四维评分模型区**：
     - 四维评分卡片：需求匹配度、购买能力、采购意向、活跃意向（0-100 分值）
     - 权重分布可视化：根据业务类型动态生成的权重占比条
     - 评分逻辑说明：各维度的定义和计算方式说明

4. 页面动画效果：
   - 各模块依次淡入上滑（fadeInUp）
   - 四维评分卡片带悬停效果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 报告不存在（404） | 跳转至初始化页面，Toast「报告不存在或已过期，请重新开始」 |
| 必要字段缺失 | 显示降级文案，Toast「AI 分析不完整」 |
| 可选字段缺失 | 隐藏对应模块，不影响其他内容展示 |
| 评分数据异常 | 使用默认值 0 替代，Toast「评分数据异常」 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 报告数据加载成功 |
| 加载中 | 报告不存在 | 接口返回 404 |
| 加载中 | 数据异常 | 接口返回 500 或数据格式错误 |

**输出**：完整业务分析报告页面。

### 场景二：四维评分权重说明

**触发条件**：用户查看评分权重配置区域。

**执行逻辑**：

1. 前端展示当前报告使用的四维权重值（从 `scoring_model.weights` 读取）
2. 权重总和校验：
   - 前端校验权重总和是否等于 1.00
   - 若不等于 1.00，前端做归一化处理或显示异常提示
3. 展示各维度定义说明：
   - **需求匹配度**：AI 分析客户与候选企业的供需关系，判断品类匹配、业务模式匹配、市场范围匹配程度
   - **购买能力**：企业规模评分（60%）+ 地理位置评分（40%），评估企业的采购预算和规模
   - **采购意向**：基于海关 HS Code 匹配度计算，应用时间衰减系数（近期记录权重更高）
   - **活跃意向**：基于展会参展记录计算，参展商身份得分更高，同样应用时间衰减系数
4. 展示业务类型标识：
   - 根据 `business_type` 显示对应的业务类型标签（外贸型/生产型/服务型/渠道型）

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 权重总和不为 1.00 | 前端做归一化处理，或显示「权重配置异常」提示 |
| 权重字段缺失 | 使用业务类型默认权重替代 |

**状态变更**：无

**输出**：权重说明模块。

### 场景三：重新分析

**触发条件**：用户点击底部"重新分析"按钮。

**执行逻辑**：

1. 弹出确认对话框：
   - 标题："确认重新分析？"
   - 内容："重新分析会清除当前报告和向量数据，需要重新选择数据来源并上传材料"
   - 按钮："取消" / "确认"

2. 用户确认后，前端调用重新分析接口：
   - 接口：`POST /api/v1/analysis/restart`
   - 请求体：`{ "bot_id": "BOT-001" }`

3. 后端处理：
   - 删除 `bot_customer_profiles` 表中该 BOT 的画像数据
   - 删除 `analysis_tasks` 和 `analysis_stages` 表中该 BOT 的历史任务记录
   - 清除相关的向量数据
   - 重置 BOT 的数据来源状态

4. 前端接收成功响应后：
   - 跳转至业务初始化页面
   - 携带 `bot_id` 参数

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 用户取消确认 | 关闭对话框，留在当前页面 |
| 重新分析接口失败 | Toast「重新分析失败，请稍后重试」 |
| 网络超时 | Toast「网络连接异常，请检查网络后重试」 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 跳转中 | 用户确认重新分析 |
| 跳转中 | 业务初始化 | 接口返回成功 |

**输出**：跳转至业务初始化页面。

### 场景四：开始搜客

**触发条件**：用户确认报告无误，点击"开始搜客"按钮（页面底部 CTA 区域）。

**执行逻辑**：

1. 前端校验：
   - 检查是否存在有效的 ICP 画像数据
   - 检查 Credits 余额是否充足（可选，具体在搜客需求输入页面校验）

2. 前端跳转到搜客需求输入页面：
   - 目标页面：`搜客需求输入.html`
   - 携带参数：`bot_id`

3. 搜客需求输入页面将使用当前报告的 ICP 画像作为默认搜索条件

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| ICP 画像不存在 | 按钮禁用，Toast「请先生成业务分析报告」 |
| bot_id 传递失败 | 跳转失败，展示错误提示 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 跳转中 | 用户点击"开始搜客" |
| 跳转中 | 搜客需求输入 | 页面跳转成功 |

**输出**：跳转至搜客需求输入页面。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 企业画像数据 | 数据查询 | 服务端根据 `bot_id` 查询 `bot_customer_profiles` 表，返回 `customer_info` 和 `scoring_model` 字段的 JSON 数据 | MySQL `bot_customer_profiles` 表 |
| 报告状态 | 状态判断 | 服务端校验 `customer_info` 和 `scoring_model` 字段的完整性，判断报告是否可用 | 实时计算，不持久化 |
| 四维评分 | 直接读取 | 从 `scoring_model.scores` 中直接读取 AI 生成的评分值 | MySQL `bot_customer_profiles` 表 |
| 四维权重 | 直接读取 | 从 `scoring_model.weights` 中直接读取根据业务类型匹配的预设权重 | MySQL `bot_customer_profiles` 表 |
| 生成时间 | 直接读取 | 从 `created_at` 字段读取 | MySQL `bot_customer_profiles` 表 |

### 重新分析处理流程

1. **接收请求**：服务端接收 `POST /api/v1/analysis/restart` 请求，携带 `bot_id`
2. **数据清除**：
   - 删除 `bot_customer_profiles` 表中该 BOT 的画像数据
   - 删除 `analysis_tasks` 和 `analysis_stages` 表中该 BOT 的历史任务记录
   - 清除相关的向量数据（如已生成）
3. **状态重置**：将该 BOT 的数据来源状态重置为未选择
4. **返回响应**：返回成功状态，前端跳转至业务初始化页面

### 数据表结构

**bot_customer_profiles 表**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| profile_id | varchar(32) | 主键，画像唯一 ID |
| bot_id | varchar(32) | 外键，关联 BOT |
| task_id | varchar(32) | 外键，关联分析任务 |
| customer_info | json | 客户信息（企业名称、定位、业务类型、主营品类、核心优势、企业规模、供应链分析、目标客户画像等） |
| scoring_model | json | 评分模型（业务类型、四维评分、四维权重） |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 报告不存在 | HTTP 404 | 返回 404 错误码 | 跳转至初始化页面，Toast「报告不存在或已过期，请重新开始」 |
| 数据异常 | 必要字段缺失或格式错误 | 返回 500 或部分数据 | 异常提示，Toast「数据异常，建议重新分析」 |
| AI 输出为空 | 四维评分字段为空 | 返回空数据 | 显示默认评分 0，Toast「AI 分析结果为空，请重新分析」 |
| AI 格式错误 | JSON 解析失败 | 返回原始数据 | 显示原始数据，Toast「AI 返回数据格式异常」 |
| AI 内容幻觉 | 与原始数据矛盾 | 返回标记数据 | 标记仅供参考，Toast「AI 分析结果仅供参考，以实际数据为准」 |
| AI 字段缺失 | 必要字段缺失 | 返回部分数据 | 使用默认值，Toast「AI 分析不完整」 |
| 评分超出范围 | 评分 <0 或 >100 | 返回异常数据 | 默认值替代，Toast「AI 评分异常，已使用默认值」 |
| 权重总和异常 | 权重和不等于 1 | 返回异常数据 | 前端归一化处理，Toast「权重配置异常，已自动校准」 |
| 网络超时 | 请求超时（>10s 无响应） | 返回 504 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | HTTP 500 / 503 | 返回对应错误码，触发告警 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | HTTP 401 | 返回 401 | 跳转登录页面，Toast「请先登录」 |
| 重新分析失败 | 清除数据失败 | 返回 500 | Toast「重新分析失败，请稍后重试」 |
| 跳转失败 | 页面跳转异常捕获 | - | 展示手动跳转按钮，Toast「页面跳转失败，请手动继续」 |
