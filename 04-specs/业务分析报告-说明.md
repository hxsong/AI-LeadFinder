## 概述

业务分析报告页面是业务初始化流程的最终展示页面，基于 P08 阶段生成的四维评分模型（需求匹配度、购买能力、采购意向、活跃意向）展示深度业务分析结果。报告包含企业定位、业务类型、四维雷达图、目标客户画像、核心优势标签等模块，供用户确认后进入搜客流程。

## 页面流转

### 入口定义

| 来源页面 | 触发条件 | 携带参数 |
|---------|---------|---------|
| AI业务分析中.html | 所有阶段处理完成后自动跳转 | `bot_id`, `task_id` |

### 出口定义

| 目标页面 | 触发条件 | 携带数据 |
|---------|---------|---------|
| 业务初始化.html | 点击"重新分析"按钮 | `bot_id` |
| 搜客需求输入.html | 点击"开始搜客"按钮 | `bot_id` |

### 页面状态及流转条件

```
[加载中] --(报告加载成功)--> [正常]
[正常] --(点击重新分析)--> [跳转至业务初始化]
[正常] --(点击开始搜客)--> [跳转至搜客需求输入]
[加载中] --(报告不存在)--> [跳转至业务初始化]
[加载中] --(向量数据丢失)--> [异常]
```

| 状态 | 触发条件 | UI 表现 |
|-----|---------|---------|
| 正常 | 报告加载成功 | 展示完整报告 |
| 加载中 | 请求发出后 | 骨架屏或 loading 动画 |
| 报告不存在 | code 404 | 跳转至初始化页面 |
| 向量数据丢失 | code 500 | 异常提示，建议重新分析 |
| AI 异常 | AI 字段异常 | AI 区域显示降级文案 |

**请求头**：`Authorization: Bearer {tenant_token}`

  "code": 200,
  "data": {
    "bot_id": "BOT-001",
    "status": "restarted",
    "redirect_to": "/business-init.html?bot_id=BOT-001"

## 处理逻辑

### 场景一：查看完整业务分析报告

**触发条件**：从 AI 业务分析中页面跳转进入。

**执行逻辑**：
1. 前端请求报告详情数据
2. 页面展示完整报告内容：
   - 企业定位区：企业名称、业务类型标签、企业定位描述
   - 四维雷达图：需求匹配度、购买能力、采购意向、活跃意向
   - 主营品类标签区：一级/二级分类
   - 核心优势标签区
   - 目标客户画像区：客户类型、市场区域、主营品类、企业规模
   - 评分权重说明

**边界条件**：
- 报告不存在时跳转至初始化页面
- 向量数据丢失时显示异常提示

**状态变更**：`加载中` → `正常` / `报告不存在`

**输出**：完整业务分析报告页面。

### 场景二：四维评分权重说明

**触发条件**：用户查看评分权重配置。

**执行逻辑**：
1. 展示当前报告使用的四维权重值
2. 展示各维度定义说明：
   - 需求匹配度：候选企业是否需要客户的产品
   - 购买能力：企业规模评分（60%）+ 地理评分（40%）
   - 采购意向：海关 HS Code 匹配度 × 时间衰减系数
   - 活跃意向：展会记录 × 时间衰减系数
3. 权重总和必须等于 100

**边界条件**：
- 权重总和不为 1.00 时前端需做归一化处理或提示异常

**状态变更**：无

**输出**：权重说明模块。

### 场景三：重新分析

**触发条件**：用户点击"重新分析"按钮。

**执行逻辑**：
1. 弹出确认提示："重新分析会清除当前报告和向量数据"
2. 用户确认后，调用重新分析接口
3. 后端清除当前报告和向量数据
4. 跳转至数据来源选择页面

**边界条件**：
- 用户取消确认则留在当前页面
- 重新分析接口失败需提示错误

**状态变更**：`正常` → 跳转至业务初始化

**输出**：跳转至业务初始化页面。

### 场景四：开始搜客

**触发条件**：用户确认报告无误，点击"开始搜客"。

**执行逻辑**：
1. 前端跳转到搜客需求输入页面
2. 使用当前报告的 ICP 画像作为搜客输入条件

**边界条件**：
- 需确保 bot_id 正确传递

**状态变更**：`正常` → 跳转至搜客需求输入

**输出**：跳转至搜客需求输入页面。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 报告不存在 | code 404 | 返回 404 | 跳转至初始化页面，Toast「报告不存在或已过期，请重新开始」 |
| 向量数据丢失 | code 500 | 返回 500 | 异常提示，Toast「数据异常，建议重新分析」 |
| AI 输出为空 | 四维评分为空 | 返回空数据 | 显示默认评分 0，Toast「AI 分析结果为空，请重新分析」 |
| AI 格式错误 | JSON 解析失败 | 返回原始数据 | 显示原始数据，Toast「AI 返回数据格式异常」 |
| AI 内容幻觉 | 与原始数据矛盾 | 返回标记数据 | 标记仅供参考，Toast「AI 分析结果仅供参考，以实际数据为准」 |
| AI 字段缺失 | 必要字段缺失 | 返回部分数据 | 使用默认值，Toast「AI 分析不完整」 |
| 评分超出范围 | 评分 <0 或 >100 | 返回异常数据 | 默认值替代，Toast「AI 评分异常，已使用默认值」 |
| 网络超时 | 请求超时 | 返回 504 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 返回 500 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | code 401 | 返回 401 | 跳转登录页面，Toast「请先登录」 |
