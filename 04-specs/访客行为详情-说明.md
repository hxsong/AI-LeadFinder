## 数据契约

### 前端展示字段映射

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 访客 ID | `visitor_log_id` | `string` | 路由参数传入 | 系统生成 UUID |
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| IP 地址（脱敏） | `visitor_identity.ip_masked` | `string` | GeoIP 脱敏处理 | 保留前3段，最后一段脱敏为 `**` |
| 地理位置 | `visitor_identity.geo_location` | `string` | GeoIP 定位数据 | 格式：城市, 国家/地区 |
| 设备类型 | `visitor_identity.device_type` | `string` | UA 解析 | Desktop/Mobile/Tablet |
| 操作系统 | `visitor_identity.os` | `string` | UA 解析 | Windows/macOS/iOS/Android |
| 流量来源 | `traffic_context.source` | `string` | 流量归因数据 | Google 搜索/直接访问等 |
| 落地页 | `traffic_context.landing_page` | `string` | 首次访问 URL 路径 | 页面路径，不含域名 |
| UTM 参数 | `traffic_context.utm_params` | `object` | URL 解析获取 | JSON 对象完整存储 source/medium/campaign 等 |
| 访问次数 | `visit_stats.visit_count` | `number` | 同一 IP 历史来访总次数 | — |
| 首次访问 | `visit_stats.first_visit` | `string` | 接口返回 | ISO 8601 日期 |
| 最近访问 | `visit_stats.last_visit` | `string` | 接口返回 | ISO 8601 日期 |
| 规则打分 | `behavior_analysis.rule_based_score` | `number` | 行为打分引擎 | 0-100 |
| 热度等级 | `behavior_analysis.heat_level` | `string` | 系统映射 | very_high/high/normal |
| AI 分析触发 | `behavior_analysis.ai_analysis_triggered` | `boolean` | 系统判断 | 规则打分 ≥ 60 时为 true |
| AI 关键洞察 | `behavior_analysis.ai_insights` | `array` | 上游处理完成 | 需做空值校验 |
| AI 综合研判 | `behavior_analysis.ai_reasoning` | `string` | 上游处理完成 | 需做空值和格式校验 |
| 行为时间戳 | `current_session.events[].timestamp` | `number` | Unix 时间戳（秒） | — |
| 行为类型 | `current_session.events[].event_type` | `string` | 探针采集 | page_view / copy_text / file_download / click / form_submit / form_abandon / session_end |
| 行为对象 | `current_session.events[].target_object` | `object` | 各类型携带不同字段 | 见下方行为对象字段说明 |
| 停留时长 | `current_session.events[].metrics.stay_duration_ms` | `number` | 毫秒 | — |
| 滚动深度 | `current_session.events[].metrics.scroll_depth` | `number` | 百分比 | 0-100 |
| AI 单行为解读 | `current_session.events[].ai_interpretation` | `string` | 上游处理完成 | — |
| 会话时长 | `current_session.duration_seconds` | `number` | 秒 | — |
| 历史会话日期 | `historical_sessions[].date` | `string` | ISO 8601 日期 | — |
| 历史会话打分 | `historical_sessions[].score` | `number` | 该会话规则打分 | — |
| 历史会话摘要 | `historical_sessions[].summary` | `string` | 会话关键行为摘要 | — |
| 是否已挖掘 | `is_mined` | `boolean` | 接口返回 | 该访客是否已触发深度挖掘 |

### 行为对象字段说明

| 行为类型 | target_object 字段 | 说明 |
|---------|-------------------|------|
| page_view | `page_title`, `url_path` | 页面标题和路径 |
| copy_text | `copied_content` | 被复制的文本内容（脱敏处理） |
| file_download | `file_name`, `file_type`, `file_size` | 文件名、类型、大小 |
| click | `element_type`, `element_text`, `click_position` | 元素类型、文本、位置 |
| form_submit | `form_id`, `form_title` | 表单 ID 和标题 |
| form_abandon | `form_id`, `form_title`, `fill_progress` | 表单 ID、标题、填表进度 |
| session_end | `exit_time` | 退出时间 |

### 数据来源深度说明

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 热度等级 | `behavior_analysis.heat_level` | `string` | 上游处理完成 | 基于 `rule_based_score` 映射：≥80 为 very_high（极高）、60-79 为 high（较高）、<60 为 normal（一般）。数据来自访客雷达页面的机器规则引擎计算 |
| 规则打分 | `behavior_analysis.rule_based_score` | `number` | 上游处理完成 | 基于访客行为数据实时计算，数据来自访客雷达页面的机器规则引擎计算 |
| AI 分析触发 | `behavior_analysis.ai_analysis_triggered` | `boolean` | 上游处理完成 | 规则判断：`rule_based_score >= 60`。触发后由访客雷达页面调用 LLM（Step 3 AI 意图分析）生成关键动作洞察和综合研判建议 |
| AI 关键洞察 | `behavior_analysis.ai_insights` | `array` | 上游处理完成 | 由访客雷达页面的 Step 3 AI 意图分析流程生成，本页面仅做展示 |
| AI 综合研判 | `behavior_analysis.ai_reasoning` | `string` | 上游处理完成 | 由访客雷达页面的 Step 3 AI 意图分析流程生成，本页面仅做展示 |
| AI 单行为解读 | `current_session.events[].ai_interpretation` | `string` | 上游处理完成 | 由访客雷达页面的 Step 3 AI 意图分析流程生成的事件级洞察 |
| IP 地址（脱敏） | `visitor_identity.ip_masked` | `string` | 系统生成 | 原始 IP 经过脱敏处理：保留前3段（IPv4）或前4组（IPv6），剩余部分替换为 `**` |
| 地理位置 | `visitor_identity.geo_location` | `string` | 系统生成 | 通过 GeoIP 数据库（MaxMind GeoIP2）查询 IP 对应的地理位置，格式化为 "城市, 国家/地区" |
| 设备类型/操作系统 | `visitor_identity.device_type` / `visitor_identity.os` | `string` | 系统生成 | 解析 User-Agent 字符串，使用 `ua-parser-js` 库提取设备类型和操作系统信息 |
| 流量来源 | `traffic_context.source` | `string` | 系统生成 | 基于 Referer 和 UTM 参数解析：识别搜索引擎（Google/Bing/百度）、直接访问、社交媒体等来源 |
| UTM 参数 | `traffic_context.utm_params` | `object` | 系统生成 | JSON 对象完整存储，包含 source/medium/campaign 等字段 |
| 历史会话 | `historical_sessions` | `array` | 系统生成 | 存储最近 90 天内该 IP 的所有历史会话记录 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 访客雷达.html | 用户点击访客卡片 | `bot_id`, `visitor_log_id`, `session_id`（可选） |

**入口前置条件**：
1. 用户已登录且拥有企业管理员权限
2. 当前账号已完成探针部署（已绑定域名且探针状态为 active）
3. 访客记录存在于系统中且未过期（90 天内）

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 访客雷达.html | 用户点击"返回访客雷达"链接 | `bot_id` |
| 动态线索池.html | 深度挖掘完成后自动跳转 | `bot_id` |
| 动态线索详情.html | 深度挖掘完成后自动跳转（已完成状态） | `bot_id`, `dynamic_lead_id` |
| 外部链接 | 用户点击 LinkedIn/Twitter 链接 | URL（新标签页打开） |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后等待数据返回 | 页面加载 | 数据返回成功或失败 |
| 正常（高分） | 数据完整返回，规则打分 ≥ 60，显示 AI 洞察区域 | 数据返回成功且 `ai_analysis_triggered` = true | 用户操作或页面跳转 |
| 正常（低分） | 数据完整返回，规则打分 < 60，不显示 AI 洞察区域 | 数据返回成功且 `ai_analysis_triggered` = false | 用户操作或页面跳转 |
| 空状态 | 访客无行为记录 | 数据返回成功但行为事件为空 | 用户返回访客雷达 |
| 深度挖掘中 | 挖掘任务创建中 | 用户点击深度挖掘按钮并确认 | 挖掘接口返回成功或失败 |
| 错误状态 | 数据加载失败 | 接口返回 4xx/5xx 错误或网络异常 | 用户点击重试或返回 |

**状态流转图**：
```
[加载中] --(数据返回成功)--> [正常（高分）]  (ai_analysis_triggered = true)
    │                           │
    │                           │--(规则打分 < 60)--> [正常（低分）]
    │                           │
    └──(数据返回失败)-----> [错误状态]

[正常（高分）] --(点击深度挖掘)--> [深度挖掘中] --(挖掘完成)--> [动态线索池]
```

## 业务处理逻辑

### 场景一：正常查看访客行为详情（高分访客）

**触发条件**：用户从访客雷达点击高分访客卡片（规则打分 ≥ 60）进入。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `visitor_log_id` 和 `bot_id`
2. 系统执行处理：
   - 前端请求 `GET /api/v1/visitors/{visitor_log_id}` 获取完整行为报告
   - 页面展示访客基础信息卡片（IP、地理位置、设备、操作系统、流量来源、落地页、访问次数、首次访问、最近访问）
   - 展示高维微观行为轨迹时间轴（按时间倒序排列每个行为节点）
   - 所有行为节点使用统一的中性样式展示：
     - 页面浏览：展示时间戳、页面标题、URL 路径、停留时长、滚动深度
     - 复制文本：展示被复制的具体内容
     - 文件下载：展示文件名、文件类型、文件大小
     - 点击行为：展示元素类型、元素文本、点击位置
     - 意图流失：展示表单 ID、表单 Title、填表进度、弃填状态
     - 会话结束：展示退出时间、总会话时长
   - 展示同一 IP 的历史访问记录（按时间倒序排列）
   - 右侧面板展示：规则打分结果、AI 关键动作洞察、AI 综合研判建议
   - 显示"深度挖掘"按钮（如未挖掘）或"已挖掘"状态（如已挖掘）
3. 系统返回响应：返回完整行为报告数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 访客不存在 | 提示并返回访客雷达列表，Toast「该访客记录不存在或已被清除」 |
| 访客数据已过期（>90天） | 提示数据已归档，建议查看历史记录 |
| AI 洞察字段为空 | 显示默认解读模板「AI 分析结果为空」 |
| 无历史会话记录 | 显示「暂无历史访问记录」空态 |
| 行为事件为空 | 显示「暂无行为记录」空态 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常（高分） | 数据返回成功且 `ai_analysis_triggered` = true |
| 加载中 | 空状态 | 数据返回成功但行为事件为空 |
| 加载中 | 错误状态 | 数据返回失败 |

**输出**：完整访客行为报告页面（高分版本）。

### 场景二：正常查看访客行为详情（低分访客）

**触发条件**：用户从访客雷达点击低分访客卡片（规则打分 < 60）进入。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `visitor_log_id` 和 `bot_id`
2. 系统执行处理：
   - 前端请求 `GET /api/v1/visitors/{visitor_log_id}` 获取完整行为报告
   - 页面展示访客基础信息卡片（与高分访客一致）
   - 展示高维微观行为轨迹时间轴（按时间倒序排列）
   - 展示历史访问记录
   - 右侧面板展示：规则打分结果
   - 右侧面板显示"AI 分析未触发"提示区域，说明规则打分未达到阈值（60分）
   - 显示建议操作列表：持续观察、等待行为分提升、当前不建议深度挖掘
   - "深度挖掘"按钮置灰禁用，提示「访客热度较低，建议持续观察」
3. 系统返回响应：返回完整行为报告数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 访客不存在 | 同场景一 |
| 访客数据已过期 | 同场景一 |
| 无行为事件 | 同场景一 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常（低分） | 数据返回成功且 `ai_analysis_triggered` = false |
| 加载中 | 空状态 | 同场景一 |
| 加载中 | 错误状态 | 同场景一 |

**输出**：完整访客行为报告页面（低分版本）。

### 场景三：行为轨迹时间轴展示

**触发条件**：访客有多个行为记录，页面正常加载完成。

**执行逻辑**：

1. 系统接收请求：行为数据已加载
2. 系统执行处理：
   - 按时间倒序展示行为节点（最新的在最上方）
   - 所有行为类型使用统一的中性 UI 样式（灰色调）
   - 每个节点展示：时间戳、行为类型标签、行为描述、相关指标
   - 高价值行为节点（复制文本、文件下载、表单交互）额外展示 AI 行为解读卡片
   - 时间轴左侧显示连接线，表示行为先后顺序
3. 系统返回响应：时间轴渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无行为记录 | 显示「暂无行为记录」空态 |
| 单一会话结束事件 | 显示简化时间轴 |
| AI 解读字段缺失 | 隐藏该行为的 AI 解读卡片 |

**状态变更**：无

**输出**：带 AI 解读的行为轨迹时间轴。

### 场景四：深度挖掘（存在有效 UTM）

**触发条件**：访客存在有效 UTM 参数（source 和 medium 同时存在），用户点击深度挖掘按钮。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 弹出深度挖掘确认弹窗，展示：
     - 计费规则说明（IP 解析 + 联系人挖掘费用）
     - 异步执行提示
     - UTM 标识检测提示「检测到该访客带有 UTM 参数，系统将优先尝试关联静态数据中已有的企业信息」
     - 当前账户 MoliAI点数余额
   - 检查 MoliAI点数余额，若余额不足禁用确认按钮并提示充值
   - 检查访客是否已挖掘，若已挖掘禁用按钮并提示「该访客已挖掘」
   - 用户确认后，前端发送 `POST /api/v1/visitors/{visitor_log_id}/reveal`
   - 系统根据 UTM 参数中的企业标识，自动关联静态数据中已有的企业信息
   - 跳过 IP 解析服务调用，直接进入联系人挖掘阶段
   - 创建动态线索占位记录
3. 系统返回响应：返回挖掘任务状态，自动跳转至动态线索池页面

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| UTM 参数无效 | 回退到分支 B（无 UTM 标识流程） |
| MoliAI点数余额 ≤ 0 | 按钮禁用，弹窗提示充值 |
| 访客已挖掘 | 按钮禁用，Toast「该访客已挖掘」 |
| 网络超时 | 弹窗提示重试 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常（高分） | 深度挖掘中 | 用户点击深度挖掘按钮并确认 |
| 深度挖掘中 | 动态线索池 | 挖掘任务创建成功，自动跳转 |
| 深度挖掘中 | 正常（高分） | 挖掘接口返回失败 |

**输出**：直接进入联系人挖掘，MoliAI点数 仅消耗联系人挖掘部分。

### 场景五：深度挖掘（无 UTM 标识）

**触发条件**：访客无有效 UTM 参数，用户点击深度挖掘按钮。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 弹出深度挖掘确认弹窗，展示计费规则、异步执行提示、MoliAI点数余额
   - 检查 MoliAI点数余额和挖掘状态
   - 用户确认后，前端发送 `POST /api/v1/visitors/{visitor_log_id}/reveal`
   - 启动 IP 解析服务（IRS）进行企业识别
   - 通过 IP 反查获取企业域名/名称
   - 验证企业信息有效性
   - 进入联系人挖掘阶段
   - 创建动态线索占位记录
3. 系统返回响应：返回挖掘任务状态，自动跳转至动态线索池页面

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| IP 无法解析到企业 | 标记为 ISP 状态，进入动态线索池展示 ISP 卡片 |
| 企业信息验证未通过 | 标记为失败状态，提示用户 |
| MoliAI点数不足 | 同场景四 |
| 访客已挖掘 | 同场景四 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常（高分） | 深度挖掘中 | 用户点击深度挖掘按钮并确认 |
| 深度挖掘中 | 动态线索池 | 挖掘任务创建成功，自动跳转 |
| 深度挖掘中 | 正常（高分） | 挖掘接口返回失败 |

**输出**：异步执行，完成后自动跳转到动态线索池页面。

### 场景六：AI 行为解读异常处理

**触发条件**：AI 模型返回空内容、格式错误、内容幻觉或字段缺失。

**执行逻辑**：

1. 系统接收请求：前端检测 AI 解读字段异常状态
2. 系统执行处理：
   - 按异常类型显示对应降级文案：
     - AI 解读为空：显示「AI 分析结果为空」
     - AI 格式错误：显示原始行为描述
     - AI 内容幻觉：标记「AI 解读仅供参考」
     - AI 字段缺失：显示「AI 分析不完整」
   - 页面其余内容正常展示
   - 记录异常日志供后续优化
3. 系统返回响应：异常处理完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| AI 解读为空 | 显示默认解读模板 |
| AI 格式错误 | 显示原始行为描述 |
| AI 内容幻觉 | 标记仅供参考 |
| AI 字段缺失 | 显示分析中状态 |

**状态变更**：无

**输出**：访客行为报告页面，AI 解读区域显示降级文案。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 访客不存在 | code 404 | 返回错误 | 提示并返回访客雷达列表，Toast「该访客记录不存在或已被清除」 |
| 访客数据已过期 | code 404 + 特定错误码 | 返回错误 | 提示数据已归档，提供查看历史记录入口 |
| 网络超时 | 请求超时 30s | 重试 1 次 | 显示错误状态，Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志 | 显示错误状态，Toast「服务暂时不可用，请稍后再试」 |
| MoliAI点数不足 | code 1002 | 拒绝挖掘请求 | 弹窗提示充值，阻止提交，Toast「MoliAI点数余额不足（当前：{balance}），请充值后再试」 |
| 访客已挖掘 | code 409 | 返回错误 | 深度挖掘按钮禁用，Toast「该访客已挖掘」 |
| 挖掘请求超时 | HTTP 超时 | — | 弹窗提示重试，Toast「挖掘请求超时，请稍后重试」 |
| 数据过期 | 缓存过期 | — | 提示刷新获取最新数据，Toast「部分数据已过期，请刷新获取最新数据」 |
| AI 解读为空 | `ai_insights` 为空数组 | — | 显示「AI 分析结果为空」，Toast「暂无解读」 |
| AI 格式错误 | JSON 解析失败 | — | 显示原始行为描述，Toast「AI 解读生成异常」 |
| AI 内容幻觉 | 与行为事实矛盾 | — | 标记「AI 解读仅供参考」，Toast「AI 解读仅供参考」 |
| AI 字段缺失 | 关键洞察缺失 | — | 显示「AI 分析不完整」，Toast「AI 分析不完整」 |
| 页面跳转失败 | 导航操作失败 | — | 提示重试，Toast「页面跳转失败，请重试」 |
| 无行为记录 | `events` 为空 | — | 显示「暂无行为记录」空态 |
| 无历史会话 | `historical_sessions` 为空 | — | 显示「暂无历史访问记录」空态 |

## 性能安全要求

### 性能指标

| 指标类型 | 指标要求 | 说明 |
|---------|---------|------|
| 页面加载时间 | 首屏加载 < 1s | 包含访客基础信息、行为轨迹、AI 洞察 |
| 接口响应时间 | 详情查询 < 500ms | 访客详情数据接口 |
| 数据实时性 | 进入页面时强制刷新 | 确保展示最新数据 |
| 图片/资源加载 | 懒加载优化 | 历史会话等非首屏内容延迟加载 |

### 安全策略

| 安全维度 | 策略要求 | 说明 |
|---------|---------|------|
| 认证授权 | JWT Token 认证 | 所有接口需携带有效 Token |
| 数据脱敏 | IP 地址脱敏展示 | 前端不展示完整 IP，仅展示脱敏后格式 |
| 访问控制 | 企业数据隔离 | 只能访问本企业访客数据，通过 `bot_id` 校验权限 |
| 参数校验 | 严格校验路由参数 | 校验 `visitor_log_id` 格式和存在性 |
| 防重复提交 | 深度挖掘按钮防抖 | 点击后禁用按钮，防止重复提交 |
