## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 绑定域名 | `domain` | `string` | 用户输入或 OAuth 自动提取 | 用户手动输入或通过 OAuth 授权自动提取的域名 |
| 探针 ID | `probe_id` | `string` | 系统生成 | 唯一标识符，格式如 `LF-X7Y8Z9` |
| 绑定类型 | `bind_type` | `string` | 系统生成 | 取值：`manual`（手动）/ `oauth`（授权） |
| 绑定时间 | `bind_at` | `string` | 系统生成 | ISO 8601 格式时间戳 |
| 脚本 URL | `script_url` | `string` | 系统生成 | CDN 地址，含探针 ID，如 `https://cdn.leadfinder.io/probe/LF-X7Y8Z9.js` |
| 嵌入代码 | `embed_code` | `string` | 系统生成 | 完整 `<script>` 标签代码片段 |
| 部署域名 | `deployed_domain` | `string` | 接口返回 | 一键部署时独立站实际域名 |
| 域名匹配 | `domain_match` | `boolean` | 接口返回 | 部署域名与绑定域名是否一致 |
| 部署状态 | `status` | `string` | 接口返回 | 取值：`active`（运行中）/ `pending`（待验证）/ `failed`（失败） |
| 验证状态 | `verified` | `boolean` | 接口返回 | 探针代码是否被检测到 |
| 请求来源域名 | `source_domain` | `string` | 接口返回 | 探针请求时携带的来源域名 |
| 已授权站点列表 | `authorized_sites` | `array` | 接口返回 | OAuth 授权独立站列表，含站点 ID、名称、域名 |
| 站点 ID | `site_id` | `string` | 接口返回 | 独立站站点唯一标识 |
| 部署时间 | `deployed_at` | `string` | 接口返回 | ISO 8601 格式时间戳 |

**数据来源深度说明**：

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 探针 ID | `probe_id` | `string` | 系统生成 | 服务端生成唯一标识符，格式为 `LF-` + 6位随机字母数字组合。生成后持久化到 `probes` 表，与 `enterprise_id` 建立一对一关系 |
| 嵌入代码 | `embed_code` | `string` | 系统生成 | 服务端基于 `probe_id` 拼接生成标准 GTM 格式追踪代码模板，将 `probe_id` 注入代码中的 `i` 参数位置 |
| 脚本 URL | `script_url` | `string` | 系统生成 | 基于 `probe_id` 拼接 CDN 地址：`https://cdn.leadfinder.io/probe/{probe_id}.js` |
| 域名匹配 | `domain_match` | `boolean` | 系统生成 | 服务端校验逻辑：提取探针请求头中的 `Origin` 或 `Referer` 域名，与 `bound_domain` 进行字符串匹配（忽略协议和 www 前缀） |
| 部署状态 | `status` | `string` | 系统生成 | 状态机流转：`pending`（初始）→ `active`（验证通过）/ `failed`（验证失败或部署异常） |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 访客雷达.html | 用户点击侧边栏"访客雷达"，系统检查未部署探针时自动跳转 | `bot_id` |

**入口前置条件**：
1. 用户已登录且拥有企业管理员权限
2. 用户点击任意页面的侧边栏"访客雷达"入口
3. 系统检查当前账号未绑定域名或未部署探针

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 访客雷达.html | 部署成功后点击"查看访客雷达"，或系统自动跳转 | `bot_id` |

**出口后置逻辑**：
- 部署成功后，用户点击"查看访客雷达"进入访客雷达页面
- 若部署流程由访客雷达页面自动触发，部署成功后应自动返回访客雷达页面

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 未部署 | 初始状态，显示部署方式选择 | 页面加载且未检测到绑定域名 | 用户选择部署方式并完成绑定 |

## 业务处理逻辑

### 场景一：OAuth 授权独立站一键安装

**触发条件**：账号已存在已授权独立站（通过独立站授权流程完成）。

**执行逻辑**：

1. 系统接收请求：页面加载，检测已授权独立站
2. 系统执行处理：
   - 系统检测已授权独立站，自动提取域名并完成绑定，生成专属探针 ID
   - 页面显示"一键安装追踪探针"主按钮
   - 用户点击按钮，弹出部署进度弹窗，实时展示：连接独立站 API → 注入追踪代码 → 校验域名一致性 → 验证部署状态
   - 部署完成后，弹窗自动关闭，页面底部显示成功卡片"数据已回传，探针已生效"
3. 系统返回响应：返回部署结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 独立站连接失败 | 部署进度弹窗显示失败，提示"无法连接独立站 API，请检查授权是否有效" |
| 代码注入失败 | 显示失败提示，提示"代码注入失败，请手动复制代码进行部署" |
| 域名不一致 | 拒绝数据上报，提示"检测到部署域名与绑定域名不一致，请检查部署位置" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| OAuth 授权流程 | 部署中 | 用户点击一键安装按钮 |
| 部署中 | 部署成功 | 部署接口返回成功且验证通过 |
| 部署中 | 部署失败 | 部署接口返回失败或验证不通过 |

**输出**：部署成功，探针生效。

---

### 场景二：手动填写域名手动埋点

**触发条件**：账号无已授权独立站，需手动输入域名。

**执行逻辑**：

1. 系统接收请求：用户手动输入网站域名
2. 系统执行处理：
   - 用户手动输入网站域名，系统验证域名格式（正则校验）与可访问性（HTTP 请求）后完成绑定
   - 系统生成专属探针 ID，展示 JS 探针代码片段，右上角提供"复制代码"按钮
   - 页面以卡片形式展示三步操作指引：① 复制上方代码 → ② 粘贴到网站 `<head>` 标签 → ③ 等待系统验证生效
   - 用户点击"验证代码是否生效"按钮，弹出验证进度弹窗，检测代码是否正确嵌入并校验请求来源域名与绑定域名一致性
3. 系统返回响应：返回验证结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 域名格式无效 | 绑定按钮禁用，提示"请输入有效的网站域名，格式如 example.com" |
| 域名已被其他账号绑定 | 拒绝绑定请求，提示"该域名已被其他账号绑定，如需协助请联系客服" |
| 账号已有绑定域名 | 阻止新绑定，提示"当前账号已绑定域名 {domain}，如需更换请联系管理员" |
| 验证冷却 | 手动验证按钮 30 秒内重复点击，返回"验证操作过于频繁"提示 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 手动部署流程 | 绑定中 | 用户点击绑定按钮 |
| 绑定中 | 手动部署流程 | 绑定接口返回失败 |
| 手动部署流程 | 部署成功 | 验证接口返回成功 |
| 手动部署流程 | 部署失败 | 验证接口返回失败 |

**输出**：域名绑定成功 + 探针代码可复制部署。

### 场景三：域名已被其他账号绑定

**触发条件**：用户输入的域名已被其他企业账号绑定。

**执行逻辑**：

1. 系统接收请求：用户输入域名
2. 系统执行处理：
   - 后端拒绝绑定请求
   - 前端显示"该域名已被其他账号绑定"提示
3. 系统返回响应：返回 409 错误

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 域名已被绑定 | 拒绝绑定请求 |

**状态变更**：无

**输出**：绑定失败，提示联系客服。

---

### 场景四：账号已有绑定域名

**触发条件**：当前企业账号已存在绑定域名。

**执行逻辑**：

1. 系统接收请求：用户尝试绑定新域名
2. 系统执行处理：
   - 阻止新域名绑定请求
   - 前端显示"当前账号已绑定域名 xxx.com，如需更换请联系管理员"提示
3. 系统返回响应：返回 409 错误

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 账号已有绑定域名 | 阻止新绑定 |

**状态变更**：无

**输出**：阻止新绑定，提示联系管理员。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 绑定域名 | 校验 | 正则校验：`^[a-zA-Z0-9][-a-zA-Z0-9]{0,62}(\.[a-zA-Z0-9][-a-zA-Z0-9]{0,62})+$`，支持多级域名，忽略协议前缀 | `probes.bound_domain` |
| 探针 ID | 生成 | 随机生成 6 位字母数字组合，前缀 `LF-`，确保唯一性（查询 `probes` 表校验） | `probes.probe_id` |
| 嵌入代码 | 生成 | 基于模板字符串替换，将 `probe_id` 注入 GTM 格式代码模板中的变量位置 | 服务端内存生成，不持久化 |
| 域名匹配 | 计算 | 提取请求头 `Origin` 或 `Referer` 中的域名，去除协议和 `www` 前缀后与 `bound_domain` 进行字符串比对 | 实时计算，不存储 |
| 部署状态 | 状态机 | `pending` → `active`（验证通过）/ `failed`（验证失败或异常） | `probes.status` |

### 计算逻辑

**域名格式校验规则**：

```
域名正则：^[a-zA-Z0-9][-a-zA-Z0-9]{0,62}(\.[a-zA-Z0-9][-a-zA-Z0-9]{0,62})+$

校验逻辑：
1. 去除用户输入中的协议前缀（http:// 或 https://）
2. 去除路径部分（/path）和查询参数（?query）
3. 去除 www 前缀（如有）
4. 应用正则表达式校验
5. 校验通过后，存储标准化域名（小写、无协议、无 www）
```

**触发时机**：用户输入域名时实时校验（前端），提交绑定时二次校验（后端）。

**存储方式**：标准化后的域名存储于 `probes.bound_domain` 字段。

## 数据流

### 域名绑定

```
POST /api/v1/domain/bind
```

**请求头**：`Authorization: Bearer {enterprise_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| domain | `string` | 是 | 网站域名 |
| bind_type | `string` | 是 | 绑定类型：`manual` / `oauth` |
| site_id | `string` | 否 | OAuth 方式时必填，独立站站点 ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "domain": "example.com",
    "probe_id": "LF-X7Y8Z9",
    "bind_type": "manual",
    "bind_at": "2024-02-26T10:00:00Z",
    "status": "pending"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 绑定成功 |
| 400 | 域名格式无效 |
| 401 | 未登录 |
| 409 | 域名已被其他账号绑定 / 账号已有绑定域名 |
| 500 | 服务异常 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 域名格式无效 | 正则校验失败 | 拒绝绑定请求，返回 400 | 绑定按钮禁用，Toast「请输入有效的网站域名，格式如 example.com」 |
| 域名已被绑定 | HTTP 409 | 拒绝绑定请求，返回 409 | Toast「该域名已被其他账号绑定，如需协助请联系客服」 |
| 账号已有绑定 | 检测到已绑定域名 | 阻止新绑定，返回 409 | Toast「当前账号已绑定域名 {domain}，如需更换请联系管理员」 |
| 独立站连接失败 | API 调用失败 | 记录错误日志，返回 500 | 部署进度弹窗显示失败，Toast「无法连接独立站 API，请检查授权是否有效」 |
| 代码注入失败 | 注入返回错误 | 记录错误日志，返回 500 | 显示失败提示，Toast「代码注入失败，请手动复制代码进行部署」 |
| 域名不一致 | domain_match false | 拒绝数据上报，返回 409 | Toast「检测到部署域名与绑定域名不一致，请检查部署位置」 |
| 未检测到代码 | 验证返回 verified=false | — | 失败提示，Toast「未检测到探针代码，请确认代码已正确嵌入 head 标签」 |
| 域名校验失败 | domain_match false | — | 失败提示，Toast「域名校验失败，探针部署域名与绑定域名不一致」 |
| 验证冷却中 | 30 秒内重复点击 | 阻止验证，返回 429 | Toast「验证操作过于频繁，请 {seconds} 秒后再试」 |
| 复制失败 | 浏览器不支持 Clipboard API | — | 提示手动复制，Toast「当前浏览器不支持自动复制，请手动选择代码」 |
| 网络错误 | 请求失败 | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | HTTP 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | Token 缺失或过期 | 返回 401 | 跳转登录页面 |
