## 概述

搜客策略预览页面是搜客流程的策略确认节点。AI 模型根据用户输入的自然语言搜客需求，结合 ICP 画像自动生成完整的接口调用策略（数据源选择、筛选条件、执行顺序、费用预估）。管理员预览策略后确认执行，策略传入下游搜客执行中页面进行瀑布流调用。

## 功能逻辑

### 场景一：AI 策略生成

**触发条件**：页面加载时，后端基于搜客需求输入生成 AI 策略。

**执行逻辑**：
1. 后端调用 AI 策略生成接口（Claude 3.5 Sonnet / GPT-4o）
2. AI 分析用户需求 + ICP 画像，输出结构化策略 JSON
3. 前端解析策略，展示三大模块：策略解读、瀑布流执行计划、费用预估明细
4. 页面组件依次淡入上滑（fadeInUp），带延迟效果
5. 顶部显示"AI 策略规划完成"绿色徽章

**输出**：策略解读 + 瀑布流 + 费用预估三大模块。

---

### 场景二：策略解读展示

**触发条件**：AI 策略生成完成。

**执行逻辑**：
1. 展示 AI 生成的策略摘要文字
2. 展示计划调用的数据源列表（出海派 / 10times / Google Maps）
3. 展示每个数据源的筛选条件说明
4. 若某数据源离线，标记该步骤异常状态

**输出**：策略解读模块。

---

### 场景三：瀑布流执行计划展示

**触发条件**：策略解读展示完成。

**执行逻辑**：
1. 展示执行计划瀑布流图（各 API 调用步骤串联）
2. 每个步骤显示：步骤序号、数据源名称、预计返回数量、预估 Credits 消耗
3. 展示执行顺序说明（串行 / 并行）
4. 最多支持单次执行 4 个 API 步骤

**输出**：瀑布流执行计划模块。

---

### 场景四：费用预估明细

**触发条件**：瀑布流展示完成。

**执行逻辑**：
1. 展示预估费用明细表：
   - 各 API 基础调用费用
   - 动态溢价（30% buffer）
   - 总预估费用（Credits）
2. 展示 Credits 余额和执行后预估余额
3. 余额规则：可为负，≤0 时禁止消耗操作；执行中不扣费，完成后按实际消耗结算

**输出**：费用预估模块。

---

### 场景五：确认执行

**触发条件**：用户确认策略无误，点击"确认执行"按钮。

**执行逻辑**：
1. 前端显示"冻结 Credits..."加载状态
2. 按钮禁用，防止重复提交
3. 调用策略执行接口，传入策略 JSON
4. 跳转至搜客执行中页面

**输出**：跳转至搜客执行中页面。

## 数据流

### 请求一：生成搜客策略

```
POST /api/v1/search/strategy/generate
```

**请求体**：

```json
{
  "bot_id": "BOT-001",
  "user_query": "欧洲户外屏代理商，有海关记录",
  "target_count": 100,
  "icp_context": { ... }
}
```

**响应体**：

```json
{
  "code": 200,
  "data": {
    "strategy_id": "STRAT-001",
    "summary": "基于用户需求，优先调用出海派获取海关数据匹配企业，再通过10times补充展会信息",
    "data_sources": [
      { "name": "出海派", "priority": 1, "estimated_results": 150, "estimated_credits": 30 },
      { "name": "10times", "priority": 2, "estimated_results": 80, "estimated_credits": 20 }
    ],
    "waterfall": [
      { "step": 1, "source": "出海派", "filters": { "hs_code_match": true, "region": "Europe" } },
      { "step": 2, "source": "10times", "filters": { "event_type": "trade_show" } }
    ],
    "cost_estimate": {
      "base": 50,
      "premium_rate": 0.3,
      "premium": 15,
      "total": 65
    },
    "credits_balance_after": 34935
  }
}
```

## AI Prompt 引用

### AI Prompt 章节

**引用位置**：场景一（AI 策略生成）之后、边界规则之前

**参考模板**：`AI业务分析中-说明.md`

本页面 AI 调用由后端处理，Prompt 模板如下：

**Prompt 模板**：

```
【系统指令】
你是一位专业的 B2B 外贸获客策略分析师。用户需要根据其 ICP 画像和自然语言需求，生成最优的搜客策略。
请严格基于用户提供的 ICP 画像和需求内容生成策略，不得捏造任何非输入数据之外的 ICP 信息。

【用户输入】
- ICP 画像：{icp_profile}
- 自然语言需求：{user_query}
- 目标企业数量：{target_count}

【输出要求】
请以 JSON 格式输出结构化搜客策略，必须包含以下字段：
- strategy_id: 策略唯一标识
- summary: 策略摘要说明（50字以内）
- data_sources: 数据源列表，每项包含 name, priority, estimated_results, estimated_credits
- waterfall: 瀑布流步骤列表，每项包含 step, source, filters
- cost_estimate: 费用预估对象，包含 base, premium_rate, premium, total

【零捏造原则】
- 策略中的 estimated_results 和 estimated_credits 仅供参考估算，实际数量以 API 返回为准
- data_sources 仅使用已知数据源：出海派、10times、Google Maps，不得虚构数据源名称
- 策略逻辑必须与用户需求一致，如存在矛盾须在 summary 中注明
```

**模型选择**：Claude 3.5 Sonnet / GPT-4o

**JSON Schema 要求**：

```json
{
  "type": "object",
  "required": ["strategy_id", "summary", "data_sources", "waterfall", "cost_estimate"],
  "properties": {
    "strategy_id": { "type": "string" },
    "summary": { "type": "string", "maxLength": 100 },
    "data_sources": {
      "type": "array",
      "maxItems": 4,
      "items": {
        "type": "object",
        "required": ["name", "priority", "estimated_results", "estimated_credits"],
        "properties": {
          "name": { "type": "string", "enum": ["出海派", "10times", "Google Maps", "Apollo.io"] },
          "priority": { "type": "integer", "minimum": 1 },
          "estimated_results": { "type": "integer", "minimum": 0 },
          "estimated_credits": { "type": "number", "minimum": 0 }
        }
      }
    },
    "waterfall": {
      "type": "array",
      "maxItems": 4,
      "items": {
        "type": "object",
        "required": ["step", "source", "filters"],
        "properties": {
          "step": { "type": "integer", "minimum": 1 },
          "source": { "type": "string" },
          "filters": { "type": "object" }
        }
      }
    },
    "cost_estimate": {
      "type": "object",
      "required": ["base", "premium_rate", "premium", "total"],
      "properties": {
        "base": { "type": "number", "minimum": 0 },
        "premium_rate": { "type": "number", "minimum": 0, "maximum": 1 },
        "premium": { "type": "number", "minimum": 0 },
        "total": { "type": "number", "minimum": 0 }
      }
    }
  }
}
```

**零捏造原则**：
- 策略必须与用户 ICP 画像和需求一致，不得捏造不存在的业务特征
- estimated_results / estimated_credits 仅作估算参考，实际以 API 返回为准
- 不得虚构数据源名称或 API 能力

**边界处理**：

| 异常类型 | 检测方式 | 系统行为 |
|---------|---------|---------|
| 输出为空 | AI 返回空策略 | 提示"AI 策略生成结果为空，请重试"，显示重试按钮 |
| JSON 格式错误 | JSON 解析失败 | 自动重试 1 次，提示"AI 返回数据格式异常，正在自动重试..." |
| 一致性校验失败 | 策略与需求矛盾 | 显示警告"AI 策略与需求不符，建议重新生成"，显示重新生成按钮 |
| 必要字段缺失 | schema 校验失败 | 自动重试 1 次 |

---

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| Credits 余额 | 余额 > 0 且 ≥ 预估消耗 | 方可执行策略 |
| 余额为负 | 余额 ≤ 0 | 禁止消耗操作；执行中不扣费，完成后按实际消耗结算 |
| 目标企业数量上限 | 最多 200 家 | 超出截断至 200 |
| API 步骤限制 | 单次执行最多 4 个 API | 超出时分批执行 |
| 数据源离线 | 依赖第三方 API 离线 | 对应步骤标记异常，不阻断其他步骤 |
| 策略生成模型 | AI 模型调用 | 超时（60s）降级返回预置策略模板 |
| 预估费用仅供参考 | - | 实际扣费按成功调用结算 |
| 策略一致性 | 策略与需求矛盾 | 触发一致性校验警告 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 网络超时 | 请求超时 30s | Toast 提示 + 重试按钮 | 「网络连接异常，请检查网络后重试」 |
| AI 策略为空 | AI 输出校验为空 | 重试按钮 | 「AI 策略生成结果为空，请重试」 |
| AI 格式错误 | JSON 解析失败 | 自动重试 1 次 | 「AI 返回数据格式异常，正在自动重试...」 |
| AI 一致性失败 | 策略与需求矛盾 | 警告提示 + 重新生成按钮 | 「AI 策略与需求不符，建议重新生成」 |
| AI 超时 | 调用超时 60s | 降级预置策略模板 | 「AI 分析超时，已使用默认策略」 |
| 数据源离线 | API 可用性检测 | 策略步骤标记异常 | 「{source} 暂不可用，策略已调整」 |
| Credits 不足 | 余额 < 预估消耗 | 执行按钮禁用 | 「Credits 余额不足（当前：{balance}），无法执行」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
