## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 任务 ID | `task_id` | `string` | 路由参数传入 | 搜客任务唯一标识 |
| 策略 ID | `strategy_id` | `string` | 系统生成 | AI 生成的策略唯一标识 |
| 策略摘要 | `summary` | `string` | AI 生成 | 策略摘要说明（50字以内） |
| 数据源名称 | `data_sources[].name` | `string` | AI 生成 | 出海派/10times/Google Maps |
| 数据源优先级 | `data_sources[].priority` | `number` | AI 生成 | 执行优先级排序 |
| 预估结果数 | `data_sources[].estimated_results` | `number` | AI 估算 | 预计返回企业数量 |
| 预估 Credits | `data_sources[].estimated_credits` | `number` | AI 估算 | 预计消耗 Credits |
| 瀑布流步骤 | `waterfall[].step` | `number` | AI 编排 | 步骤序号 |
| 执行筛选条件 | `waterfall[].filters` | `object` | AI 生成 | 各步骤筛选条件 |
| 基础费用 | `cost_estimate.base` | `number` | 后端计算 | API 基础调用费用 |
| 动态溢价 | `cost_estimate.premium` | `number` | 后端计算 | base × 30% |
| 总预估费用 | `cost_estimate.total` | `number` | 后端计算 | base + premium |
| 执行后余额 | `credits_balance_after` | `number` | 后端计算 | 执行后预估余额 |
| Credits 余额 | `credits_balance` | `number` | 接口返回 | 当前租户 Credits 余额 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 搜客需求输入.html | 用户点击"确认搜客"按钮 | `bot_id`, `task_id` |

**入口前置条件**：搜客任务已创建且状态为 pending。

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 搜客执行中.html | 用户确认策略后点击执行 | `bot_id`, `task_id` |
| 搜客需求输入.html | 用户点击"返回修改"按钮 | `bot_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | AI 策略生成中 | 页面加载 | 策略生成完成或失败 |
| 正常 | 策略生成成功，展示策略内容 | 策略接口返回成功 | 用户点击执行或返回 |
| 策略异常 | AI 策略生成失败或超时 | 策略接口返回失败或超时 | 用户点击重试或返回 |
| 执行中 | 用户已点击确认执行 | 用户点击"确认执行" | 执行接口返回成功或失败 |

**状态流转图**：
```
加载中 --(生成成功)--> 正常 --(点击执行)--> 执行中 --(成功)--> 搜客执行中.html
加载中 --(生成失败)--> 策略异常 --(点击重试)--> 加载中
正常 --(点击返回)--> 搜客需求输入.html
```

## 数据流

### 生成搜客策略

```
POST /api/v1/search/strategy/generate
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | BOT 实例 ID |
| task_id | `string` | 是 | 搜客任务 ID |
| user_query | `string` | 否 | 用户输入的自然语言需求 |
| target_count | `number` | 否 | 目标企业数量 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "strategy_id": "STRAT-001",
    "summary": "基于用户需求，优先调用出海派获取海关数据匹配企业，再通过10times补充展会信息",
    "data_sources": [
      { "name": "出海派", "priority": 1, "estimated_results": 150, "estimated_credits": 30 },
      { "name": "10times", "priority": 2, "estimated_results": 80, "estimated_credits": 20 }
    ],
    "waterfall": [
      { "step": 1, "source": "出海派", "filters": { "hs_code_match": true, "region": "Europe" } },
      { "step": 2, "source": "10times", "filters": { "event_type": "trade_show" } }
    ],
    "cost_estimate": {
      "base": 50,
      "premium_rate": 0.3,
      "premium": 15,
      "total": 65
    },
    "credits_balance_after": 34935
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 策略生成成功 |
| 400 | 参数错误 |
| 404 | 任务不存在 |
| 504 | AI 策略生成超时 |
| 500 | 服务异常 |

### 确认执行搜客

```
POST /api/v1/search/execute
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | `string` | 是 | BOT 实例 ID |
| task_id | `string` | 是 | 搜客任务 ID |
| strategy_id | `string` | 是 | 策略唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "TASK-202401-001",
    "status": "running",
    "redirect_to": "/搜客执行中.html?bot_id=BOT-001&task_id=TASK-202401-001"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 执行成功，任务开始运行 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 404 | 任务或策略不存在 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：AI 策略生成

**触发条件**：页面加载时，后端基于搜客需求输入生成 AI 策略。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id` 和 `task_id`
2. 系统执行处理：
   - 后端调用 AI 策略生成接口
   - AI 分析用户需求 + ICP 画像，输出结构化策略 JSON
   - 前端解析策略，展示三大模块：策略解读、瀑布流执行计划、费用预估明细
   - 页面组件依次淡入上滑（fadeInUp），带延迟效果
   - 顶部显示"AI 策略规划完成"绿色徽章
3. 系统返回响应：返回策略详情数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| AI 策略生成超时（60s） | 降级返回预置策略模板 |
| AI 输出为空 | 提示"AI 策略生成结果为空，请重试"，显示重试按钮 |
| JSON 格式错误 | 自动重试 1 次，提示"AI 返回数据格式异常，正在自动重试..." |
| 策略与需求矛盾 | 显示警告"AI 策略与需求不符，建议重新生成"，显示重新生成按钮 |
| 必要字段缺失 | 自动重试 1 次 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 策略生成成功 |
| 加载中 | 策略异常 | 策略生成失败或超时 |
| 策略异常 | 加载中 | 用户点击重试 |

**输出**：策略解读 + 瀑布流 + 费用预估三大模块。

---

### 场景二：策略解读展示

**触发条件**：AI 策略生成完成，进入正常状态。

**执行逻辑**：

1. 系统接收请求：策略数据加载完成
2. 系统执行处理：
   - 展示 AI 生成的策略摘要文字
   - 展示计划调用的数据源列表（出海派 / 10times / Google Maps）
   - 展示每个数据源的筛选条件说明
   - 若某数据源离线，标记该步骤异常状态
3. 系统返回响应：策略解读模块渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 数据源离线 | 对应步骤标记异常状态，不阻断其他步骤 |
| 策略摘要为空 | 显示默认摘要"基于 ICP 画像的默认搜客策略" |

**状态变更**：无

**输出**：策略解读模块。

---

### 场景三：瀑布流执行计划展示

**触发条件**：策略解读展示完成。

**执行逻辑**：

1. 系统接收请求：策略数据已加载
2. 系统执行处理：
   - 展示执行计划瀑布流图（各 API 调用步骤串联）
   - 每个步骤显示：步骤序号、数据源名称、预计返回数量、预估 Credits 消耗
   - 展示执行顺序说明（串行 / 并行）
   - 最多支持单次执行 4 个 API 步骤
3. 系统返回响应：瀑布流执行计划模块渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 步骤超过 4 个 | 超出部分标记为分批执行 |
| 某步骤数据源离线 | 该步骤标记为异常，其他步骤正常展示 |

**状态变更**：无

**输出**：瀑布流执行计划模块。

---

### 场景四：费用预估明细展示

**触发条件**：瀑布流展示完成。

**执行逻辑**：

1. 系统接收请求：策略数据已加载
2. 系统执行处理：
   - 展示预估费用明细表：
     - 各 API 基础调用费用
     - 动态溢价（30% buffer）
     - 总预估费用（Credits）
   - 展示 Credits 余额和执行后预估余额
   - 余额规则：可为负，≤0 时禁止消耗操作；执行中不扣费，完成后按实际消耗结算
3. 系统返回响应：费用预估模块渲染完成

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| Credits 余额 < 预估消耗 | "确认执行"按钮禁用，提示充值 |
| 余额 ≤ 0 | 禁止消耗操作 |

**状态变更**：无

**输出**：费用预估模块。

---

### 场景五：确认执行

**触发条件**：用户确认策略无误，点击"确认执行"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击"确认执行"按钮
2. 系统执行处理：
   - 前端显示"冻结 Credits..."加载状态
   - 按钮禁用，防止重复提交
   - 调用 `POST /api/v1/search/execute`，传入策略 JSON
   - 接口返回成功后跳转至搜客执行中页面
3. 系统返回响应：返回执行结果和跳转地址

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| Credits 余额不足 | 按钮禁用，提示"Credits 余额不足，无法执行" |
| 策略已过期 | 提示"策略已过期，请重新生成" |
| 重复点击 | 按钮 loading 状态防止重复提交 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 执行中 | 用户点击"确认执行" |
| 执行中 | 正常 | 执行接口返回失败 |

**输出**：跳转至搜客执行中页面。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 30s | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| AI 策略为空 | AI 输出校验为空 | 返回空数据 | 重试按钮，Toast「AI 策略生成结果为空，请重试」 |
| AI 格式错误 | JSON 解析失败 | 自动重试 1 次 | Toast「AI 返回数据格式异常，正在自动重试...」 |
| AI 一致性失败 | 策略与需求矛盾 | 返回警告标记 | 警告提示 + 重新生成按钮，Toast「AI 策略与需求不符，建议重新生成」 |
| AI 超时 | 调用超时 60s | 降级预置策略模板 | Toast「AI 分析超时，已使用默认策略」 |
| 数据源离线 | API 可用性检测 | 标记异常步骤 | 策略步骤标记异常，Toast「{source} 暂不可用，策略已调整」 |
| Credits 不足 | 余额 < 预估消耗 | 拒绝执行请求 | 执行按钮禁用，Toast「Credits 余额不足（当前：{balance}），无法执行」 |
| 服务不可用 | code 500 | 记录错误日志 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 任务不存在 | code 404 | 返回错误 | Toast「任务不存在或已过期」 |

## AI 提示词

### AI 模型选择

**优先使用国内模型**：

| 模型厂商 | 推荐模型 | 适用场景 |
|---------|---------|---------|
| 通义千问 (Qwen) | `qwen-max` / `qwen-plus` | 通用对话、策略生成 |
| Kimi (Moonshot) | `kimi-k2.5` | 长文本处理、复杂推理 |

**备选国际模型**：

| 模型厂商 | 推荐模型 | 适用场景 |
|---------|---------|---------|
| OpenAI | `gpt-4o` / `gpt-4o-mini` | 复杂推理、代码生成 |
| Anthropic | `claude-3.5-sonnet` | 长文本分析、安全性要求高 |

### 搜客策略生成 Prompt

**模型**：`qwen-max` / `kimi-k2.5`（优先国内模型）/ `gpt-4o`（备选）

**强制设置**：`response_format: { "type": "json_object" }`

**系统提示词**：

```
你是一位专业的 B2B 外贸获客策略分析师。用户需要根据其 ICP 画像和自然语言需求，生成最优的搜客策略。
请严格基于用户提供的 ICP 画像和需求内容生成策略，不得捏造任何非输入数据之外的 ICP 信息。

## 输出要求
请以 JSON 格式输出结构化搜客策略，必须包含以下字段：
- strategy_id: 策略唯一标识
- summary: 策略摘要说明（50字以内）
- data_sources: 数据源列表，每项包含 name, priority, estimated_results, estimated_credits
- waterfall: 瀑布流步骤列表，每项包含 step, source, filters
- cost_estimate: 费用预估对象，包含 base, premium_rate, premium, total

## 零捏造原则
- 策略中的 estimated_results 和 estimated_credits 仅供参考估算，实际数量以 API 返回为准
- data_sources 仅使用已知数据源：出海派、10times、Google Maps，不得虚构数据源名称
- 策略逻辑必须与用户需求一致，如存在矛盾须在 summary 中注明
```

**用户提示词模板**：

```
请根据以下信息生成搜客策略：

[ICP 画像]
{{icp_profile}}
[/ICP 画像]

[用户需求]
{{user_query}}
[/用户需求]

[目标企业数量]
{{target_count}}
[/目标企业数量]

请严格按照系统提示词中的 JSON Schema 返回结果。
```
