## 概述

静态线索池是管理静态数据获客来源企业线索的核心工作台，承载联系人挖掘任务的追踪与已完成线索的统一管理。用户在搜客结果列表点击联系人挖掘后，系统在此页面创建占位记录，后台异步执行联系人挖掘（基于 Apollo.io），完成后展示完整联系人信息。

## 功能逻辑

### 场景一：查看线索列表

**触发条件**：用户进入静态线索池页面。

**执行逻辑**：
1. 前端请求线索列表（默认按入库时间倒序）
2. 展示两类卡片：
   - 挖掘中状态：卡片边框高亮（蓝色）+ "挖掘中"标签（带旋转动画）+ 进度条 + 剩余时间预估
   - 已完成状态：展示联系人头像和姓名 + Credits 消耗
3. 每张卡片核心信息：企业名称、企业简介、联系人数量、Credits 消耗、入库时间

**输出**：静态线索卡片列表。

---

### 场景二：筛选与排序

**触发条件**：用户切换筛选条件。

**执行逻辑**：
1. 筛选维度：挖掘状态（全部/挖掘中/已完成）、入库时间范围
2. 排序方式：入库时间倒序（默认）/ 评分倒序
3. 筛选后列表实时刷新

**输出**：按条件过滤后的线索列表。

---

### 场景三：查看线索详情

**触发条件**：用户点击线索卡片。

**执行逻辑**：
1. 无论挖掘中还是已完成，点击卡片均跳转至线索详情.html
2. 挖掘中时展示当前进度和已有数据
3. 已完成时展示完整企业信息和联系人矩阵

**输出**：跳转至线索详情页面。

---

### 场景四：导出线索列表

**触发条件**：用户点击"导出线索列表"按钮。

**执行逻辑**：
1. 前端请求导出接口
2. 后端生成 Excel 文件，返回下载链接

**输出**：线索列表 Excel 文件下载。

---

### 场景五：联系人挖掘流程（P22-P27）

**触发条件**：在搜客结果列表点击深度挖掘。

**执行逻辑**：
1. 基于企业域名或名称查询 Apollo.io 联系人
   - 有域名：使用 organization_domains 精准锁定
   - 无域名：使用 q_organization_name 模糊匹配
2. 数据过滤：仅保留有 email 的记录
3. 邮箱揭示（reveal_personal_emails）：每个有效邮箱 1 Credit
4. 电话揭示（reveal_phone_number）：每个有效电话 1 Credit
5. 基础联系人数据不消耗 Credits
6. 无联系人结果：0 Credits

**输出**：联系人数据存储，线索卡片更新为已完成。

## 数据流

### 请求一：创建联系人挖掘占位记录

```
POST /api/v1/leads/static
```

**请求体**：

```json
{
  "enterprise_id": "ENT-001",
  "company_name": "Alpine Gear International",
  "website": "https://alpine-gear.com",
  "match_score": 88,
  "bot_id": "BOT-001"
}
```

### 请求二：获取静态线索列表

```
GET /api/v1/leads/static
```

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | string | 是 | BOT ID |
| status | string | 否 | all / digging / completed |
| sort | string | 否 | created_desc / score_desc |
| page | number | 否 | 页码，默认 1 |
| limit | number | 否 | 每页数量，默认 20 |

**响应体**：

      {

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 页面关闭继续执行 | 用户关闭页面 | 挖掘任务在后台异步执行，不中断 |
| 接口超时 | 单次 API 调用超时 | 1 分钟超时，跳过本次调用继续执行 |
| 任务不阻塞 | 单个接口失败 | 确保任务最终完成 |
| 并发挖掘限制 | 单租户同时挖掘数 | 最多 3 个，超出需等待完成 |
| 线索去重 | 同一企业域名 | 不可重复入库，已有线索显示"已存在" |
| Credits 余额 | 余额可为负，≤0 | 禁止联系人挖掘操作，挖掘中不扣费，完成后结算 |
| 联系人挖掘上限 | 单企业最多挖掘 | 5 次 |
| 联系人数据有效期 | 有效期 | 90 天 |

## API 清单

| 接口 | 方法 | 说明 | 调用时机 |
|------|------|------|---------|
| /api/v1/leads/static | POST | 创建联系人挖掘占位记录 | 深度挖掘触发时 |
| /api/v1/leads/static | GET | 获取静态线索列表 | 页面加载、筛选变化 |
| /api/v1/leads/{lead_id}/digging-status | GET | 查询挖掘进度 | 轮询挖掘中任务 |
| /api/v1/leads/{lead_id}/finalize | POST | 确认线索入库 | 挖掘完成后调用 |
| /api/v1/leads/static/export | GET | 导出线索列表 Excel | 点击导出按钮 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 线索已存在 | 同一域名重复入库 | 错误提示 | 「该企业已在线索池中，不可重复入库」 |
| 线索不存在 | code 404 | 错误提示 | 「该线索不存在」 |
| 挖掘失败 | Apollo.io 返回错误 | 卡片显示失败状态 | 「联系人挖掘失败，请重试」 |
| Credits 余额不足 | 余额 ≤ 0 | 按钮禁用 | 「Credits 余额不足（当前：{balance}），无法执行挖掘」 |
| 挖掘已达上限 | 已挖掘 5 次 | 按钮禁用 | 「该企业已达到最大挖掘次数（5 次）」 |
| 并发挖掘超限 | 同时进行 3 个任务 | 按钮禁用 | 「同时进行中的任务已达上限（3 个），请等待完成后重试」 |
| 导出失败 | 后端异常 | 失败提示 | 「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | 失败提示 | 「当前筛选条件下无数据可导出」 |
| AI 评分为空 | ai_score 为空 | 默认评分 0 | 「AI 评分结果为空，请重试」 |
| AI 格式错误 | JSON 解析失败 | 自动重试 | 「AI 返回数据格式异常，正在自动重试...」 |
| AI 评分异常 | 评分与数据矛盾 | 标记仅供参考 | 「AI 评分存在异常，建议重新分析」 |
| 评分超出范围 | 评分 <0 或 >100 | 默认值 0 | 「AI 评分异常，已使用默认值」 |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |

## 字段映射

| 前端展示字段 | 后端字段名 | 数据来源说明 |
|-------------|-----------|------------|
| 线索 ID | lead_id | 系统生成 UUID |
| 企业 ID | enterprise_id | 来源任务中的企业 ID |
| 企业名称 | company_name | Apollo.io 返回或搜客结果 |
| 企业官网 | website | Apollo.io 返回或搜客结果 |
| 线索状态 | status | digging / completed / failed |
| 挖掘进度 | progress | 0-100 整数 |
| 预计剩余时间 | estimated_time | 后端估算 |
| 联系人数量 | contacts_count | Apollo.io 返回总数 |
| 联系人预览 | contacts_preview | 前 1-3 个联系人简要信息 |
| Credits 消耗 | credits_consumed | Apollo.io 实际消耗合计 |
| 入库时间 | created_at | ISO 8601 时间戳 |
| 匹配度评分 | match_score | 来自搜客任务，0-100 |
