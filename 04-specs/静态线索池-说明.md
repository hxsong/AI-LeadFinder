## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识 |
| 线索 ID | `lead_id` | `string` | 系统生成 | 线索唯一标识 |
| 企业 ID | `enterprise_id` | `string` | 接口返回 | 来源任务中的企业 ID |
| 企业名称 | `company_name` | `string` | Apollo.io 或三方 API 返回 | — |
| 企业官网 | `website` | `string` | Apollo.io 或搜客结果 | — |
| 线索状态 | `status` | `string` | 接口返回 | digging / completed / failed |
| 挖掘进度 | `progress` | `number` | 接口返回 | 0-100 整数 |
| 预计剩余时间 | `estimated_time` | `string` | 后端估算 | 如"约 2 分钟" |
| 联系人数量 | `contacts_count` | `number` | 接口返回 | Apollo.io 返回总数 |
| 联系人预览 | `contacts_preview` | `array` | 接口返回 | 前 1-3 个联系人简要信息 |
| Credits 消耗 | `credits_consumed` | `number` | 接口返回 | Apollo.io 实际消耗合计 |
| 入库时间 | `created_at` | `string` | 接口返回 | ISO 8601 时间戳 |
| 匹配度评分 | `match_score` | `number` | 接口返回 | 来自搜客任务，0-100 |

## 数据流

### 创建联系人挖掘占位记录

```
POST /api/v1/leads/static
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| enterprise_id | `string` | 是 | 企业 ID |
| company_name | `string` | 是 | 企业名称 |
| website | `string` | 否 | 企业官网 |
| match_score | `number` | 否 | 匹配度评分 |
| bot_id | `string` | 是 | BOT 实例 ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "lead_id": "LEAD-001",
    "status": "digging"
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 500 | 服务异常 |

## 处理逻辑

### 场景一：查看线索列表

**触发条件**：用户进入静态线索池页面。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id`
2. 系统执行处理：
   - 前端请求线索列表（默认按入库时间倒序）
   - 展示两类卡片：
     - 挖掘中状态：卡片边框高亮（蓝色）+ "挖掘中"标签（带旋转动画）+ 进度条 + 剩余时间预估
     - 已完成状态：展示联系人头像和姓名 + Credits 消耗
   - 每张卡片核心信息：企业名称、企业简介、联系人数量、Credits 消耗、入库时间
3. 系统返回响应：返回线索列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无线索数据 | 显示空态引导文案 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功 |
| 加载中 | 空态 | 数据为空 |

**输出**：静态线索卡片列表。

---

### 场景二：筛选与排序

**触发条件**：用户切换筛选条件。

**执行逻辑**：

1. 系统接收请求：用户选择筛选条件
2. 系统执行处理：
   - 筛选维度：挖掘状态（全部/挖掘中/已完成）、入库时间范围
   - 排序方式：入库时间倒序（默认）/ 评分倒序
   - 筛选后列表实时刷新
3. 系统返回响应：返回过滤后的线索列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按条件过滤后的线索列表。

---

### 场景三：查看线索详情

**触发条件**：用户点击线索卡片。

**执行逻辑**：

1. 系统接收请求：用户点击线索卡片
2. 系统执行处理：
   - 无论挖掘中还是已完成，点击卡片均跳转至线索详情
   - 挖掘中时展示当前进度和已有数据
   - 已完成时展示完整企业信息和联系人矩阵
3. 系统返回响应：页面跳转

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 线索不存在 | 提示"该线索不存在" |

**状态变更**：无

**输出**：跳转至线索详情页面。

---

### 场景四：导出线索列表

**触发条件**：用户点击"导出线索列表"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击导出按钮
2. 系统执行处理：
   - 前端请求导出接口 `GET /api/v1/leads/static/export`
   - 后端生成 Excel 文件，返回下载链接
3. 系统返回响应：返回下载链接

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 当前筛选条件下无数据 | 提示"当前筛选条件下无数据可导出" |
| 导出失败 | 提示"导出失败，请稍后重试" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 导出中 | 用户点击导出按钮 |
| 导出中 | 正常 | 导出完成或失败 |

**输出**：线索列表 Excel 文件下载。

---

### 场景五：联系人挖掘流程

**触发条件**：在搜客结果列表点击深度挖掘。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 基于企业域名或名称查询 Apollo.io 联系人
     - 有域名：使用 organization_domains 精准锁定
     - 无域名：使用 q_organization_name 模糊匹配
   - 数据过滤：仅保留有 email 的记录
   - 邮箱揭示（reveal_personal_emails）：每个有效邮箱 1 Credit
   - 电话揭示（reveal_phone_number）：每个有效电话 1 Credit
   - 基础联系人数据不消耗 Credits
   - 无联系人结果：0 Credits
3. 系统返回响应：返回挖掘任务状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 同一企业域名重复入库 | 提示"该企业已在线索池中，不可重复入库" |
| 单租户同时挖掘数超过 3 个 | 提示"同时进行中的任务已达上限（3 个），请等待完成后重试" |
| 页面关闭 | 挖掘任务在后台异步执行，不中断 |
| 接口超时 | 1 分钟超时，跳过本次调用继续执行 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 挖掘中 | 创建挖掘任务 |
| 挖掘中 | 已完成 | 挖掘成功 |
| 挖掘中 | 失败 | 挖掘失败 |

**输出**：联系人数据存储，线索卡片更新为已完成。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 线索已存在 | 同一域名重复入库 | 拒绝入库 | Toast「该企业已在线索池中，不可重复入库」 |
| 线索不存在 | code 404 | 返回错误 | Toast「该线索不存在」 |
| 挖掘失败 | Apollo.io 返回错误 | 记录错误日志 | 卡片显示失败状态，Toast「联系人挖掘失败，请重试」 |
| Credits 余额不足 | 余额 ≤ 0 | 拒绝挖掘请求 | 按钮禁用，Toast「Credits 余额不足（当前：{balance}），无法执行挖掘」 |
| 挖掘已达上限 | 已挖掘 5 次 | — | 按钮禁用，Toast「该企业已达到最大挖掘次数（5 次）」 |
| 并发挖掘超限 | 同时进行 3 个任务 | — | 按钮禁用，Toast「同时进行中的任务已达上限（3 个），请等待完成后重试」 |
| 导出失败 | 后端异常 | — | Toast「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | — | Toast「当前筛选条件下无数据可导出」 |
| 网络超时 | 请求超时 | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
