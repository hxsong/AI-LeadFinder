## 业务处理逻辑

### 场景一：查看资源下发详情

**触发条件**：系统接收到页面加载请求，携带 `transaction_id` 参数

**执行逻辑**：

1. 系统接收请求：校验 `transaction_id` 参数有效性（非空、格式正确）
2. 系统执行处理：
   - 查询交易详情（状态应为`pending`）
   - 校验用户权限（是否为平台管理员、是否有权操作该租户）
   - 获取租户名称、当前到期时间、当前 BOT 席位数
   - 解析新增年数和增购席位
   - 计算计费年数、系统服务费、新到期时间
3. 系统返回响应：返回完整的下发详情数据结构

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| transaction_id 缺失 | 返回 400 错误，提示参数缺失 |
| transaction_id 格式不正确 | 返回 400 错误，提示格式错误 |
| 交易不存在 | 返回 404 错误，提示记录不存在 |
| 用户无权限 | 返回 401 错误，拒绝访问 |
| 交易状态为 active | 正常返回详情，但确认按钮应禁用，提示"已生效" |
| 交易状态为 cancelled | 正常返回详情，但页面提示已取消，按钮禁用 |

**状态变更**：无

**输出**：资源下发详情数据

---

### 场景二：确认资源生效

**触发条件**：用户点击"确认生效"按钮，系统接收到确认请求，携带 `transaction_id` 和 `tenant_id`

**执行逻辑**：

1. 系统接收请求：校验请求参数完整性（`transaction_id`、`tenant_id` 非空）
2. 系统执行处理：
   - 开启数据库事务
   - 查询交易记录（加锁防止并发），校验交易状态为 `pending`
   - 若状态校验通过，执行实际资源下发：
     - 更新系统服务到期时间
     - 更新 BOT 席位总数
   - 更新交易状态为 `active`，记录生效时间
   - 记录操作日志
   - 提交事务
3. 系统返回响应：返回生效结果及更新后的资源状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 交易状态为 active | 幂等处理，返回已生效结果（不重复执行资源下发） |
| 交易状态为 cancelled | 返回 409 错误，提示已取消，无法确认 |
| 并发重复确认请求 | 数据库行锁保证幂等，返回相同结果 |
| 资源更新失败 | 事务回滚，返回 500 错误，租户资源保持原状 |
| 租户已过期 | 正常执行，从当前日期开始计算新到期时间 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| pending | active | 确认请求校验通过且资源更新成功 |
| 正常 | 已生效 | 接口返回成功，页面展示生效状态 |

**输出**：资源生效结果（新到期时间、新 BOT 席位总数、生效时间）

---

### 场景三：取消下发

**触发条件**：用户点击"取消下发"按钮并确认后，系统接收到取消请求，携带 `transaction_id` 和 `tenant_id`

**执行逻辑**：

1. 系统接收请求：校验请求参数完整性
2. 系统执行处理：
   - 开启数据库事务
   - 查询交易记录（加锁防止并发），校验交易状态为 `pending`
   - 若状态校验通过：
     - 更新交易状态为 `cancelled`，记录取消时间
     - 记录操作日志
     - 不执行任何资源变更操作，仅标记订单为取消状态
   - 提交事务
3. 系统返回响应：返回取消结果
4. 前端自动跳转回租户资源充值页面

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 交易状态为 active | 返回 409 错误，提示已生效不可取消 |
| 交易状态为 cancelled | 幂等处理，返回相同结果 |
| 并发重复取消请求 | 数据库行锁保证幂等 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| pending | cancelled | 取消请求校验通过 |
| 正常 | 已取消 | 接口返回成功，页面自动跳转 |

**输出**：撤销成功结果（交易ID、新状态、取消时间），页面跳转至租户资源充值页面

---

### 场景四：返回修改

**触发条件**：用户点击"返回修改"按钮

**执行逻辑**：

1. 前端记录返回操作日志（可选）
2. 前端执行页面跳转：返回租户资源充值页面
3. 无后端数据变更操作

**边界条件**：无

**状态变更**：无

**输出**：页面跳转至租户资源充值页面，携带 `tenant_id` 参数

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 计费年数 | 计算 | `ceil((到期时间 - 当前时间) / 365 + 新增年数)` | 内存计算，接口返回 |
| 计算逻辑公式 | 字符串拼接 | `(当前BOT×新增年数 + 增购席位×计费年数) × 6,999` | 内存计算，接口返回 |
| 系统服务费 | 计算 | `(current_bot_seats × service_years + bot_seats_add × billing_years) × 6999` | 内存计算，接口返回 |
| 续费后到期时间 | 日期计算 | `DATE_ADD(current_expiry_date, INTERVAL service_years YEAR)` | 内存计算，接口返回 |

### 数据查询逻辑

**交易详情查询**：
```sql
SELECT 
  t.id as transaction_id,
  t.tenant_id,
  t.status,
  t.request_data,
  t.activated_at,
  t.cancelled_at,
  tn.name as tenant_name,
  ts.expiry_date as current_expiry_date,
  tc.bot_seats_total as current_bot_seats
FROM transactions t
LEFT JOIN tenants tn ON t.tenant_id = tn.id
LEFT JOIN tenant_subscriptions ts ON t.tenant_id = ts.tenant_id
LEFT JOIN tenant_configs tc ON t.tenant_id = tc.tenant_id
WHERE t.id = ? AND t.type = 'resource_provision'
```

### 费用计算逻辑

**系统服务费计算**：
```
剩余年数 = (到期时间 - 当前时间) / 365
计费年数 = ceil(剩余年数 + 新增年数)
系统服务费 = (当前BOT席位数 × 新增年数 + 增购席位 × 计费年数) × 6999
```

### 资源生效逻辑

**数据库更新顺序**（事务内执行）：

1. 更新 `tenant_subscriptions`：
   ```sql
   UPDATE tenant_subscriptions 
   SET expiry_date = DATE_ADD(expiry_date, INTERVAL ? YEAR),
       updated_at = NOW()
   WHERE tenant_id = ?
   ```

2. 更新 `tenant_configs`：
   ```sql
   UPDATE tenant_configs 
   SET bot_seats_total = bot_seats_total + ?,
       updated_at = NOW()
   WHERE tenant_id = ?
   ```

3. 更新 `transactions`：
   ```sql
   UPDATE transactions 
   SET status = 'active',
       activated_at = NOW(),
       updated_at = NOW()
   WHERE id = ? AND status = 'pending'
   ```

### 并发控制

- 使用数据库行锁（`SELECT FOR UPDATE`）防止并发确认/取消操作
- 事务隔离级别：READ COMMITTED
- 幂等设计：同一交易的重复确认返回相同结果

### 缓存策略

- 交易详情数据不缓存（确保状态实时性）

## 数据流

### 获取下发详情

```
GET /internal/v1/tenant/resources/detail?transaction_id={transaction_id}
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| transaction_id | `string` | 是 | 交易流水号，格式：TXN-YYYYMM-XXX |

**响应结构**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "transaction_id": "TXN-202401-001",
    "tenant_id": "tenant_001",
    "tenant_name": "A企业",
    "current_expiry_date": "2027-03-15",
    "status": "pending",
    "resources": {
      "service_fee": {
        "service_years": 2,
        "current_bot_seats": 3,
        "bot_seats_add": 2,
        "billing_years": 4,
        "formula": "(3 × 2 + 2 × 4) × 6,999",
        "fee": 97986,
        "new_expiry_date": "2029-03-15"
      }
    },
    "fee_summary": {
      "service_fee": 97986
    }
  }
}
```

**响应码说明**：

| code | 含义 | 后端处理 |
|------|------|---------|
| 200 | 成功 | 返回下发详情 |
| 400 | 参数错误 | transaction_id 缺失或格式不正确 |
| 401 | 未授权 | 用户无权限查看该租户资源 |
| 404 | 交易不存在 | 返回记录不存在或已过期 |
| 500 | 服务异常 | 返回通用错误 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 资源已生效 | 状态校验 | 返回 409 状态码冲突错误 | Toast「该资源已生效，无需重复确认」 |
| 资源不存在 | 数据库查询 | 返回 404 错误 | Toast「该下发记录不存在或已过期」并跳转充值页 |
| 网络超时 | 请求超时检测（30s） | 无 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | 服务端异常捕获 | 返回 500 错误 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 交易已取消 | 状态校验 | 返回 409 状态码冲突错误 | Toast「该交易已取消，请重新下单」并跳转充值页 |
| 重复确认 | 请求去重/幂等 | 幂等处理返回相同结果 | 按钮 loading 状态防止重复点击 |
| 无权限操作 | JWT Token 校验 | 返回 401 错误 | Toast「您没有权限执行此操作」 |
| 参数缺失 | 参数校验 | 返回 400 错误 | Toast「请求参数错误，请刷新页面重试」 |
| 数据库连接失败 | 连接池检测 | 返回 500 错误 | Toast「服务暂时不可用，请稍后再试」 |
| 事务执行失败 | 异常捕获 | 事务回滚，返回 500 错误 | Toast「资源下发失败，请重试或联系管理员」 |
| 超时无操作 | 前端定时器（30分钟） | 无 | 提示「页面已过期，请返回充值页重新操作」 |
