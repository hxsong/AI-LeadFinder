## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 租户名称 | `tenant_name` | `string` | 租户服务 | 租户注册时填写的企业名称 |
| Tenant ID | `tenant_id` | `string` | 系统生成 | 租户唯一标识符 |
| 交易流水号 | `transaction_id` | `string` | 系统生成 | 本次充值交易的唯一流水号 |
| 当前系统到期时间 | `current_expiry_date` | `string` | 租户服务 | 格式：YYYY-MM-DD |
| 新增服务年数 | `resources.service_fee.service_years` | `number` | 充值请求数据 | 用户选择的续费年数 |
| 当前 BOT 席位数 | `resources.service_fee.current_bot_seats` | `number` | 租户配置 | 租户当前已购买的 BOT 席位数量 |
| 增购 BOT 席位 | `resources.service_fee.bot_seats_add` | `number` | 充值请求数据 | 本次增购的 BOT 席位数量 |
| 计费年数 | `resources.service_fee.billing_years` | `number` | 后端计算 | ceil(剩余年数+新增年数) |
| 计算逻辑公式 | `resources.service_fee.formula` | `string` | 后端计算 | 费用计算公式的字符串展示 |
| 系统服务费 | `resources.service_fee.fee` | `number` | 后端计算 | 单位：元 |
| 续费后到期时间 | `resources.service_fee.new_expiry_date` | `string` | 后端计算 | 格式：YYYY-MM-DD |
| Credits 包类型 | `resources.credits_packages[].type` | `string` | 充值请求数据 | 枚举：basic/standard/professional/enterprise |
| Credits 额度 | `resources.credits_packages[].amount` | `number` | 配置数据 | 资源包包含的 Credits 数量 |
| Credits 折扣 | `resources.credits_packages[].discount` | `string` | 配置数据 | 折扣描述，如"9折"，无折扣时为 null |
| Credits 包价格 | `resources.credits_packages[].price` | `number` | 后端计算 | 单位：元 |
| 费用汇总-系统服务费 | `fee_summary.service_fee` | `number` | 后端计算 | 单位：元 |
| 费用汇总-Credits费用 | `fee_summary.credits_fee` | `number` | 后端计算 | 单位：元 |
| 费用汇总-总费用 | `fee_summary.total_fee` | `number` | 后端计算 | 单位：元 |
| 下发状态 | `status` | `string` | 系统状态 | 枚举：pending/active/cancelled |
| 生效时间 | `activated_at` | `string` | 系统时间 | 确认后生成，ISO8601格式 |
| 取消时间 | `cancelled_at` | `string` | 系统时间 | 取消后生成，ISO8601格式 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 租户资源充值.html | 点击"确认购买"按钮后跳转 | `tenant_id`, `transaction_id` |

**入口前置条件**：充值交易已创建且状态为 pending

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 多BOT工作台.html | 点击"确认生效"按钮 | `tenant_id` |
| 租户资源充值.html | 点击"返回修改"或"取消下发" | `tenant_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后等待响应 | 页面加载或提交操作 | 接口返回成功或失败 |
| 正常 | 数据完整返回，等待用户操作 | 详情接口返回成功 | 触发确认/取消/返回操作 |
| 确认中 | 点击确认按钮后等待 | 用户点击确认生效 | 确认接口返回 |
| 已生效 | 确认成功 | 确认接口返回成功 | 自动跳转工作台 |
| 已取消 | 取消操作成功 | 取消接口返回成功 | 自动跳转充值页 |

**状态流转图**：
```
加载中 --成功--> 正常 --确认--> 确认中 --成功--> 已生效 --跳转--> 多BOT工作台.html
                      |
                      +--取消--> 已取消 --跳转--> 租户资源充值.html
                      |
                      +--返回修改--> 租户资源充值.html
```

## 数据流

### 获取下发详情

```
GET /internal/v1/tenant/resources/detail?transaction_id={transaction_id}
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| transaction_id | `string` | 是 | 交易流水号 |

**响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "transaction_id": "TXN-202401-001",
    "tenant_id": "tenant_001",
    "tenant_name": "A企业",
    "current_expiry_date": "2027-03-15",
    "status": "pending",
    "resources": {
      "service_fee": {
        "service_years": 2,
        "current_bot_seats": 3,
        "bot_seats_add": 2,
        "billing_years": 4,
        "formula": "(3 × 2 + 2 × 4) × 3,000",
        "fee": 42000,
        "new_expiry_date": "2029-03-15"
      },
      "credits_packages": [
        {
          "type": "standard",
          "amount": 5000,
          "discount": "9折",
          "price": 4500
        },
        {
          "type": "basic",
          "amount": 1000,
          "discount": null,
          "price": 1000
        }
      ]
    },
    "fee_summary": {
      "service_fee": 42000,
      "credits_fee": 5500,
      "total_fee": 47500
    }
  }
}
```

**响应码说明**：

| code | 含义 | 后端处理 |
|------|------|---------|
| 200 | 成功 | 返回下发详情 |
| 400 | 参数错误 | transaction_id 缺失或格式不正确 |
| 401 | 未授权 | 用户无权限查看该租户资源 |
| 404 | 交易不存在 | 返回记录不存在或已过期 |
| 500 | 服务异常 | 返回通用错误 |

### 确认资源下发

```
POST /internal/v1/tenant/resources/confirm
```

**请求头**：`Authorization: Bearer {admin_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| transaction_id | `string` | 是 | 交易流水号 |
| tenant_id | `string` | 是 | 租户ID |

**响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "transaction_id": "TXN-202401-001",
    "status": "active",
    "activated_at": "2024-01-20T10:30:00Z",
    "new_expiry_date": "2029-03-15",
    "new_bot_seats_total": 5,
    "new_credits_balance": 76000
  }
}
```

**响应码说明**：

| code | 含义 | 后端处理 |
|------|------|---------|
| 200 | 成功 | 更新交易状态为 active，执行资源下发 |
| 400 | 参数错误 | 返回具体错误字段 |
| 401 | 未授权 | 拒绝请求 |
| 409 | 交易状态冲突 | 已生效或已取消，无法重复确认 |
| 500 | 服务异常 | 返回通用错误 |

---

## 处理逻辑

### 场景一：查看资源下发详情

**触发条件**：系统接收到页面加载请求，携带 transaction_id 参数

**执行逻辑**：

1. 系统接收请求：校验 transaction_id 参数有效性
2. 系统执行处理：
   - 查询交易详情
   - 组装资源下发数据（租户信息、服务续费、BOT增购、Credits包）
   - 计算费用汇总
3. 系统返回响应：返回完整的下发详情数据结构

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| transaction_id 缺失 | 返回 400 错误 |
| 交易不存在 | 返回 404 错误 |
| 用户无权限 | 返回 401 错误 |

**状态变更**：无

**输出**：资源下发详情数据

---

### 场景二：确认资源生效

**触发条件**：系统接收到确认请求，携带 transaction_id 和 tenant_id

**执行逻辑**：

1. 系统接收请求：校验请求参数完整性
2. 系统执行处理：
   - 校验交易状态为 pending
   - 执行资源生效：
     - 系统服务：更新到期时间
     - BOT 席位：增加可用席位
     - Credits：增加租户余额
   - 更新交易状态为 active
   - 记录生效时间
3. 系统返回响应：返回生效结果及更新后的状态

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 交易状态为 active | 返回 409 错误，提示已生效 |
| 交易状态为 cancelled | 返回 409 错误，提示已取消 |
| 重复确认请求 | 幂等处理，返回相同结果 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| pending | active | 确认请求校验通过 |

**输出**：资源生效结果

---

### 场景三：取消下发

**触发条件**：系统接收到取消请求，携带 transaction_id 和 tenant_id

**执行逻辑**：

1. 系统接收请求：校验请求参数完整性
2. 系统执行处理：
   - 校验交易状态为 pending
   - 撤销本次下发记录
   - 更新交易状态为 cancelled
   - 记录取消时间
3. 系统返回响应：返回取消结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 交易状态为 active | 返回 409 错误，提示已生效不可取消 |
| 交易状态为 cancelled | 幂等处理，返回相同结果 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| pending | cancelled | 取消请求校验通过 |

**输出**：撤销成功结果

---

### 场景四：返回修改

**触发条件**：系统接收到返回请求

**执行逻辑**：

1. 系统接收请求：记录返回操作日志
2. 系统执行处理：无数据变更操作
3. 系统返回响应：返回跳转目标信息

**边界条件**：无

**状态变更**：无

**输出**：返回目标页面路径

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 资源已生效 | 状态校验 | 返回 409 状态码冲突错误 | Toast「该资源已生效，无需重复确认」并跳转工作台 |
| 资源不存在 | 数据库查询 | 返回 404 错误 | Toast「该下发记录不存在或已过期」并跳转充值页 |
| 网络超时 | 请求超时检测 | 无 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | 服务端异常捕获 | 返回 500 错误 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 交易已取消 | 状态校验 | 返回 409 状态码冲突错误 | Toast「该交易已取消，请重新下单」并跳转充值页 |
| 重复确认 | 请求去重/幂等 | 幂等处理返回相同结果 | 按钮 loading 状态防止重复点击 |
| 超时无操作 | 前端定时器 | 无 | 30分钟无操作提示返回工作台 |
