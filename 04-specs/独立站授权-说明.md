### 数据来源说明

#### 接口返回字段

| 字段类别 | 数据来源 | 加工规则 | 缓存策略 |
|---------|---------|---------|---------|
| 站点列表 | 独立站系统 API | 调用 `GET /api/sites?tenant_id={tenant_id}` 获取，字段映射后返回 | 不缓存，实时查询 |
| 站点详情 | 独立站系统 API | 调用 `GET /api/sites/{site_id}/detail` 获取，提取企业介绍、主营品类、SEO 关键词等 | 不缓存，实时查询 |

#### 系统生成字段

| 字段名 | 生成规则 | 触发时机 | 存储位置 |
|-------|---------|---------|---------|
| authorization_status | 用户确认授权后设为 `authorized` | 用户点击"确认授权并分析"按钮 | site_authorizations 表 |
| task_id | 格式：`TASK-{timestamp}-{random}`，timestamp 为 YYYYMMDD 格式，random 为 3 位随机数 | 授权成功后生成 | analysis_tasks 表 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 业务初始化.html | 用户点击"授权独立站"卡片 | `bot_id` |

**入口前置条件**：
- 用户已登录且租户已绑定至少一个独立站
- 用户已在业务初始化页面选择"授权独立站"作为数据来源

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| AI业务分析中.html | 授权成功后点击"确认授权并分析"按钮 | `bot_id`, `task_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 正在请求站点列表数据 | 页面初始化，发送获取站点列表请求 | 接口返回成功或失败 |
| 正常 | 展示站点列表，等待用户选择 | 站点列表数据加载成功且数据非空 | 用户选择站点或页面跳转 |

| 同步中 | 正在同步站点数据 | 用户选中站点后自动触发 | 同步完成或同步失败 |

| 错误 | 数据加载或同步发生错误 | 接口返回错误码或网络异常 | 用户点击重试或返回 |

## 业务处理逻辑

### 场景一：站点列表展示与选择

**触发条件**：用户进入独立站授权页面。

**执行逻辑**：

1. 系统接收页面初始化请求：
   - 从 URL 参数获取 `bot_id`
   - 从请求头获取租户身份信息

2. 系统执行处理：
   - 调用独立站系统 API 获取当前租户绑定的站点列表
   - 解析响应数据，映射字段格式
   - 返回标准化站点列表

3. 前端渲染：
   - 每个站点卡片显示：站点名称、站点域名、状态标签（已上线/未上线）
   - 用户点击站点卡片，选中后显示橙色边框 + 勾选图标
   - 展开数据同步预览区域

4. 系统返回响应：站点列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无可授权站点 | 显示空态引导文案，提示「暂无已绑定的独立站，请先在建站系统中添加站点」|
| 未上线站点 | 可选但显示警告提示「站点未上线，数据可能不完整」|
| 已选择其他站点 | 自动取消之前的选择，仅保留最新选中状态 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 站点列表数据加载成功 |
| 加载中 | 无站点 | 站点列表返回空数组 |
| 正常 | 选中站点 | 用户点击站点卡片 |

**输出**：已选站点的数据预览区域展示。

---

### 场景二：数据同步预览

**触发条件**：用户选中某个站点后。

**执行逻辑**：

1. 系统接收同步请求：
   - 提取选中的 `site_id`
   - 验证站点是否存在且属于当前租户

2. 系统执行处理：
   - 调用独立站系统 API 获取站点详细数据
   - 提取企业介绍、成立信息、目标市场、资质证书
   - 提取主营品类（一级/二级分类）
   - 提取 SEO 关键词（核心/长尾）

3. 前端展示：
   - 数据预览区展示：公司简介、成立时间/规模、目标市场、资质证书
   - 主营品类以标签形式展示（一级/二级分类）
   - SEO 关键词以标签形式展示（核心/长尾）
   - 同步状态指示：正在同步... → 同步完成

4. 系统返回响应：结构化站点数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 同步超时（>30秒）| 提示「数据同步超时，请稍后重试」，提供重试按钮 |
| 同步失败 | 提示「数据同步失败，请稍后重试」，保留选中状态 |
| 部分字段缺失 | 正常展示，缺失字段显示「暂无数据」|
| 独立站系统不可用 | 提示「建站系统服务暂时不可用，请稍后再试」|

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 选中站点 | 同步中 | 用户选中站点后自动触发 |
| 同步中 | 同步完成 | 数据同步接口返回成功 |
| 同步中 | 选中站点 | 数据同步失败 |

**输出**：站点数据预览摘要。

---

### 场景三：确认授权

**触发条件**：用户确认站点选择，点击"确认授权并分析"按钮。

**执行逻辑**：

1. 系统接收授权请求：
   - 提取 `bot_id` 和 `site_id`
   - 验证请求参数合法性

2. 系统执行处理：
   - 查询 `site_authorizations` 表，检查是否已存在授权记录
   - 若已存在且状态为 `authorized`：
     - 查询对应的 `task_id`
     - 直接返回已有的 `task_id`，不重复创建
   - 若不存在：
     - 创建授权记录，状态设为 `authorized`
     - 生成新的 `task_id`，格式：`TASK-{timestamp}-{random}`
     - 在 `analysis_tasks` 表创建任务记录，状态为 `pending`

3. 系统返回响应：
   - 返回 `task_id` 和跳转地址
   - 跳转地址格式：`/ai-analysis.html?bot_id={bot_id}&task_id={task_id}`

4. 前端跳转：
   - 自动跳转至 AI业务分析中页面

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 已授权站点重新授权 | 返回已有的 `task_id`，不重复创建任务 |
| 授权接口超时 | 提示「授权请求超时，请稍后重试」，保留当前状态 |
| 授权接口失败 | 提示「授权失败，请稍后重试」，保留同步完成状态 |
| 站点已被删除 | 提示「站点不存在或已被删除」，返回正常状态 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 同步完成 | 授权中 | 用户点击"确认授权并分析"按钮 |
| 授权中 | 跳转至 AI业务分析中 | 授权成功，获取到 `task_id` |
| 授权中 | 同步完成 | 授权失败 |

**输出**：授权完成，跳转至分析处理页。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 站点列表 | 外部接口查询 | 服务端调用独立站系统 API 获取当前租户绑定的站点列表，实时返回，不缓存 | 不存储，实时查询 |
| 站点状态 | 状态映射 | 将独立站系统的发布状态映射为 `online`（已发布）/ `offline`（未发布）| 不存储，实时计算 |
| 公司简介 | 外部接口查询 | 调用独立站系统 API 获取企业简介文本，实时返回 | 不存储，实时查询 |
| 主营品类 | 外部接口查询 | 调用独立站系统 API 获取一级/二级分类数据，实时返回 | 不存储，实时查询 |
| SEO 关键词 | 外部接口查询 | 调用独立站系统 API 获取核心/长尾关键词，实时返回 | 不存储，实时查询 |
| 授权状态 | 数据持久化 | 用户确认授权后，将授权记录写入 `site_authorizations` 表 | `site_authorizations` 表 |
| 任务 ID | 系统生成 | 用户点击"确认授权并分析"后生成，格式：`TASK-{timestamp}-{random}`| `analysis_tasks` 表 |

### 独立站系统 API 调用流程

**获取站点列表**：
1. 接收前端请求，提取租户身份信息
2. 调用独立站系统 API：`GET {独立站系统域名}/api/sites?tenant_id={tenant_id}`
3. 解析响应数据，映射字段格式
4. 返回标准化站点列表给前端

**同步站点数据**：
1. 接收前端请求，提取 `site_id`
2. 调用独立站系统 API：`GET {独立站系统域名}/api/sites/{site_id}/detail`
3. 解析响应数据，提取企业介绍、主营品类、SEO 关键词等
4. 返回结构化数据给前端

### 授权处理流程

1. 接收前端授权请求，提取 `bot_id` 和 `site_id`
2. 查询是否已存在授权记录：
   - 若存在且状态为 `authorized`，返回已有的 `task_id`
   - 若不存在，创建新的授权记录
3. 生成新的 `task_id`，格式：`TASK-{timestamp}-{random}`
4. 创建 AI 分析任务记录，状态为 `pending`
5. 返回 `task_id` 和跳转地址给前端

## 数据流

### API 一：获取站点列表

```
GET /api/v1/tenant/sites
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：无

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "sites": [
      {
        "site_id": "SITE-001",
        "site_name": "XX科技官方网站",
        "site_domain": "xx-tech.example.com",
        "status": "online"
      },
      {
        "site_id": "SITE-002",
        "site_name": "XX科技海外站",
        "site_domain": "xx-tech-global.example.com",
        "status": "online"
      }
    ]
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| `sites[].site_id` | 站点 ID | string | 建站系统生成的站点唯一标识 |
| `sites[].site_name` | 站点名称 | string | 建站系统中配置的站点名称 |
| `sites[].site_domain` | 站点域名 | string | 建站系统中配置的站点域名 |
| `sites[].status` | 站点状态 | string | `online`（已发布）/ `offline`（未发布）|

**响应码说明**：

| code | 含义 | 处理方式 |
|------|------|---------|
| 200 | 成功 | 正常渲染站点列表 |
| 401 | 未登录 | 跳转登录页面，Toast「请先登录」|
| 403 | 无权限 | Toast「无权限执行此操作」|
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」|
| 504 | 网关超时 | Toast「网络连接异常，请检查网络后重试」|

### API 三：确认授权

```
POST /api/v1/sites/{site_id}/authorize
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `site_id` | string | 是 | 路径参数，站点 ID |
| `bot_id` | string | 是 | 请求体参数，BOT ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "site_id": "SITE-001",
    "authorization_status": "authorized",
    "task_id": "TASK-20240115-001",
    "redirect_to": "/ai-analysis.html?bot_id=BOT-001&task_id=TASK-20240115-001"
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| `site_id` | 站点 ID | string | 已授权站点 ID |
| `authorization_status` | 授权状态 | string | `pending` / `authorized` |
| `task_id` | 任务 ID | string | AI 分析任务 ID |
| `redirect_to` | 跳转地址 | string | 授权成功后跳转目标 |

**响应码说明**：

| code | 含义 | 处理方式 |
|------|------|---------|
| 200 | 成功 | 跳转至 AI业务分析中页面 |
| 401 | 未登录 | 跳转登录页面 |
| 403 | 无权限 | Toast「无权限执行此操作」|
| 404 | 站点不存在 | Toast「站点不存在或已被删除」|
| 409 | 已授权 | 返回已有的 `task_id`，不重复创建 |
| 500 | 授权失败 | Toast「授权失败，请稍后重试」|

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 站点列表为空 | `sites` 数组为空 | 返回空数组 | 显示空态引导，Toast「暂无已绑定的独立站，请先在建站系统中添加站点」|
| 站点同步失败 | sync 接口返回非 200 | 返回对应错误码和错误信息 | 显示失败提示，Toast「数据同步失败，请稍后重试」，提供重试按钮 |
| 授权失败 | authorize 接口返回非 200 | 返回对应错误码和错误信息 | 显示错误提示，Toast「授权失败，请稍后重试」|
| 网络超时 | 请求超时（>30秒）| 返回 504 | Toast「网络连接异常，请检查网络后重试」|
| 服务不可用 | code 500 | 返回 500 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」|
| 未登录 | code 401 | 返回 401 | 跳转登录页面，Toast「请先登录」|
| 无权限 | code 403 | 返回 403 | Toast「无权限执行此操作」|
| 站点不存在 | code 404 | 返回 404 | Toast「站点不存在或已被删除」|
| 独立站系统不可用 | 调用外部 API 失败 | 返回 503 | Toast「建站系统服务暂时不可用，请稍后再试」|
| 重复授权 | code 409 | 返回 409 和已有的 `task_id` | 正常跳转至 AI业务分析中页面 |

## 性能安全要求

### 性能指标

| 指标项 | 目标值 | 说明 |
|-------|-------|------|
| 站点列表加载 | < 2s | 从页面初始化到展示站点列表 |
| 站点数据同步 | < 5s | 从选中站点到展示数据预览 |
| 授权请求响应 | < 1s | 从点击确认到返回 task_id |
| 接口超时时间 | 30s | 独立站系统 API 调用超时时间 |

### 安全策略

| 策略项 | 说明 |
|-------|------|
| 身份认证 | 所有 API 请求需携带有效的租户 Token |
| 权限校验 | 验证站点是否属于当前租户 |
| 防重放攻击 | 授权接口需校验是否已存在授权记录 |
| 数据隔离 | 租户只能访问自己绑定的站点数据 |
