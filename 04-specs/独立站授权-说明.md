## 概述

独立站授权页面利用企业已有的建站系统，通过 API 毫秒级获取高纯度、高结构化的客户商业意图数据。用户授权后，系统从独立站 API 自动拉取企业介绍、主营品类、SEO 关键词等结构化数据，无需手动上传文件。

## 数据流

### API 一：获取站点列表

```
GET /api/v1/tenant/sites
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：无

**响应结构**：

```json
{
  "code": 200,
  "data": {
    "sites": [
      {
        "site_id": "SITE-001",
        "site_name": "XX科技官方网站",
        "site_domain": "xx-tech.example.com",
        "status": "online"
      }
    ]
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| sites[].site_id | 站点 ID | string | 建站系统生成 |
| sites[].site_name | 站点名称 | string | 建站系统配置 |
| sites[].site_domain | 站点域名 | string | 建站系统配置 |
| sites[].status | 站点状态 | string | online / offline |

**响应码说明**：

| 响应码 | 说明 | 处理方式 |
|-------|------|---------|
| 200 | 成功 | 正常渲染站点列表 |
| 401 | 未登录 | 跳转登录页面，Toast「请先登录」 |
| 403 | 无权限 | Toast「无权限执行此操作」 |
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」 |

### API 二：同步站点数据

```
POST /api/v1/sites/{site_id}/sync
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| site_id | string | 是 | 路径参数，站点 ID |

**响应结构**：

```json
{
  "code": 200,
  "data": {
    "site_id": "SITE-001",
    "sync_status": "completed",
    "company_profile": "专注 LED 显示屏研发与制造...",
    "founded_info": "2015年成立，200人规模",
    "target_markets": ["北美", "欧洲"],
    "certifications": ["ISO9001", "CE"],
    "categories": {
      "primary": ["LED 显示屏", "商业广告屏"],
      "secondary": ["控制系统", "安装配件"]
    },
    "seo_keywords": {
      "core": ["LED screen", "digital signage"],
      "long_tail": ["outdoor LED display", "rental LED panel"]
    }
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| site_id | 站点 ID | string | 站点唯一标识 |
| sync_status | 同步状态 | string | pending / in_progress / completed / failed |
| company_profile | 公司简介 | string | 企业简介文本 |
| founded_info | 成立信息 | string | 成立时间和规模 |
| target_markets | 目标市场 | array | 目标市场列表 |
| certifications | 资质证书 | array | 资质证书列表 |
| categories.primary | 一级分类 | array | 主营一级品类 |
| categories.secondary | 二级分类 | array | 主营二级品类 |
| seo_keywords.core | 核心关键词 | array | SEO 核心关键词 |
| seo_keywords.long_tail | 长尾关键词 | array | SEO 长尾关键词 |

**响应码说明**：

| 响应码 | 说明 | 处理方式 |
|-------|------|---------|
| 200 | 成功 | 展示数据预览摘要 |
| 401 | 未登录 | 跳转登录页面 |
| 404 | 站点不存在 | Toast「站点不存在或已被删除」 |
| 500 | 同步失败 | Toast「数据同步失败，请稍后重试」 |

### API 三：确认授权

```
POST /api/v1/sites/{site_id}/authorize
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| site_id | string | 是 | 路径参数，站点 ID |

**响应结构**：

```json
{
  "code": 200,
  "data": {
    "site_id": "SITE-001",
    "authorization_status": "authorized",
    "task_id": "TASK-202401-001",
    "redirect_to": "/ai-analysis.html?bot_id=BOT-001&task_id=TASK-202401-001"
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| site_id | 站点 ID | string | 已授权站点 ID |
| authorization_status | 授权状态 | string | pending / authorized |
| task_id | 任务 ID | string | AI 分析任务 ID |
| redirect_to | 跳转地址 | string | 授权成功后跳转目标 |

**响应码说明**：

| 响应码 | 说明 | 处理方式 |
|-------|------|---------|
| 200 | 成功 | 跳转至 AI业务分析中页面 |
| 401 | 未登录 | 跳转登录页面 |
| 403 | 无权限 | Toast「无权限执行此操作」 |
| 500 | 授权失败 | Toast「授权失败，请稍后重试」 |

## 处理逻辑

### 场景一：站点列表展示与选择

**触发条件**：用户进入独立站授权页面。

**执行逻辑**：
1. 前端调用接口获取当前租户下已绑定的站点列表
2. 每个站点卡片显示：站点名称、站点域名、状态标签（已上线/未上线）
3. 用户点击站点卡片，选中后显示橙色边框 + 勾选图标
4. 展开数据同步预览区域，显示该站点的企业信息摘要
5. 选择限制：每次仅可选 1 个站点作为核心主站

**边界条件**：
- 若无可授权站点，显示空态引导文案
- 未上线站点可选但显示警告提示

**状态变更**：`加载中` → `正常` / `无站点`

**输出**：已选站点的数据预览。

### 场景二：数据同步预览

**触发条件**：用户选中某个站点后。

**执行逻辑**：
1. 前端自动调用数据同步接口，实时获取站点数据
2. 摘要卡片展示：公司简介、成立时间/规模、目标市场、资质证书、主营品类（一/二级分类标签）、SEO 关键词
3. 同步状态指示：正在同步... → 同步完成

**边界条件**：
- 同步失败时提示可重试
- 同步超时需有超时处理

**状态变更**：`选中站点` → `同步中` → `同步完成`

**输出**：站点数据预览摘要。

### 场景三：确认授权

**触发条件**：用户确认站点选择，点击"确认授权"按钮。

**执行逻辑**：
1. 前端调用建站系统内部只读 API 网关
2. 毫秒级获取高纯度、高结构化的客户商业意图数据
3. 数据包括：企业介绍（含目标市场、资质证书）、主营品类、SEO 关键词
4. 无需用户手动输入，自动完成数据采集
5. 后端存储数据，触发 AI 分析任务
6. 跳转到 AI 业务分析中页面

**边界条件**：
- 已授权站点重新授权返回已有 task_id，不重复创建
- 授权接口返回失败或超时需有重试机制

**状态变更**：`同步完成` → `授权中` → 跳转至 AI业务分析中

**输出**：授权完成，跳转至分析处理页。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 站点列表为空 | sites 数组为空 | 返回空数组 | 空态引导，Toast「暂无已绑定的独立站，请先在建站系统中添加站点」 |
| 站点同步失败 | sync 接口返回非 200 | 返回对应错误码 | 失败提示，Toast「数据同步失败，请稍后重试」 |
| 授权失败 | authorize 接口返回非 200 | 返回对应错误码 | 错误提示，Toast「授权失败，请稍后重试」 |
| 网络超时 | 请求超时 | 返回 504 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 返回 500 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | code 401 | 返回 401 | 跳转登录页面，Toast「请先登录」 |
| 无权限 | code 403 | 返回 403 | Toast「无权限执行此操作」 |
| 站点不存在 | code 404 | 返回 404 | Toast「站点不存在或已被删除」 |
