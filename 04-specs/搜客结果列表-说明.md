## 概述

搜客结果列表是静态数据获客流程的结果展示页面，展示经过 AI 分析和 ICP 匹配打分排序的企业线索列表。用户可浏览、筛选、导出结果，点击企业卡片查看原始数据证据，并对高价值企业执行联系人挖掘操作。

## 功能逻辑

### 场景一：浏览搜客结果列表

**触发条件**：用户进入搜客结果列表页面。

**执行逻辑**：
1. 前端请求搜客结果数据
2. 以卡片形式展示候选企业列表，默认按评分倒序
3. 每张卡片展示：匹配度环形评分图、企业名称、官网链接（超链接）、徽章标签、企业简介、AI 推荐理由、员工规模
4. 评分颜色：≥80 绿色（S级）/ 60-79 蓝色（A/B级）/ <60 灰色（C级）

**输出**：按评分排序的企业卡片列表。

#### AI 调用一：match_score 计算（ICP 匹配评分）

**触发时机**：搜客执行中页面，P19-P20 ICP 匹配打分阶段。

**Prompt 模板**：

```
【系统指令】
你是一位专业的 B2B 外贸 ICP 匹配分析师。请根据企业的原始数据（海关记录、展会记录、企业信息）与客户的 ICP 画像，计算供需匹配评分。
严格基于输入的原始数据进行分析，不得捏造任何不存在于原始数据中的信息。

【输入数据】
- 企业原始数据：{enterprise_raw_data}
- ICP 画像：{icp_profile}
- 四维权重：需求匹配度 {weight_demand} / 购买能力 {weight_ability} / 采购意向 {weight_intent} / 活跃意向 {weight_activity}

【输出要求】
请以 JSON 格式输出评分结果，必须包含以下字段：
- match_score: 0-100 的整数加权总分
- dimension_scores: 各维度原始分（需求匹配度、购买能力、采购意向、活跃意向）
- reasoning: 评分理由（100字以内，必须基于原始数据）

【零捏造原则】
- reasoning 中的每一条理由必须可追溯至输入的原始数据
- 不得捏造企业的海关记录、展会记录或其他业务行为
- 评分必须在 0-100 范围内，超出则强制截断
```

**模型选择**：Claude 3.5 Sonnet / GPT-4o

**JSON Schema 要求**：

```json
{
  "type": "object",
  "required": ["match_score", "dimension_scores", "reasoning"],
  "properties": {
    "match_score": { "type": "integer", "minimum": 0, "maximum": 100 },
    "dimension_scores": {
      "type": "object",
      "properties": {
        "需求匹配度": { "type": "integer", "minimum": 0, "maximum": 100 },
        "购买能力": { "type": "integer", "minimum": 0, "maximum": 100 },
        "采购意向": { "type": "integer", "minimum": 0, "maximum": 100 },
        "活跃意向": { "type": "integer", "minimum": 0, "maximum": 100 }
      }
    },
    "reasoning": { "type": "string", "maxLength": 200 }
  }
}
```

**零捏造原则**：
- match_score 和 reasoning 必须基于企业原始数据（海关记录、展会记录、企业信息）
- 不得捏造不存在于输入数据中的业务行为或资质信息

**边界处理**：

| 异常类型 | 检测方式 | 系统行为 |
|---------|---------|---------|
| AI 输出为空 | 校验 null | 默认评分 0，不阻断流程 |
| 评分超出范围 | <0 或 >100 | 强制截断至 0-100 |
| JSON 格式错误 | 解析失败 | 自动重试 1 次 |
| 评分与原始数据矛盾 | 一致性校验失败 | 标记仅供参考 |

#### AI 调用二：ai_reasoning 生成（AI 推荐理由）

**触发时机**：搜客执行中页面，ICP 匹配打分完成后。

**Prompt 模板**：

```
【系统指令】
你是一位专业的外贸获客顾问。请根据企业的原始数据与 ICP 画像，为 B2B 销售团队生成一条推荐理由，帮助判断该企业是否为优质潜在客户。
严格基于输入的原始数据撰写推荐理由，不得捏造任何数据。

【输入数据】
- 企业名称：{company_name}
- 企业原始数据：{enterprise_raw_data}
- ICP 画像：{icp_profile}
- 匹配评分：{match_score}

【输出要求】
请以 JSON 格式输出推荐理由，必须包含以下字段：
- ai_reasoning: 推荐理由文本（50-500字），必须包含以下维度中已有数据的说明：
  1. 企业与客户产品的供需关系
  2. 企业资质与规模印证
  3. 活跃度信号（展会、海关等）
  4. 综合判断与转化率预估
- confidence: 置信度（high/medium/low），反映推荐理由的可信程度

【零捏造原则】
- ai_reasoning 中提及的每一条信息必须存在于原始数据中
- 不得捏造企业的展会参与记录、海关进出口记录或其他商业行为
- 如原始数据不足某个维度，该维度可不提及，不得虚构

【超长截断策略】
- 超过 500 字时，截断至 500 字并在末尾添加"..."
- 优先保留前三维度内容
```

**模型选择**：Claude 3.5 Sonnet / GPT-4o

**JSON Schema 要求**：

```json
{
  "type": "object",
  "required": ["ai_reasoning", "confidence"],
  "properties": {
    "ai_reasoning": { "type": "string", "minLength": 10, "maxLength": 500 },
    "confidence": { "type": "string", "enum": ["high", "medium", "low"] }
  }
}
```

**零捏造原则**：
- 推荐理由中的所有事实性陈述必须可追溯至原始数据
- 不得捏造企业未参与的展会、未存在的海关记录

**边界处理**：

| 异常类型 | 检测方式 | 系统行为 |
|---------|---------|---------|
| ai_reasoning 为空 | 校验空字符串 | 显示默认文案"暂无推荐理由" |
| ai_reasoning 超过 500 字 | 字符串长度校验 | 截断至 500 字 + "..." |
| JSON 格式错误 | 解析失败 | 自动重试 1 次 |
| 幻觉检测 | 内容与原始数据矛盾 | 标记"仅供参考"，前端显示警告文案 |

### 场景三：导出搜客结果

**触发条件**：用户点击"导出结果"按钮。

**执行逻辑**：
1. 前端请求导出接口
2. 后端生成 Excel 文件，返回下载链接（有效期 24 小时）
3. 导出字段：企业名称、官网、匹配度评分、AI推荐理由、徽章标签、企业简介、员工规模、联系方式、海关记录、展会记录、地理位置
4. 无版本限制，所有版本均可使用

**输出**：Excel 文件下载。

---

### 场景四：查看企业原始数据

**触发条件**：用户点击企业卡片的"查看详情"。

**执行逻辑**：
1. 弹出原始数据查看弹窗
2. 展示四个数据来源的原始证据：
   - 海关数据：HS Code、产品描述、进口次数、最近日期、供应商国家（来源：出海派）
   - 展会数据：展会名称、日期、地点、参与类型（来源：10times）
   - 地图数据：企业名称、地址、Google评分、营业状态、坐标（来源：Google Maps）
   - 网站数据：域名、内容摘要（来源：Jina Reader）

**输出**：原始数据证据弹窗。

---

### 场景五：联系人挖掘

**触发条件**：用户点击企业卡片的"深度挖掘"按钮。

**执行逻辑**：
1. 弹出联系人挖掘弹窗，展示计费规则
2. Credits 规则：单次挖掘消耗 15 Credits，每获取 1 个验证邮箱 = 2 Credits，附加 LinkedIn = +1 Credit，附加 Twitter/FB = +1 Credit/个
3. 用户确认后，任务异步执行
4. 前端跳转至静态线索池页面查看进度

**输出**：联系人挖掘任务创建，跳转至静态线索池。

## 边界规则

| 规则 | 条件 | 系统行为 |
|------|------|---------|
| 数据保留期限 | 搜索结果保留 | 90 天 |
| 导出功能 | 无版本限制 | 所有版本均可使用 |
| 分页限制 | 每页最大 20 条记录 | 总结果最大 200 条 |
| 联系人挖掘消耗 | 单次挖掘 | 15 Credits |
| 联系人邮箱揭示 | 每获取 1 个验证邮箱 | 2 Credits |
| LinkedIn 附加 | 附加 LinkedIn 主页 | +1 Credit |
| 社交媒体附加 | 附加 Twitter/FB | +1 Credit/个 |
| 单企业挖掘次数 | 最多次数 | 5 次 |
| 联系人数据有效期 | 有效期 | 90 天 |
| Credits 余额规则 | 余额可为负 | ≤0 禁止消耗操作，挖掘中不扣费，完成后结算 |
| AI 推荐理由超长 | 超过 500 字 | 截断 + "查看更多"展开 |
| AI 推荐理由幻觉 | 与企业信息矛盾 | 标记仅供参考 | 「推荐理由仅供参考」 |

## 异常处理

| 异常场景 | 检测方式 | 前端表现 | Toast 文案 |
|---------|---------|---------|-----------|
| 任务不存在 | code 404 | 错误提示并返回 | 「该任务不存在或已过期」 |
| 结果数据为空 | items 数组为空 | 空态引导 | 「未找到符合条件的候选企业」 |
| Credits 不足 | 余额 ≤ 0 | 按钮禁用 | 「Credits 余额不足（当前：{balance}），无法执行挖掘」 |
| 挖掘已达上限 | 已挖掘 5 次 | 按钮禁用 | 「该企业已达到最大挖掘次数（5 次）」 |
| 导出失败 | code 非 200 | 失败提示 | 「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | 失败提示 | 「当前筛选条件下无数据可导出」 |
| AI 推荐理由为空 | ai_reasoning 为空 | 显示默认文案 | 「暂无推荐理由」 |
| AI 推荐理由格式错误 | JSON 解析失败 | 显示原始摘要 | 「推荐理由生成异常」 |
| AI 推荐理由超长 | 超过 500 字 | 截断展示 | - |
| 网络超时 | 请求超时 | 错误提示 | 「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 全局错误处理 | 「服务暂时不可用，请稍后再试」 |
