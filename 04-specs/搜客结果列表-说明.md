## 数据契约

### 前端展示字段映射

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 任务 ID | `task_id` | `string` | 路由参数传入 | 搜客任务唯一标识，格式：`TASK-{timestamp}-{random}` |
| BOT ID | `bot_id` | `string` | 路由参数传入 | 当前 BOT 唯一标识，用于权限校验 |
| 结果总数 | `total` | `number` | 接口返回 | 从 `search_task_results` 表统计该任务的记录总数 |
| 高潜力数 | `high_potential_count` | `number` | 接口返回 | 评分 ≥80 的企业数量，从 `search_task_results` 表统计 `match_score >= 80` 的记录数 |
| 企业 ID | `enterprise_id` | `string` | 接口返回 | `search_task_results` 表的 `id` 字段 |
| 企业名称 | `company_name` | `string` | 接口返回 | 从 `search_task_results` 表的 `company_name` 字段获取 |
| 企业官网 | `website` | `string` | 接口返回 | 从 `search_task_results` 表的 `website` 字段获取 |
| 匹配度评分 | `match_score` | `number` | 接口返回 | 从 `search_task_results` 表的 `match_score` 字段获取，由上游 ICP 匹配打分阶段计算并存储 |
| 徽章标签 | `badges` | `array` | 系统生成 | 根据 `data_sources` 字段自动生成：有海关记录/有参展记录/有地图数据等 |
| 企业简介 | `company_profile` | `string` | 接口返回 | 从 `search_task_results` 表的 `candidate_icp_profile.positioning` 字段获取 |
| 员工规模 | `employee_scale` | `string` | 接口返回 | 从 `search_task_results` 表的 `candidate_icp_profile.scale` 字段获取 |
| AI 推荐理由 | `ai_verdict` | `string` | 接口返回 | 从 `search_task_results` 表的 `ai_verdict` 字段获取，由上游 ICP 匹配打分阶段生成并存储 |
| 海关数据来源 | `data_sources.customs` | `boolean` | 接口返回 | 从 `search_task_results` 表的 `structured_data` 判断是否存在海关数据 |
| 展会数据来源 | `data_sources.exhibition` | `boolean` | 接口返回 | 从 `search_task_results` 表的 `structured_data` 判断是否存在展会数据 |
| 地图数据来源 | `data_sources.maps` | `boolean` | 接口返回 | 从 `search_task_results` 表的 `structured_data` 判断是否存在地图数据 |
| 网站数据来源 | `data_sources.website` | `boolean` | 接口返回 | 从 `search_task_results` 表的 `website_content` 判断是否不为空 |
| 已挖掘状态 | `is_dug` | `boolean` | 接口返回 | 查询 `static_leads` 表，判断该企业是否已存在对应线索记录 |
| 四维评分 | `dimension_scores` | `object` | 接口返回 | 包含需求匹配度、购买能力、采购意向、活跃意向四个维度的原始分 |

### 数据来源深度说明

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含技术处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 结果总数 | `total` | `number` | 接口返回 | 服务端执行 `SELECT COUNT(*) FROM search_task_results WHERE task_id = ?` 实时统计，无缓存 |
| 高潜力数 | `high_potential_count` | `number` | 接口返回 | 服务端执行 `SELECT COUNT(*) FROM search_task_results WHERE task_id = ? AND match_score >= 80` 实时统计 |
| 徽章标签 | `badges` | `array` | 系统生成 | 服务端根据 `data_sources` 对象各字段值动态生成：海关数据存在时添加"有海关数据"，展会数据存在时添加"有参展记录"，地图数据存在时添加"有地图数据" |
| 已挖掘状态 | `is_dug` | `boolean` | 接口返回 | 服务端遍历结果列表时，对每个企业执行 `SELECT EXISTS(SELECT 1 FROM static_leads WHERE enterprise_id = ?)` 查询判断 |
| 四维评分 | `dimension_scores` | `object` | 接口返回 | 服务端将 `match_score`、`buying_power_score`、`purchase_intent_score`、`active_intent_score` 四个字段组合为对象返回 |

## 页面流转

### 入口

| 来源 | 触发条件 | 携带参数 |
|------|---------|---------|
| 搜客记录.html | 点击已完成的任务记录 | `bot_id`, `task_id` |
| 搜客执行中.html | 所有步骤完成后自动跳转 | `bot_id`, `task_id` |

**入口前置条件**：
1. 搜客任务已存在于 `search_tasks` 表且状态为 `completed`
2. 任务结果数据已写入 `search_task_results` 表
3. 当前用户有权限访问该 BOT 的任务

### 出口

| 目标 | 触发条件 | 携带数据 |
|------|---------|---------|
| 搜客需求输入.html | 点击"返回"或"重新搜索"按钮 | `bot_id` |
| 静态线索池.html | 点击"深度挖掘"确认后跳转 | `bot_id`, `lead_id` |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 加载中 | 请求发出后等待数据返回 | 页面加载 | 数据返回或请求失败 |
| 正常 | 数据加载成功，展示结果列表 | 接口返回数据且 items 不为空 | 用户操作或页面跳转 |

## 业务处理逻辑

### 场景一：浏览搜客结果列表

**触发条件**：用户进入搜客结果列表页面。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id` 和 `task_id`
2. 系统执行处理：
   - 前端请求搜客结果数据
   - 以卡片形式展示候选企业列表，默认按评分倒序
   - 每张卡片展示：匹配度环形评分图、企业名称、官网链接（超链接）、徽章标签、企业简介、AI 推荐理由、员工规模、四维评分
   - 评分颜色：≥80 绿色 / 60-79 蓝色 / <60 灰色
3. 系统返回响应：返回结果列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无结果数据 | 显示空态引导文案，提供"重新搜索"按钮 |
| AI 推荐理由为空 | 显示默认文案"暂无推荐理由" |
| AI 推荐理由超过 500 字 | 截断展示，提供"查看更多"展开 |
| 企业官网为空 | 不显示官网链接 |
| 员工规模为空 | 不显示规模信息 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功且 items 不为空 |
| 加载中 | 空态 | items 为空数组 |
| 加载中 | 错误 | 接口返回错误 |

**输出**：按评分排序的企业卡片列表。

---

### 场景二：筛选与排序

**触发条件**：用户切换筛选或排序条件。

**执行逻辑**：

1. 系统接收请求：用户选择筛选或排序条件
2. 系统执行处理：
   - 筛选维度：
     - 数据来源：有海关记录、有参展记录、有地图数据、有网站数据
   - 排序方式：评分倒序（默认）、评分正序、企业名称 A-Z
   - 筛选后列表实时刷新，重置到第一页
3. 系统返回响应：返回过滤后的结果列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案"当前筛选条件下无数据" |
| 组合筛选条件冲突 | 按 AND 逻辑处理，返回同时满足所有条件的数据 |

**状态变更**：无

**输出**：按条件过滤后的企业卡片列表。

---

### 场景三：导出搜客结果

**触发条件**：用户点击"导出结果"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击导出按钮
2. 系统执行处理：
   - 前端请求导出接口
   - 后端生成 Excel 文件，返回下载链接（有效期 24 小时）
   - 导出字段：企业名称、官网、匹配度评分、AI推荐理由、徽章标签、企业简介、员工规模、海关记录、展会记录、地理位置
   - 无版本限制，所有版本均可使用
3. 系统返回响应：返回下载链接

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 当前筛选条件下无数据 | 提示"当前筛选条件下无数据可导出" |
| 导出失败 | 提示"导出失败，请稍后重试" |
| 生成文件超时 | 提示"文件生成中，请稍后刷新页面下载" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 导出中 | 用户点击导出按钮 |
| 导出中 | 正常 | 导出完成或失败 |

**输出**：Excel 文件下载。

### 场景四：查看企业原始数据

**触发条件**：用户点击企业卡片的"查看原始数据"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击查看原始数据
2. 系统执行处理：
   - 弹出原始数据查看弹窗
   - 从 `search_task_results` 表的 `raw_data` 和 `structured_data` 字段获取数据
   - 展示四类原始数据证据：
     - 海关数据：HS Code、产品描述、进口次数、最近日期、供应商国家
     - 展会数据：展会名称、日期、地点、参与类型
     - 地图数据：企业名称、地址、地图评分、营业状态、坐标
     - 网站数据：域名、内容摘要
3. 系统返回响应：弹窗展示原始数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 某类数据不存在 | 对应区域显示"暂无数据" |
| 数据格式异常 | 显示原始 JSON，标记格式异常 |

**状态变更**：无

**输出**：原始数据证据弹窗。

### 场景五：联系人挖掘

**触发条件**：用户点击企业卡片的"深度挖掘"按钮（仅对未挖掘企业）。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 校验该企业是否已挖掘
   - 若已挖掘：Toast 提示"该企业已挖掘，请前往静态线索列表查看"，终止流程
   - 若未挖掘：
     - 弹出联系人挖掘弹窗，展示计费规则
     - MoliAI点数规则：单次挖掘消耗 15 MoliAI点数，每获取 1 个验证邮箱 = 1 MoliAI点数，附加 LinkedIn = 0 MoliAI点数（免费），附加 Twitter/FB = 0 MoliAI点数/个（免费）
     - 用户确认后，调用联系人挖掘接口
     - 任务创建成功后，标记该企业为已挖掘状态
     - 任务异步执行，前端跳转至静态线索池页面查看进度
3. 系统返回响应：返回挖掘任务 ID 或提示信息

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| MoliAI点数余额 ≤ 0 | 按钮禁用，提示"MoliAI点数余额不足，无法执行挖掘" |
| 该企业已挖掘 | 按钮显示"已挖掘"且禁用，点击时 Toast 提示"该企业已挖掘，请前往静态线索列表查看" |
| 该企业已在线索池中 | 提示"该企业已在线索池中" |
| 并发限制 | 提示"当前挖掘任务过多，请等待其他任务完成" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 挖掘中 | 用户点击深度挖掘按钮 |
| 挖掘中 | 正常 | 挖掘任务创建失败 |

**输出**：联系人挖掘任务创建，跳转至静态线索池。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| `total` | 查询 | `SELECT COUNT(*) FROM search_task_results WHERE task_id = ?` | 实时查询，无缓存 |
| `high_potential_count` | 查询 | `SELECT COUNT(*) FROM search_task_results WHERE task_id = ? AND match_score >= 80` | 实时查询，无缓存 |
| `badges` | 系统生成 | 根据 `data_sources` 各字段值动态生成标签数组 | 内存组装，不持久化 |
| `is_dug` | 关联查询 | `SELECT EXISTS(SELECT 1 FROM static_leads WHERE enterprise_id = ?)` | 实时查询，无缓存 |
| `dimension_scores` | 字段映射 | 将四个维度评分字段组合为对象返回 | 内存组装 |

### 数据表结构

**search_task_results 表（搜客任务结果表）**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | 主键，作为 enterprise_id 使用 |
| `task_id` | `string` | 任务 ID，外键关联 search_tasks 表 |
| `company_name` | `string` | 企业名称 |
| `website` | `string` | 企业官网 |
| `structured_data` | `json` | 原始搜索的结构化数据（海关、展会、地图数据） |
| `website_content` | `text` | Jina Reader 抓取的企业官网内容 |
| `candidate_icp_profile` | `json` | 候选企业 ICP 画像：企业名称、定位、业务类型、主营品类、核心优势、规模 |
| `match_score` | `number` | 需求匹配度评分（0-100），由上游 ICP 匹配打分阶段计算 |
| `buying_power_score` | `number` | 购买力评分（0-100），由上游 ICP 匹配打分阶段计算 |
| `purchase_intent_score` | `number` | 采购意向评分（0-100），由上游 ICP 匹配打分阶段计算 |
| `active_intent_score` | `number` | 活跃意向评分（0-100），由上游 ICP 匹配打分阶段计算 |
| `ai_verdict` | `string` | AI 综合判词/推荐理由，由上游 ICP 匹配打分阶段生成 |
| `raw_data` | `json` | 原始数据（海关、展会、地图详细数据） |
| `created_at` | `timestamp` | 创建时间 |

**static_leads 表（静态线索表）**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | 主键，线索唯一标识 |
| `enterprise_id` | `string` | 关联的搜客结果企业 ID |
| `task_id` | `string` | 来源搜客任务 ID |
| `bot_id` | `string` | BOT ID |
| `status` | `string` | 线索状态：`pending`/`digging`/`completed` |
| `created_at` | `timestamp` | 创建时间 |

### 徽章标签生成规则

| 标签 | 生成条件 |
|------|---------|
| 有海关数据 | structured_data->>'customs' IS NOT NULL AND jsonb_array_length(structured_data->'customs') > 0 |
| 有参展记录 | structured_data->>'exhibition' IS NOT NULL AND jsonb_array_length(structured_data->'exhibition') > 0 |
| 有地图数据 | structured_data->>'maps' IS NOT NULL AND structured_data->'maps'->>'name' IS NOT NULL |

### 数据来源判断逻辑

**海关数据存在判断**：
```sql
structured_data->>'customs' IS NOT NULL 
AND jsonb_array_length(structured_data->'customs') > 0
```

**展会数据存在判断**：
```sql
structured_data->>'exhibition' IS NOT NULL 
AND jsonb_array_length(structured_data->'exhibition') > 0
```

**地图数据存在判断**：
```sql
structured_data->>'maps' IS NOT NULL 
AND structured_data->'maps'->>'name' IS NOT NULL
```

**网站数据存在判断**：
```sql
website_content IS NOT NULL 
AND LENGTH(website_content) > 0
```

### 缓存策略

| 数据 | 缓存方式 | 过期时间 | 说明 |
|------|---------|---------|------|
| 搜客结果列表 | Redis | 5分钟 | Key: `search:results:{task_id}:{page}:{sort}` |
| 企业详情 | 无 | - | 实时查询 |
| 导出文件 | OSS/S3 | 24小时 | 临时下载链接 |

## 数据流

### 获取搜客结果

```
GET /api/v1/search/results/{task_id}
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | `string` | 是 | 路径参数，搜客任务唯一标识 |
| sort_by | `string` | 否 | 排序字段：`match_score`/`company_name`，默认 `match_score` |
| sort_order | `string` | 否 | 排序方向：`desc`/`asc`，默认 `desc` |
| filter_source | `string` | 否 | 数据来源筛选：`customs`/`exhibition`/`maps`/`website`，多选用逗号分隔 |
| page | `number` | 否 | 页码，默认 1 |
| page_size | `number` | 否 | 每页数量，默认 20，最大 100 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 任务不存在 | code 404 | 返回错误 | Toast「该任务不存在或已过期」 |
| 结果数据为空 | items 数组为空 | — | 空态引导，Toast「未找到符合条件的候选企业」 |
| MoliAI点数不足 | 余额 ≤ 0 | 拒绝挖掘请求 | 按钮禁用，Toast「MoliAI点数余额不足（当前：{balance}），无法执行挖掘」 |
| 企业已挖掘 | 已挖掘 | — | 按钮显示「已挖掘」且禁用，点击时 Toast「该企业已挖掘，请前往静态线索列表查看」 |
| 导出失败 | code 非 200 | — | Toast「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | — | Toast「当前筛选条件下无数据可导出」 |
| AI 推荐理由为空 | ai_verdict 为空 | — | 显示默认文案，Toast「暂无推荐理由」 |
| 网络超时 | 请求超时 | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |
| 无权限访问 | code 403 | 返回错误 | Toast「您无权访问该任务」 |
