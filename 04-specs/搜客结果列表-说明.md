## 数据流

### 获取搜客结果

```
GET /api/v1/search/results/{task_id}
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | `string` | 是 | 路径参数，搜客任务唯一标识 |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "task_id": "TASK-202401-001",
    "total": 98,
    "high_potential_count": 8,
    "items": [
      {
        "enterprise_id": "ENT-001",
        "company_name": "Alpine Gear International",
        "website": "https://alpine-gear.com",
        "match_score": 88,
        "score_grade": "S",
        "badges": ["S级潜力", "有海关数据"],
        "company_profile": "Wholesale distributor specializing in outdoor equipment...",
        "employee_scale": "50-200人",
        "ai_reasoning": "该企业长期进口户外攀岩设备，与客户主营品类高度匹配...",
        "confidence": "high",
        "is_dug": false,
        "data_sources": { "customs": true, "exhibition": false, "maps": true, "website": true }
      }
    ]
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 查询成功 |
| 401 | 未登录 |
| 404 | 任务不存在 |
| 500 | 服务异常 |

### 联系人挖掘

```
POST /api/v1/contacts/dig
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`application/json`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| enterprise_id | `string` | 是 | 企业 ID |
| bot_id | `string` | 是 | BOT 实例 ID |
| task_id | `string` | 是 | 搜客任务 ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "lead_id": "LEAD-001",
    "status": "digging",
    "estimated_credits": 15
  }
}
```

**响应码说明**：

| code | 含义 |
|------|------|
| 200 | 挖掘任务创建成功 |
| 400 | 参数错误 |
| 1002 | Credits 余额不足 |
| 404 | 企业不存在 |
| 500 | 服务异常 |

**请求头**：`Authorization: Bearer {tenant_token}`

| 字段 | 类型 | 必填 | 说明 |

## 处理逻辑

### 场景一：浏览搜客结果列表

**触发条件**：用户进入搜客结果列表页面。

**执行逻辑**：

1. 系统接收请求：页面加载，携带 `bot_id` 和 `task_id`
2. 系统执行处理：
   - 前端请求搜客结果数据 `GET /api/v1/search/results/{task_id}`
   - 以卡片形式展示候选企业列表，默认按评分倒序
   - 每张卡片展示：匹配度环形评分图、企业名称、官网链接（超链接）、徽章标签、企业简介、AI 推荐理由、员工规模
   - 评分颜色：≥80 绿色（S级）/ 60-79 蓝色（A/B级）/ <60 灰色（C级）
3. 系统返回响应：返回结果列表数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 无结果数据 | 显示空态引导文案 |
| AI 推荐理由为空 | 显示默认文案"暂无推荐理由" |
| AI 推荐理由超过 500 字 | 截断展示，提供"查看更多"展开 |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 加载中 | 正常 | 数据返回成功 |
| 加载中 | 空态 | items 为空 |

**输出**：按评分排序的企业卡片列表。

---

### 场景二：筛选与排序

**触发条件**：用户切换筛选或排序条件。

**执行逻辑**：

1. 系统接收请求：用户选择筛选或排序条件
2. 系统执行处理：
   - 筛选维度：评分等级、数据来源、徽章标签
   - 排序方式：评分倒序（默认）/ 评分正序 / 企业名称
   - 筛选后列表实时刷新
3. 系统返回响应：返回过滤后的结果列表

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 筛选后无数据 | 显示空态引导文案 |

**状态变更**：无

**输出**：按条件过滤后的企业卡片列表。

---

### 场景三：导出搜客结果

**触发条件**：用户点击"导出结果"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击导出按钮
2. 系统执行处理：
   - 前端请求导出接口 `GET /api/v1/search/results/{task_id}/export`
   - 后端生成 Excel 文件，返回下载链接（有效期 24 小时）
   - 导出字段：企业名称、官网、匹配度评分、AI推荐理由、徽章标签、企业简介、员工规模、联系方式、海关记录、展会记录、地理位置
   - 无版本限制，所有版本均可使用
3. 系统返回响应：返回下载链接

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 当前筛选条件下无数据 | 提示"当前筛选条件下无数据可导出" |
| 导出失败 | 提示"导出失败，请稍后重试" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 导出中 | 用户点击导出按钮 |
| 导出中 | 正常 | 导出完成或失败 |

**输出**：Excel 文件下载。

### 场景四：查看企业原始数据

**触发条件**：用户点击企业卡片的"查看详情"。

**执行逻辑**：

1. 系统接收请求：用户点击查看详情
2. 系统执行处理：
   - 弹出原始数据查看弹窗
   - 展示四个数据来源的原始证据：
     - 海关数据：HS Code、产品描述、进口次数、最近日期、供应商国家（来源：出海派）
     - 展会数据：展会名称、日期、地点、参与类型（来源：10times）
     - 地图数据：企业名称、地址、Google评分、营业状态、坐标（来源：Google Maps）
     - 网站数据：域名、内容摘要（来源：Jina Reader）
3. 系统返回响应：弹窗展示原始数据

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 某类数据不存在 | 对应区域显示"暂无数据" |

**状态变更**：无

**输出**：原始数据证据弹窗。

### 场景五：联系人挖掘

**触发条件**：用户点击企业卡片的"深度挖掘"按钮。

**执行逻辑**：

1. 系统接收请求：用户点击深度挖掘按钮
2. 系统执行处理：
   - 校验该企业是否已挖掘（查询 `is_dug` 字段），若已挖掘则提示并终止
   - 弹出联系人挖掘弹窗，展示计费规则
   - Credits 规则：单次挖掘消耗 15 Credits，每获取 1 个验证邮箱 = 1 Credit，附加 LinkedIn = 0 Credit（免费），附加 Twitter/FB = 0 Credit/个（免费）
   - 用户确认后，调用 `POST /api/v1/contacts/dig`
   - 任务创建成功后，标记该企业为已挖掘状态（`is_dug: true`）
   - 任务异步执行，前端跳转至静态线索池页面查看进度
3. 系统返回响应：返回挖掘任务 ID

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| Credits 余额 ≤ 0 | 按钮禁用，提示"Credits 余额不足，无法执行挖掘" |
| 该企业已挖掘 | 按钮显示"已挖掘"且禁用，提示"该企业已挖掘，不能重复挖掘" |
| 该企业已在线索池中 | 提示"该企业已在线索池中" |

**状态变更**：

| 原状态 | 新状态 | 触发条件 |
|--------|--------|----------|
| 正常 | 挖掘中 | 用户点击深度挖掘按钮 |
| 挖掘中 | 正常 | 挖掘任务创建失败 |

**输出**：联系人挖掘任务创建，跳转至静态线索池。

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 任务不存在 | code 404 | 返回错误 | Toast「该任务不存在或已过期」 |
| 结果数据为空 | items 数组为空 | — | 空态引导，Toast「未找到符合条件的候选企业」 |
| Credits 不足 | 余额 ≤ 0 | 拒绝挖掘请求 | 按钮禁用，Toast「Credits 余额不足（当前：{balance}），无法执行挖掘」 |
| 企业已挖掘 | 已挖掘 | — | 按钮显示「已挖掘」且禁用，Toast「该企业已挖掘，不能重复挖掘」 |
| 导出失败 | code 非 200 | — | Toast「导出失败，请稍后重试」 |
| 无数据可导出 | 筛选条件下无数据 | — | Toast「当前筛选条件下无数据可导出」 |
| AI 推荐理由为空 | ai_reasoning 为空 | — | 显示默认文案，Toast「暂无推荐理由」 |
| AI 推荐理由格式错误 | JSON 解析失败 | — | 显示原始摘要，Toast「推荐理由生成异常」 |
| AI 推荐理由超长 | 超过 500 字 | — | 截断展示 |
| 网络超时 | 请求超时 | 重试 1 次 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 记录错误日志 | Toast「服务暂时不可用，请稍后再试」 |

## AI 提示词

### AI 模型选择

**优先使用国内模型**：

| 模型厂商 | 推荐模型 | 适用场景 |
|---------|---------|---------|
| 通义千问 (Qwen) | `qwen-max` / `qwen-plus` | 通用对话、评分计算 |
| Kimi (Moonshot) | `kimi-k2.5` | 长文本处理、复杂推理 |

**备选国际模型**：

| 模型厂商 | 推荐模型 | 适用场景 |
|---------|---------|---------|
| OpenAI | `gpt-4o` / `gpt-4o-mini` | 复杂推理、代码生成 |
| Anthropic | `claude-3.5-sonnet` | 长文本分析、安全性要求高 |

### ICP 匹配评分 Prompt

**模型**：`qwen-max` / `kimi-k2.5`（优先国内模型）/ `gpt-4o`（备选）

**强制设置**：`response_format: { "type": "json_object" }`

**系统提示词**：

```
你是一位专业的 B2B 外贸 ICP 匹配分析师。请根据企业的原始数据（海关记录、展会记录、企业信息）与客户的 ICP 画像，计算供需匹配评分。
严格基于输入的原始数据进行分析，不得捏造任何不存在于原始数据中的信息。

## 输出要求
请以 JSON 格式输出评分结果，必须包含以下字段：
- match_score: 0-100 的整数加权总分
- dimension_scores: 各维度原始分（需求匹配度、购买能力、采购意向、活跃意向）
- reasoning: 评分理由（100字以内，必须基于原始数据）

## 零捏造原则
- reasoning 中的每一条理由必须可追溯至输入的原始数据
- 不得捏造企业的海关记录、展会记录或其他业务行为
- 评分必须在 0-100 范围内，超出则强制截断
```

**用户提示词模板**：

```
请根据以下信息计算 ICP 匹配评分：

[企业原始数据]
{{enterprise_raw_data}}
[/企业原始数据]

[ICP 画像]
{{icp_profile}}
[/ICP 画像]

[四维权重]
需求匹配度: {{weight_demand}}
购买能力: {{weight_ability}}
采购意向: {{weight_intent}}
活跃意向: {{weight_activity}}
[/四维权重]

请严格按照系统提示词中的 JSON Schema 返回结果。
```

### AI 推荐理由 Prompt

**模型**：`qwen-max` / `kimi-k2.5`（优先国内模型）/ `gpt-4o`（备选）

**强制设置**：`response_format: { "type": "json_object" }`

**系统提示词**：

```
你是一位专业的外贸获客顾问。请根据企业的原始数据与 ICP 画像，为 B2B 销售团队生成一条推荐理由，帮助判断该企业是否为优质潜在客户。
严格基于输入的原始数据撰写推荐理由，不得捏造任何数据。

## 输出要求
请以 JSON 格式输出推荐理由，必须包含以下字段：
- ai_reasoning: 推荐理由文本（50-500字），必须包含以下维度中已有数据的说明：
  1. 企业与客户产品的供需关系
  2. 企业资质与规模印证
  3. 活跃度信号（展会、海关等）
  4. 综合判断与转化率预估
- confidence: 置信度（high/medium/low），反映推荐理由的可信程度

## 零捏造原则
- ai_reasoning 中提及的每一条信息必须存在于原始数据中
- 不得捏造企业的展会参与记录、海关进出口记录或其他商业行为
- 如原始数据不足某个维度，该维度可不提及，不得虚构

## 超长截断策略
- 超过 500 字时，截断至 500 字并在末尾添加"..."
- 优先保留前三维度内容
```

**用户提示词模板**：

```
请根据以下信息生成 AI 推荐理由：

[企业名称]
{{company_name}}
[/企业名称]

[企业原始数据]
{{enterprise_raw_data}}
[/企业原始数据]

[ICP 画像]
{{icp_profile}}
[/ICP 画像]

[匹配评分]
{{match_score}}
[/匹配评分]

请严格按照系统提示词中的 JSON Schema 返回结果。
```
