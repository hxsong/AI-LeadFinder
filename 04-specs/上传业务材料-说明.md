## 数据契约

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明 |
|-------------|-----------|------|----------|------|
| 文件名称 | `file_name` | `string` | 用户输入 | 用户选择文件时读取，展示在文件列表中 |
| 文件大小 | `file_size` | `number` | 用户输入 | 前端字节数，用于展示和校验（最大 5MB） |
| 文件类型 | `file_type` | `string` | 用户输入 | MIME 类型，用于文件图标展示和格式校验 |
| 存储 URL | `storage_url` | `string` | 接口返回 | AWS S3 返回的文件访问地址，服务端生成预签名 URL |
| 上传状态 | `upload_status` | `string` | 接口返回 | pending / uploading / completed / failed |
| 材料 ID | `material_id` | `string` | 接口返回 | 系统生成唯一 ID，格式：MAT-{timestamp}-{random} |
| BOT ID | `bot_id` | `string` | 路由参数传入 | 从 URL 参数获取，关联到当前 BOT |
| 文件列表 | `files` | `array` | 本地状态 | 前端维护的已上传文件数组，最多 3 个 |

**数据来源深度说明**：

| 前端展示字段 | 后端字段名 | 类型 | 来源/存储 | 说明（含服务端处理逻辑） |
|-------------|-----------|------|----------|------------------------|
| 存储 URL | `storage_url` | `string` | 接口返回 | 服务端接收文件后上传至 AWS S3，存储路径格式：`materials/{bot_id}/{material_id}/{file_name}`，返回预签名访问 URL（有效期 7 天），同时将文件元数据写入 `bot_materials` 表 |
| 材料 ID | `material_id` | `string` | 系统生成 | 服务端使用 Snowflake 算法生成唯一 ID，格式：MAT-YYYYMMDD-XXXXX，确保全局唯一性 |
| 上传状态 | `upload_status` | `string` | 接口返回 | 服务端根据 S3 上传结果返回：pending（待上传）、uploading（上传中）、completed（已完成）、failed（失败） |

## 页面流转

### 入口

| 来源页面 | 触发条件 | 携带参数 |
|---------|---------|---------|
| 业务初始化.html | 用户点击"上传业务材料"卡片 | `bot_id` |

**入口前置条件**：
- 用户已在业务初始化页面选择"上传业务材料"作为数据来源
- URL 中包含有效的 `bot_id` 参数
- 用户已登录（携带有效 tenant_token）

### 出口

| 目标页面 | 触发条件 | 携带数据 |
|---------|---------|---------|
| AI业务分析中.html | 用户点击"提交分析"按钮且所有文件上传成功 | `bot_id`, `task_id` |
| 业务初始化.html | 用户点击"返回上一步"按钮 | - |

### 页面状态

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 空态 | 未上传任何文件 | 页面初始化或所有文件被删除 | 用户选择文件开始上传 |
| 拖拽悬停 | 文件拖入上传区域 | 用户拖拽文件至上传区域 | 用户放开文件或移出区域 |
| 上传中 | 文件正在上传 | 用户选择文件并通过校验 | 上传完成（成功或失败） |
| 上传完成 | 文件上传成功 | 服务端返回上传成功 | 用户删除文件或提交分析 |
| 上传失败 | 文件上传失败 | 格式校验失败或网络错误 | 用户删除失败文件重新上传 |
| 满额 | 已上传 3 个文件 | 成功上传第 3 个文件 | 用户删除文件 |

**状态流转图**：

```
[空态] --(点击上传/拖拽文件)--> [上传中]
[空态] --(拖拽悬停)--> [拖拽悬停] --(放开文件)--> [上传中]
[上传中] --(上传成功)--> [上传完成]
[上传中] --(上传失败)--> [上传失败]
[上传完成] --(点击提交分析)--> [跳转至AI业务分析中]
[上传完成] --(已达3个文件)--> [满额]
[上传完成] --(删除文件)--> [空态] (仅剩1个文件时)
[上传失败] --(删除文件)--> [空态] (无其他文件时)
[满额] --(删除文件)--> [上传完成]
```

## 业务处理逻辑

### 场景一：文件上传（点击方式）

**触发条件**：用户点击上传区域触发文件选择器。

**执行逻辑**：

1. 用户点击上传区域，系统打开系统文件选择器
2. 用户选中文件后，前端进行本地校验：
   - 校验文件格式：PDF、DOC、DOCX、MD、TXT
   - 校验文件大小：单文件最大 5MB
   - 校验文件数量：最多 3 个文件
3. 校验通过后，将文件添加到上传队列，状态变更为"上传中"
4. 前端调用 `/api/v1/materials/upload` 接口，使用 multipart/form-data 格式上传
5. 上传过程中显示进度条动画
6. 服务端接收文件后：
   - 再次校验文件格式和大小
   - 生成 material_id
   - 上传至 AWS S3
   - 写入 bot_materials 表
   - 返回文件元数据
7. 前端接收响应后，状态变更为"上传完成"，显示文件卡片

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 文件格式不支持 | 前端拦截，显示错误状态，Toast 提示「不支持的文件格式」 |
| 文件超过 5MB | 前端拦截，显示错误状态，Toast 提示「文件大小超出限制」 |
| 已达 3 个文件上限 | 前端禁用上传区域，Toast 提示「已上传 3 个文件，达上传上限」 |
| 网络中断 | 上传失败，显示重试按钮 |
| 服务端返回错误 | 根据错误码显示对应 Toast 提示 |

**状态变更**：`空态` → `上传中` → `上传完成`

**输出**：文件进入上传队列，显示在文件列表中。

---

### 场景二：文件上传（拖拽方式）

**触发条件**：用户将文件拖拽至上传区域。

**执行逻辑**：

1. 用户拖拽文件进入上传区域，前端监听 `dragover` 事件
2. 上传区域显示蓝色边框 + 渐变背景（拖拽悬停状态）
3. 用户放开文件，触发 `drop` 事件
4. 前端获取 `dataTransfer.files` 中的文件列表
5. 后续流程同点击方式：本地校验 → 添加到队列 → 调用上传接口 → 显示结果

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 拖拽非文件对象 | 不响应，保持空态 |
| 拖拽多个文件 | 逐个校验并处理，最多取前 3 个有效文件 |
| 拖拽时移出区域 | 恢复上传区域默认样式 |

**状态变更**：`空态` → `拖拽悬停` → `上传中` → `上传完成`

**输出**：拖拽文件进入上传队列。

---

### 场景三：多文件上传

**触发条件**：用户一次选择多个文件或多次选择文件。

**执行逻辑**：

1. 用户一次选择多个文件（通过文件选择器的多选功能）
2. 前端遍历文件列表，逐个进行本地校验
3. 每个文件独立验证格式和大小，任一文件验证失败不影响其他文件
4. 校验通过的文件并行发起上传请求
5. 每个文件独立显示上传进度
6. 上传完成后，文件按上传完成顺序显示在列表中
7. 达到 3 个文件上限后，上传区域禁用

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 选择超过 3 个文件 | 只取前 3 个有效文件，超出部分忽略 |
| 部分文件校验失败 | 失败的文件显示错误状态，成功的文件继续上传 |
| 已达上限后继续上传 | 前端拦截，Toast 提示「已达上传上限」 |

**状态变更**：`空态` → `上传中` → `上传完成` / `满额`

**输出**：多个文件并行上传，显示在文件列表中。

---

### 场景四：删除已上传文件

**触发条件**：用户点击文件卡片上的删除按钮。

**执行逻辑**：

1. 用户点击删除按钮
2. 前端从本地 `uploadedFiles` 数组中移除对应文件
3. 重新渲染文件列表
4. 如果删除后文件数量 < 3，解除上传区域禁用状态
5. 如果删除后无文件，切换至空态
6. 更新提交按钮状态（无文件时禁用）

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 删除最后一个文件 | 切换至空态，禁用提交按钮 |
| 删除后从满额恢复 | 解除上传区域禁用，显示正常上传引导 |

**状态变更**：`上传完成` / `满额` → `上传完成` / `空态`

**输出**：文件从列表中移除。

---

### 场景五：提交分析任务

**触发条件**：用户点击"提交分析"按钮。

**执行逻辑**：

1. 用户点击"提交分析"按钮
2. 前端校验：
   - 至少上传了 1 个文件
   - 所有文件状态均为 completed
3. 按钮进入 loading 状态，显示"上传中..."
4. 前端调用 `/api/v1/materials/submit` 接口
5. 服务端校验：
   - 校验 bot_id 有效性
   - 校验 material_ids 对应的文件是否都存在且状态为 completed
   - 生成 task_id
   - 创建分析任务记录
   - 异步触发 AI 分析流程（发送消息队列）
6. 服务端返回 task_id 和跳转地址
7. 前端按钮显示"上传成功"，800ms 后跳转至 AI业务分析中页面

**边界条件**：

| 条件 | 系统行为 |
|------|---------|
| 未上传文件 | 提交按钮禁用，无法点击 |
| 有文件上传失败 | Toast 提示「请先删除上传失败的文件」 |
| 网络超时 | Toast 提示「网络连接异常，请检查网络后重试」 |
| 服务端返回错误 | 根据错误码显示对应 Toast 提示，按钮恢复可点击状态 |

**状态变更**：`上传完成` → 跳转至 AI业务分析中

**输出**：分析任务创建成功，跳转至分析处理页。

## 技术处理逻辑

### 数据加工规则

| 前端字段 | 加工类型 | 技术处理逻辑 | 存储位置 |
|---------|---------|--------------|---------|
| 文件存储 | 文件上传 | 1. 接收前端 multipart/form-data 请求<br>2. 校验文件格式（PDF/DOC/DOCX/MD/TXT）和大小（≤5MB）<br>3. 生成 material_id（Snowflake 算法）<br>4. 上传至 AWS S3，路径：`materials/{bot_id}/{material_id}/{file_name}`<br>5. 生成预签名 URL（7天有效期）<br>6. 写入 `bot_materials` 表 | AWS S3 + MySQL `bot_materials` 表 |
| 提交分析 | 任务创建 | 1. 校验至少上传 1 个文件<br>2. 校验所有文件状态为 completed<br>3. 生成 task_id（Snowflake 算法）<br>4. 创建分析任务记录，状态为 processing<br>5. 异步触发 AI 分析流程（P08 文本提取）<br>6. 返回 task_id 和跳转地址 | MySQL `analysis_tasks` 表 |

### 数据表结构

**bot_materials 表**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| material_id | varchar(32) | 主键，材料唯一 ID |
| bot_id | varchar(32) | 外键，关联 BOT |
| file_name | varchar(255) | 原始文件名 |
| file_size | bigint | 文件大小（字节） |
| file_type | varchar(100) | MIME 类型 |
| storage_url | varchar(500) | S3 存储路径 |
| upload_status | tinyint | 0-pending, 1-completed, 2-failed |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**analysis_tasks 表**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| task_id | varchar(32) | 主键，任务唯一 ID |
| bot_id | varchar(32) | 外键，关联 BOT |
| material_ids | json | 关联的材料 ID 列表 |
| status | varchar(20) | pending/processing/completed/failed |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

## 数据流

### API 一：上传业务材料

```
POST /api/v1/materials/upload
```

**请求头**：`Authorization: Bearer {tenant_token}`
**Content-Type**：`multipart/form-data`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 上传文件（PDF/WORD/MD/TXT） |
| bot_id | string | 是 | BOT ID |

**响应结构**：

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "material_id": "MAT-20240525-00123",
    "file_name": "产品手册.pdf",
    "file_size": 2097152,
    "file_type": "application/pdf",
    "storage_url": "https://s3.amazonaws.com/bucket/materials/BOT-001/MAT-20240525-00123/产品手册.pdf",
    "upload_status": "completed"
  }
}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| material_id | 材料 ID | string | 系统生成唯一 ID |
| file_name | 文件名称 | string | 原始文件名 |
| file_size | 文件大小 | number | 字节数 |
| file_type | 文件类型 | string | MIME 类型 |
| storage_url | 存储 URL | string | S3 预签名访问地址 |
| upload_status | 上传状态 | string | completed 表示上传成功 |

**响应码说明**：

| 响应码 | 说明 | 处理方式 |
|-------|------|---------|
| 200 | 上传成功 | 显示文件卡片，绿色勾选 |
| 400 | 文件格式不支持或超出大小限制 | Toast「不支持的文件格式，请上传 PDF/WORD/MD/TXT」或「文件大小超出限制，最大支持 5MB」 |
| 401 | 未登录 | 跳转登录页面，Toast「请先登录」 |
| 413 | 文件过大 | Toast「文件大小超出限制，最大支持 5MB」 |
| 415 | 不支持的文件类型 | Toast「不支持的文件格式，请上传 PDF/WORD/MD/TXT」 |
| 500 | 服务异常 | Toast「服务暂时不可用，请稍后再试」 |
| 504 | 网关超时 | Toast「网络连接异常，请检查网络后重试」 |

### API 二：提交分析任务

```
POST /api/v1/materials/submit
```

**请求头**：`Authorization: Bearer {tenant_token}`

**请求体**：

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |

```json
{
  "code": 200,
  "message": "分析任务提交成功",
  "data": {
    "task_id": "TASK-20240525-00123",
    "status": "processing",
    "redirect_to": "/ai-analysis.html?bot_id=BOT-001&task_id=TASK-20240525-00123"

}
```

**字段映射**：

| 响应字段 | 前端展示字段 | 类型 | 说明 |
|---------|-------------|------|------|
| task_id | 任务 ID | string | AI 分析任务 ID |
| status | 状态 | string | processing 表示处理中 |
| redirect_to | 跳转地址 | string | 跳转至 AI业务分析中页面 |

**响应码说明**：

| 响应码 | 说明 | 处理方式 |
|-------|------|---------|
| 200 | 成功 | 跳转至 AI业务分析中页面 |
| 400 | 参数错误 | Toast「请至少上传 1 个文件」 |
| 401 | 未登录 | 跳转登录页面 |
| 500 | 提交失败 | Toast「分析任务提交失败，请稍后重试」 |

## 异常兜底

| 异常场景 | 检测方式 | 后端处理 | 前端表现 |
|---------|---------|---------|---------|
| 文件格式不支持 | 前端 MIME 类型校验 + 后端二次校验 | 返回 400/415 错误码 | 错误提示，Toast「不支持的文件格式，请上传 PDF/WORD/MD/TXT」 |
| 文件超过 5MB | 前端文件大小校验 + 后端二次校验 | 返回 400/413 错误码 | 错误提示，Toast「文件大小超出限制，最大支持 5MB」 |
| 上传数量已达上限 | 前端文件计数校验 | 前端拦截，不上传 | 上传区域禁用，Toast「已上传 3 个文件，达上传上限」 |
| 网络超时 | 请求超时检测（30s） | 返回 504 | Toast「网络连接异常，请检查网络后重试」 |
| 服务不可用 | code 500 | 返回 500 | 全局错误处理，Toast「服务暂时不可用，请稍后再试」 |
| 未登录 | Token 校验失败 | 返回 401 | 跳转登录页面，Toast「请先登录」 |
| S3 上传失败 | S3 SDK 异常捕获 | 返回 500，记录错误日志 | 文件显示上传失败状态，可重试 |
| 数据库写入失败 | SQL 异常捕获 | 返回 500，记录错误日志 | Toast「上传失败，请稍后重试」 |
| 分析任务提交失败 | 数据库写入异常 | 返回 500，记录错误日志 | Toast「分析任务提交失败，请稍后重试」 |
| 无效 BOT ID | 数据库查询不存在 | 返回 400 | Toast「参数错误，请刷新页面重试」 |
