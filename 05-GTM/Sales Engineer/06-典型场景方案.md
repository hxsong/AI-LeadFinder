# AI LeadFinder 典型场景方案

## 场景方案选择指南

| 客户痛点 | 推荐方案 | 核心功能 |
|----------|----------|----------|
| 新市场没客户 | 场景一 | AI 搜客 + 海关数据 |
| 独立站没转化 | 场景二 | 访客识别 + 行为分析 |
| 销售效率低 | 场景三 | 自动评分 + 线索管理 |
| 多业务难管理 | 场景四 | 多 BOT + 数据隔离 |
| 展会效果差 | 场景五 | 展前搜客 + 展后跟进 |

### 方案组合建议

| 客户类型 | 推荐组合 | 预期效果 |
|----------|----------|----------|
| 初创外贸企业 | 场景一 + 场景二 | 快速获客 + 流量转化 |
| 成长型企业 | 场景一 + 场景二 + 场景三 | 全面获客 + 销售赋能 |
| 集团企业 | 场景四 + 场景一/二/三 | 统一管理 + 业务定制 |
| 展会依赖型企业 | 场景五 + 场景一 | 展会增效 + 日常获客 |

## 场景二：独立站流量转化

### 客户背景

| 维度 | 描述 |
|------|------|
| **客户类型** | B2B 工业品企业 |
| **当前状态** | 独立站月访问量 5000+，但转化率 < 0.5% |
| **核心痛点** | 不知道访客是谁，高价值访客流失 |
| **目标** | 识别访客企业，转化高意向访客 |

### 解决方案

```
┌─────────────────────────────────────────────────────────────┐
│                  AI LeadFinder 独立站转化方案                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  阶段一：探针部署 (1天)                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 选择核心主站（企业官网）                            │   │
│  │ • 复制 JS 探针代码                                    │   │
│  │ • 嵌入网站 <head> 标签                                │   │
│  │ • 验证数据上报正常                                    │   │
│  │ • 探针体积 <10KB，对网站性能影响 <1%                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  阶段二：访客识别 - 动态数据获客 (实时)                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • IP 地址反查企业信息                                 │   │
│  │ • 匹配多源企业数据库（工商、海关、展会）              │   │
│  │ • 记录访客实时行为（页面、停留时长、访问频次）        │   │
│  │ • 识别率：15-25% 的访客可识别企业身份                 │   │
│  │ • 价值：实时捕捉客户意向，判断最佳跟进时机            │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  阶段三：行为分析 (自动)                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 访问深度分析：看了哪些页面、停留多久                │   │
│  │ • 实时意向识别：产品页、价格页、联系页访问            │   │
│  │ • 四维评分：需求匹配+购买能力+采购意向+活跃意向       │   │
│  │ • 高意向标记：多次访问、深度浏览、价格页停留          │   │
│  │ • AI 洞察：复制技术参数=做竞品对比表；表单弃填=有顾虑 │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  阶段四：访客转化                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 高意向访客自动转线索                                │   │
│  │ • 消耗 Credits 揭秘企业身份（3 Credits/IP）           │   │
│  │ • 联系人挖掘：获取访客企业关键人邮箱                  │   │
│  │ • 销售基于访客行为数据主动跟进                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 关键效果指标

| 指标 | 使用前 | 使用后 |
|------|--------|--------|
| 访客识别率 | < 5% | 15-25% |
| 可识别访客数 | 250/月 | 750-1250/月 |
| 高意向访客 | 未知 | 约 20% |
| 转化率 | < 0.5% | 3-5% |

## 场景三：销售团队赋能

### 客户背景

| 维度 | 描述 |
|------|------|
| **客户类型** | 贸易公司 |
| **当前状态** | 销售团队 15 人，人均月产出 8 个有效线索 |
| **核心痛点** | 销售 70% 时间花在找客户，线索质量差 |
| **目标** | 提升销售效率，让销售专注跟进高质量客户 |

## 场景五：展会营销增效

### 客户背景

| 维度 | 描述 |
|------|------|

│  │ • 联系人挖掘：获取关键人邮箱                          │   │

| 单场展会获客成本 | 5-20 万 | 3-12 万 |
