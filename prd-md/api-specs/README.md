## 1. 概述

本文档汇总 AI-LeadFinder 系统接入的所有第三方 API 服务，提供统一的服务索引、调用流程说明和接入规范。各 API 详细规格请参见对应子文档。

## 2. 第三方服务总览

| 服务名称 | 用途 | 调用时机 | 关联 PRD 节点 | 成本 |
|---------|------|---------|--------------|------|
| 出海派海关 API | 获取企业进出口记录，判断采购意向 | 搜客执行中（P18） | P12-P21 | 5 Credits/批（100域名） |
| 10Times 展会 API | 获取企业展会参展记录，判断活跃意向 | 搜客执行中（P18） | P12-P21 | 3 Credits/批（100域名） |
| Apollo.io API | 获取企业联系人数据（邮箱/电话/社媒） | 线索挖掘（P24/P42） | P22-P27, P39-P46 | 2 Credits/邮箱 + 1 Credit/社媒 |
| Google Maps API | 获取企业地理位置和基础信息 | 搜客执行中（P18） | P12-P21 | 2 Credits/次（20条） |
| Import Genius API | 获取进口商数据 | 搜客执行中（补充） | P12-P21 | 视调用量计费 |
| IPInfo API | IP 地址反查企业身份 | 动态访客深度挖掘（P41） | P39-P46 | 固定 5 Credits/次 |
| Jina Reader API | 抓取企业官网正文内容 | 搜客执行中富化（P18） | P12-P21 | 1 Credit/域名 |

## 3. API 调用流程

### 静态数据获客（P12-P21）

```
搜客策略生成（P13）
    ↓
按策略调用 API 序列：
  ① Google Maps API → 获取企业基础列表
  ② 出海派海关 API → 批量验证进出口记录
  ③ 10Times 展会 API → 批量验证展会记录
  ④ Jina Reader API → 富化官网内容（可选）
    ↓
四维评分（P19）
    ↓
返回企业数据分析结果列表（P20）
```

### 动态数据获客（P28-P46）

```
访客行为上报（P35）
    ↓
行为意向评分（P36，仅规则评分，不调用外部 API）
    ↓
访客雷达列表展示（P38）
    ↓
用户触发深度挖掘（P39）
    ↓
  ├─ UTM 标识命中 → 直接关联已有静态线索
  └─ 无 UTM → IPInfo API → 企业身份解析
    ↓
Apollo API → 挖掘联系人（P42-P43）
    ↓
动态线索入库（P45）
```

## 4. API 调用顺序与依赖

- Google Maps API 为起手数据源（除非策略指定优先海关数据）
- 出海派和 10Times 均为批量验证 API，应攒批后一次性调用（100 域名/批），而非逐个调用
- Jina Reader API 在有 URL 的企业上使用，无 URL 的跳过
- Apollo API 在联系人挖掘环节调用，独立于搜客执行流

## 8. API 依赖关系说明

### 8.1 静态数据获客链路（P12-P21）

```
[搜客策略生成] → Google Maps Places API → 出海派海关 API（可选） → 10Times 展会 API（可选） → Import Genius API（可选） → Jina Reader API（可选） → [四维评分] → [搜客结果列表 P20]
```

## 9. 统一错误码规范

### 9.1 标准 HTTP 状态码

| 状态码 | 含义 | 处理建议 |
|--------|------|---------|
| 200 OK | 请求成功 | 解析响应数据 |
| 400 Bad Request | 请求参数错误 | 检查参数格式/必填项，返回友好提示 |
| 401 Unauthorized | 认证失败 | 检查 API Key 是否正确/未过期 |
| 402 Payment Required | Credits 不足 | 提示用户充值或减少调用量 |
| 403 Forbidden | 无权限 | 检查账户权限或配额限制 |
| 404 Not Found | 资源不存在 | 跳过该记录，记录日志 |
| 429 Too Many Requests | 请求超限（限流） | 按 Retry-After 延迟重试 |
| 500 Internal Server Error | 服务端错误 | 记录日志，触发告警，延时重试 |

### 9.2 各 API 特有错误码

**出海派海关 API**
| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 1001 | 账户余额不足 | 停止调用，提示充值 |
| 1002 | 域名格式错误 | 过滤无效域名后重试 |
| 1003 | 批次超限（>100） | 自动拆分为多个批次 |
| 1004 | 无查询结果 | 返回空数据，继续流程 |

**Apollo.io API**

| 401 | API Key 无效 | 检查配置 |
| 429 | 查询频率超限 | 降低并发，延时重试 |
| 404 | 未找到联系人 | 返回空列表 |
| 500 | 服务异常 | 重试最多 3 次 |

**Google Maps Places API**
| 错误码 | 含义 | 处理方式 |

| INVALID_REQUEST | 请求格式错误 | 检查 query 参数 |
| NOT_FOUND | 未找到地点 | 跳过该记录 |

**IPInfo API**

**Jina Reader API**

|--------|------|---------|

**Import Genius API**

**10Times 展会 API**
| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 401 | 认证失败 | 检查 API Key |

| 404 | 未找到展会/企业记录 | 返回空数据 |

### 9.3 错误处理通用原则

**幂等性**
- GET 类查询操作天然幂等，可安全重试
- 写操作（如线索入库）需确保幂等键（trace_id）唯一，避免重复写入

**重试策略**
- 429 响应：读取 `Retry-After` 头，无头时使用指数退避（1s → 2s → 4s → 8s，最大 60s）
- 500 响应：最多重试 3 次，间隔 2s/4s/8s
- 400 响应：不重试，修正请求后重试
- 所有重试需记录日志，包含原始请求和响应

**Credits 不足 402 处理**
- 检测到 402 立即停止当前批次调用
- 通知用户当前消耗量和剩余额度
- 提供"继续执行（需充值）"和"暂停"两个选项
- 记录本次已消耗 Credits 用于后续对账

**日志记录**
- 每一次 API 调用记录：时间戳、API 名称、请求参数（脱敏）、响应状态、Credits 消耗、耗时（ms）
- 错误日志包含完整堆栈和响应体（前 500 字符）

## 10. API 限流说明

### 10.1 各 API 限流规则

| API | 限流维度 | 限制值 | 超限行为 |
|-----|---------|--------|---------|
| 出海派海关 API | 每天/账户 | 视套餐而定（标准套餐约 5000 次/天） | 返回 402 或 429，次日恢复 |
| Apollo.io API | 每分钟/Key | 约 200 次/分钟 | 返回 429，需退避重试 |
| Google Maps Places API | 每天/项目 | 标准套餐 100,000 次/天 | 返回 429，按配额管控 |
| IPInfo API | 每月/Key | 按套餐（标准 50,000 次/月） | 超出后降级或停止 |
| Jina Reader API | 每分钟/IP | 约 60 次/分钟 | 返回 429，延时重试 |
| Import Genius API | 每天/账户 | 视套餐而定 | 返回 429 或 403 |
| 10Times 展会 API | 每小时/IP | 约 100 次/小时 | 返回 429，延时重试 |

### 10.2 全局限流策略

**Credits 保护机制**
- 每次 API 调用前检查剩余 Credits 是否充足
- Credits < 10% 时发出告警（站内通知）
- Credits < 5% 时自动暂停非关键调用（仅保留四维评分）
- Credits = 0 时停止所有外部 API 调用

**并发控制**
- 单 API 并发数上限：5（Apollo 并发 3，其他 5）
- 全局并发数上限：15
- 超出上限的请求进入等待队列（FIFO）

**退避重试 429 策略**
```
首轮退避：1 秒
二轮退避：2 秒
三轮退避：4 秒
四轮退避：8 秒
超过四轮：记录失败，标记跳过，流程继续
```

**配额监控**
- 每小时导出各 API 调用次数报告
- 每日 23:00 核查当天消耗，预测次日是否触发限流
- 接近日配额 80% 时自动触发告警
