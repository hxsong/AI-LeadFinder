# AI-LeadFinder 第三方 API 集成规范

> **版本**：v1.0.0
> **状态**：N（初稿）
> **最后更新**：2026-04-10

---

## 1. 概述

本文档汇总 AI-LeadFinder 系统接入的所有第三方 API 服务，提供统一的服务索引、调用流程说明和接入规范。各 API 详细规格请参见对应子文档。

## 2. 第三方服务总览

| 服务名称 | 用途 | 调用时机 | 关联 PRD 节点 | 成本 |
|---------|------|---------|--------------|------|
| 出海派海关 API | 获取企业进出口记录，判断采购意向 | 搜客执行中（P18） | P12-P21 | 5 Credits/批（100域名） |
| 10Times 展会 API | 获取企业展会参展记录，判断活跃意向 | 搜客执行中（P18） | P12-P21 | 3 Credits/批（100域名） |
| Apollo.io API | 获取企业联系人数据（邮箱/电话/社媒） | 线索挖掘（P24/P42） | P22-P27, P39-P46 | 2 Credits/邮箱 + 1 Credit/社媒 |
| Google Maps API | 获取企业地理位置和基础信息 | 搜客执行中（P18） | P12-P21 | 2 Credits/次（20条） |
| Import Genius API | 获取进口商数据 | 搜客执行中（补充） | P12-P21 | 视调用量计费 |
| IPInfo API | IP 地址反查企业身份 | 动态访客深度挖掘（P41） | P39-P46 | 固定 5 Credits/次 |
| Jina Reader API | 抓取企业官网正文内容 | 搜客执行中富化（P18） | P12-P21 | 1 Credit/域名 |

## 3. API 调用流程

### 静态数据获客（P12-P21）

```
搜客策略生成（P13）
    ↓
按策略调用 API 序列：
  ① Google Maps API → 获取企业基础列表
  ② 出海派海关 API → 批量验证进出口记录
  ③ 10Times 展会 API → 批量验证展会记录
  ④ Jina Reader API → 富化官网内容（可选）
    ↓
四维评分（P19）
    ↓
返回企业数据分析结果列表（P20）
```

### 动态数据获客（P28-P46）

```
访客行为上报（P35）
    ↓
行为意向评分（P36，仅规则评分，不调用外部 API）
    ↓
访客雷达列表展示（P38）
    ↓
用户触发深度挖掘（P39）
    ↓
  ├─ UTM 标识命中 → 直接关联已有静态线索
  └─ 无 UTM → IPInfo API → 企业身份解析
    ↓
Apollo API → 挖掘联系人（P42-P43）
    ↓
动态线索入库（P45）
```

## 4. API 调用顺序与依赖

- Google Maps API 为起手数据源（除非策略指定优先海关数据）
- 出海派和 10Times 均为批量验证 API，应攒批后一次性调用（100 域名/批），而非逐个调用
- Jina Reader API 在有 URL 的企业上使用，无 URL 的跳过
- Apollo API 在联系人挖掘环节调用，独立于搜客执行流
