# 产品非功能性需求清单

> 本文档从用户体验角度定义 LeadFinder 产品的非功能性需求，所有指标均与功能说明文档保持一致。

***

## 一、页面响应性能

| 需求类型                  | 建议指标     | 上限指标     | 说明         |
| --------------------- | -------- | -------- | ---------- |
| 常规页面首屏加载（列表页/详情页/工作台） | ≤ 1.5 秒  | ≤ 2 秒    | 用户可接受的等待时间 |
| 列表分页 / 筛选 / 搜索操作      | ≤ 500 ms | ≤ 1 秒    | 交互操作需即时反馈  |
| 按钮点击响应（提交/确认/跳转）      | ≤ 300 ms | ≤ 500 ms | 用户感知为即时响应  |

***

## 二、异步任务性能

### 2.1 任务执行时长

| 需求类型    | 建议指标       | 上限指标                      | 说明                      |
| ------- | ---------- | ------------------------- | ----------------------- |
| AI 业务分析 | 1 分钟       | 5 分钟                      | 单任务最长执行时间 10 分钟         |
| 搜客执行    | 每个接口调用 10秒 | 按 API 调用次数预估，每次接口调用 1 分钟， | 接口调用超时自动跳过，任务不阻塞，确保最终完成 |
| 静态线索池挖掘 | 每个接口调用 10秒 | 按 API 调用次数预估，每次接口调用 1 分钟  | 接口调用超时自动跳过，任务不阻塞，确保最终完成 |
| 动态线索池挖掘 | 每个接口调用 10秒 | 按 API 调用次数预估，每次接口调用 1 分钟  | 接口调用超时自动跳过，任务不阻塞，确保最终完成 |

### 2.2 接口超时机制

| 需求类型     | 建议指标            | 上限指标 | 说明              |
| -------- | --------------- | ---- | --------------- |
| 单次接口调用超时 | 每个接口调用 10秒      | 1 分钟 | 单次 API 调用最大等待时间 |
| 超时处理     | 跳过本次调用，继续执行后续步骤 | -    | 确保任务不阻塞         |
| 任务阻塞保护   | 确保任务最终完成        | -    | 不会因单个接口失败而阻塞    |

| 同时动态线索池挖掘任务 | 5 个 / 单BOT | 5 个 / 单BOT | 单BOT同时最多 5 个挖掘中任务           |

## 四、数据保留策略

| 需求类型    | 建议指标   | 上限指标   | 说明            |

## 六、余额规则

| 需求类型   | 建议指标               | 上限指标       | 说明              |
| ------ | ------------------ | ---------- | --------------- |
| 余额可为负  | 允许余额为负数            | 无上限        | 允许余额为负数         |
| 操作限制   | 余额 > 0 时允许操作       | 余额 ≤ 0 时禁止 | 禁止启动任何需要消耗余额的操作 |
| 预估消耗检查 | 余额 ≥ 预估消耗          | 余额 ≥ 预估消耗  | 有预估消耗的场景需满足此条件  |
| 执行中不扣费 | 任务执行过程中不扣除 Credits | -          | 完成后按实际消耗结算      |
| 完成后结算  | 按实际消耗结算，允许余额为负     | -          | 任务完成后统一结算       |

***

## 七、错误处理与重试

### 7.1 异步任务失败处理

| 需求类型        | 建议指标          | 上限指标 | 说明        |
| ----------- | ------------- | ---- | --------- |
| 单个接口调用失败    | 跳过继续执行，记录错误日志 | -    | 不影响整体任务执行 |
| 任务整体失败      | 标记失败状态        | -    | 确保用户知情    |
| AI 业务分析失败重试 | 最多重试 3 次      | 3 次  | 超过需联系客服   |

### 7.2 用户提示

| 需求类型   | 建议指标                | 上限指标 | 说明          |
| ------ | ------------------- | ---- | ----------- |
| 任务超时   | 前端提示"任务超时，点击重试"     | -    | 用户可主动重试     |
| 余额不足   | 弹窗提示充值，阻止任务提交       | -    | 防止余额不足时启动任务 |
| 文件格式错误 | 弹窗提示"不支持的文件格式"      | -    | 引导用户上传正确格式  |
| 文件过大   | 弹窗提示"文件过大，最大支持 5MB" | -    | 引导用户压缩文件    |

## 八、JS 探针性能（客户独立站影响）

| 需求类型                 | 建议指标     | 上限指标     | 说明              |
| -------------------- | -------- | -------- | --------------- |
| 探针脚本体积（gzip 后）       | ≤ 10 KB  | ≤ 15 KB  | 减少对客户网站加载影响     |
| 探针初始化耗时              | ≤ 30 ms  | ≤ 50 ms  | 不阻塞页面渲染         |
| 对独立站 Lighthouse 分数影响 | 降幅 ≤ 2 分 | 降幅 ≤ 3 分 | 不可导致客户网站性能可感知下降 |
| 上报批次间隔               | 30 秒     | 30 秒     | 批量上报减少请求次数      |

> **底线原则**：探针不可导致客户独立站性能可感知下降。违规指标阻断发布。

***

## 九、可用性与数据可靠性

| 需求类型           | 建议指标      | 上限指标      | 说明         |
| -------------- | --------- | --------- | ---------- |
| 平台整体可用性（月度）    | ≥ 99.9%   | ≥ 99.5%   | 核心业务可用性保障  |
| Credits 账务一致性  | 100%（强一致） | 100%（强一致） | 余额操作必须准确无误 |
| 线索数据持久化成功率     | ≥ 99.99%  | ≥ 99.99%  | 确保线索数据不丢失  |
| 异步任务最终完成率（含重试） | ≥ 99%     | ≥ 98%     | 确保任务可靠执行   |

***

## 十、安全基线

| 需求类型          | 建议指标                  | 上限指标     | 说明       |
| ------------- | --------------------- | -------- | -------- |
| 全站强制 HTTPS    | TLS 1.3               | TLS 1.2+ | 确保数据传输安全 |
| 探针上报接口鉴权      | probe\_id + 域名白名单校验   | -        | 防止非法数据上报 |
| Credits 操作幂等性 | 相同 request\_id 不可重复扣费 | -        | 防止重复扣费   |
| 三方 API Key 存储 | 加密存储于密钥管理服务           | -        | 防止密钥泄露   |
| 租户数据隔离        | 不同租户数据严格隔离            | -        | 防止数据越权访问 |

***

## 十一、监控告警（最小集）

| 需求类型         | 建议指标               | 上限指标             | 说明            |
| ------------ | ------------------ | ---------------- | ------------- |
| API 错误率飙升    | > 3% 持续 3 分钟       | > 5% 持续 3 分钟     | Critical 级别告警 |
| 异步任务队列堆积     | > 1,000 条          | > 10,000 条       | Critical 级别告警 |
| Credits 余额为负 | 任意出现               | 任意出现             | Critical 级别告警 |
| 探针事件接收异常     | 错误率 > 0.5% 持续 5 分钟 | 错误率 > 1% 持续 5 分钟 | Critical 级别告警 |
