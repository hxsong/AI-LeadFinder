# 数据保留策略

> **版本**：v1.0.0
> **状态**：N（初稿）
> **作者**：产品团队
> **最后更新**：2026-04-10

---

## 1. 概述

本文档定义 AI-LeadFinder 各数据类型在系统中的保留时长、归档规则和清理策略，确保数据合规性的同时控制存储成本。

## 2. 数据分类保留策略

| 数据类型 | 保留时长 | 归档规则 | 清理规则 |
|---------|---------|---------|---------|
| 业务分析报告 | 永久 | - | 跟随租户账号，账号停用后不删除 |
| BOT 配置信息 | 永久 | - | 跟随租户账号 |
| 搜客记录 | 90 天 | 超期自动归档至冷存储 | 账号停用后保留 30 天后删除 |
| 访客行为日志（未挖掘） | 90 天 | 超期自动归档至冷存储 | 已挖掘访客永久保留，未挖掘超期后删除 |
| 已挖掘动态线索 | 永久 | - | 企业信息永久保留，联系人数据重新挖掘后覆盖 |
| 静态线索 | 永久 | - | 跟随租户账号 |
| 联系人数据 | 永久 | - | 重新挖掘后覆盖旧数据 |
| 用户操作日志 | 180 天 | 超期自动清理 | 用于审计追溯 |

## 3. 各模块数据生命周期

### 3.1 账号与资源管理（P01-P11）

| 数据 | 保留时长 | 说明 |
|------|---------|------|
| 租户账号信息 | 永久 | 账号停用后转为冻结状态，保留 2 年后视情况删除 |
| Credits 余额流水 | 永久 | 用于财务对账和审计 |
| BOT 实例配置 | 永久 | 每个 BOT 的独立配置数据 |

### 3.2 静态数据获客（P12-P27）

| 数据 | 保留时长 | 说明 |
|------|---------|------|
| 搜客任务记录 | 90 天 | 含任务配置、执行日志、结果摘要 |
| 搜客结果（企业列表） | 90 天 | 含匹配分、AI 理由、原始数据 |
| 静态线索 | 永久 | 进入线索池后永久保留 |
| 联系人数据 | 永久 | 有效期 90 天，超期需重新挖掘 |
| 导出记录 | 180 天 | 用于审计追溯 |

### 3.3 动态数据获客（P28-P46）

| 数据 | 保留时长 | 说明 |
|------|---------|------|
| 访客行为日志（原始） | 90 天 | 超期自动归档至冷存储 |
| 已挖掘访客数据 | 永久 | 挖掘后永久保留，含企业信息 |
| 动态线索 | 永久 | 进入线索池后永久保留 |
| 联系人数据 | 永久 | 有效期 90 天 |

## 4. 账号停用后的数据处理

| 数据类型 | 账号停用后处理 |
|---------|--------------|
| 业务分析报告 | 保留，可恢复 |
| 搜客记录 | 保留 30 天后删除 |
| 访客行为日志 | 保留 30 天后删除 |
| 静态/动态线索 | 保留，可恢复 |
| 联系人数据 | 保留，可恢复 |
| Credits 余额 | 清零，不可恢复 |

## 5. 归档与清理机制

- **自动归档**：距今超过 90 天的数据自动从热存储迁移至冷存储
- **自动清理**：超出保留期后，数据进入删除队列，每周执行一次批量清理
- **删除前通知**：距删除 7 天前通过邮件/短信通知租户管理员
- **数据导出**：支持用户在数据删除前自助导出（CSV/Excel 格式）

## 6. 合规说明

- 所有数据处理符合 GDPR 相关要求
- 访客行为数据在采集时已获得用户同意授权
- 联系人数据的使用需遵循各数据来源方的使用条款
