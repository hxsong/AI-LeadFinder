> **版本**：v1.0.0
> **状态**：N（初稿）
> **作者**：产品团队
> **最后更新**：2026-04-10

---

## 1. 概述

本文档定义 AI-LeadFinder 系统中所有前后端接口的统一响应码规范。所有接口必须遵循此响应码体系，确保前端 Toast 提示、错误处理和用户反馈的一致性。

## 2. 响应码定义

| 响应码 | 含义 | Toast 类型 | 说明 |
|--------|------|-----------|------|
| 200 | 操作成功 | success | 接口正常、业务成功 |
| 400 | 请求参数错误 | error | 前端传参有误，参数缺失或格式错误 |
| 401 | 未登录/过期 | error | Token 过期或无效，前端清除 Token 后跳转登录页 |
| 403 | 无权限 | warn | 当前用户无此操作权限 |
| 404 | 接口不存在 | error | 请求路径错误或资源不存在 |
| 500 | 服务器异常 | error | 后端代码异常或服务不可用 |
| 1001 | 业务提示 | info | 普通提示说明类信息，不阻塞操作 |
| 1002 | 业务警告 | warn | 风险提示类信息，可能影响操作结果 |
| 1003 | 加载中状态 | loading | 异步处理进行中，不弹出 Toast |

## 3. 统一响应格式

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

### 错误响应

```json
{
  "code": 400,
  "message": "请求参数错误：xxx",
  "data": null
}
```

## 4. 前端 Toast 显示规则

- Toast 显示位置：页面顶部居中
- Toast 自动消失时间：3 秒
- success/info 类型：自动消失
- error/warn 类型：需用户手动关闭或操作后消失

## 5. 401 统一跳转处理

所有接口返回 401 时，前端必须执行：

1. 清除本地存储的 Token
2. 弹出"登录已过期，请重新登录"提示
3. 跳转到登录页面

## 6. 各模块响应码使用场景

### 账号与资源管理（P01-P11）

- 开通/充值操作：200 成功 / 500 失败
- BOT 创建：200 成功 / 400 名称重复 / 500 失败
- 业务材料上传：1003 上传中 / 200 成功 / 400 格式错误

### 静态数据获客（P12-P27）

- 搜客策略生成：200 成功 / 1002 Credits 不足 / 500 失败
- 联系人挖掘：200 成功 / 1002 已达上限 / 500 失败

### 动态数据获客（P28-P46）

- 访客数据上报：200 成功（异步，不阻塞）
- 深度挖掘：200 成功 / 1002 Credits 不足 / 500 服务异常
