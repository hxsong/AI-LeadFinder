# Credits 余额规则

> **版本**：v1.0.0
> **状态**：N（初稿）
> **作者**：产品团队
> **最后更新**：2026-04-10

---

## 1. 概述

Credits（全局获客算力积分）是 AI-LeadFinder 的核心计费单位，采用"预冻结 + 后结算"模式，支撑 SaaS 订阅 + 用量付费的商业化混合模式。

## 2. 基础规则

### 2.1 余额允许为负

- 系统允许账户余额为负数，无下限
- 余额为负时系统不自动暂停账户

### 2.2 操作限制

| 条件 | 允许操作 | 禁止操作 |
|------|---------|---------|
| 余额 > 0 | 所有消耗操作 | - |
| 余额 = 0 | 浏览/查看（只读） | 搜客/挖掘等消耗操作 |
| 余额 < 0 | 浏览/查看（只读） | 所有消耗操作 |

### 2.3 预估消耗检查

- 有预估消耗的场景（如搜客、联系人挖掘）需满足：`余额 >= 预估消耗`
- 预估消耗不含 30% 动态溢价，实际扣费可能低于预估

## 3. 冻结与结算机制

### 3.1 冻结时机

- 用户确认发起消耗操作时（如点击"确认开始搜客"），立即冻结预估 Credits
- 冻结金额 = 预估消耗（含 30% 溢价）
- 冻结期间余额仍显示为可用余额，但扣除冻结部分后不足则无法发起新操作

### 3.2 执行中不扣费

- 任务执行过程中（搜客执行、联系人挖掘等）不扣除实际 Credits
- 所有 API 调用费用在任务完成后统一结算

### 3.3 完成结算

- 任务完成后，按实际 API 成功调用次数重新核算
- 实际消耗 = min(冻结金额, 真实消耗)
- 退还冻结差额：冻结金额 - 实际消耗，可为负数

### 3.4 结算示例

```
预估消耗（含30%溢价）= 200 Credits → 冻结 200 Credits
实际 API 消耗 = 150 Credits
退还差额 = 200 - 150 = 50 Credits（退回到余额）
净扣费 = 150 Credits
```

## 4. 各场景 Credits 消耗规则

### 4.1 搜客执行（P12-P21）

| API 服务 | 单次消耗 | 说明 |
|---------|---------|------|
| Google Maps API | 2 Credits/次 | 单次返回 20 条基础记录 |
| 出海派海关 API | 5 Credits/次 | 批量查询 100 个域名 |
| 10Times 展会 API | 3 Credits/次 | 批量查询 100 个域名 |
| Jina Reader API | 1 Credit/次 | 单域名网页解析 |

**预估公式**：`预估 Credits = Σ(API预计调用次数 × API单价) × 1.3（溢价）`

### 4.2 静态联系人挖掘（P22-P27）

| 操作 | 消耗 | 说明 |
|------|------|------|
| 挖掘任务创建 | 冻结 15 Credits | 预估固定值 |
| 每获取 1 个验证邮箱 | 2 Credits | 实际按成功数扣费 |
| 附加 LinkedIn 主页 | +1 Credit | 可选附加 |
| 附加 Twitter/FB | +1 Credit/个 | 可选附加 |

### 4.3 动态访客深度挖掘（P39-P46）

| 操作 | 消耗 | 说明 |
|------|------|------|
| IP 企业解析 | 5 Credits | 固定消耗，无论成功与否 |
| 每获取 1 个验证邮箱 | 2 Credits | 实际按成功数扣费 |
| 附加 LinkedIn 主页 | +1 Credit | 可选附加 |
| 附加 Twitter/FB | +1 Credit/个 | 可选附加 |

## 5. 充值与扩展

- 充值入口：全局导航栏 Credits 余额区 → "+购买算力"
- 扩展 BOT 席位：多 BOT 工作台 → "+扩展 BOT 席位"
- 充值和扩展均通过线下销售引导，不支持线上直接支付

## 6. 异常与边界

| 场景 | 余额状态 | 前端表现 | 后端处理 |
|------|---------|---------|---------|
| 发起搜客 | 余额 < 预估 | 按钮禁用 + 提示文案 | 不发起请求 |
| 发起挖掘 | 余额 ≤ 0 | 弹窗提示充值 | 不发起请求 |
| 任务进行中余额耗尽 | 允许继续执行 | - | 完成后结算，余额为负 |
| 余额不足提示 | 余额 = 0 或 < 预估 | Toast 提示"额度不足" | - |
