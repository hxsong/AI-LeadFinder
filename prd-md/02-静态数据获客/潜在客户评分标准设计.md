# 潜在客户评分标准设计

## 一、设计背景

### 1.1 核心问题

在 B2B 获客场景中，如何判断一个企业是否是潜在客户？传统的"企业相似度"方法存在根本性缺陷：

```
❌ 错误思路：找和我相似的企业
   - 轮胎厂找轮胎厂 → 同行，不是客户
   - 语义相似度高，但不是潜在客户

✅ 正确思路：找需要我产品的企业
   - 轮胎厂找汽修厂 → 下游客户
   - 语义相似度可能低，但是潜在客户
```

### 1.2 设计原则

**回归销售本质**：模拟真实销售判断潜在客户的逻辑

| 销售问题 | 对应维度 | 判断依据 |
|----------|----------|----------|
| "这个企业需要我的产品吗？" | 需求匹配度 | AI 模型供需关系判断 |
| "这个企业买得起吗？" | 购买能力 | 规模 + 地理范围 |
| "这个企业最近在采购吗？" | 采购意向 | 海关进口记录 |
| "这个企业活跃吗？" | 活跃意向 | 展会参展记录 |

## 二、四维评分框架

### 2.1 维度定义

| 维度 | 维度说明 | 数据来源 | 判断逻辑 |
|------|----------|----------|----------|
| **需求匹配度** | 企业是否处于客户下游 | AI 模型供需关系判断 | 供需关系映射 |
| **购买能力** | 企业是否有购买能力 | 地图数据 + 网络抓取 | 规模阈值判断 |
| **采购意向** | 企业是否有采购行为 | 海关数据 | 海关记录命中 |
| **活跃意向** | 企业是否市场活跃 | 展会数据 | 展会记录命中 |

### 2.2 动态权重生成

权重不是固定的，而是根据客户的业务类型动态生成。

#### 2.2.1 业务类型分类

系统首先识别客户的业务类型：

| 业务类型 | 特征描述 | 典型客户示例 |
|----------|----------|--------------|
| **外贸型** | 专注出口贸易，需要跨境采购客户 | 外贸公司、跨境电商服务商 |
| **生产型** | 自有工厂，需要分销渠道 | 制造商、品牌商 |
| **服务型** | 提供服务，需要终端客户 | 咨询公司、技术服务商 |
| **渠道型** | 分销代理，需要供应商 | 分销商、代理商 |

#### 2.2.2 权重生成规则

**规则一：外贸型企业权重配置**

外贸型企业的核心诉求是找到有跨境采购需求的客户，因此"采购意向"权重最高。

## 三、各维度评分细则

### 3.1 需求匹配度（AI 模型计算）

#### 核心逻辑

需求匹配度通过 AI 模型判断候选企业是否是客户的潜在客户。AI 模型基于客户业务信息和候选企业信息，推理供需关系并给出匹配度分数。

#### 统一信息维度框架

| 维度 | 客户企业信息 | 候选企业信息 |
|------|--------------|--------------|
| 企业名称 | 客户原始内容提取 | 三方数据内容提取 |
| 企业定位 | 客户原始内容提取 | 三方数据内容提取 |
| 业务类型 | 客户原始内容提取 | 三方数据内容提取 |
| 主营品类 | 客户原始内容提取 | 三方数据内容提取 |
| 核心优势 | 客户原始内容提取 | 三方数据内容提取 |
| 企业规模 | 客户原始内容提取 | 三方数据内容提取 |
| 供应链分析 | 客户原始内容提取 | - |
| 目标客户类型 | 客户原始内容提取 | - |
| 目标客户市场区域 | 客户原始内容提取 | - |
| 目标客户主营品类 | 客户原始内容提取 | - |
| 目标客户企业规模 | 客户原始内容提取 | - |

### 3.2 购买能力（规模和地理）

#### 核心逻辑

购买能力判断企业是否有足够的预算和规模进行采购。

#### 3.2.2 规模评分（满分 60 分）

| 规模指标 | 得分 | 判断依据 |
|----------|------|----------|
| 大型企业（500人以上） | 60 | 员工数、营收规模 |
| 中型企业（50-500人） | 45 | 员工数、营收规模 |
| 小型企业（10-50人） | 30 | 员工数、营收规模 |
| 微型企业（10人以下） | 15 | 员工数、营收规模 |
| 无数据 | 30 | 默认中等偏下 |

**规模数据来源**：
- 地图数据：部分地图 API 返回企业规模信息
- 网络抓取：从官网"关于我们"页面提取员工数
- 海关数据：进出口金额可间接反映企业规模

#### 地理评分（满分 40 分）

**地理匹配的定义**：

| 地理指标 | 得分 | 定义说明 |
|----------|------|----------|
| 在目标区域内 | 40 | 企业注册地址或主要经营地址在目标区域内 |
| 不在目标区域内 | 0 | 企业地址不在目标区域 |
| 无数据 | 20 | 默认中等 |

**目标区域的定义**：

目标区域由客户在搜客需求输入时指定，支持以下层级：

| 层级 | 示例 | 判断方式 |
|------|------|----------|
| 大洲级 | "欧洲"、"北美" | 企业地址在国家代码列表中 |
| 国家级 | "德国"、"美国" | 企业地址的国家代码匹配 |
| 区域级 | "加州"、"巴伐利亚州" | 企业地址的州/省匹配 |
| 城市级 | "慕尼黑"、"洛杉矶" | 企业地址的城市名匹配 |

**判断逻辑**：

```
Step 1: 获取客户指定的目标区域
目标区域 = "欧洲"

Step 2: 获取候选企业的地址信息
企业地址 = "Munich, Germany"

Step 3: 解析企业地址的国家代码
国家代码 = "DE"（德国）

Step 4: 判断是否在目标区域内
目标区域国家代码列表 = ["DE", "FR", "GB", "IT", "ES", "NL", "BE", "AT", "CH", "SE", ...]
"DE" in 目标区域国家代码列表 → True

结果：在目标区域内，得分 = 40
```

**数据来源**：
- 地图 API 返回的企业地址信息（包含国家代码、州/省、城市）
- 网络抓取的企业联系页面地址信息

#### 计算方法

```
购买能力得分 = (规模得分 × 0.6) + (地理得分 × 0.4)
```

### 3.3 采购意向（海关记录）

#### 核心逻辑

海关进口记录是企业有采购行为的直接证据。通过 HS Code 匹配判断企业是否采购过相关产品。

#### HS Code 匹配规则

| 相关程度 | 判断标准 | 示例 |
|----------|----------|------|
| 对口 HS Code | 前4位相同，或产品直接对应 | 6306.22 vs 6306.29 |
| 相关 HS Code | 前2位相同（同大类），或上下游关系 | 6306.22 vs 6302.91 |
| 不相关 HS Code | 前2位不同，且无上下游关系 | 6306.22 vs 8708.29 |

| 海关记录情况 | 得分 | 说明 |

## 四、综合评分计算

### 4.1 计算公式

```
潜在客户得分 = 需求匹配度 × W1 
             + 购买能力 × W2 
             + 采购意向 × W3 
             + 活跃意向 × W4

其中 W1 + W2 + W3 + W4 = 1.0
权重根据客户业务类型动态生成
```

## 五、数据可行性分析

### 5.1 各维度数据来源

| 维度 | 数据来源 | 数据覆盖率 | 数据可靠性 |
|------|----------|------------|------------|
| 需求匹配度 | 网络抓取 + AI 推断 | ~60% | 中等（依赖企业官网信息） |
| 购买能力 | 地图数据 + 网络抓取 | ~50% | 中等（规模数据可能不完整） |
| 采购意向 | 海关数据 | ~20% | 高（官方数据） |
| 活跃意向 | 展会数据 | ~15% | 高（官方数据） |

### 5.2 数据缺失处理

| 数据缺失情况 | 处理策略 |
|--------------|----------|
| 无海关数据 | 采购意向得分 = 0，但不排除该企业 |
| 无展会数据 | 活跃意向得分 = 0，但不排除该企业 |
| 无规模数据 | 购买能力得分 = 默认中等（30分） |
| 无官网数据 | 需求匹配度得分 = 仅基于企业类型判断 |

### 5.3 评分可信度标识

系统应在展示评分时，同时展示数据完整度：

## 七、总结

### 7.1 核心改进

| 改进点 | 原方案 | 新方案 |
|--------|--------|--------|
| 评分逻辑 | 企业相似度（语义匹配） | 供需关系（AI 模型判断） |
| 权重配置 | 固定权重 | 动态权重（根据业务类型） |
| 维度定义 | 企业类型、交易意向、活跃意向、地理规模 | 需求匹配度、购买能力、采购意向、活跃意向 |
| 数据处理 | 假设数据完整 | 明确数据缺失处理策略 |
| 知识依赖 | 需要行业知识图谱 | AI 模型内置行业知识 |

### 7.2 AI 模型计算需求匹配度的优势

| 优势 | 说明 |
|------|------|
| 无需维护知识图谱 | 大模型已内置行业知识，无需额外构建和维护 |
| 灵活适应新行业 | 不受预定义行业分类限制，可处理任意行业 |
| 语义理解能力强 | 能理解企业业务描述，判断供需关系 |
| 可解释性好 | AI 输出包含判断理由，便于用户理解 |

3. **数据补充**：探索更多数据源提高数据覆盖率
4. **成本优化**：优化批量计算策略，降低 AI 调用成本
5. **反馈学习**：收集客户反馈，持续优化评分准确性
