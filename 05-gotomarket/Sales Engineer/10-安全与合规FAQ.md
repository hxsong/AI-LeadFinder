# AI LeadFinder 安全与合规 FAQ

## 数据安全

### Q1：数据在传输和存储过程中如何保护？

**A**：多层安全防护体系：

**传输安全**：
- HTTPS/TLS 1.3 加密传输
- 证书 Pinning 防止中间人攻击

**存储安全**：
- AES-256 数据加密存储
- 数据库访问控制
- 敏感字段（邮箱、电话）独立加密

**访问控制**：
- RBAC 角色权限管理
- 双因素认证（2FA）
- 登录审计日志

**数据隔离**：
- 租户级数据隔离
- BOT 级数据隔离
- 数据库行级安全（RLS）

---

### Q2：探针采集访客数据是否安全？会泄露吗？

**A**：探针设计遵循"最小采集、最大安全"原则：

| 采集项 | 是否采集 | 说明 |
|--------|----------|------|
| 页面浏览 | ✅ | 公开页面访问记录 |
| 滚动深度 | ✅ | 阅读行为分析 |
| 悬停时长 | ✅ | 兴趣点识别 |
| 复制文本 | ✅ | 仅限网页公开文本 |
| 点击元素 | ✅ | 按钮/链接交互 |
| 表单输入内容 | ❌ | **绝不采集** |
| 个人信息 | ❌ | **绝不采集** |

**技术保障**：
- 探针异步加载，不阻塞页面
- 体积 < 10KB，性能影响 < 1%
- CDN 全球分发，99.9% 可用性
- 数据上报采用加密通道

---

### Q3：我们的业务数据和客户线索会不会被其他用户看到？

**A**：完全隔离，绝不可能。

- **租户级隔离**：每个企业拥有独立的数据库 schema
- **BOT 级隔离**：同一企业内的不同 BOT 数据也完全隔离
- **访问控制**：所有数据查询严格绑定当前用户的租户 ID 和 BOT ID
- **审计日志**：所有数据访问行为均有日志记录

## 隐私合规

### Q4：是否符合 GDPR（欧盟通用数据保护条例）？

**A**：完全符合 GDPR 要求。

| GDPR 要求 | AI LeadFinder 措施 |
|-----------|-------------------|
| 数据主体知情权 | 探针部署页面显示隐私声明 |
| 访问权 | 支持用户查询其个人数据 |
| 更正权 | 支持数据更正请求 |
| 删除权（被遗忘权） | 支持数据删除，90 天内彻底清除 |
| 数据处理协议（DPA） | 提供标准 DPA 模板 |
| 隐私影响评估（PIA） | 已完成系统级 PIA |
| 数据泄露通知 | 72 小时内通知受影响用户 |
| 欧盟数据本地化 | 企业版支持数据存储在欧盟区域 |

---

### Q5：访客数据的保留期限是多久？

**A**：按数据类型分级保留：

| 数据类型 | 保留期限 | 说明 |
|---------|----------|------|
| 搜客记录 | 90 天 | 自动清理 |
| 搜客结果 | 7 天 | 可导出备份 |
| 访客数据 | 90 天 | 超期归档 |
| 线索数据 | 永久 | 客户主动删除除外 |
| 联系人数据 | 90 天 | 超期需重新验证 |

**客户可控**：
- 随时导出全部数据
- 随时删除指定数据
- 企业版可自定义保留策略

---

### Q6：如果不再使用服务，数据如何处理？

**A**：支持完整的数据导出和删除：

1. **数据导出**：提供一键导出全部线索、访客记录、业务报告
2. **账户注销**：提交注销申请后，90 天内完成数据彻底清除
3. **备份清除**：注销 180 天后，所有备份数据永久删除

## 系统安全

**A**：99.9% SLA 保障。

**架构保障**：
- 多可用区部署

|----------|----------|------|

---

### Q9：如何防止 API 滥用和 Credits 盗刷？

**A**：多层防护机制：

- **频率限制**：单账号 API 调用频率受限
- **异常检测**：系统自动检测异常消耗模式
- **额度预警**：Credits 余额不足时自动提醒
- **操作审计**：所有 Credits 消耗均有明细记录
- **IP 白名单**：企业版支持 API 调用 IP 限制
