## 阶段一：业务建模（AI 理解您的业务）

**目标**：让 AI 理解企业业务，自动生成理想客户画像（ICP）和评分模型。

### 功能模块

| 功能           | 说明                 | 输入         | 输出             |
| ------------ | ------------------ | ---------- | -------------- |
| **业务材料上传**   | 上传 PDF、PPT、图片等企业资料 | 产品画册、公司介绍  | 结构化业务数据        |
| **独立站授权**    | 授权绑定企业独立站          | 网站域名/API   | 站点 SEO/产品/内容数据 |
| **AI 业务分析**  | 大模型深度分析企业定位        | 上传的材料/站点数据 | 业务分析报告         |
| **ICP 画像生成** | 自动生成理想客户画像         | 业务分析结果     | ICP 定义、评分权重    |

### 业务分析报告内容

- **企业介绍**：一句话定位、核心背书
- **主营产品清单**：产品类别、核心技术壁垒
- **行业供应链**：上游依赖、自身生态位、下游高潜客群
- **竞争优势**：5-8 个行业辨识标签（用于 AI 供需匹配）
- **ICP 建模**：四维评分权重配置（根据业务类型动态生成）

## 阶段二：静态数据获客（主动挖掘）

**目标**：基于多源数据全网搜寻潜在客户，生成静态线索。

### 功能模块

| 功能          | 说明                        | 核心价值   |
| ----------- | ------------------------- | ------ |
| **自然语言搜客**  | 用自然语言描述需求，AI 自动生成搜客策略     | 零门槛使用  |
| **AI 策略生成** | AI 规划 API 调用路径，预估费用       | 透明、高效  |
| **多源数据抓取**  | 海关 + 展会 + 地图 + 网页数据融合     | 数据全面   |
| **四维评分计算**  | 需求匹配 + 购买能力 + 采购意向 + 活跃意向 | 精准筛选   |
| **AI 判词生成** | 自动解释每个线索的推荐理由             | 降低判断成本 |
| **联系人挖掘**   | 获取关键决策人邮箱 + 社媒            | 直达决策层  |
| **静态线索池**   | 统一管理所有静态线索                | 集中管理   |

### 数据源覆盖

| 数据源  | 覆盖范围        | 更新频率 | 核心价值          |
| ---- | ----------- | ---- | ------------- |
| 海关数据 | 全球 100+ 国家  | 月度   | 识别有真实进出口记录的企业 |
| 展会数据 | 年度 5000+ 展会 | 实时   | 定位活跃的行业参与者    |
| 地图数据 | 全球企业 POI    | 实时   | 验证企业实体存在和规模   |
| 网页数据 | 全球企业网站      | 实时抓取 | 获取企业业务和产品信息   |

## 阶段三：动态数据获客（被动捕捉）

**目标**：追踪独立站访客行为，识别高意向采购者，生成动态线索。

### 功能模块

| 功能          | 说明                    | 核心价值       |
| ----------- | --------------------- | ---------- |
| **探针部署**    | JS 探针嵌入网站 `<head>` 标签 | 1 天完成部署    |
| **访客识别**    | IP 解析 + 多源数据库匹配       | 识别率 15-25% |
| **行为采集**    | 点击、滚动、复制、表单交互         | 微观行为洞察     |
| **AI 行为分析** | 推理访客采购心态和阶段           | 精准判断意向     |
| **动态评分**    | 基于行为特征打分（0-100）       | 实时筛选高意向    |
| **动态线索池**   | 统一管理动态线索              | 访客转化管理     |

### 探针技术特性

| 特性   | 规格           | 说明         |
| ---- | ------------ | ---------- |
| 加载方式 | 异步加载         | 不阻塞页面渲染    |
| 体积   | < 10KB（gzip） | 极轻量        |
| 性能影响 | < 1%         | 对页面加载几乎无影响 |
| 合规性  | GDPR 标准      | 不采集表单输入内容  |
| 可用性  | 99.9%        | CDN 全球分发   |

## 多 BOT 管理体系

**目标**：支持集团型企业多业务线/多独立站的独立隔离运营。

```
                    ┌─────────────────┐
                    │   企业管理员     │
                    │  (全局资源看板)  │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              ▼              ▼
       ┌──────────┐   ┌──────────┐   ┌──────────┐
       │ BOT-业务A │   │ BOT-业务B │   │ BOT-业务C │
       │  独立站A  │   │  独立站B  │   │  独立站C  │
       └────┬─────┘   └────┬─────┘   └────┬─────┘
            │              │              │
    ┌───────┴───────┐ ┌────┴────┐  ┌─────┴──────┐
    │ • 独立 ICP    │ │ • 独立  │  │ • 独立     │
    │ • 独立线索池  │ │   ICP   │  │   ICP     │
    │ • 独立访客数据│ │ • 独立  │  │ • 独立     │
    │ • 数据隔离    │ │   线索池│  │   线索池  │
    └───────────────┘ └─────────┘  └──────────┘

       共享资源：Global Credits 资金池
```

**核心特性**：

- 每个 BOT 对应一个独立站或业务线
- BOT 间数据完全隔离（ICP、线索池、访客数据）
- 全局 Credits 资金池共享
- 统一管理，独立运营
