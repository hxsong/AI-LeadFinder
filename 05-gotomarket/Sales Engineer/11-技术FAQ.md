# AI LeadFinder 技术 FAQ

## 数据相关

### Q1：数据来源有哪些？覆盖范围如何？

**A**：AI LeadFinder 整合多维度数据源，形成**双引擎数据体系**：

**静态数据引擎（历史数据）**：

| 数据类型 | 来源                    | 更新频率 | 覆盖范围        | 核心价值          |
| ---- | --------------------- | ---- | ----------- | ------------- |
| 海关数据 | 官方进出口报关数据             | 月度更新 | 全球 100+ 国家  | 识别有真实进出口记录的企业 |
| 展会数据 | 全球主流展会参展商             | 实时更新 | 年度 5000+ 展会 | 定位活跃的行业参与者    |
| 地图数据 | Google Maps、Bing Maps | 实时更新 | 全球企业 POI    | 验证企业实体存在和规模   |
| 网页数据 | 企业官网公开信息              | 实时抓取 | 全球企业网站      | 获取企业业务和产品信息   |

**动态数据引擎（实时意向）**：

| 数据类型 | 来源                 | 更新频率 | 核心价值            |
| ---- | ------------------ | ---- | --------------- |
| 访客行为 | 独立站 JS 探针采集        | 实时   | 识别高意向采购者和最佳跟进时机 |
| 社媒数据 | LinkedIn、Twitter 等 | 实时更新 | 补充联系人信息和社交动态    |

**多源数据融合优势**：

- 静态历史数据完成初次触达和背景调研
- 动态实时行为判断客户意向和跟进时机
- 双引擎结合生成全景客户画像，显著提高线索质量和转化成功率

### Q2：数据准确性如何保证？

**A**：多层数据质量保证机制：

1. **数据源验证**：只接入权威数据源
2. **交叉验证**：多源数据交叉比对（如海关记录 vs 展会记录 vs 地图信息）

5. **联系人验证**：邮箱 SMTP 实时验证，确保可送达

**准确率指标**：

| 数据类型       | 准确率  |
| ---------- | ---- |
| 企业基础信息     | 95%+ |
| 海关数据       | 98%+ |

## AI 相关

### Q5：AI 业务建模是如何工作的？

**A**：AI 业务建模分为两个阶段：

**阶段 1：文本提取与结构化（P07-P08）**

- 解析上传文档（PDF/Word/TXT/图片）
- 提取关键信息：产品、服务、优势、客户群体
- 结构化存储为机器可读格式
- 使用模型：GPT-4o-mini / Gemini 1.5 Flash

**阶段 2：深度分析与 ICP 建模（P09）**

- 基于提取信息分析企业定位
- 识别业务类型（外贸/生产/服务/渠道）
- 生成理想客户画像（ICP）
- 输出四维评分权重配置
- 使用模型：Claude 3.5 Sonnet / GPT-4o

***

### Q6：四维评分模型是如何计算的？

**A**：

```
潜在客户得分 = 需求匹配度 × W1 
             + 购买能力 × W2 
             + 采购意向 × W3 
             + 活跃意向 × W4
```

<br />

**各维度计算逻辑**：

- **需求匹配度**：AI 模型判断候选企业是否是潜在客户（供需关系分析）
- **购买能力**：企业规模评分（60%）+ 地理评分（40%）
- **采购意向**：海关 HS Code 匹配度 × 时间衰减系数
- **活跃意向**：展会记录 × 时间衰减系数
