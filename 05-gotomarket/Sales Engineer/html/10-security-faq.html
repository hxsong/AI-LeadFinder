Data Security

数据安全

Q1：数据在传输和存储过程中如何保护？

多层安全防护体系：

传输安全：

  • HTTPS/TLS 1.3 加密传输
  • 证书 Pinning 防止中间人攻击

存储安全：

  • AES-256 数据加密存储
  • 数据库访问控制
  • 敏感字段（邮箱、电话）独立加密

访问控制：

  • RBAC 角色权限管理
  • 双因素认证（2FA）
  • 登录审计日志

数据隔离：

  • 租户级数据隔离
  • BOT 级数据隔离
  • 数据库行级安全（RLS）
数据保护

多层安全防护：传输安全、存储安全、访问控制、数据隔离。

Q2：探针采集访客数据是否安全？会泄露吗？

探针设计遵循"最小采集、最大安全"原则：

采集项 是否采集 说明
页面浏览 ✅ 公开页面访问记录
滚动深度 ✅ 阅读行为分析
悬停时长 ✅ 兴趣点识别
复制文本 ✅ 仅限网页公开文本
点击元素 ✅ 按钮/链接交互
表单输入内容 ❌ 绝不采集
个人信息 ❌ 绝不采集

技术保障：

  • 探针异步加载，不阻塞页面
  • 体积 < 10KB，性能影响 < 1%
  • CDN 全球分发，99.9% 可用性
  • 数据上报采用加密通道
探针安全

最小采集原则，不采集表单输入和个人信息。

Q3：我们的业务数据和客户线索会不会被其他用户看到？

完全隔离，绝不可能。

  • 租户级隔离：每个企业拥有独立的数据库 schema
  • BOT 级隔离：同一企业内的不同 BOT 数据也完全隔离
  • 访问控制：所有数据查询严格绑定当前用户的租户 ID 和 BOT ID
  • 审计日志：所有数据访问行为均有日志记录
数据隔离

租户级和BOT级双重隔离，严格访问控制。

Q5：访客数据的保留期限是多久？

按数据类型分级保留：

数据类型 保留期限 说明
搜客记录 90 天 自动清理
搜客结果 7 天 可导出备份
访客数据 90 天 超期归档
线索数据 永久 客户主动删除除外
联系人数据 90 天 超期需重新验证

客户可控：

  • 随时导出全部数据
  • 随时删除指定数据
  • 企业版可自定义保留策略
数据保留期限

按数据类型分级保留，客户可控导出和删除。

Q6：如果不再使用服务，数据如何处理？

支持完整的数据导出和删除：

  1. 数据导出：提供一键导出全部线索、访客记录、业务报告
  2. 账户注销：提交注销申请后，90 天内完成数据彻底清除
  3. 备份清除：注销 180 天后，所有备份数据永久删除
数据删除

支持完整数据导出，90天内彻底清除，180天后备份删除。

Q9：如何防止 API 滥用和 Credits 盗刷？

多层防护机制：

  • 频率限制：单账号 API 调用频率受限
  • 异常检测：系统自动检测异常消耗模式
  • 额度预警：Credits 余额不足时自动提醒
  • 操作审计：所有 Credits 消耗均有明细记录
  • IP 白名单：企业版支持 API 调用 IP 限制
API 防护

频率限制、异常检测、额度预警、操作审计、IP白名单。
