# Requirements: AI-LeadFinder 文档体系重构

**Defined:** 2026-04-16
**Core Value:** 建立一套职责分明、双向索引、自我更新的产品文档结构，让产品经理、研发、测试在任何时候都能快速定位所需信息

## v1 Requirements

### 文档结构重构

- [ ] **DOC-01**: 完成目录结构重组，将根目录扁平化设计文档移入 docs/design/，建立按产品阶段的 prd-md/ 子目录结构
- [ ] **DOC-02**: 重写 21 个功能说明文档（prototype/docs/），按新模板统一格式：页面入口/出口、可见逻辑、不可见逻辑（API）、边界规则、异常处理、字段映射、AI Prompt 引用、版本元数据
- [ ] **DOC-03**: 建立 prototype/ 与 prd-md/ 的完整双向索引映射表，实现 PRD 节点与原型页面的双向可追溯
- [ ] **DOC-04**: 统一 22 个原型页面样式规范，确保所有页面引用 global.css，视觉一致
- [ ] **DOC-05**: 整理修订日志体系，建立 docs/changelog/v1.0.0.md 和 v1.1.0.md
- [ ] **DOC-06**: 完善 API 规范总览（prd-md/api-specs/README.md），包含调用流程图和依赖关系说明
