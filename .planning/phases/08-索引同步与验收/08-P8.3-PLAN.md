## 目标

修复 Phase 8 UAT 发现的 4 个链接缺失问题，使文档体系具备完整的可点击交叉导航。

## 问题汇总

| # | 问题 | 根因 | 涉及文件 |
|---|------|------|---------|
| G1 | 03-prototypes/README.md 页面目录列表无链接 | 表格用纯文本文件名，未加链接格式 | 03-prototypes/README.md |
| G2 | 04-specs/README.md 功能说明文档索引无链接 | 表格用纯文本文件名，未加链接格式 | 04-specs/README.md |
| G3 | 04-specs/README.md 缺少模块映射表入口章节 | 缺少"模块映射表入口"章节 | 04-specs/README.md |
| G4 | 模块 README 映射表 spec 路径非链接 | 功能说明文档列用纯文本路径 | 02-modules/*/README.md |

## 修复计划

### Fix 1: 03-prototypes/README.md 页面目录列表加链接

**文件**: `03-prototypes/README.md`

**操作**: 将"页面目录列表"表格（第 23-44 行）的第一列从纯文本改为 Markdown 链接，指向 `./pages/` 目录：

```
| 业务分析报告.html | P11 业务分析报告页面 |
```
改为
```
| [业务分析报告.html](./pages/业务分析报告.html) | P11 业务分析报告页面 |
```

其余 19 行同理，共 20 个链接。

**验收标准**: 点击表格中任意文件名，跳转至对应 03-prototypes/pages/*.html 文件。

---

### Fix 2: 04-specs/README.md 功能说明文档索引加链接

**文件**: `04-specs/README.md`

**操作**: 将"功能说明文档索引"表格（第 20-40 行）的第一列从纯文本改为 Markdown 链接，指向当前目录：

```
| 租户资源充值-说明.md | 租户资源充值.html | P01 |
```
改为
```
| [租户资源充值-说明.md](./租户资源充值-说明.md) | 租户资源充值.html | P01 |
```

其余 20 行同理，共 21 个链接。

**验收标准**: 点击表格中任意文件名，跳转至对应 04-specs/*.md 文件。

---

### Fix 4: 模块 README 映射表 spec 路径改为链接

**文件**: `02-modules/01-账号/README.md`、`02-modules/02-静态数据/README.md`、`02-modules/03-企业管理/README.md`

**操作**: 将各模块 README"文档索引"表格的"功能说明文档"列从纯文本路径改为 Markdown 链接数组。

以 01-账号 为例（第 20-21 行），将：
```
| P01-P03 | 租户资源充值.html、资源下发确认.html、多BOT工作台.html | ../04-specs/租户资源充值-说明.md、../04-specs/资源下发确认-说明.md、../04-specs/多BOT工作台-说明.md |
```
改为：
```
| P01-P03 | 租户资源充值.html、资源下发确认.html、多BOT工作台.html | [租户资源充值-说明.md](../04-specs/租户资源充值-说明.md)、[资源下发确认-说明.md](../04-specs/资源下发确认-说明.md)、[多BOT工作台-说明.md](../04-specs/多BOT工作台-说明.md) |
```

**验收标准**: 在任意模块 README 中点击功能说明文档列的链接，能跳转至对应 04-specs/*.md 文件。
