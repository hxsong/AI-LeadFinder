# Phase 7 迁移执行标准

**依据：** Phase 6 P6.1 目录结构规范（06-目录结构规范.md）
**目的：** 为 Phase 7 内容迁移提供可逐条执行的校验标准

---

## 1. 迁移前完整性校验

在执行任何迁移前，先执行以下校验：

```bash
# Step 1: 记录当前 docs/ 文件清单
find docs/ -type f | sort > /tmp/docs-files.txt
wc -l /tmp/docs-files.txt

# Step 2: 记录当前 prd-md/ 文件清单
find prd-md/ -type f | sort > /tmp/prd-md-files.txt
wc -l /tmp/prd-md-files.txt

# Step 3: 记录当前 prototype/pages/ 文件清单
find prototype/pages/ -type f | sort > /tmp/prototype-pages.txt
wc -l /tmp/prototype-pages.txt

# Step 4: 记录当前 prototype/docs/ 文件清单
find prototype/docs/ -type f | sort > /tmp/prototype-docs.txt
wc -l /tmp/prototype-docs.txt
```

**记录校验点：** 各文件数量，后续 Step 5 验证必须完全匹配。

---

## 2. 迁移步骤

### Step 1: 创建目标目录

```bash
mkdir -p 01-concept/design/核心业务流
mkdir -p 01-concept/design/实体关系说明
mkdir -p 01-concept/api-specs
mkdir -p 02-modules/{00-概述与定义,01-账号,02-静态数据,03-企业管理,04-AILead管理,05-销售线索,06-任务协作,07-全局规范,api-specs}
mkdir -p 03-prototypes/{pages,styles,components}
mkdir -p 04-specs/{00-概述与定义,01-账号,02-静态数据,03-企业管理,04-AILead管理,05-销售线索,06-任务协作,07-全局规范}
mkdir -p 05-shared/{changelog,misc}
mkdir -p resources/{images,fonts}
```

### Step 2: 执行文件迁移

**docs/ → 01-concept + 05-shared**
```bash
# changelog（05-shared/changelog/）
cp -r docs/changelog/* 05-shared/changelog/

# design/（01-concept/design/）
cp -r docs/design/核心业务流 01-concept/design/
cp -r docs/design/实体关系说明 01-concept/design/

# 其余 docs/ 文件（05-shared/misc/）
find docs/ -maxdepth 1 -type f -exec cp {} 05-shared/misc/ \;
```

**prd-md/ → 02-modules/**
```bash
# 逐模块迁移（去掉 -md 后缀，改目录名）
cp -r prd-md/00-概述与定义/. 02-modules/00-概述与定义/
cp -r prd-md/01-账号/. 02-modules/01-账号/
cp -r prd-md/02-静态数据/. 02-modules/02-静态数据/
cp -r prd-md/03-企业管理/. 02-modules/03-企业管理/
cp -r prd-md/04-AILead管理/. 02-modules/04-AILead管理/
cp -r prd-md/05-销售线索/. 02-modules/05-销售线索/
cp -r prd-md/06-任务协作/. 02-modules/06-任务协作/
cp -r prd-md/07-全局规范/. 02-modules/07-全局规范/
cp -r prd-md/api-specs/. 02-modules/api-specs/
```

**prototype/pages/ → 03-prototypes/pages/**
```bash
cp -r prototype/pages/* 03-prototypes/pages/
```

**prototype/docs/ → 04-specs/**
```bash
cp -r prototype/docs/* 04-specs/
```

**注意：** prototype/styles/ 和 prototype/components/ 不移动，它们已经在新结构的 03-prototypes/ 树中：
```bash
# 仅确保它们在正确位置（理论上已在 03-prototypes/ 下，无需操作）
ls 03-prototypes/styles/
ls 03-prototypes/components/
```

## 3. 迁移后完整性校验

```bash
# Step 5: 验证文件数量（必须与 Step 1 完全匹配）
find 05-shared/changelog/ -type f | sort > /tmp/migrated-changelog.txt
find 01-concept/design/ -type f | sort > /tmp/migrated-design.txt
find 05-shared/misc/ -type f | sort > /tmp/migrated-misc.txt
find 02-modules/ -type f | sort > /tmp/migrated-prd.txt
find 03-prototypes/pages/ -type f | sort > /tmp/migrated-pages.txt
find 04-specs/ -type f | sort > /tmp/migrated-specs.txt

# 比较：docs/ 文件数应完全出现在新目录中
# changelog 文件
echo "changelog: $(wc -l < /tmp/docs-files.txt | xargs) → $(wc -l < /tmp/migrated-changelog.txt)"
```

**关键校验：**
1. `05-shared/changelog/` 文件数 = 原始 docs/changelog/ 文件数
2. `02-modules/` 下 prd-md 全部文件（.md）均存在
3. `03-prototypes/pages/` 文件数 = 原始 prototype/pages/ 文件数
4. `04-specs/` 文件数 = 原始 prototype/docs/ 文件数

---

## 4. 内容不变性校验

迁移原则：**内容不变，仅改变路径**。

```bash
# 抽样校验：比较迁移前后文件 MD5
# 示例

md5sum 05-shared/changelog/变更日志.md
# 两个 MD5 必须一致
```
