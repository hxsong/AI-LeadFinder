## 1. 目标

本次目录体系重构的核心目标：

- **职责分明**：每个目录有明确的职责边界，不重复、不遗漏
- **双向索引**：PRD 节点 ↔ 原型页面 ↔ 功能说明文档，三向映射可双向追溯
- **自我更新**：各层 README.md 包含维护指南，新增文档时同步更新索引

## 3. 各目录职责边界说明

### 01-concept/

**职责：** 核心概念设计资产和全局规范文档的统一入口，与产品模块无关的全局性知识资产。

**包含内容：**
- 设计图（ER 图、业务流图等）
- API 规范总览
- 术语表（glossary.md）

**与其他层的关系：** 全局共享，被 02-modules/ 各模块的 README.md 引用；与产品模块 PRD 文档分离存放，避免职责混乱。

**维护责任人：** PM

---

### 02-modules/

**职责：** 按产品阶段分模块的 PRD 文档集合，每个子目录对应一个产品模块。

**包含内容：**
- 各产品阶段的 PRD 文档（含字段映射、使用说明、API 调用方式）
- 每个模块独立的三向映射表（PRD 节点 ↔ 原型页面 ↔ 功能说明文档）
- api-specs/ 目录（各模块独立的 API 调用说明）

**与其他层的关系：** 被顶层 README.md 和 03-prototypes/、04-specs/ 交叉引用；PRD 节点编号（P01-P80）作为全局索引标识。

**维护责任人：** PM

---

### 03-prototypes/

**职责：** 承载全部 HTML 原型页面，以及支撑原型运行的样式和组件资源。

**包含内容：**
- `pages/`：各产品模块的 HTML 页面（20 个）
- `styles/`：global.css 设计 Token（保持在原型目录下，不外迁）
- `components/`：组件库（保持在原型目录下，不外迁）

**与其他层的关系：** 被 02-modules/ 各模块的三向映射表引用；pages/ 中的页面是功能说明文档的原型依据；styles/ 和 components/ 与原型页面紧邻，确保原型可独立运行。

**维护责任人：** 前端 / PM

---

### 04-specs/

**职责：** 功能说明文档的统一入口，对应各产品模块的具体功能实现说明。

**包含内容：**
- 各产品模块的功能说明文档（21 个，迁移自 prototype/docs/）
- 每个模块独立的 README.md 索引

**与其他层的关系：** 被 02-modules/ 各模块的三向映射表引用；功能说明文档对应 03-prototypes/pages/ 中的具体页面。

**维护责任人：** PM / 研发

---

### 05-shared/

**职责：** 跨产品模块的共享知识资产，所有不属于特定模块的通用文档集中存放。

**包含内容：**
- `changelog/`：变更历史记录（v1.0.0 历史 + v1.1.0 规划）
- `misc/`：杂项文档（迁移自 docs/ 除 changelog/、design/ 外的文件）

**与其他层的关系：** 各模块 README.md 的"变更历史"章节引用 changelog/ 中的内容；顶层 README.md 引用 misc/ 内容。

**维护责任人：** PM

---

### resources/

**职责：** 顶层非设计类共享资源（图片、字体等），与 03-prototypes/ 中的设计资源分离。

**包含内容：**
- `images/`：文档用图片、截图等
- `fonts/`：字体文件

**与其他层的关系：** 被各层 Markdown 文档引用；与 03-prototypes/styles/（原型样式）和 03-prototypes/components/（原型组件）保持分离，避免资源职责混淆。

**维护责任人：** PM / 研发

## 7. 各层 README 索引格式规范

### 7.1 三向映射表格式

三向映射表是跨层索引的核心，实现 **PRD 需求 ↔ 原型页面 ↔ 功能说明文档** 的双向追溯。

**格式：**

```markdown
## {模块名} — 三向映射表

| PRD 节点 | 原型页面 | 功能说明文档 |
|----------|----------|-------------|
| [节点名称] | [页面名](../03-prototypes/pages/模块/xxx.html) | [文档名](../04-specs/模块/xxx.md) |
| ... | ... | ... |
```

**规则：**
- 链接使用 Markdown 相对路径（如 `../03-prototypes/pages/01-账号/login.html`）
- 从任一列可点击跳转到对应文件
- 节点名称取自 PRD 文档的标题（不含编号）
- 每个模块独立一张映射表

### 7.2 各层 README 结构模板

#### 02-modules/{模块}/README.md

```markdown
# {模块名}

## 职责说明

[本模块的职责边界、包含哪些 PRD 文档、与其他模块的关系]

## PRD 文档

| 文档名 | 说明 |
|--------|------|
| xxx.md | [描述] |

## 三向映射表

| PRD 节点 | 原型页面 | 功能说明文档 |
|----------|----------|-------------|
| ... | ... | ... |

## 变更历史

[来自 ../../05-shared/changelog/ 的摘要]

## 维护指南

- 新增页面时：在原型页面列补充一行，同步更新 03-prototypes/pages/
- 新增功能说明时：在功能说明文档列补充一行，同步更新 04-specs/
- 新增 PRD 节点时：在 PRD 节点列补充一行
```
