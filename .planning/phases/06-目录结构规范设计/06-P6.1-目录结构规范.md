## 7. 各层 README 索引格式规范

### 7.1 三向映射表格式

三向映射表是跨层索引的核心，实现 PRD 需求 ↔ 原型页面 ↔ 功能说明文档 的双向追溯。

**格式：**

```markdown
## {模块名} — 三向映射表

| PRD 节点 | 原型页面 | 功能说明文档 |
|----------|----------|-------------|
| [节点名称] | [页面名](../03-prototypes/pages/模块/xxx.html) | [文档名](../04-specs/模块/xxx.md) |
| ... | ... | ... |
```

**规则：**
- 链接使用 Markdown 相对路径（如 `../03-prototypes/pages/01-账号/login.html`）
- 从任一列可点击跳转到对应文件
- 节点名称取自 PRD 文档的标题（不含编号）
- 每个模块独立一张映射表

### 7.2 各层 README 结构模板

#### 02-modules/{模块}/README.md

```markdown
# {模块名}

## 职责说明

[本模块的职责边界、包含哪些 PRD 文档、与其他模块的关系]

## PRD 文档

| 文档名 | 说明 |
|--------|------|
| xxx.md | [描述] |

## 三向映射表

| PRD 节点 | 原型页面 | 功能说明文档 |
|----------|----------|-------------|
| ... | ... | ... |

## 变更历史

[来自 ../../05-shared/changelog/ 的摘要]

## 维护指南

- 新增页面时：在原型页面列补充一行，同步更新 03-prototypes/pages/
- 新增功能说明时：在功能说明文档列补充一行，同步更新 04-specs/
- 新增 PRD 节点时：在 PRD 节点列补充一行
