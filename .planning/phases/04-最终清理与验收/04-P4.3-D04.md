## 发现的问题汇总

### 问题 1：功能说明文档扩展名不一致（中等优先级）

**位置**：
- `prototype/README.md` 映射表（全部 20 个功能说明文档映射行）
- `docs/changelog/v1.1.0.md` 变更文件清单（第 29-49 行）

**现象**：映射表和 changelog 中引用功能说明文档时使用 `.html` 扩展名，但实际文件均为 `.md` 文件。

**影响**：文档读者点击链接将遇到 404 错误；影响索引链路 A 和链路 C 的可用性。

**修复建议**：
```diff
# prototype/README.md 映射表，所有功能说明文档列
- 租户资源充值-说明.html
+ 租户资源充值-说明.md

# docs/changelog/v1.1.0.md 变更文件清单
- prototype/docs/AI业务分析中-说明.html
+ prototype/docs/AI业务分析中-说明.md
（其余 20 个文件同理）
```

---

### 问题 2：原型 README 映射表缺少第 21 项（高优先级）

**位置**：`prototype/README.md` 第 2 节映射表

**现象**：表格共 20 行，缺少第 21 项（动态线索详情）。

| 缺失项 | 应填内容 |
|--------|---------|
| 原型页面 | 动态线索详情.html |
| 功能说明文档 | 动态线索详情-说明.md |
| 对应 PRD 节点 | P47-P70（预留） |

**影响**：21 个功能说明文档中有 1 个无法通过 README 定位到原型页面。

**修复建议**：在映射表末尾补充第 21 行。

---

### 问题 3：Changelog 声称创建的文件不存在（低优先级）

**位置**：`docs/changelog/v1.1.0.md` 影响文件清单

**现象**：changelog 声称"新增"两个文件，但均不存在：
1. `docs/changelog/v1.1.0.md` → "新增：`docs/changelog/v1.1.0.md`"（已存在 ✓，但修改文件清单中声称"新增"本身有问题）
2. `docs/双向索引映射表.md`（Phase 2 产物）— 不存在
3. `prototype/docs/全局索引映射表.html`（changelog 第 129 行）— 不存在

**影响**：changelog 的准确性受损，读者无法通过 changelog 索引定位到文档。

**修复建议**：删除 changelog 中对不存在文件的引用，或确认这些文件是否应被创建。
