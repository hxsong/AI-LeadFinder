## 功能目标

本模块实现动态流量数据的采集、分析和线索转化全流程。核心目标包括：

1. **探针部署**：支持 JS 探针和 IP 解析两种数据采集方式
2. **访客雷达**：展示独立站访客实时访问数据，包括 IP 地址、地理位置、访问页面、停留时长等
3. **访客行为详情**：展示单个访客的完整访问轨迹和行为数据
4. **动态线索池**：管理动态数据获客来源的企业线索，支持 IP 解析和访客转线索
5. **动态线索详情**：展示动态线索的完整分析报告，包含访客行为数据、评分、联系人矩阵

**我想要** 部署 JS 探针到独立站以采集访客数据

## 业务规则

### 规则一：探针类型选择

- **JS 探针**：通过嵌入 JavaScript 代码实时采集访客行为数据
- **IP 解析**：通过 IP 地址反向解析获取企业信息

### 规则二：JS 探针部署规则

- 每次仅可选 1 个站点作为核心主站
- 选中新的自动取消旧的
- 无独立站时，探针部署卡片置灰禁用
- 探针代码需嵌入目标站点 HTML 的 `<head>` 标签内

### 规则三：IP 解析计费

- 单次 IP 解析消耗 5 Credits
- IP 解析结果有效期 7 天

### 规则四：访客数据保留

- 访客数据保留期限：90 天
- 超过 90 天的数据自动归档

### 规则五：访客转线索规则

- 访客可手动转为线索
- 转线索后，访客数据与线索数据关联
- 转线索后，该访客不再出现在访客雷达列表

### 规则六：动态线索四维评分

动态线索评分包含四维：
- **需求匹配度**：AI 模型判断访客企业是否是客户的潜在客户
- **购买能力**：企业规模评分（60%）+ 地理评分（40%）
- **采购意向**：访客行为评分（页面深度、停留时长、访问频次）
- **活跃意向**：近期访问活跃度评分

### 规则七：评分等级划分

| 评分范围 | 等级 | 颜色 |
|---------|------|------|
| ≥80 | S级 | 绿色 |
| 60-79 | A/B级 | 蓝色 |
| <60 | C级 | 灰色 |

### 规则八：联系人挖掘计费

- 单次挖掘消耗 15 Credits
- 每获取 1 个验证邮箱 = 2 Credits
- 附加 LinkedIn = +1 Credit
- 附加 Twitter/FB = +1 Credit/个
- 单企业最多挖掘 5 次

## 用户操作流程

### 场景一：选择探针类型（P28-P29）

1. 页面展示两张选择卡片：JS 探针、IP 解析
2. 用户点击选择探针类型
3. 系统记录用户选择
4. 系统跳转至对应页面

### 场景二：JS 探针部署（P29-P32）

1. 系统展示当前租户下已绑定的站点列表
2. 每个站点卡片显示：站点名称、站点域名、状态标签
3. 用户点击站点卡片，选中后显示橙色边框 + 勾选图标
4. 展开探针代码区域，显示 JavaScript 代码
5. 提供复制代码按钮
6. 用户将代码嵌入目标站点 HTML 的 `<head>` 标签内
7. 用户点击"我已部署"完成部署

### 场景三：IP 解析（P29-P38）

1. 用户输入 IP 地址
2. 用户点击"解析"按钮
3. 系统进行 IP 解析
4. 系统展示解析结果：企业名称、地理位置、ISP 信息等
5. 用户可以将解析结果转为线索

### 场景四：查看访客雷达（P29-P38）

1. 用户进入访客雷达页面
2. 系统展示访客列表（默认按访问时间倒序）
3. 每张访客卡片展示：
   - IP 地址（脱敏处理）
   - 地理位置（国家/城市）
   - 访问页面列表
   - 停留时长
   - 访问时间
   - 是否为回访客户
4. 用户可以筛选（时间范围、访问页面、地理位置）
5. 用户可以排序（访问时间、停留时长）
6. 用户可以导出访客数据

### 场景五：查看访客行为详情（P39-P42）

1. 用户点击访客卡片进入详情页面
2. 系统展示：
   - 访客基本信息（IP、地理位置、ISP）
   - 访问轨迹时间轴（页面、停留时长、访问时间）
   - 行为分析（访问深度、兴趣页面、回访次数）
3. 用户可以将访客转为线索

### 场景六：访客转线索（P39-P46）

1. 用户在访客雷达或访客详情页面点击"转为线索"按钮
2. 系统弹出确认弹窗，展示计费规则
3. 用户确认后，系统开始执行线索生成任务
4. 系统执行 IP 解析、AI 评分、生成线索记录
5. 用户跳转至动态线索池页面查看进度

### 场景七：查看动态线索池（P39-P46）

1. 用户进入动态线索池页面
2. 系统展示线索列表（默认按入库时间倒序）
3. 展示两类卡片：
   - 解析中状态：卡片边框高亮 + "解析中"标签 + 进度条
   - 已完成状态：展示企业信息和评分 + Credits 消耗
4. 用户可以筛选（解析状态、入库时间范围、评分等级）
5. 用户可以排序（入库时间倒序/评分倒序）
6. 用户可以导出线索数据

### 场景八：查看动态线索详情（P47-P70）

1. 用户点击线索卡片进入详情页面
2. 系统展示：
   - 企业核心信息区（名称、域名、所在地、规模）
   - 最终线索评分与等级
   - AI 判词
   - 访客行为数据（访问轨迹、停留时长、访问频次）
   - 联系人矩阵（姓名、职位、邮箱、验证状态、社媒链接）
   - 账单摘要（Credits 消耗明细）
3. 用户可以进行智能触达操作

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 无独立站 | 探针部署卡片置灰，提示绑定独立站 |
| IP 解析失败 | 提示解析失败，支持重试 |
| Credits 余额不足 | 按钮禁用，提示充值 |
| 访客数据为空 | 显示空态引导 |
| 探针部署失败 | 提示部署失败，提供排查指南 |
| 线索已存在 | 提示不可重复入库 |
| 网络超时 | 提示网络异常，支持重试 |
