## 功能目标

本模块实现动态流量数据的采集、分析和线索转化全流程。核心目标包括：

1. **探针部署**：部署 JS 探针到企业独立站，实时采集访客行为数据（每个账号仅支持绑定一个域名）
2. **访客雷达**：展示独立站访客实时访问数据，包括 IP 地址、地理位置、访问页面、停留时长、行为热度评分等
3. **访客行为详情**：展示单个访客的完整访问轨迹、微观行为数据和 AI 行为洞察
4. **动态线索池**：管理通过"深度挖掘"访客生成的企业线索，展示挖掘进度和联系人信息
5. **动态线索详情**：展示动态线索的完整信息，包含企业信息、访客行为证据、联系人矩阵和账单摘要

## 用户故事

### 故事一：企业管理员部署 JS 探针

**作为** 企业管理员  
**我想要** 部署 JS 探针到独立站以采集访客数据  
**以便于** 实时监控网站访客行为

**验收条件**：
- 可以通过 OAuth 授权独立站实现一键安装探针
- 可以手动填写域名并获取探针代码
- 每个账号仅能绑定一个网站域名
- 可以复制探针代码并嵌入网站 `<head>` 标签
- 可以验证探针是否生效

### 故事二：企业管理员深度挖掘访客

**作为** 企业管理员  
**我想要** 对访客进行深度挖掘以获取企业信息和联系人  
**以便于** 将访客转化为可跟进的销售线索

**验收条件**：
- 可以在访客雷达中查看访客热度评分
- 可以对任意访客点击"深度挖掘"按钮（不限分数）
- 可以查看挖掘计费规则（IP 解析 + 联系人挖掘）
- 确认后系统自动执行 IP 解析和联系人挖掘
- 可以在动态线索池查看挖掘进度

### 故事三：企业管理员查看访客雷达

**作为** 企业管理员  
**我想要** 查看独立站的实时访客数据  
**以便于** 了解访客行为和来源

**验收条件**：
- 可以查看访客列表（IP、地理位置、访问页面、停留时长）
- 可以筛选和排序访客数据
- 可以导出访客数据
- 可以将访客转为线索

### 故事四：企业管理员查看访客行为详情

**作为** 企业管理员  
**我想要** 查看单个访客的详细行为数据  
**以便于** 深入了解访客兴趣

**验收条件**：
- 可以查看访客基本信息
- 可以查看访问轨迹时间轴
- 可以查看行为分析（访问深度、兴趣页面）
- 可以将访客转为线索

### 故事五：企业管理员管理动态线索

**作为** 企业管理员  
**我想要** 查看和管理动态数据获客的线索  
**以便于** 跟进高价值潜在客户

**验收条件**：
- 可以查看动态线索列表（包含挖掘中、已完成、ISP 宽带三种状态）
- 可以查看线索挖掘进度（IP 解析 → 企业验证 → 联系人挖掘）
- 可以查看已完成线索的企业信息和联系人数量
- 可以导出线索数据

### 故事六：企业管理员查看动态线索详情

**作为** 企业管理员  
**我想要** 查看动态线索的完整信息  
**以便于** 制定针对性的跟进策略

**验收条件**：
- 可以查看企业核心信息（名称、域名、所在地、规模）
- 可以查看原始访客行为证据（热度评分、行为轨迹）
- 可以查看联系人列表（姓名、职位、邮箱、社媒链接）
- 可以查看账单摘要（Credits 消耗明细）
- 可以进行智能触达

## 业务规则

### 规则一：JS 探针部署规则

| 规则项 | 说明 |
|--------|------|
| 一账号一域名 | 每个企业账号仅能绑定一个网站域名 |
| 域名唯一性 | 同一域名仅能被一个账号绑定 |
| 部署方式 | 支持 OAuth 授权一键安装或手动填写域名部署 |
| 嵌入位置 | 探针代码需嵌入目标站点 HTML 的 `<head>` 标签内 |
| 域名校验 | 探针部署后系统会校验请求来源域名与绑定域名一致性 |
| GDPR 合规 | 探针不采集 input 输入内容，仅记录"发生过输入行为"，需用户 Cookie 同意后触发 |
| 验证冷却 | 手动验证按钮 30 秒内重复点击，返回"验证操作过于频繁"提示 |

### 规则二：访客数据保留

| 规则项 | 说明 |
|--------|------|
| 保留期限 | 访客数据保留期限 90 天 |
| 归档规则 | 超过 90 天的数据自动归档 |

### 规则三：访客热度评分规则

| 规则项 | 说明 |
|--------|------|
| 评分计算 | 基于访客行为数据（页面深度、停留时长、访问频次、交互行为）规则打分 |
| 热度等级 | 极高热度（≥80分，红色）、较高热度（60-79分，橙色）、一般热度（<60分，绿色） |
| AI 分析触发 | 规则打分 ≥60 分时触发 AI 行为洞察和综合研判 |

### 规则四：深度挖掘计费规则

| 计费项 | Credits 消耗 |
|--------|-------------|
| IP 企业解析 | 1 Credit |
| 每获取 1 个验证邮箱 | 1 Credit |
| 附加 LinkedIn 主页 | 0 Credit（免费） |
| 附加 Twitter/FB | 0 Credit/个（免费） |
| 单企业挖掘上限 | 最多 5 次 |

### 规则五：动态线索状态规则

| 状态 | 说明 |
|------|------|
| Resolving（挖掘中） | IP 解析、企业验证、联系人挖掘进行中 |
| Completed（已完成） | 成功获取企业信息和联系人 |
| ISP（宽带匿名） | IP 解析结果为公共宽带，无法定位企业主体 |

## 用户操作流程

### 场景一：探针部署（P28-P32）

1. 用户进入探针部署页面
2. 系统检测是否存在已授权独立站：
   - **分支 A（已授权）**：显示 OAuth 授权站点列表，提供"一键安装探针"按钮
   - **分支 B（未授权）**：显示手动域名输入框
3. 用户选择部署方式：
   - **OAuth 一键安装**：选择站点 → 点击一键部署 → 系统注入追踪代码 → 验证部署状态
   - **手动部署**：输入域名 → 系统验证 → 生成探针代码 → 复制代码 → 嵌入网站 `<head>` → 手动验证
4. 部署成功后显示"探针已生效"状态，可跳转访客雷达

### 场景二：查看访客雷达（P33-P38）

1. 用户进入访客雷达页面
2. 系统展示统计概览：今日访客数、高热度访客数、已挖掘访客数
3. 系统展示访客列表（默认按访问时间倒序）
4. 每张访客卡片展示：
   - 热度评分（规则打分）和热度等级标签
   - IP 地址（脱敏处理）
   - 地理位置（国家/城市）
   - 设备类型和流量来源
   - 高维微观行为轨迹（关键行为事件）
   - AI 关键动作洞察（≥60 分时显示）
   - AI 综合研判建议（≥60 分时显示）
5. 用户可以筛选（热度等级、地区、时间范围）
6. 用户可以对任意访客点击"深度挖掘"（不限分数）

### 场景三：查看访客行为详情（P39-P42）

1. 用户点击访客卡片进入详情页面
2. 系统展示：
   - 访客基础信息（IP、地理位置、设备、操作系统、流量来源）
   - 热度评分和等级
   - 高维微观行为轨迹时间轴（页面浏览、复制文本、文件下载、点击行为、意图流失等）
   - 每个行为事件的 AI 解读
   - 历史来访记录
3. 高分访客（≥60 分）显示"深度挖掘"按钮
4. 低分访客（<60 分）显示"AI 分析未触发"提示

### 场景四：深度挖掘访客（P39-P46）

1. 用户在访客雷达或访客详情页面点击"深度挖掘"按钮
2. 系统弹出深度挖掘弹窗，展示：
   - 计费规则（IP 解析 + 联系人挖掘费用）
   - 异步执行提示
   - UTM 标识检测提示（如有）
   - 当前账户 Credits 余额
3. 用户确认后，系统根据 UTM 标识执行不同分支：
   - **分支 A（存在有效 UTM）**：自动关联静态数据中的企业信息，跳过 IP 解析，直接进入联系人挖掘
   - **分支 B（无 UTM 标识）**：启动 IP 解析服务进行企业识别，再进入联系人挖掘
4. 系统创建动态线索占位记录
5. 系统自动跳转至动态线索池页面查看挖掘进度

### 场景五：查看动态线索池（P45-P46）

1. 用户进入动态线索池页面
2. 系统展示线索列表（默认按入库时间倒序）
3. 展示三种状态卡片：
   - **Resolving（挖掘中）**：高亮边框 + 进度动画 + 实时展示 IP 解析/企业验证/联系人挖掘进度
   - **Completed（已完成）**：展示企业名称、域名、联系人数量、Credits 消耗和 AI 动态意向分析结果
   - **ISP（宽带匿名）**：展示 ISP 运营商名称和提示信息
4. 用户可以筛选（状态、入库时间）
5. 用户可以导出线索数据

### 场景六：查看动态线索详情（P46）

1. 用户点击线索卡片进入详情页面
2. 系统展示：
   - 企业核心信息区（名称、域名、所在地、规模）
   - AI 判词（AI 动态意向分析结果）
   - 联系人列表（姓名、职位、验证邮箱、社媒链接、立即触达按钮）
   - 原始动态行为证据（访客 IP、热度评分、行为轨迹时间轴）
   - 关键微观事件摘要
   - 账单摘要（Credits 消耗明细）
3. 用户可以进行智能触达操作

> **注**：P47-P50 属于阶段五"全景画像融合"

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 无独立站 | 探针部署卡片置灰，提示绑定独立站 |
| IP 解析失败 | 提示解析失败，支持重试 |
| Credits 余额不足 | 按钮禁用，提示充值 |
| 访客数据为空 | 显示空态引导 |
| 探针部署失败 | 提示部署失败，提供排查指南 |
| 线索已存在 | 提示不可重复入库 |
| 网络超时 | 提示网络异常，支持重试 |
