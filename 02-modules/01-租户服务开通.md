## 功能目标

本模块实现租户账号的开通与资源分配，以及多 BOT 工作空间的管理。核心目标包括：

1. **系统服务管理**：平台管理员为入驻租户购买系统服务年限，控制功能入口权限
2. **资源下发**：平台管理员为入驻租户分配 Global Credits 和 BOT 引擎席位
3. **资源确认**：在资源正式发放前展示下发详情，供管理员确认
4. **多 BOT 管理**：企业管理员登录后的首个指挥中心，聚合展示全局资源和各 BOT 实例状态

## 用户故事

### 故事一：平台管理员为租户购买系统服务

**作为** 平台管理员
**我想要** 为入驻租户购买系统服务年限和增购 BOT 席位
**以便于** 租户可以持续使用 LeadFinder 进行获客

**验收条件**：
- 可以查看当前系统到期时间和 BOT 席位使用情况
- 可以购买系统服务年限，费用按当前 BOT 席位数 × 年数 × 3,000 元计算
- 可以增购 BOT 席位，费用按当前服务剩余年数 × 席位 × 3,000 元计算
- 可以购买 Credits 资源包补充算力
- 充值成功后租户资源实时更新

### 故事二：企业管理员查看资源状态

**作为** 企业管理员
**我想要** 查看账号的全局资源使用情况
**以便于** 了解剩余 Credits、BOT 席位和系统服务到期时间

**验收条件**：
- 可以查看系统服务到期时间
- 可以查看全局 Credits 余额
- 可以查看已使用/剩余的 BOT 席位数量
- 可以查看各 BOT 实例的状态和消耗情况

### 故事三：企业管理员创建新 BOT

**作为** 企业管理员
**我想要** 在 BOT 席位充足时创建新的 BOT 实例
**以便于** 开展不同业务线的获客工作

**验收条件**：
- BOT 席位充足时可以创建新 BOT
- BOT 席位不足时创建入口被禁用
- 创建后可以进入 BOT 工作空间

### 故事四：平台管理员减少 BOT 席位

**作为** 平台管理员
**我想要** 在必要时减少租户的 BOT 席位数量
**以便于** 管理租户资源配额

**验收条件**：
- 可以通过独立工具减少席位
- 减少时若已创建 BOT 超出目标席位，提示需先删除 BOT
- 减少操作不退费

## 业务规则

### 规则一：资源类型定义

| 资源类型 | 属性 | 作用 |
|---------|------|------|
| 系统服务 | 时效型资产 | 按年计费，控制功能入口权限和访客雷达数据接收能力 |
| BOT Seats | 并发限制型资产 | 限制该企业账号最多可同时创建并运行几个独立的获客 BOT |
| Global Credits | 消耗型资产 | 全账号名下所有 BOT 共享此资金池，用于支付全网搜客、IP 解析、联系人挖掘等动作的 API 与 Token 成本 |

### 规则二：费用计算规则

| 计费项 | 单价 | 计算公式 |
|--------|------|---------|
| 系统服务费 | 3,000 元/年/BOT | (当前BOT×新增年数 + 增购席位×总年数) × 3,000 |
| 年数取整 | - | 向上取整（1.2年、1.9年都按2年计算） |
| Credits 基础包 | 1,000 元/1,000 Credits | 固定价格 |
| Credits 标准包 | 4,500 元/5,000 Credits | 固定价格（9折） |
| Credits 专业包 | 16,000 元/20,000 Credits | 固定价格（8折） |
| Credits 企业包 | 70,000 元/100,000 Credits | 固定价格（7折） |

**系统服务费计算示例**：
- 当前 BOT 席位：3个，剩余年数：1.9年
- 新增年数：2年，增购席位：2个
- 总年数 = ceil(1.9 + 2) = 4年
- 系统服务费 = (3 × 2 + 2 × 4) × 3,000 = 42,000 元

### 规则三：Credits 资源包配置

- **基础包**：1,000 Credits / 1,000 元
- **标准包**：5,000 Credits / 4,500 元（9折）
- **专业包**：20,000 Credits / 16,000 元（8折）
- **企业包**：100,000 Credits / 70,000 元（7折）

### 规则四：BOT 额度限制

- 仅支持增购，不支持减购（减购需通过独立"减少席位"工具）
- 增购后总席位 = 原总席位 + 增购数量
- BOT 席位用完时，创建新 BOT 卡片置灰禁用

### 规则五：系统服务到期规则

- 系统服务费按年计费，到期后系统入口关闭，所有功能不可用
- BOT 席位本身无到期时间，但受系统服务期限制
- 系统到期后，已购买的 Credits 不会清除，但无法使用（需续费系统服务后才能恢复）
- 功能开关控制：系统入口权限 + 访客雷达数据上传能力
- 到期后或手动关闭后，访客雷达不再接受上传数据

### 规则六：减少席位工具规则

- 减少席位为独立工具，入口在功能开关旁
- 弹窗展示已创建 BOT 列表，支持勾选删除
- 实时校验逻辑：
  - 删除后剩余 BOT 数 = 已创建数 - 选中删除数
  - 仅当目标席位 ≥ 删除后剩余 BOT 数时，才允许确认
  - 若目标席位 < 删除后剩余 BOT 数，显示警告并提示需删除的数量
- 删除的 BOT 数据不可恢复
- 减少操作不退费
- 减少后已创建 BOT 不受影响（只要不超过新席位上限）

### 规则七：数据隔离

- 点击进入任意 BOT 后，该 BOT 成为当前激活状态
- 此后该用户所有操作被严格锁定在该 BOT 的数据作用域内
- 不同 BOT 的配置互不交叉污染

### 规则八：BOT 状态自动同步

- 状态判断以动态数据获客（JS 探针）运行状态为准
- 最近 500 次上报中错误率恢复至 ≤5%，自动重置为"正常运行"
- 24 小时内恢复数据上报，自动重置为"正常运行"

## 用户操作流程

### 场景一：系统服务费购买（P01）

1. 管理员进入充值页面，查看当前系统到期时间和 BOT 席位
2. 横向设置两项参数：
   - 新增服务年数（1-5年）
   - 增购 BOT 席位数量
3. 系统实时计算系统服务费：
   - 总年数 = ceil(剩余年数 + 新增年数)
   - 系统服务费 = (当前 BOT 席位数 × 新增年数 + 增购席位 × 总年数) × 3,000 元
4. 页面展示计算逻辑公式
5. 确认后系统延长到期时间并增加 BOT 席位

### 场景二：Credits 资源包购买（P01）

1. 展示四种 Credits 资源包
2. 管理员选择需要的资源包（支持多选）
3. 系统实时计算 Credits 费用
4. 确认后增加租户 Credits 余额

### 场景三：资源下发执行（P01-P02）

1. 管理员确认资源购买信息
2. 系统显示资源下发详情供确认
3. 确认后系统执行资源下发
4. 下发成功后显示成功状态，租户资源更新

### 场景四：功能开关控制（P01）

1. 管理员点击功能开关
2. 开关控制：系统入口权限 + 访客雷达数据上传能力
3. 到期后自动关闭

### 场景五：减少席位（P01）

1. 管理员点击"减少席位"按钮
2. 弹窗展示：
   - 当前席位和已创建 BOT 数量
   - 目标席位数输入框
   - 已创建 BOT 列表（名称、创建时间、线索数、状态）
3. 勾选需要删除的 BOT
4. 系统实时计算删除后剩余 BOT 数
5. 若目标席位 < 删除后剩余 BOT 数，显示警告提示需继续删除
6. 仅当目标席位 ≥ 删除后剩余 BOT 数时，确认按钮启用
7. 确认后执行删除和席位减少（不退费）

### 场景六：查看全局资源看板（P03）

1. 企业管理员进入多 BOT 工作台页面
2. 查看系统服务到期时间
3. 查看全局 Credits 余额（全账号共享资金池）
4. 查看 BOT 席位使用情况（已创建 / 总额度）
5. 查看 BOT 实例列表，每个卡片显示：名称、状态、累计线索数、Credits 消耗

### 场景七：创建新 BOT 实例（P03）

1. 系统检测 BOT 席位是否充足
2. 若额度充足，点击创建卡片弹出创建表单，输入 BOT 名称
3. 提交后系统创建 BOT 并初始化
4. 显示初始化进度，初始化成功后跳转业务初始化页面

### 场景八：进入已有 BOT 工作空间（P03）

1. 用户点击 BOT 实例卡片
2. 系统将该 BOT 设置为当前激活状态
3. 此后该用户所有操作被严格锁定在该 BOT 的数据作用域内
4. 不同 BOT 的配置互不交叉污染

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 充值失败 | 显示失败提示，不扣费 |
| 网络超时 | 提示网络异常，支持重试 |
| BOT 创建失败 | 显示失败提示，支持重试 |
| BOT 初始化失败 | 显示失败提示，提供重试按钮 |
| 无 BOT 额度 | 创建卡片禁用，提示联系销售 |
| 资源已生效 | 提示无需重复确认 |
| 资源不存在 | 提示记录不存在或已过期 |
| 减少席位超限 | 提示需先删除多余 BOT |
| 功能开关失败 | 开关回滚，提示重试 |
