## 功能目标

本模块实现 BOT 业务建模的完整流程，从数据来源选择到 ICP 画像生成。核心目标包括：

1. **数据来源选择**：支持上传业务材料或授权独立站两种方式输入企业数据
2. **业务材料上传**：接收用户本地的非结构化业务凭证（PDF/WORD/MD/TXT）
3. **独立站授权**：利用企业已有的建站系统，毫秒级获取结构化数据
4. **AI 业务分析**：将原始输入数据转换为结构化的客户信息（ICP 画像）
5. **分析报告展示**：基于四维评分模型展示深度业务分析结果

## 用户故事

### 故事一：企业管理员选择数据来源

**作为** 企业管理员  
**我想要** 选择上传业务材料或授权独立站作为数据来源  
**以便于** 为 BOT 提供企业信息进行业务建模

**验收条件**：
- 可以选择上传业务材料方式
- 可以选择授权独立站方式（需已有绑定站点）
- 选择后数据来源不可更改

### 故事二：企业管理员上传业务材料

**作为** 企业管理员  
**我想要** 上传企业的业务凭证文件  
**以便于** 让 AI 分析企业业务并生成 ICP 画像

**验收条件**：
- 支持上传 PDF、WORD、MD、TXT 格式文件
- 单文件不超过 5MB
- 最多上传 3 个文件
- 上传后可查看文件列表并提交分析

### 故事三：企业管理员授权独立站

**作为** 企业管理员  
**我想要** 授权已绑定的独立站作为数据来源  
**以便于** 快速获取企业结构化信息

**验收条件**：
- 可以查看已绑定的站点列表
- 可以选择一个站点作为核心主站
- 可以预览站点数据摘要
- 确认授权后进入分析流程

### 故事四：企业管理员查看业务分析报告

**作为** 企业管理员  
**我想要** 查看 AI 生成的业务分析报告  
**以便于** 了解企业定位和 ICP 画像

**验收条件**：
- 可以查看企业定位、业务类型、主营品类、核心优势
- 可以查看四维评分展示
- 可以查看目标客户画像
- 可以重新分析或开始搜客

## 业务规则

### 规则一：数据来源互斥

| 规则项 | 说明 |
|--------|------|
| 互斥原则 | 上传业务材料与授权独立站两种数据输入方式互斥 |
| 锁定机制 | 选择完成后数据来源标记不可更改，防止实体信息冲突 |

### 规则二：文件上传限制

| 限制项 | 规则说明 |
|--------|---------|
| 支持格式 | PDF、WORD（DOC/DOCX）、MD、TXT |
| 单文件大小 | 上限 5MB |
| 文件数量 | 最多 3 个 |

### 规则三：独立站授权规则

| 规则项 | 说明 |
|--------|------|
| 单选限制 | 每次仅可选 1 个站点作为核心主站 |
| 切换规则 | 选中新的自动取消旧的 |
| 无站处理 | 无独立站时，授权卡片置灰禁用 |

### 规则四：AI 分析阶段

| 阶段 | 步骤 | 说明 | 前置条件 |
|------|------|------|---------|
| 提交分析任务 | P07 | 提交全量材料，启动异步分析任务 | 数据来源已确认 |
| 文本提取与结构化 | P08 | 提取并结构化处理输入数据（ETL 数据清洗） | P07 任务已提交 |
| 深度分析与 ICP 建模 | P09 | 基于结构化数据进行商业分析，生成 ICP 画像 | P08 必填字段完整 |
| 中断规则 | - | P08 必填字段缺失时，中断流程，提示客户补充，不进入 P09 | - |

### 规则五：四维评分权重动态生成

根据业务类型动态生成权重配置：

| 业务类型 | 需求匹配度 | 购买能力 | 采购意向 | 活跃意向 |
|---------|-----------|---------|---------|---------|
| 外贸型 | 32% | 20% | 35% | 13% |
| 生产型 | 40% | 25% | 20% | 15% |
| 服务型 | 35% | 25% | 15% | 25% |
| 渠道型 | 35% | 20% | 30% | 15% |

### 规则六：重新分析规则

| 规则项 | 说明 |
|--------|------|
| 数据清除 | 重新分析会清除当前报告数据 |
| 流程重置 | 需重新选择数据来源 |

## 用户操作流程

### 场景一：检查初始化状态（P04）

1. 用户进入业务初始化页面
2. 系统显示当前数据来源选择状态
3. 若已选择数据来源，自动跳转至对应页面
4. 若已选择但无独立站，授权卡片置灰禁用

### 场景二：选择数据来源（P04）

1. 页面展示两张选择卡片：上传业务材料、授权独立站
2. 用户点击卡片后，系统记录用户选择（后续不可更改）
3. 系统跳转至对应页面

### 场景三：文件上传（P05）

1. 用户点击上传区域或拖拽文件触发文件选择
2. 系统验证文件格式和大小，不通过时提示用户
3. 文件添加到上传队列，显示上传进度
4. 上传完成后显示文件列表
5. 用户点击"开始分析"提交文件

### 场景四：独立站授权（P05-P06）

1. 系统展示当前租户下已绑定的站点列表
2. 每个站点卡片显示：站点名称、站点域名、状态标签
3. 用户点击站点卡片，选中后显示橙色边框 + 勾选图标
4. 展开数据同步预览区域，显示站点数据摘要
5. 用户点击"确认授权"完成授权（P06 自动同步站点数据）

### 场景五：AI 业务分析处理（P07-P09）

1. **提交分析任务（P07）**：
   - 系统提交当前 BOT 的全量材料进行 AI 智能业务分析
   - 启动异步分析任务

2. **文本提取与结构化阶段（P08）**：
   - 系统解析上传文件或独立站数据
   - 系统提取并结构化处理数据（ETL 数据清洗）
   - 若必填字段缺失，系统提示用户补充，不进入 P09

3. **深度分析与 ICP 建模阶段（P09）**：
   - 系统基于结构化数据进行 AI 分析
   - 输出企业定位、业务类型、主营品类、核心优势等
   - 根据业务类型生成动态四维评分权重
   - 生成 ICP 画像和特征向量
   - 客户信息存储至系统

### 场景六：处理进度实时展示（P07-P09）

1. 三个阶段并行/串行展示：P07 提交分析 → P08 文本提取与结构化 → P09 深度分析与 ICP 建模
2. 步骤状态切换：等待中 → 执行中 → 已完成
3. 进度条动画实时更新
4. 所有步骤完成后自动跳转至业务分析报告页面

### 场景七：查看业务分析报告（P10-P11）

1. 系统返回当前 BOT 的专属业务分析报告（P10）
2. 用户进入业务分析报告页面（P11）
3. 系统展示完整报告内容：
   - 企业定位区：企业名称、业务类型标签、企业定位描述
   - 四维评分展示：需求匹配度、购买能力、采购意向、活跃意向
   - 主营品类标签区：一级/二级分类
   - 核心优势标签区
   - 目标客户画像区
   - 供应链分析及示例

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 租户未登录 | 跳转至登录页面 |
| 租户无独立站 | 授权卡片置灰，提示选择上传材料 |
| 已选择数据来源 | 自动跳转至对应页面 |
| 文件格式不支持 | 提示不支持的文件格式 |
| 文件超过 5MB | 提示文件大小超出限制 |
| 上传数量达上限 | 上传区域禁用 |
| P08 必填字段缺失 | 中断流程，提示补充，不进入 P09 |
| AI 分析超时 | 提示超时，支持重试 |
| 网络超时 | 提示网络连接异常，支持重试 |
| 服务不可用 | 提示服务暂时不可用，请稍后再试 |
| AI 输出为空 | 自动重试或提示重新分析 |
| AI 格式错误 | 提示数据格式异常 |
| 报告不存在 | 跳转至初始化页面 |
