# 01-concept 概念层设计规范

本目录存放产品基础信息设计文档，是后续所有模块设计的概念基础。

---

## 职责定位

| 维度 | 说明 |
|------|------|
| **核心问题** | 产品是什么？为什么做？ |
| **关注点** | 产品定义、价值主张、业务架构 |
| **描述方式** | 概念性、战略性、全局性 |
| **读者对象** | 产品经理、架构师、决策层 |
| **变更频率** | 低（稳定优先） |

---

## 文件结构

```
01-concept/
├── 01-概念设计.md          # 产品定义、核心价值主张、用户画像
├── 02-商业化说明.md        # 定价策略、付费模式
├── 03-角色组件设计.md      # 系统角色定义、组件架构图
├── 04-实体关系设计.md      # ER 图、实体定义
└── 05-业务流设计.md        # 核心业务流程图
```

---

## 编写原则

1. **概念先行**：只描述"是什么"和"为什么"，不涉及"怎么做"
2. **全局视角**：内容覆盖整个产品，不局限于单个功能模块
3. **稳定优先**：概念层一旦确定，变更成本较高，需谨慎编写
4. **定义明确**：所有术语和概念必须有明确定义，避免歧义

---

## 文档内容规范

### 01-概念设计.md

| 章节 | 必填 | 内容定义 |
|------|------|----------|
| 产品定位 | ✓ | 一句话定义产品是什么 |
| 核心价值主张 | ✓ | 产品解决什么问题，带来什么价值 |
| 目标用户画像 | ✓ | 目标用户的特征描述 |
| 用户痛点与需求 | ✓ | 用户面临的问题和需求 |
| 产品边界 | ✓ | 产品做什么、不做什么 |

### 02-商业化说明.md

| 章节 | 必填 | 内容定义 |
|------|------|----------|
| 商业模式 | ✓ | 盈利模式说明 |
| 定价策略 | ✓ | 价格体系和套餐设计 |
| 付费模式 | ✓ | 付费方式和周期 |
| 推广策略 | ○ | 市场推广计划 |

### 03-角色组件设计.md

| 章节 | 必填 | 内容定义 |
|------|------|----------|
| 系统角色定义 | ✓ | 所有参与角色的定义 |
| 角色权限矩阵 | ✓ | 各角色的权限范围 |
| 组件架构图 | ✓ | 系统组件及其关系 |
| 组件职责说明 | ✓ | 每个组件的职责边界 |

**角色定义格式**：
```markdown
### 角色名称（缩写）

**定义**：角色的明确定义

**特征**：
- 特征1
- 特征2

**权限范围**：
- 可执行操作A
- 可执行操作B
```

### 04-实体关系设计.md

| 章节 | 必填 | 内容定义 |
|------|------|----------|
| 核心实体定义 | ✓ | 关键业务实体的定义 |
| 实体属性 | ✓ | 每个实体的核心属性 |
| 实体关系图 | ✓ | 实体间的关系 |
| 数据流转规则 | ✓ | 数据如何在实体间流转 |

**实体定义格式**：
```markdown
### 实体名称

**定义**：实体的明确定义

**核心属性**：
| 属性名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| id | string | 唯一标识 | "XXX-001" |

**生命周期**：
创建条件 → 状态变化 → 销毁条件
```

### 05-业务流设计.md

| 章节 | 必填 | 内容定义 |
|------|------|----------|
| 流程阶段划分 | ✓ | 业务流程的阶段分解 |
| 阶段目标 | ✓ | 每个阶段的核心目标 |
| 流程步骤 | ✓ | 每个阶段的详细步骤 |
| 阶段产出 | ✓ | 每个阶段的产出物 |
| 流程时序图 | ✓ | 完整的时序图 |

**流程步骤表格格式**：
```markdown
| 步骤 | 动作 | 参与者 | 说明 |
|------|------|--------|------|
| P01 | 动作描述 | 参与者 | 详细说明 |
```

**步骤编号规则**：
- 格式：`P{两位数字}`
- 全局连续编号
- 便于跨文档引用
