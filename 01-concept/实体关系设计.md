## 核心实体说明

### 租户（Tenant）

系统顶级账号单位，对应一个 B2B 外贸企业客户。

| 属性 | 说明 |
|------|------|
| tenant_id | 租户唯一标识 |
| tenant_name | 企业名称 |
| status | 账号状态（正常/冻结） |
| credit_balance | 当前 Credits 余额（可为负） |
| created_at | 创建时间 |
| expired_at | 服务到期时间 |

### BOT 实例（BOT）

租户下的独立获客服务实例，每个 BOT 有独立的 ICP 画像、数据和任务空间。

| 属性 | 说明 |
|------|------|
| bot_id | BOT 唯一标识 |
| tenant_id | 所属租户 |
| icp_profile | ICP 画像（JSON，AI 业务分析结果） |
| status | BOT 状态 |
| created_at | 创建时间 |

### 企业（Company）

系统中的企业主体信息，从多个数据源汇聚。

| 属性 | 说明 |
|------|------|
| company_id | 企业唯一标识 |
| company_name | 企业名称 |
| domain | 企业域名 |
| industry | 行业 |
| scale | 规模（微/小/中/大） |
| country | 注册国家 |
| source | 数据来源（搜客/访客/手动导入） |

### 线索（Lead）

经过 AI 分析评估的企业客户信息，分静态、动态、全景三种类型。

| 属性 | 说明 |
|------|------|
| lead_id | 线索唯一标识 |
| bot_id | 所属 BOT |
| company_id | 关联企业 |
| lead_type | 类型（static/dynamic/panorama） |
| overall_score | 综合评分（0-100） |
| score_breakdown | 四维评分明细（JSON） |
| contact_id | 关联联系人 |
| status | 线索状态 |
| created_at | 创建时间 |

### 联系人（Contact）

企业的关键联系人信息。

| 属性 | 说明 |
|------|------|
| contact_id | 联系人唯一标识 |
| company_id | 所属企业 |
| name | 姓名 |
| position | 职位 |
| email | 邮箱 |
| phone | 电话 |
| source | 来源（ECS/手动） |

### 访客记录（Visitor）

通过 JS 探针采集的访客行为记录。

| 属性 | 说明 |
|------|------|
| visitor_id | 访客唯一标识（基于 IP 哈希） |
| bot_id | 所属 BOT |
| ip_address | IP 地址 |
| country | 国家 |
| first_visit_at | 首次访问时间 |
| last_visit_at | 最近访问时间 |
| visit_count | 访问次数 |
| utm_source | UTM 来源标识（如有） |

### 任务（Task）

异步执行的任务记录。

| 属性 | 说明 |
|------|------|
| task_id | 任务唯一标识 |
| bot_id | 所属 BOT |
| task_type | 类型（analysis/search/dig/analysis_auto） |
| status | 状态（pending/running/completed/failed） |
| started_at | 开始时间 |
| completed_at | 完成时间 |
| credit_consumed | 实际消耗 Credits |

## 实体关系描述

### 租户-BOT 关系（一对多）

```
Tenant ──1:N──► BOT
```
一个租户可拥有多个 BOT 实例，每个 BOT 独立运营，互不干扰。

### BOT-ICP 画像（一对一）

```
BOT ──1:1──► ICP Profile
```
每个 BOT 有一个 ICP 画像（JSON），由 AI 业务分析服务生成，是搜客策略的基础。

### BOT-线索关系（一对多）

```
BOT ──1:N──► Lead
```
每个 BOT 的线索数据严格隔离，按类型分为静态线索、动态线索、全景线索。

### 线索-联系人关系（多对一）

```
Lead ──N:1──► Contact
```
一个线索对应一个主要联系人（可重新挖掘覆盖）。

### 线索-企业关系（多对一）

```
Lead ──N:1──► Company
```
多个线索（静/动/全景）可关联同一企业，通过 domain 匹配实现静动态数据关联。

### BOT-访客关系（一对多）

```
BOT ──1:N──► Visitor
```
每个 BOT 记录其独立站访客，访客数据按 BOT 隔离。

### 企业-联系人关系（一对多）

```
Company ──1:N──► Contact
```
一个企业可有多个联系人记录。

## 数据隔离规则

| 维度 | 隔离方式 |
|------|----------|
| 租户间 | 强制隔离，不可跨租户查询 |
| BOT 间 | ICP 画像、线索、访客数据均按 BOT 空间隔离 |
| 任务并发 | 单 BOT 同类型任务有并发数限制（见业务流设计） |
