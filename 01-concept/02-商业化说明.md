## 商业模式

### 盈利模式

采用 **"SaaS 订阅 + 用量付费"** 的混合模式，通过双重收入流实现可持续盈利：

1. **订阅收入（基础收入）**：按月/年收取固定订阅费，提供基础功能和 BOT 席位
2. **用量收入（增值收入）**：按实际使用量收取 Credits 费用，覆盖高价值 AI 服务

### 收入来源构成

| 收入类型 | 占比预估 | 说明 |
|----------|----------|------|
| 订阅费 | 40% | 基础版、专业版、企业版订阅收入 |
| Credits 用量费 | 50% | 搜客、联系人挖掘、AI 内容生成等 |
| 算力加油包 | 10% | 基础 Credits 用完后的补充购买 |

### 成本结构

| 成本类型 | 说明 |
|----------|------|
| 三方数据 API 成本 | 海关数据、企业信息、联系人数据等 |
| AI 模型调用成本 | GPT-4、Embedding 等模型 Token 费用 |
| 基础设施成本 | 服务器、数据库、CDN 等 |
| 获客成本 | 市场推广、销售团队 |

## 定价策略

### 订阅套餐

| 版本 | 年付价格 | BOT 席位 | 月度 Credits |
|------|----------|----------|--------------|
| **基础版** | 6,999 元 | 1 个 | 待定 |
| **专业版** | 20,997 元 | 3 个 | 待定 |
| **企业版** | 69,990 元 | 10 个 | 待定 |

> **定价说明**：BOT 席位单价为 **6,999 元/年/BOT**，Credits 单价为 **0.1 元/个**。

### 价值导向的后置计费

采用**"见效计费，多维溢价"**，让客户为真实的商业价值买单。

**计费规则**：
- 获取 1 个带有验证邮箱的联系人 = 1 Credit（0.1 元）
- 附加 LinkedIn 主页 = 0 Credit（免费）
- 附加 Twitter/FB = 0 Credit（免费）

**保底策略**：
- 单次无上限获取，后台精准算人头
- 若 API 仅返回空列表（未查到邮箱），则本次操作扣费为 0
- 企业的基础静态信息仍会免费入库保存

## 付费模式

采用 **"SaaS 订阅 + 用量付费"** 的混合模式。

### 订阅费（门槛）

降低入门门槛，包含基础的访客识别功能，快速圈占市场的用户。

### 用量费（增收）

针对高端用户，对"AI 线索深度挖掘次数"、"AI 邮件生成量"等高价值功能设置消耗点数（Credits），对标 ZoomInfo 的 Credit 模式但价格更亲民。

## Credits 计费体系

### 资源类型定义

| 资源类型 | 属性 | 作用 |
|----------|------|------|
| **Global Credits** | 消耗型资产 (Consumable) | 全账号名下所有 BOT 共享此资金池，用于支付全网搜客、IP 解析、联系人挖掘等动作的 API 与 Token 成本 |
| **BOT Seats** | 并发限制型资产 (Concurrency Limit) | 限制该企业账号最多可同时创建并运行几个独立的获客 BOT（不同 BOT 对应不同的独立站或业务线） |

### Credits 消耗场景

| 功能场景 | 计费规则 | 说明 |
|----------|----------|------|
| **全网搜客** | 按实际 API 调用次数计费 | 预估调用次数 → 后台加权折算 → 实际执行后多退少补 |
| **IP 解析** | 1 Credit/次（0.1 元） | 解析访客 IP 对应的企业信息 |
| **联系人挖掘** | 1 Credit/有效邮箱（0.1 元） | 获取 1 个带有验证邮箱的联系人 |
| **附加社媒** | 0 Credit/个（免费） | 附加 LinkedIn 主页、Twitter/FB 等 |
| **AI 内容生成** | 按 Token 消耗计费 | 生成个性化开发信、推广消息等 |

#### 系统消耗明细

| API 服务 | 单次调用 Credits 消耗 | 说明 |
|----------|----------------------|------|
| 地图 API (Google Maps) | 1 Credit | 搜客-地图数据调用 |
| 海关数据 API (出海派) | 1 Credit | 搜客-海关数据调用 |
| 展会数据 API (10Times) | 1 Credit | 搜客-展会数据调用 |
| 网页抓取 API (Jina Reader) | 0 Credit（免费） | 搜客-网页数据抓取 |
| IP 解析 | 1 Credit | 访客 IP 企业信息解析 |
| 验证邮箱获取 | 1 Credit/个 | 联系人挖掘核心计费项 |
| LinkedIn 主页附加 | 0 Credit（免费） | 联系人挖掘附加服务 |
| Twitter/Facebook 附加 | 0 Credit/个（免费） | 联系人挖掘附加服务 |

### 计费原则

计费基于实际 API 调用次数,系统按真实调用核算扣费。

### 实际扣费机制

在检索结束后，系统按真实的 API 成功调用次数重新核算，从用户余额中扣除实际点数 (Credits)，预估时锁定的多余 Credits 将退回账户。

## 推广策略

- 提供 14 天免费试用期
- 试用期内赠送 200 Credits
- 限制最多创建 1 个 BOT

### 阶梯优惠

- 年付享受 8 折优惠
- 连续订阅 12 个月享受额外 9 折
- 推荐新用户双方各获赠 500 Credits 奖励

- 大客户（年消费 $10,000+）可享受专属定价
- 支持自定义 Credits 包购买
