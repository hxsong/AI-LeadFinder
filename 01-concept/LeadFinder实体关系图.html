Tenant

企业租户

tenant_id PK company_name admin_user_id FK global_credits_balance bot_seats_total bot_seats_used service_expiry_date account_status

核心实体，代表一个企业客户账号，管理全局资源池和BOT席位

🤖

Bot

获客服务BOT

bot_id PK tenant_id FK bot_name bot_language business_analysis_id FK icp_model_vector automation_enabled status

业务隔离单元，每个BOT对应一个独立站或业务线的获客引擎实例

🔍

SearchRecord

搜客记录

search_record_id PK bot_id FK task_name query target_count result_count credits_used data_sources status

搜客任务记录，追踪每次搜客的需求、消耗和结果

📊

BusinessAnalysis

业务分析报告

analysis_id PK bot_id FK industry_type upstream_industries downstream_industries icp_profile feature_vectors analysis_report

AI生成的业务分析报告，包含行业定位、ICP画像、特征向量等

📁

StaticLead

静态线索

static_lead_id PK bot_id FK search_record_id FK company_name company_domain match_score lead_grade contact_list source_channel

通过全网搜客获取的潜在客户线索，来源于海关数据、社媒数据等

🔥

DynamicLead

动态线索

dynamic_lead_id PK bot_id FK visitor_id FK company_name intent_score behavior_summary contact_list status

通过访客行为分析挖掘的高意向线索，来源于独立站访客追踪

👤

Contact

联系人

contact_id PK lead_id FK lead_type full_name job_title email linkedin_url is_primary

企业关键联系人，通过Apollo API挖掘获取

👀

Visitor

访客

visitor_id PK bot_id FK website_id FK ip_address geo_location total_engagement_score visit_count status

独立站匿名访客，通过JS探针采集行为数据

🎯

VisitorBehavior

访客行为

behavior_id PK visitor_id FK bot_id FK event_type event_target page_url session_id context_data

访客微观行为事件，包括页面浏览、复制、悬停、表单交互等

🌍

Website

独立站

website_id PK bot_id FK domain_name site_name js_probe_code probe_installed authorized_at

企业独立站配置，用于部署JS探针采集访客数据

👤

User

用户

user_id PK tenant_id FK role_type username email permissions

系统用户，区分Admin和Member角色

💰

CreditTransaction

积分流水

transaction_id PK tenant_id FK transaction_type amount balance_after used_by_bot_id used_for_action

积分消费记录，追踪每次积分变动

📦

ResourcePackage

资源包

package_id PK tenant_id FK package_type credits_amount bot_seats purchase_date expiry_date

购买的资源包，包含积分和BOT席位

🏢

租户层级关系

1:NTenant → Bot（一个租户可创建多个BOT实例）
1:NTenant → User（一个租户可拥有多个用户）
1:NTenant → CreditTransaction（积分流水记录）
1:NTenant → ResourcePackage（资源包购买记录）
🤖

BOT 业务隔离关系

1:1Bot → BusinessAnalysis（专属业务分析报告）
1:1Bot → Website（绑定一个独立站）
1:NBot → BusinessMaterial（多个业务材料）
1:NBot → SearchRecord（多条搜客记录）
1:NBot → StaticLead/DynamicLead（产生线索）
📁

线索关系

1:NSearchRecord → StaticLead（产生多条静态线索）
1:NStaticLead → Contact（包含多个联系人）
1:NDynamicLead → Contact（包含多个联系人）
转化Visitor → DynamicLead（访客转化为动态线索）
👀

访客追踪关系

1:NWebsite → Visitor（接待多个访客）
1:NVisitor → VisitorBehavior（产生多个行为事件）
转化Visitor → DynamicLead（挖掘后转化为动态线索）
⚡

核心业务规则

账号级资源隔离：Global Credits 全账号共享，BOT Seats 限制并发BOT数量

BOT级数据隔离：每个BOT的业务数据、线索、访客等严格隔离

线索类型区分：静态线索来源于全网搜索，动态线索来源于访客追踪

访客数据分离：访客数据独立存储，仅在挖掘成功后才提拔至线索池

数据模型设计说明

架构特点

  • ●采用"账号级统管算力资金池 + BOT级绝对隔离业务数据"架构
  • ●支撑多品牌、多产品线的企业级客户场景
  • ●两种线索类型（静态/动态）独立存储，通过domain关联
  • ●访客数据与线索数据分离，仅在挖掘成功后才提拔至线索池

核心服务映射

  • ●企业分析服务(EAI)：生成BusinessAnalysis和ICPModel
  • ●搜客服务(SDA)：生成SearchRecord、StaticLead和Contact
  • ●动态数据分析服务(DDA)：生成Visitor、VisitorBehavior、DynamicLead
